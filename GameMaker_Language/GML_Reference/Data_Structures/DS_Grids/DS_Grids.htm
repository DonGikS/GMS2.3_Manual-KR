DS 그리드

DS 그리드는 기본적으로 2 차원 배열 유형입니다. 정수 너비와 높이를 설정하여 그리드를 정의한 다음 그리드 내의 총 "셀"수 (너비와 높이를 곱한 값)를 설정 한 다음이 셀을 사용하여 다른 데이터 값 (실수 또는 문자열)을 보유합니다.

이 유형의 구조를 사용하면 x 및 y 인덱스를 제공하여 그리드의 셀 값을 설정하고 검색 할 수 있습니다 (x 및 y 축 모두에 대해 0으로 시작하고 너비-1 및 높이까지 올라갑니다) -1). 그러나 특정 그리드 영역에서 값을 설정하고 단일 값을 추가하며 주어진 영역에서 합계, 최대 값, 최소값 및 평균값을 검색하여이 데이터 구조를 사용 가능한 모든 것 중에서 가장 다목적으로 사용할 수 있습니다.

DS 그리드 데이터 구조에 접근 할 때는 항상 셀 위치에 정수 값 을 사용해야하며, 그렇지 않으면 정수가 아닌 모든 인덱스는 GameMaker Studio 2에 의해 계산됩니다. 그것이 당신이 필요하지 않은 경우, 확인하려는 색인을 전달하기 전에 미리 반올림을 수행해야합니다.

참고 : 모든 동적 리소스와 마찬가지로 데이터 구조는 메모리를 차지하므로 더 이상 메모리 누수를 방지하기 위해 더 이상 필요하지 않을 때는 항상 파괴되어 게임 속도가 느려지고 게임이 중단됩니다.

참고 : 데이터 구조가 있는지 확인해야하는 경우 ds_exists () 함수를 사용할 수 있습니다.

제외하고는 전혀 사용할 수 있습니다 아래에 나열된 특정 기능에서 식을 (AN라는 접근 추가하거나 DS 그리드의 내용을 수정을). 이 접근자는 다음 구문을 사용하여 2D 배열과 유사합니다.

grid_index[# xpos, ypos]

당신의 GML 개요 페이지에서 더 많은 정보 플러스 예를 찾을 수 있습니다 접근 자 .
