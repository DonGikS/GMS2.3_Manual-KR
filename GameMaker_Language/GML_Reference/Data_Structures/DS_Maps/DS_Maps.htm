DS 맵

DS 맵 데이터 구조는 키 와 값 쌍을 저장할 수 있으므로 매우 유용합니다. 예를 들어, 게임에서 캐릭터는 다른 항목 (의 양 가질 수 키 ) 및 각 개별 항목에 대해 그는 그들 (의 수를 가질 수 값을 10 개 개의 건강의 묘약, 5 개 마나 물약을 할 수있는 롤 플레잉처럼) 100 골드 지도는 이러한 쌍을 모두 한 곳에 모으고지도에 쌍을 추가하고 일부 간단한 기능을 사용하여 특정 키에 해당하는 값을 검색 할 수 있습니다. 그러나지도를 사용하기 전에 알아야 할 몇 가지 사항이 있습니다! 지도는 (인식 할 수있는) 방식으로 정렬되어 있지 않습니다 . 즉, 특정 키를 찾으려면 모든 것을 반복해야 할 수도 있습니다 ( 매우 느립니다). 동일한 두 개의 키를 보유 할 방법도없고 하나의 키에 두 개의 값을 할당 할 수도 없습니다.

참고 : 모든 동적 리소스와 마찬가지로 데이터 구조는 메모리를 차지하므로 더 이상 메모리 누수를 방지하기 위해 더 이상 필요하지 않을 때는 항상 파괴되어 게임 속도가 느려지고 게임이 중단됩니다.

참고 : 데이터 구조가 있는지 확인해야하는 경우 ds_exists () 함수를 사용할 수 있습니다.

제외하고는 전혀 사용할 수 있습니다 아래에 나열된 특정 기능에서 식을 (AN라는 접근 추가하거나 DS의 내용이지도 수정을). 이 접근자는 다음 구문을 사용하여 1D 배열과 유사합니다.

map_index[? key]

당신의 GML 개요 페이지에서 더 많은 정보 플러스 예를 찾을 수 있습니다 접근 자 .

  • ds_map_size
  • ds_map_find_first
  • ds_map_find_last
  • ds_map_find_next
  • ds_map_find_previous
  • ds_map_find_value
  • ds_map_find_all
  • ds_map_set
  • ds_map_read
  • ds_map_write
  • ds_map_destroy

 

DS 맵을 저장하고로드하는 데 사용할 수있는 추가 기능이 4 가지 있습니다. 이러한 기능은 맵을 난독 처리하고 파일 형식을 사용하여 대상 플랫폼의 안전한 위치에 저장합니다. 즉, 최종 파일을 장치간에 전송할 수 없습니다.

 

  • ds_map_secure_save
  • ds_map_secure_save_buffer
  • ds_map_secure_load
  • ds_map_secure_load_buffer

 

DS 맵에서 사용하는 용도 중 하나는 JSON으로 작업 할 때 사용되므로 특정 기능이 있습니다.

 

  • ds_map_add_list
  • ds_map_add_map
  • ds_map_replace_list
  • ds_map_replace_map
  • ds_map_is_list
  • ds_map_is_map

참고 : 이 함수를 사용하면 맵 내에 목록과 맵을 추가 할 수 있지만 JSON 이외의 다른 기능에는 사용할 수 없으며 디스크에 쓰거나 다른 방식으로 액세스하면 중첩 된 맵과 목록이 올바르게 읽히지 않습니다.
