게임을 만들 때 언젠가는 텍스트를 사용해야합니다. 게임의 텍스트는 문자열 함수 (문자열은 텍스트 줄을 말하는 또 다른 방법 임)를 사용하여 처리되며 GameMaker Studio 2에는 하나의 삽입을 포함하여 다양한 방법으로 문자열을 조작 할 수있는 완전한 기능 세트가 있습니다 문자열 복사, 문자열 복사 및 포함 된 숫자 또는 문자에 대한 문자열 구문 분석 기능. 일반적으로 큰 따옴표 ""안에 텍스트를 추가 해야만 문자열을 생성 할 수 있으며 작은 따옴표 문자열은 허용 되지 않으며 문자열을 여러 줄로 나누거나 줄 바꿈이 줄 바꿈 인 것처럼 GameMaker Studio 2에서 렌더링하지 않을 수도 있습니다 ( 아래에 설명 된대로 문자열 리터럴 @ 식별자가 사용됩니다).

문자열을 만들 때 사용할 수있는 특정 규칙이 있으며 대부분 이스케이프 문자 사용과 관련이 있습니다. " \\ "기호가 앞에 오는 문자입니다. 예를 들어 문자열 안에 따옴표를 넣으려면 다음과 같이하십시오.

str = "Hello\"World\"";

GameMaker Studio 2는 또한 전체 4 바이트 폭의 유니 코드 문자를 지원하므로 표준의 상한 (이모티콘 포함)에서 유니 코드 문자를 디코딩하고 인코딩 할 수 있습니다. 유니 코드 문자를 처리하려면 " \\ "를 사용하여 유니 코드 리터럴 앞에 16 진수의 숫자가 " u "앞에 오도록 합니다 (예 : " Ã¡ "에 대한 " \\ u00e2 "). 여기서 숫자는 유니 코드 문자의 수입니다. . 이런 방식으로 유니 코드로 작업 할 때 GameMaker Studio 2가 " u "뒤에있는 모든 숫자를 해석한다는 사실을 알고 있어야합니다. 예를 들어 "Ã"를 쓰려면 다음을 사용해야합니다.

"\u00e2\a"

또는

"\u00e2\u61"

또는

"\u00e2" + "a"

" \\ u00e2a "를 사용하면 실제로 유니 코드 문자 " à¸ª "(실질적으로 " \\ ue2a "가 됨)가됩니다.

또한 GameMaker Studio 2는 16 진수 리터럴을 처리 할 수 ​​있습니다. 일반적으로 " 0x "다음에 16 진수로 기록됩니다 (예 : " 0xff "). 여기서 숫자는 사용할 문자 수입니다. GameMaker Studio 2에서는 " \\ x "를 사용하여 16 진수 값을 사용합니다. 이들 및 기타 사전 정의 된 이스케이프 문자는 아래 표에 나열되어 있습니다.

일정한 기술
\\엔 줄 바꿈
\\아르 자형 캐리지 리턴
\\비 백 스페이스 (0x08)
\\에프 용지 공급 (0x0c)
\\티 가로 탭 (0x09)
\\V 세로 탭 (0x0b)
\\\\ 백 슬래시 자체 (0x5c)
\\ㅏ 경고 (0x07)
\\ u [16 진수] 유니 코드 문자 삽입
\\ x [육각수] 16 진 리터럴 문자 삽입
\\ [소수] 8 진 유니 코드 문자 삽입

 

참고 : 문자열은 용지 공급, 세로 탭 등을 지원하지만 렌더링을 의미하는 것은 아니며 문자열을 그릴 때 이러한 문자는 무시 될 수 있습니다.

전체 문자열 앞에 @ 문자를 붙여서 그대로 문자열 리터럴 을 만들 수도 있습니다.

var test = @"

";

위의 코드는 줄 바꿈 이스케이프 문자가 포함 된 것처럼 여러 줄에 문자열을 렌더링합니다. 완전 문자열 리터럴은 이전 GameMaker 버전 문자열 리터럴과 유사하지만 큰 따옴표 나 작은 따옴표를 사용해야하며 @ 기호가 앞에 붙어야합니다. 코드 파일에서 여러 줄로 나눌 수 있으며 이스케이프 된 문자, 즉 @ "를 지원하지 않습니다. Hello \\ World " 는 W on World를 벗어나려고 하지 않으며 그대로 저장됩니다. 이와 같은 문자열 리터럴을 사용할 때 문자열의 일부로 따옴표를 포함하려면 문자열을 분리해야합니다.

var test = @"Hello " + "\"" + @"World" + "\""
