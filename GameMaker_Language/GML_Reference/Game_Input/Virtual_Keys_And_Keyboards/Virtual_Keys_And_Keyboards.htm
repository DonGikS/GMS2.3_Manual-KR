가상 키 및 가상 키보드

터치 스크린 장치 용 게임을 개발할 때 사용자로부터 키보드 입력을받는 것이 종종 중요합니다. GameMaker Studio 2를 사용할 때 다음 두 가지 방법 중 하나를 수행 할 수 있습니다.

  • 화면상의 가상 키 를 만들어 입력하십시오.
  • OS 가상 키보드 를 띄우고 입력을받습니다

사용하는 것은 게임과 필요한 것에 크게 좌우되지만, 일반적으로 플레이어 제어 또는 메뉴 용 화면 버튼을 원할 때는 가상 키를 사용하고 플레이어 입력이 필요할 때는 가상 키보드를 사용합니다 이름 가져 오기 또는 값 설정과 같은

터치 스크린이있는 모바일 장치에서 게임이 제대로 작동하려면 사용자가 터치 할 때 특정 항목에 반응하도록 화면 영역을 할당해야 할 수 있습니다. 내부적으로 매핑 된 표준 키보드 키에 "가상 키"를 할당 한 다음 생성 된 키보드 이벤트를 사용하여 응용 프로그램을 제어하면됩니다. 이 "가상 키"는 룸 위치가 아닌 화면 위치에 매핑되므로 x / y 값은 디스플레이 크기에 따라 절대적입니다. 즉, 뷰 또는 상대 룸 좌표 사용에 대해 걱정할 필요가 없으며 객체의 GUI 이벤트 그리기 에서 키 스프라이트를 그리고 가상 키를 간단하게 정의 할 수 있습니다.

참고 : 이러한 기능은 터치 스크린을 사용하는 Windows, Ubuntu (Linux) 또는 MacOS 대상 플랫폼에서는 작동하지 않지만 마우스 클릭은 "터치"로 표시됩니다.

사용 가능한 가상 키 기능은 다음과 같습니다.

 

  • virtual_key_add
  • virtual_key_show
  • virtual_key_hide
  • virtual_key_delete

 

게임에서 터치 스크린 장치의 텍스트 또는 숫자 입력이 필요한 경우 제공된 가상 키보드 기능을 사용하여 장치 OS 키보드를 사용할 수 있습니다. 이 함수는 호출 또는 OS 키보드를 숨기고 동의 그것에서 입력, 그러나 그것은 할 것 없는 일반 키보드 이벤트를 트리거, 오히려 업데이트 keyboard_string의 사실로 인해 사용자 입력 (이은과 변수를 가상 키보드 입력 할 수있다 전체가 단일 문자가 아닌 단어).

가상 키보드를 열거 나 닫으면 시스템 비동기 이벤트 가 생성되어 async_load DS 맵에 다음 키 / 값 쌍이 채워집니다.

  • " event_type "-수신 된 시스템 이벤트 유형으로, 가상 키보드의 경우 " 가상 키보드 상태 "가됩니다.
  • " screen_height "-가상 키보드의 높이 (픽셀). 키보드가 보이지 않으면 0이됩니다.
  • " keyboard_status "- 키보드 의 현재 상태로, 다음 문자열 중 하나로 반환됩니다. "숨김"
  • "숨겨진"
  • "전시"
  • "명백한"

사용 가능한 가상 키보드 기능은 다음과 같습니다.

참고 : 이 기능은 (tvOS 포함) iOS 및 UWP 대상 플랫폼 (아마존 화재, FireTV 및 AndroidTV 포함) 안드로이드에만 유효합니다.

 

  • keyboard_virtual_show
  • keyboard_virtual_hide
  • keyboard_virtual_status
