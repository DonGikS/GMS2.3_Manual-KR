입자

GameMaker 스튜디오 2 복잡한 것들에 대한 당신은 일반적으로했을 객체를 만들 경우 방 주위 해당 개체의를. 그러나 그래픽 효과의 경우 모든 인스턴스에 포함 된 변수와 다른 이벤트에 포함 된 코드로 인해 처리시 "비용"이 발생하므로 비용이 많이들 수 있습니다. 그래픽 요구 사항을 그리기 위해 타일 ​​맵 레이어 와 자산 레이어 로 전환하면 이러한 비용을 줄일 수 있지만 일반적으로 정적이며 변경하거나 많이 이동할 수 없습니다. 그러나 게임에서 빠르고 다양한 그래픽 효과를 그릴 수있는 또 다른 옵션이 있으며 입자 를 사용 하는 것 입니다.

파티클 은 파티클 시스템 내에 정의 된 특정 속성을 가진 그래픽 리소스입니다. 이러한 속성은 개별 입자에 대해 직접 조작 할 수 없지만 개별 입자와 그 입자가 속한 시스템을 정의하는 데 사용되는 코드를 통해 변경됩니다. 인스턴스를 사용하는 CPU 오버 헤드없이 게임에서 폭발, 데칼, 비, 눈, 별 필드 및 잔해와 같은 아름답고 화려한 효과 (또는 미묘하고 신중한 효과)를 만드는 데 매우 유용합니다.

파티클 시스템의 기본 설정은 세 단계로 이루어지며, 세 번째 단계는 방 안에서 파티클 효과를 만드는 방법에 따라 선택 사항입니다.

  • 파티클 시스템 만들기 : 파티클 시스템은 다양한 파티클 유형을 준비하기 위해 사용할 컨테이너와 같습니다. 우리는 코드를 사용하여 입자에 대한 일련의 시각적 측면과 동작을 정의한 다음 "컨테이너"에 배치하여 나중에 필요할 때 언제 어디서나 사용할 수 있도록합니다.
  • 입자 유형 만들기 : 입자 유형은 그래픽 효과 자체입니다. 다양한 색상, 알파, 크기 및 움직임을 가진 다양한 유형을 가질 수 있지만 개별 입자를 제어 할 수 는 없습니다 . 일련의 매개 변수를 정의하면 입자가 선택되어 임의의 동작 비헤이비어가 선택됩니다.
  • 이미 터 만들기 : 이미 터는 매우 명확하게 정의 된 한계 내에서 입자를 버스트하거나 스트리밍하는 데 사용할 수있는 옵션입니다. part_particles_create () 및 part_particles_create_colour () 함수를 사용하여 모든 인스턴스에서 파티클을 생성 할 수 있기 때문에 선택 사항이지만 모든 상황에 항상 적합한 것은 아닙니다.

파티클은 효과를 만들기위한 훌륭한 도구이지만, 게임 성능이 저하되거나 충돌이 발생하지 않도록하려는 경우를 제외하고 따라야하는 특정 제한 사항과 모범 사례가 있습니다.

  • 파티클 시스템, 파티클 및 이미 터는 메모리를 차지하므로 메모리 누수를 유발하여 게임 속도가 느려지고 결국 충돌 할 수 있으므로 사용 방법에 매우주의해야합니다. 이것에 대처하는 한 가지 방법은 게임 시작시 정의되고 모든 것이 제거되는 글로벌 시스템을 갖는 것입니다. 그러나 동적 시스템을 원한다면 각 파티클과 이미 터 (및 시스템 자체)를 순간에 파괴해야합니다 필요하지 않습니다.
  • 파티클은 CPU에서 빠르고 가벼울 수 있지만 여전히 약간의 처리가 필요하므로 한 번에 40,000 개가 화면을 가로 질러 터지지 않아야합니다. 특정 효과를 달성하는 데 필요한 것으로 제한하십시오.
  • 14 개의 포함 된 스프라이트 중 하나를 사용하는 대신 자신의 입자 스프라이트를 정의하는 경우 필요한 효과를 달성하기 위해 가능한 작은 크기로 유지해야합니다.
  • 입자는 아무것도 상호 작용하지 않습니다 . 사용자가 게임에서 사용자 또는 다른 인스턴스와 상호 작용해야하는 경우, 입자는 단순한 그래픽이므로 인스턴스를 사용하는 것이 좋습니다.
  • 한 게임에서 만들 수있는 시스템, 이미 터 및 파티클의 양에는 기술적 제한이 없지만 메모리 사용을 가능한 한 낮게 유지하려면 가능한 한 모든 것을 최소한으로 제한해야합니다.
  • 휴대 기기로서 입자 알아서 도면 들은 (감속의 주요 원인 중 하나 인 휴대 기기 위에 그리기) 화면의 큰 영역을 커버하는 경우 그들 저하 될 수있다.
  • HTML5를 사용하면 추가 블렌딩이 없으며 WebGL을 사용하도록 설정하지 않으면 색상 블렌딩을 수행 할 수 없습니다.

GameMaker Studio 2로 게임에서 파티클을 생성하는 가장 쉬운 방법은 내장 효과 메커니즘을 사용하는 것입니다. 이러한 효과는 기본적으로 그래픽 효과 만 그리는 매우 빠른 방법 인 내부 파티클 시스템을 사용하여 만들어 지므로 이러한 기능을 사용할 때 메모리 관리와 같은 모든 세부 사항에 대해 걱정할 필요가 없습니다. 효과 유형, 생성 위치, 원하는 크기 및 색상을 지정하면 GameMaker Studio 2에서 모든 작업을 수행 할 수 있습니다.

내장 된 입자 효과를 만들기 위해 다음과 같은 기능이 있습니다.

 

  • effect_create_below
  • effect_create_above
  • effect_clear

 

이러한 효과는 범위와 사용자 지정에 제한이 있지만 적은 노력으로 간단하고 훌륭한 효과를 만드는 데 계속 사용할 수 있습니다. 예를 들어, 각 단계에서 움직이는 미사일 아래에 작은 회색 연기 퍼프를 만들면 꽤 설득력있는 연기 흔적이 만들어 지므로 입자 전문가 일지라도 이러한 효과는 여전히 일부를 저장할 수 있기 때문에 존재한다는 것을 기억해야합니다 시각.

그러나 파티클의 진정한 잠재력은 GameMaker Language가 제공하는 강력한 기능을 사용하여 자신 만의 효과를 만들고 생성하는 것입니다. 다음 각 섹션에서 자세한 내용을 확인할 수 있습니다.
