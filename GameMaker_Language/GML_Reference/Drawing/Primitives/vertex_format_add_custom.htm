vertex_format_add_custom

이 기능을 사용하면 작성중인 새 정점 형식의 일부로 특정 정점 형식 속성에 대한 사용자 정의 데이터 유형을 사용할 수 있습니다. 사용 가능한 값은 아래에 나열된 선택한 데이터 유형 상수로 정의됩니다.

 

일정한 기술
vertex_type_float1 단일 부동 소수점 값
vertex_type_float2 두 개의 부동 소수점 값
vertex_type_float3 3 개의 부동 소수점 값
vertex_type_float4 4 개의 부동 소수점 값
vertex_type_colour 4 가지 성분 값 (r, g, b, a)
vertex_type_ubyte4 4 개의 구성 요소 부호없는 바이트 값 (0 ~ 255)

이러한 상수를 사용하는 경우에도 생성중인 셰이더 내에서 값을 올바르게 "바인딩"할 수 있도록 정의해야합니다. 이것은 DX와 OpenGL이 다른 요구 사항을 가지고 있기 때문에 필요하기 때문에 제대로 바인딩하지 않으면 셰이더에서 제대로 전달되지 않습니다. 선택할 수있는 사용 가능한 상수는 다음과 같으며 사용하는 사용 상수는 생성되는 셰이더의 특성에 따라 다릅니다.

 

일정한 기술
vertex_usage_position 위치 값 (x, y, z)
vertex_usage_colour 색상 값 (r, g, b, a)
vertex_usage_normal 정점 법선 값 (nx, ny, nz)
vertex_usage_textcoord UV 좌표 (u, v)
vertex_usage_blendweight 입력 행렬의 블렌드 웨이트 (예 : 스켈 레탈 애니메이션)
꼭짓점 _ 블렌드 사용할 행렬의 인덱스 (예 : 골격 애니메이션)
vertex_usage_depth 정점 깊이 버퍼 값
vertex_usage_tangent 탄젠트 값
vertex_usage_binormal 비정규 값
꼭짓점 _usage_fog 안개 값
vertex_usage_sample 샘플러 인덱스

다음과 같은 사용자 정의 형식을 사용할 때주의해야 할 사항이 있습니다.

  • vertex_format_add_custom () 함수는 GLSL 셰이더를 사용할 때 vertex_usage_position , vertex_usage_colour , vertex_usage_normal 및 vertex_usage_textcoord 만 지원합니다. 이것들은 각각 in_Position , in_Colour [0-...] , in_Normal 셰이더 속성에 매핑됩니다 (예 : 텍스처 좌표-정의한 속성에 세 가지 속성 중 하나가 아닌 모든 속성을 매핑 할 수 있음).
  • 일반적으로 vertex_usage_blendweight 및 vertex_usage_tangent 와 같은 유형은 대부분의 셰이더 언어에서 더 이상 사용되지 않으며 올바르게 변환되지 않으므로 가능한 경우 모든 추가 매개 변수에 vertex_usage_textcoord 를 사용해야합니다. 대신 vec , vec2 , vec3 또는 vec4 유형 vertex_usage_textcoord 를 사용하면 모든 것이 잘 작동합니다.
  • GLSL ES는 정수 속성을 지원하지 않으므로 ivec4를 전달해도 작동하지 않습니다 (이 유형은 일반적으로 vertex_usage_blendindices를 전달할 때 사용됨 ). 텍스처 좌표를 전달한 다음 셰이더에서 다음과 같이 ivec4 로 변환하면됩니다.

 

attribute vec3 in_Position;
attribute vec4 in_BlendIndices;
attribute vec4 in_BlendWeights;

varying vec4 v_vColour;
varying mat4 v_mat;

void main()
{
    gl_Position = gm_Matrices[MATRIX_WORLD_VIEW_PROJECTION] * vec4( in_Position.xyz, 1.0);
    v_vColour = in_BlendWeights;
     ivec4 t = ivec4(in_BlendIndices);
     v_mat = gm_Matrices[ t.x ];
}

  • 블렌드 가중치는 일반적으로 배열에 저장되고 블렌드 인덱스를 사용하여 액세스되지만 여기서는 in_BlendIndices 를 ivec4 속성으로 정의하는 대신 vec4 이며 코드에서 ivec4 로 캐스팅됩니다. 그런 다음 gm_Matrix 를 사용하여 생성 된 배열을 인덱싱하는 데 사용됩니다 (float가 아닌 INT 값을 사용하여 배열에만 액세스 할 수 있음).

통사론:

vertex_format_add_custom(type, usage);

예:

vertex_format_begin();
vertex_format_add_textcoord();
vertex_format_add_custom(vertex_type_float3, vertex_usage_position);
my_format = vertex_format_end();

위의 코드는 텍스처와 위치에 대한 3 개의 사용자 정의 부동 소수점 값으로 새로운 정점 형식을 만듭니다. 그런 다음 형식 ID를 변수 "my_format"에 저장합니다.
