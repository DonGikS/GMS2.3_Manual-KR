os_get_info

이 함수는 게임이 실행되는 OS에 대한 자세한 정보가 포함 된 DS Map 을 반환합니다. 반환되는 정확한 정보는 OS와 장치에 따라 다르므로 show_debug_message () 와 같은 함수를 사용하여 반환하려는 값을 대상으로하고 출력하려는 ​​모든 플랫폼에서이 함수를 테스트하는 것이 좋습니다. DS 맵 은 메모리에서 자동으로 지워지지 않으므로 더 이상 포함 된 데이터가 필요하지 않은 경우 ds_map_destroy () 함수를 사용해야합니다.

다음은 다음 플랫폼과 관련된 DS 맵에 제공되는 추가 정보에 대한 세부 정보입니다.

 

Windows / UWP Windows / UWP

Windows Desktop 및 Windows UWP의 경우 DS map에서 다음 키를 사용할 수있는 DX11의 추가 정보를 얻을 수 있습니다 (확장에 유용).

  • video_d3d11_device -DX11 장치에 대한 포인터
  • video_d3d11_context -DX11 컨텍스트에 대한 포인터
  • video_adapter_vendorid- 어댑터 공급 업체 ID가있는 문자열
  • video_adapter_deviceid- 어댑터 장치 ID가있는 문자열
  • video_adapter_subsysid- 어댑터 서브 시스템 ID가있는 문자열
  • video_adapter_revision- 어댑터 개정 번호가있는 문자열
  • video_adapter_dedicatedvideomemory- 비디오 메모리 크기의 문자열
  • video_adapter_dedicatedsystemmemory- 시스템 메모리 크기의 문자열 (DX11 어댑터에서 사용)

안드로이드 / 아마존 파이어 안드로이드 / 아마존 파이어

Android 플랫폼에서는 DS 맵에서 다음 키를 사용할 수있는 다양한 API에서 추가 정보를 얻을 수 있습니다.

  • android_tv - 장치 (또는 아마존 화재 타겟 용으로 컴파일 할 경우), 또는 그렇지 않은 경우는 false 안드로이드 TV 장치 인 것으로 감지되는 경우이 true를 돌려줍니다
  • GL_VERSION- 드라이버가보고 한 OpenGL 버전
  • GL_VENDOR- 운전자가보고 한 OpenGL 공급 업체
  • GL_RENDERER- 드라이버에 의해보고 된 OpenGL의 렌더러
  • GL_EXTENSIONS- 드라이버가보고 한대로 사용 가능한 OpenGL의 확장
  • GL_SHADING_LANGUAGE_VERSION- 이 OpenGL 드라이버가 지원하는 GLSL 버전
  • GL_MAX_TEXTURE_SIZE- 이 OpenGL 드라이버가 지원하는 최대 텍스처 크기
  • SDK_INT -android.os.Build.VERSION.SDK_INT의 값 자세한 내용은 여기 를 참조하십시오.
  • 릴리스-android.os.Build.VERSION.RELEASE의 값 자세한 내용은 여기 를 참조하십시오.
  • 모델-android.os.Build.MODEL의 값 자세한 내용은 여기 를 참조하십시오.
  • DEVICE -android.os.Build.DEVICE의 값 자세한 내용은 여기 를 참조하십시오.
  • MANUFACTURER -android.os.Build.MANUFACTURER의 값 자세한 내용은 여기 를 참조하십시오.
  • CPU_ABI -android.os.Build.CPU_ABI의 값 자세한 내용은 여기 를 참조하십시오.
  • CPU_ABI2 -android.os.Build.CPU_ABI2의 값 자세한 내용은 여기 를 참조하십시오.
  • BOOTLOADER -android.os.Build.BOOTLOADER의 값 자세한 내용은 여기 를 참조하십시오.
  • BOARD -android.os.Build.BOARD의 값 자세한 내용은 여기 를 참조하십시오.
  • VERSION -Android System.getProperty 메소드의 os.version 값
  • REGION -Android System.getProperty 메소드의 user.region 값
  • VERSION_NAME- 이 패키지의 값 versionName-자세한 내용은 여기 를 참조하십시오.
  • PHYSICAL_KEYBOARD- 실제 키보드를 사용할 수 있다고 생각되면 문자열 "TRUE", 그렇지 않으면 "FALSE"

 

참고 : 이 기능은 Android, iOS, Mac, Ubuntu 및 Windows (Desktop + UWP)에서만 작동하며 다른 모든 대상 플랫폼에서는 DS Map 대신 -1을 반환합니다.

 

통사론:

os_get_info()

 

보고:

Real (ds_map index)

 

예:

os_map = os_get_info();
if (os_map != -1)
   {
   var size = ds_map_size(os_map);
   var key = ds_map_find_first(os_map);
   for (var i = 0; i < size - 1; i++;)
      {
      map_data[i] = ds_map_find_value(os_map, key);
      key = ds_map_find_next(os_map, key);
      }
   ds_map_destroy(os_map);
   }

위 코드는 OS 정보를 확인하여 DS Map 이 반환되는지 확인합니다. 그렇다면 코드가 맵을 반복하고 모든 값을 배열에 할당합니다.
