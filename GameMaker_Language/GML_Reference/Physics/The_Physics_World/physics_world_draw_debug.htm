physics_world_draw_debug

이 기능은 물리 장면이 올바르게 작동하는 데 어려움이있을 때 디버깅에 도움을주기 위해 방의 물리 세계 시스템을 나타냅니다. 때로는 게임의 렌더링 부분에 문제가있을 수 있습니다. 예를 들어 스프라이트가 잘못된 위치 나 회전에 그려져 잘못된 물리학 모양이 나타나거나 계산이 정확하지 않을 수 있습니다. 어느 쪽이든이 기능을 사용하면 물리 시스템에서 진행중인 작업을 그려 문제를 찾고 해결하는 데 도움이됩니다. 이렇게하려면 시스템의 어느 부분을 그릴 지 알려야하며, 다양한 "플래그"상수에 저장된 비트에서 생성 된 값을 전달하면됩니다.

 

논의 기술
phy_debug_render_aabb 이것은 룸 축과 관련하여 각 조명기의 절대 경계 상자를 보여줍니다
phy_debug_render_collision_pairs 이것은 현재 충돌중인 조명기를 보여줍니다
phy_debug_render_coms 이것은 방에있는 각 조명기의 무게 중심을 표시합니다
phy_debug_render_core_shapes 방의 비품을 구성하는 기본 모양을 보여줍니다.
phy_debug_render_joints 이것은 방에있는 모든 비품의 각 관절을 그립니다.
phy_debug_render_obb 이것은 방에있는 조명기의 상대적인 경계 상자를 보여줍니다
phy_debug_render_shapes 이것은 실내의 비품을 구성하는 실제 모양을 보여줍니다

 

따라서 두 가지 옵션 중 하나를 그리려면 두 가지에 대해 비트 단위 "또는" (GameMaker Studio 2에서 |로 표시됨)을 수행하고 함수에 사용할 결과 변수를 저장해야합니다. 다음은 룸에서 질량 중심과 모든 조명기의 조인트 만 그리는 함수에 사용할 변수를 설정하는 방법의 예입니다.

flag = phy_debug_render_coms | phy_debug_render_shapes;

어느 것이 당신에게 이런 것을 줄 것입니다 :

위의 코드에서 볼 수 있듯이 화면에 모양과 질량 중심 만 그려집니다 (상태를 나타내는 색상-자세한 내용은 Box 2D 설명서를 온라인으로 확인하십시오). 같은 방법으로 디버그하고 싶은 상수의 전부 또는 전부를 표시 할 수 있습니다. 한 가지 더주의해야 할 점-이 함수는 디버그하려는 인스턴스보다 깊이가 낮은 인스턴스의 그리기 이벤트 에서만 호출해야합니다. 그렇지 않으면 디버그 정보를 볼 수 없습니다.

 

통사론:

physics_world_draw_debug(flag)

논의 기술
깃발 다른 디버그 기능을 표시하기위한 플래그 값

 

보고:

N/A

 

예:

flag = phy_debug_render_aabb | phy_debug_render_collision_pairs | phy_debug_render_obb;
physics_world_draw_debug(flag);

위의 코드는 현재 물리 시스템 내에서 충돌하는 고정구뿐만 아니라 모든 절대 경계 상자, 상대 경계 상자를 그립니다.
