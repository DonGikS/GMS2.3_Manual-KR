트랙 구조

트랙 구조체는 최상위 레벨에 사용할 수 있습니다 자산 트랙 또는 하위 트랙 파라미터 트랙 및 트랙 의 동작은 이름 과 유형 의 두 가지로 정의됩니다. 트랙 구조체 는 sequence_track_new () 함수를 사용하여 생성되며 사용 가능한 속성은 다음과 같습니다.

  • type : 다음 상수는 시퀀스에서 생성되는 트랙의 유형을 정의하는 데 사용할 수 있습니다 (이 상수는 키 프레임 및 키 프레임 데이터를 생성 할 때도 사용되며 아래 섹션에서 설명).
일정한 기술 값
seqtracktype_graphic 그래픽 (스프라이트) 자산 트랙입니다. 1
seqtracktype_audio 이것은 오디오 자산 트랙입니다. 2
seqtracktype_instance 이것은 인스턴스 자산 트랙입니다. 14
seqtracktype_sequence 시퀀스 자산 트랙입니다. 7
seqtracktype_clipmask 클립 마스크 그룹 자산 트랙입니다. 8
seqtracktype_clipmask_mask 클립 마스크 생성에 사용되는 클립 마스크 스프라이트 자산 트랙입니다. 9
seqtracktype_clipmask_subject 이것은 마스크되고있는 클립 마스크 스프라이트 자산 트랙입니다. 10
seqtracktype_group 이 그룹 폴더 자산 추적 11
seqtracktype_colour 이것은 컬러 데이터 파라미터 트랙입니다. 4
seqtracktype_real 이것은 실수 값 파라미터 트랙입니다. 삼
seqtracktype_message 방송 메시지 트랙입니다. 15
seqtracktype_moment 이벤트 / 모멘트 트랙입니다. 16
seqtracktype_bool 현재 사용되지 않습니다. 5
seqtracktype_string 현재 사용되지 않습니다. 6
seqtracktype_spriteframes 현재 사용되지 않습니다. 13
seqtracktype_empty 현재 사용되지 않습니다. 12

 

  • name : 애셋 트랙을 생성 할 때 여기에 지정한 이름은 트랙을 식별하는 데 필요한 문자열 일 수 있지만, 파라미터 트랙의 경우 특정 문자열을 지정하여 GameMaker Studio 2에 생성하는 파라미터 트랙의 종류를 알려야합니다 ( 이 문자열은 seqtracktype_real 및 seqtracktype_colour 매개 변수 트랙 유형에만 적용됩니다. " position "-유형이 seqtracktype_real 인 트랙에 설정되면이 트랙의 내용이 위치 데이터로 사용되어야 함을 나타냅니다.
  • " scale "-유형이 seqtracktype_real 인 트랙에 설정되면이 트랙의 내용이 스케일링 데이터로 사용되어야 함을 나타냅니다.
  • " rotation " -seqtracktype_real 유형의 트랙에 설정되면이 트랙의 내용이 회전 데이터로 사용되어야 함을 나타냅니다.
  • " blend_multiply "-유형이 seqtracktype_colour 인 트랙에 설정되면이 트랙의 내용이 색상 곱셈 데이터로 사용되어야 함을 나타냅니다. 이러한 트랙의 키 프레임 데이터는 [A, R, G, B] 형식의 4 개 값으로 구성된 배열이어야합니다. 각 구성 요소의 값은 0과 1 사이로 표현됩니다. 여기서 0은 HEX 값 # 00에 해당하고 1은 HEX 값 #FF에 해당합니다 (시퀀스에서 이미지 블렌드 트랙의 색상 선택기에 표시되는 0-255). 편집자).
  • "image_speed는"- 형 seqtracktype_real이다 트랙 설정할 때이 트랙의 콘텐츠가 화상의 속도 값으로 사용하는 것을 나타낸다.
  • "image_index는"- 형 seqtracktype_real이다 트랙 설정할 때이 트랙의 콘텐츠가 화상의 인덱스 값으로 사용하는 것을 나타낸다.
  • "image_angle는"- 형 seqtracktype_real이다 트랙 설정할 때이 트랙의 콘텐츠가 화상의 각도 값으로 사용하는 것을 나타낸다.
  • " image_blend "-유형이 seqtracktype_colour 인 트랙에 설정되면이 트랙의 내용이 이미지 블렌드 값으로 사용되어야 함을 나타냅니다. 이러한 트랙의 키 프레임 데이터는 [A, R, G, B] 형식의 4 개 값으로 구성된 배열이어야합니다. 각 구성 요소의 값은 0과 1 사이로 표현됩니다. 여기서 0은 HEX 값 # 00에 해당하고 1은 HEX 값 #FF에 해당합니다 (시퀀스의 색상 곱하기 트랙에 대한 색상 선택기에 표시된대로 0-255). 편집자).
  • tracks :이 속성을 사용하면이 트랙의 자식 인 트랙 목록에 액세스 할 수 있습니다. 이 속성을 가져 오면 트랙 구조체 배열 이 반환되고이 속성을 설정할 때 트랙 구조체 배열을 지정해야합니다.
  • visible :이 트랙이 표시되는지 (값이 true ) 아닌지 (값이 false ) 여부를 나타냅니다. 이 값을 가져 오거나 설정할 수 있으며 트랙이 보이지 않으면 하위 트랙 중 어느 것도 그려지지 않습니다.
  • keyframes :이 속성을 사용하면 트랙의 키 프레임 구조체 목록에 액세스 할 수 있습니다. 이 속성을 가져 오면 키 프레임 구조체의 배열이 반환되고이 속성을 설정할 때 키 프레임 구조체의 배열을 지정해야합니다.
