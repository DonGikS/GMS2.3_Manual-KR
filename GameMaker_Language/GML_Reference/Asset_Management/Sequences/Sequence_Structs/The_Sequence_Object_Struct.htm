시퀀스 객체 구조

시퀀스 오브젝트는 게임의 자산 브라우저에 정의 된 시퀀스 자산에서 검색된 구조체에 부여 된 이름입니다. 이 구조체 에는 다음과 같은 속성을 변경할 수 있습니다.

  • name : 문자열의 시퀀스 이름이며 필요에 따라이 값을 가져 오거나 설정할 수 있습니다. sequence_create () 함수를 사용하여 생성 된 시퀀스 는 이름을 갖지 않으며 단순히 빈 문자열 ""이됩니다.
  • loopmode : 시퀀스 객체의 재생 모드이며 가져 오거나 설정할 수 있습니다. 다음 상수를 반환하거나 설정할 수 있습니다.
일정한 기술 값
seqplay_oneshot 시퀀스는 한 번 재생 된 후 완료되면 중지됩니다. 0
seqplay_loop 재생 영역의 끝에 도달하면 재생 헤드가 시작 부분으로 돌아 가면서 시퀀스가 ​​반복됩니다. 1
seqplay_pingpong 재생 영역 끝에 도달하면 재생 헤드가 방향을 바꾸면서 시퀀스가 ​​반복됩니다. 2

 

  • playbackSpeed : 시퀀스의 재생 속도를 지정 합니다 . 이는 PlaybackSpeedType에 따라 초당 프레임 또는 게임당 프레임으로 해석 됩니다 (아래 참조). 이 값을 얻거나 설정할 수 있습니다.
  • playbackSpeedType : playbackSpeed의 해석 방법을 지정하고이 값을 얻거나 설정할 수 있습니다. 다음 상수 중 하나를 사용하여 값을 설정할 수 있습니다 (또는 속성을 확인하면 이러한 상수 중 하나가 반환 됨).
일정한 기술 값
spritespeed_framespersecond playbackSpeed 가 초당 프레임으로 해석되도록 지정합니다 0
spritespeed_framespergameframe playbackSpeed 가 게임당 프레임으로 해석되도록 지정합니다. 1

 

  • length : 시퀀스의 길이 (프레임). 이 값을 가져 오거나 설정할 수 있지만이 시퀀스를 참조하는 시퀀스 인스턴스에 재생 헤드가 새 길이를지나도록 설정된 경우 시퀀스를 짧게 만들면 문제가 발생할 수 있습니다.
  • volume : 시퀀스의 모든 오디오 트랙의 음량을 조절하는 데 사용되는 0에서 1까지의 스칼라 값입니다. 이 값을 가져 오거나 설정할 수 있으며 모든 트랙의 글로벌 오디오 출력을 수정합니다. 예를 들어, 볼륨이 0.8 인 오디오 트랙이 있고 시퀀스 볼륨 속성을 0.5로 설정하면 오디오 트랙이 최종 0.4의 부피.
  • xorigin : 이것은 X 축을 따른 시퀀스의 원점입니다. 이 값을 얻거나 설정할 수 있습니다.
  • yorigin : Y 축을 따른 시퀀스의 원점입니다. 이 값을 얻거나 설정할 수 있습니다.
  • messageEventKeyframes : 시퀀스의 메시지 이벤트 키 프레임에 액세스 할 수 있습니다. 이러한 메시지 이벤트를 가져 오거나 설정할 수 있으며이 속성을 가져올 때 키 프레임 구조체 배열이 반환되며 속성을 설정하려면 키 프레임 구조체 배열을 제공해야합니다. 자세한 내용은 시퀀스 이벤트 및 모멘트 페이지를 참조하십시오.
  • momentKeyframes : 시퀀스의 모멘트 이벤트 키 프레임에 액세스 할 수 있습니다. 이러한 모멘트 이벤트를 가져 오거나 설정할 수 있으며이 속성을 가져올 때 키 프레임 구조체 배열이 반환되고 속성을 설정하려면 키 프레임 구조체 배열을 제공해야합니다. 자세한 내용은 시퀀스 이벤트 및 모멘트 페이지를 참조하십시오.
  • tracks : 시퀀스의 최상위 레벨에있는 자산 트랙 목록에 액세스 할 수 있습니다. 이 속성을 가져 오거나 설정할 수 있으며이 속성을 가져올 때 트랙 구조체 배열이 반환되며 속성을 설정하려면 트랙 구조체 배열을 제공해야합니다. 자세한 내용은 아래의 트랙 구조 섹션을 참조하십시오.

자산 브라우저에서 생성 된 시퀀스의 속성에 액세스하려면 먼저 자산 인덱스에서 sequence_get () 함수를 호출하여 시퀀스 객체 구조체를 검색해야합니다. 또한이 시퀀스 구조를 변경하면이 시퀀스 자산의 모든 추가 인스턴스에도 변경 사항이 적용되며 game_restart () 함수를 호출하더라도 게임이 실행되는 동안 유지됩니다.
