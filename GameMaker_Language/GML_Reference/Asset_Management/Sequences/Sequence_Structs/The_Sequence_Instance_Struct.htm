시퀀스 인스턴스 구조

방에 배치 된 시퀀스는 레이어에 요소 로 배치되며이 요소에는 시퀀스 인스턴스가 포함 됩니다. 이 시퀀스 인스턴스는 구조체에 다음과 같은 속성을 갖습니다.

  • sequence : 시퀀스 인스턴스가 생성 된 시퀀스 객체 구조체의 ID입니다 (자세한 내용은 여기 참조). 이 구조체를 얻거나 설정할 수 있으므로 인스턴스가 참조하는 기본 시퀀스를 변경하십시오.
  • headPosition : 시퀀스 인스턴스의 현재 재생 헤드 위치 (프레임 단위)입니다. 이 값을 가져 오거나 설정할 수 있지만 현재 정의 된 재생 영역보다 크거나 작은 값으로 설정하면 시퀀스에 설정된 재생 유형에 따라 다른 효과가 나타납니다. 자세한 내용은 layer_sequence_headpos () 를 참조하십시오.
  • headDirection : 시퀀스 인스턴스의 현재 재생 방향입니다. 다음 상수를 사용하여이 값을 가져 오거나 설정할 수 있습니다 (레이어 함수 layer_sequence_get_headdir () 및 layer_sequence_headdir () 을 사용하여 구조체에 직접 액세스하지 않고도이 값을 가져 오거나 설정할 수 있습니다).
일정한 기술 값
seqdir_right 시퀀스는 왼쪽에서 오른쪽으로 점진적으로 프레임을 재생합니다 1
seqdir_left 시퀀스는 오른쪽에서 왼쪽으로 감소하는 순서로 프레임을 재생합니다 -1

 

  • speedScale :이 속성을 사용하여 재생 속도 스케일을 가져 오거나 설정할 수 있습니다. 속도 스케일은 승수입니다 . 여기서 1은 기본 재생 속도이고 1보다 작은 값은 재생 속도를 늦추고 1보다 큰 값은 속도를 높입니다 (예 : 0.5 값은 절반 재생 속도, 2 값은 2) 재생 속도가 두 배가됩니다. 레이어 함수 layer_sequence_get_speedscale () 및 layer_sequence_speedscale () 을 사용하여 구조체에 직접 액세스하지 않고도이 값을 가져 오거나 설정할 수 있습니다.
  • paused :이 속성을 확인하여 시퀀스가 ​​일시 중지되었는지 여부를 확인할 수 있으며, 시퀀스가 ​​있으면 true , 그렇지 않으면 false 입니다. 이 속성은 읽기 전용 속성이지만 레이어 함수 layer_sequence_pause () 를 사용하여 재생을 일시 중지하고 layer_sequence_play () 를 사용하여 재생을 다시 시작할 수 있습니다. 구조체의 속성을 직접 확인하는 대신 layer_sequence_is_paused () 레이어 함수를 사용하여이 속성을 확인할 수도 있습니다.
  • finished :이 속성을 확인하여 시퀀스 재생이 완료되었는지 여부를 확인하고 재생이 완료되면 true를 반환하고 그렇지 않으면 false를 반환 할 수 있습니다. 이것은 읽기 전용 속성이며 루프 또는 탁구로 설정되지 않은 트랙에 대해서만 true 를 반환 합니다 . 구조체의 속성을 직접 확인하는 대신 layer_sequence_is_finished () 레이어 함수를 사용하여이 속성을 확인할 수도 있습니다.
  • elementID :이 속성은 시퀀스 요소 의 ID를 보유합니다. 이 iD는 룸 편집기에서 레이어와 연관된 간단한 식별 값이며, 적절한 레이어 기능 과 함께 사용하여 시퀀스가 ​​할당 된 레이어를 찾거나 처리하지 않고도 특정 시퀀스 속성을 변경할 수 있습니다 구조체.
  • activeTracks :이 속성은 시퀀스 (그래픽, 시퀀스, 오디오 등)에서 각 자산 트랙 의 현재 상태에 대한 정보를 포함하는 "평가"구조체 배열을 보유합니다. 이들은 최상위 트랙 일 뿐이며, 반환 된 배열은 읽기 전용 입니다. 각 배열 항목에 반환 된 자산 트랙 구조의 내용은 아래 섹션을 참조하십시오.

다음 목록에는 트랙 구조체에 액세스 할 때 사용할 수있는 속성이 포함되어 있습니다 (상위 시퀀스 구조체의 activeTracks 속성에 의해 반환 됨). 각 자산 트랙 구조에는 트랙이 참조하는 자산 유형에 따라 다음 속성 중 일부 또는 전부가있을 수 있습니다 (읽기 전용으로 지정된 경우를 제외하고 이러한 속성을 모두 가져 오거나 설정할 수 있음). 현재 재생 헤드 위치의 :

  • matrix : 트랙 (스프라이트, 인스턴스 및 시퀀스 트랙)을 그리는 데 사용되는 변환 매트릭스입니다.
  • posx : 현재 재생 헤드 위치 (모든 자산)에서 트랙의 X 축을 따라 시퀀스의 자산 위치.
  • posy : 현재 재생 헤드 위치 (모든 자산)에서 트랙의 Y 축을 따라 시퀀스의 자산 위치.
  • scalex : 현재 재생 헤드 위치 (스프라이트, 인스턴스 및 시퀀스 트랙)에서 트랙의 X 축을 따라 시퀀스의 자산 스케일입니다.
  • scaley : 현재 재생 헤드 위치 (스프라이트, 인스턴스 및 시퀀스 트랙)의 트랙에 대해 Y 축을 따라 시퀀스의 자산 스케일입니다.
  • xorigin : 트랙에 대한 자산의 X 원점 (모든 자산).
  • yorigin : 트랙에 대한 자산의 Y 원점 (모든 자산).
  • width : 트랙에 대한 자산의 너비 (픽셀)입니다 (스프라이트, 인스턴스 및 시퀀스 트랙).
  • height : 트랙에 대한 자산의 높이 (픽셀)입니다 (스프라이트, 인스턴스 및 시퀀스 트랙).
  • imageindex : 현재 재생 헤드 위치 (스프라이트 및 인스턴스 트랙)의 시퀀스에서 트랙의 자산에 대한 이미지 인덱스입니다.
  • imagespeed : 현재 재생 헤드 위치 (스프라이트 및 인스턴스 트랙)에서 시퀀스의 트랙에있는 자산의 이미지 속도입니다.
  • colourMultiply : 현재 재생 헤드 위치 (스프라이트, 인스턴스 및 시퀀스 트랙)에서 시퀀스의 트랙에있는 자산의 색상 곱하기 값입니다. 이 값은 [A, R, G, B] 형식의 4 개의 ARGB 값으로 구성된 배열 입니다. 각 구성 요소의 값은 0과 1 사이로 표현됩니다. 여기서 0은 HEX 값 # 00에 해당하고 1은 HEX 값 #FF에 해당합니다 (시퀀스의 색상 곱하기 트랙에 대한 색상 선택기에 표시된대로 0-255). 편집자).
  • gain : 현재 재생 헤드 위치 (오디오 트랙)에서 트랙의 게인 (볼륨).
  • pitch : 현재 재생 헤드 위치에서 트랙의 피치 (오디오 트랙).
  • track :이 트랙이 기반으로하는 트랙 데이터 구조 (모든 자산, 읽기 전용 값).
  • parent : 트랙의 부모 시퀀스 인스턴스 ID (모든 자산, 읽기 전용 값).
  • activeTracks : 자산 트랙에 포함 된 각 매개 변수 트랙 (모든 자산, 읽기 전용 값)에 대한 평가 구조의 배열입니다. 배열에있는 각 구조체의 내용은 Track Struct 매개 변수 섹션에 나열됩니다.
