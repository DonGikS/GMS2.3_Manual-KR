키 프레임 데이터 구조

트랙에 대한 키 프레임 구조 를 정의 할 때 해당 트랙과 관련된 데이터도 정의해야합니다. 이것은 서로 다른 채널 로 구성되며, 키 프레임 데이터를 설정하는 트랙 유형에 따라 각 채널에 특정 유형의 데이터가 제공 될 수 있습니다. 이러한 속성은 다음과 같습니다.

  • channel : 키 프레임 데이터를 적용 할 채널입니다. 0부터 시작하는 양의 정수 값이며 "position"또는 "scale"키 프레임에 대한 매개 변수 트랙을 만들 때 매우 특정한 채널 값을 사용해야한다는 점에 주목할 가치가 있습니다. 채널 0은 X 위치 또는 X 스케일이고 채널 1은 Y 위치 또는 Y 스케일입니다.
  • spriteIndex : 트랙에 사용할 스프라이트 자산의 인덱스입니다. 이 속성은 seqtracktype_graphic 유형의 트랙에만 사용할 수 있으며 가져 오거나 설정할 수 있습니다.
  • soundIndex : 트랙에 사용할 오디오 자산의 인덱스입니다. 이 속성은 seqtracktype_audio 유형의 트랙에만 사용할 수 있으며 가져 오거나 설정할 수 있습니다.
  • playbackMode : 사운드의 재생 모드. 이 속성은 seqtracktype_audio 유형의 트랙에만 사용할 수 있으며 가져 오거나 설정할 수 있습니다. 특성은 다음 상수 중 하나 여야합니다.
일정한 기술 값
seqaudiokey_loop 재생하면 소리가 반복됩니다. 0
seqaudiokey_oneshot 소리는 한 번만 재생 된 다음 정지됩니다. 1

 

  • curve :이 프로퍼티는 애니메이션 커브 구조체가 필요하며 (자세한 내용은 여기 를 참조하십시오) seqtracktype_real 유형의 트랙에만 사용할 수 있습니다. 곡선 구조체를 사용하지 않으면이 속성의 값은 -1이됩니다.
  • value :이 속성은 단순히 키 프레임 데이터 채널과 관련된 값이며 곡선 구조체가 제공되지 않은 경우 seqtracktype_real 유형의 트랙에만 사용할 수 있습니다.
  • color :이 속성은 [A, R, G, B] 형식으로 키 프레임의 색상 값에 대한 배열 을 반환합니다 (설정된 경우 필요). 이것은 seqtracktype_colour 유형의 트랙에만 사용할 수 있습니다. 각 구성 요소의 값은 0과 1 사이로 표현해야합니다. 여기서 0은 HEX 값 # 00에 해당하고 1은 HEX 값 #FF에 해당합니다 (시퀀스의 색상 트랙에 대한 색상 선택기에 표시되는 0-255). 편집자).
  • sequence :이 속성은 시퀀스 객체 구조체 를 반환하거나 설정해야하며 seqtracktype_sequence 유형의 트랙에만 사용할 수 있습니다.
  • objectIndex :이 속성은 객체 인덱스를 반환하거나 설정되어 있으면 seqtracktype_instance 유형의 트랙에만 사용할 수 있습니다.
  • events :이 속성을 사용하면 키 프레임 데이터 구조와 관련된 이벤트 및 브로드 캐스트 메시지에 액세스 할 수 있습니다. 이 속성을 가져 오거나 설정할 수 있으며, 가져 오면 문자열 배열이 반환되고 설정시 문자열 배열을 지정해야합니다. 이벤트에 대한 자세한 내용은 시퀀스 이벤트 및 순간 섹션을 참조하십시오. 이 특성은 seqtracktype_message 유형의 트랙에만 사용 가능합니다.
  • event :이 속성은 키 프레임 데이터 구조체와 관련된 메서드 를 반환하거나 설정할 수 있습니다. 메소드를 지정하지 않았거나 메소드를 제거하려는 경우 특성은 -1이어야합니다. 이 속성은 seqtracktype_moment 유형의 트랙에만 사용할 수 있습니다.
