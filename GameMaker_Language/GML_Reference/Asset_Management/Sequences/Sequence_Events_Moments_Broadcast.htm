이벤트, 순간 및 방송 메시지

시퀀스 구조체에 대해 여기 에 나열된 모든 속성 외에도 특별한 "액션"속성을 시퀀스에 추가 할 수도 있습니다. 이 "조치"속성에는 시퀀스 객체 인 Events에 추가 할 수있는 세 가지 유형이 있습니다. 모멘트 및 브로드 캐스트 메시지 -이들 각각에 대한 자세한 내용은 아래 섹션에서 설명합니다.

 

시퀀스 이벤트

시퀀스 객체에는 이벤트가 추가 될 수 있으며 이러한 이벤트는 객체 자산이 사용하는 이벤트와 유사합니다. 시퀀스 객체에 할당 한 이벤트는 시퀀스 수명주기에서 정의 된 순간에 한 번 또는 시퀀스가 ​​실행중인 모든 프레임에서 특정 순서로 발생합니다. 이 섹션에서는 이러한 이벤트를 코드를 사용하여 처리하는 방법에 대해 설명하지만 시퀀스 편집기를 사용하여 추가 할 수도 있습니다. 여기에 나열된 이벤트 순서는 재생 헤드 방향의 영향을받지 않으며 시퀀스가 ​​뒤로 재생 되더라도 이벤트는 아래에 주어진 순서대로 계속 실행됩니다.

사용 가능한 이벤트는 다음과 같습니다.

  • Create : 시퀀스 인스턴스가 레이어에 생성 될 때 트리거됩니다.
  • Destroy : 레이어의 시퀀스 인스턴스가 파괴 될 때 트리거됩니다.
  • 정리 : 시퀀스 인스턴스가 가비지 수집 될 때 트리거됩니다.
  • Step : 일반적인 객체 단계 이벤트 직전에 트리거됩니다. 중대한! 시퀀스가 일시 정지되면 시퀀스 재생이 다시 시작된 후 다음 단계까지이 이벤트가 트리거되지 않습니다.
  • Begin Step : 객체 시작 단계 이벤트 직전에 트리거됩니다. 중대한! 시퀀스가 일시 정지되면 시퀀스 재생이 다시 시작된 후 다음 단계까지이 이벤트가 트리거되지 않습니다.
  • End Step : 객체 종료 단계 이벤트 직전에 트리거됩니다. 중대한! 시퀀스가 일시 정지되면 시퀀스 재생이 다시 시작된 후 다음 단계까지이 이벤트가 트리거되지 않습니다.
  • 비동기 시스템 : 모든 객체 비동기 시스템 이벤트 핸들러와 동일하게 트리거됩니다. 중대한! 시퀀스가 일시 중지되면이 이벤트는 트리거되지 않습니다.
  • 브로드 캐스트 메시지 : 브로드 캐스트 메시지 키 프레임이 발생하여 시퀀스 업데이트 중에 트리거됩니다.

모든 이벤트는 스크립트에 정의 된 함수와 연관되어야하므로 이벤트를 설정하려면 메소드를 변수 를 올바른 이름으로 특성에 지정하여 함수를 호출해야하는 이벤트를 시퀀스에 알려야합니다. 이에 대한 구문은 다음과 같습니다.

<sequence_struct>.<event_property> = method(<sequence_struct>, <function_name>);

다른 이벤트에 대한 시퀀스 구조체 속성은 다음과 같습니다.

  • event_create
  • event_destroy
  • event_clean_up
  • event_step
  • event_step_begin
  • event_step_end
  • event_async_system
  • event_broadcast_message

사용 예제를 제공하기 위해 마우스 왼쪽 버튼 클릭이 감지 될 때 시퀀스가 ​​반대 방향으로 돌아가고 싶다고 가정 해 봅시다. 먼저 호출하려는 함수를 사용하여 다음과 같은 스크립트를 작성해야합니다.

/// @function seq_reverse();

function seq_reverse()
{
if mouse_check_button_pressed(mb_left)
    {
    if self.headDirection == seqdir_right
        {
        self.headDirection = seqdir_left;
        }
    else self.headDirection = seqdir_right;
    }
}

이 기능에서 우리가하는 일은 마우스 누름을 듣고 나서 메소드가 바인딩 된 시퀀스의 재생 방향을 반대로하는 것입니다. 현재는 이러한 속성에 연결된 함수에 매개 변수를 전달할 수 없으며, 모든 경우에 함수 범위 내에서 자체 변수가 시퀀스 인스턴스로 지정되어 있습니다.

참고 : 위 스크립트에서 자체 식별자는 생략 할 수 있으며이 예제에 사용되는 메소드 변수의 범위를 명시 적으로 표시하는 데만 사용됩니다.

이것을 시퀀스에 할당하려면 다음과 같이하십시오 :

var _seq = sequence_get(Sequence1);
_seq.event_step = method(_seq, seq_reverse);

이제 순서로 공정 이벤트 지정한 오브젝트를 마우스 누를 때를 검출하고, 시퀀스의 헤드의 방향을 변경하는 기능 인스턴스 가 실내에 생성 된 후에.

 

시퀀스 모멘트 는 시퀀스가 ​​재생 될 때 하나 이상의 지정된 프레임에서 시작되는 고유 한 코드 작업입니다. 이러한 코드 모멘트는 이벤트 (위에서 설명)와 마찬가지로 주어진 프레임에서 트리거되는 미리 정의 된 메소드 변수 입니다. 예를 들어, 시퀀스가 ​​애니메이션의 특정 프레임에서 "글 머리 기호"인스턴스를 작성하려고한다고 가정하십시오. 먼저이 조치에 사용할 함수를 작성해야합니다.

/// @function seq_shoot();

seq_shoot = function()
    {
    instance_create_layer(sequence.xorigin, sequence.yorigin, "Instances", obj_Bullet);
    }

그런 다음 이것을 시퀀스 구조체의 momentKeyframes 속성에있는 프레임에 추가합니다. 이것은 본질적으로이 목적으로 만 사용되는 "특별한"트랙이므로 시퀀스에 원하는 순간 동작과 관련된 키 프레임 데이터 만 사용할 수 있습니다. 이 트랙은 키 프레임 구조를 생성하고 키 프레임 데이터로 채워서 다른 트랙을 만들 때와 같은 방식으로 만들어집니다. 여기서 채널 0은 사용할 수있는 유일한 채널이며 트랙 유형은 seqtracktype_moment 입니다. 아래는 위에서 정의한 함수를 사용하여 주어진 순간 (프레임)에서 실행되는 방법의 예입니다.

var _seq = sequence_get(my_Seq);
var _k = array_create(3);
_k[0] = sequence_keyframe_new(seqtracktype_moment);
_k[0].frame = 60;
_k[1] = sequence_keyframe_new(seqtracktype_moment);
_k[1].frame = 120;
_k[2] = sequence_keyframe_new(seqtracktype_moment);
_k[2].frame = 180;
var _d = array_create(1);
_d[0] = sequence_keyframedata_new(seqtracktype_moment);
_d[0].channel = 0;
_d[0].event = method(_d[0], seq_shoot);
_k[0].channels = _d;
_k[1].channels = _d;
_k[2].channels = _d;
_seq.momentKeyframes = _k;

위의 코드는 단순히 "moment"트랙의 프레임 60, 120 및 180을 설정하여 seq_shoot 메소드를 호출합니다. 현재 각 순간에 사용되는 함수에 매개 변수를 전달할 수 없으며 모든 경우에 함수 범위 내에서 자체 변수가 시퀀스 인스턴스로 할당됩니다 (일반적으로 스스로 사용해야하지만 특별한 경우에는 유용 ​​할 수 있습니다).

 

애니메이션 타임 라인을 따라 특정 프레임에 추가되는 간단한 문자열 인 브로드 캐스트 메시지 를 생성하는 시퀀스를 만들 수 있으며 타임 라인의 해당 지점에 도달하면 문자열은이를 수신하는 모든 객체 인스턴스에 브로드 캐스트됩니다. 시퀀스 편집기를 사용하여 IDE에 이러한 메시지를 추가 할 수 있지만 코드를 사용하여 작성 및 편집 할 수도 있습니다.

GML을 사용하여 시퀀스에서 브로드 캐스트 메시지를 만들려면 시퀀스 구조체의 messageEventKeyframes 속성에서 프레임에 브로드 캐스트 메시지를 추가해야합니다. 이것은 본질적으로이 목적으로 만 사용되는 "특별한"트랙이므로 시퀀스에 원하는 브로드 캐스트 메시지와 관련된 키 프레임 데이터 만 사용할 수 있습니다. 이 트랙은 키 프레임 구조를 생성하고 키 프레임 데이터로 채워서 다른 트랙을 만들 때와 같은 방식으로 만들어집니다. 여기서 채널 0은 사용할 수있는 유일한 채널이며 트랙 유형은 seqtracktype_message 입니다. 아래는 이것이 어떻게 만들어지는 예입니다.

var _seq = sequence_get(my_Seq);
var _k = array_create(2);
_k[0] = sequence_keyframe_new(seqtracktype_message);
_k[0].frame = 60;
_k[1] = sequence_keyframe_new(seqtracktype_message);
_k[1].frame = 120;
var _m1 = array_create(2);
_m1[0] = "Hello";
_m1[1] = "World";
var _m2 = array_create(1);
_m2[0] = "This is a Broadcast Message";
var _d1 = array_create(1);
_d1[0] = sequence_keyframedata_new(seqtracktype_message);
_d1[0].channel = 0;
_d1[0].events = _m1;
var _d2 = array_create(1);
_d2[0] = sequence_keyframedata_new(seqtracktype_message);
_d2[0].channel = 0;
_d2[0].events = _m2;
_k[0].channels = _d1;
_k[1].channels = _d2;
_seq.messageEventKeyframes = _k;

메시지 텍스트가 키 프레임 데이터 채널에 배열로 전달됩니다. 이는 동일한 프레임에서 여러 메시지를 브로드 캐스트 할 수 있고 서로 다른 객체 인스턴스가 해당 메시지를 수신하고 실제로 예상하는 메시지에 따라 다르게 반응하도록하기 때문입니다.

시퀀스에서 브로드 캐스트 메시지가 생성되면 오브젝트 인스턴스에서 기타 > 브로드 캐스트 메시지 이벤트를 사용하여 구문 분석 할 수 있습니다 (자세한 설명은 여기 참조) .
