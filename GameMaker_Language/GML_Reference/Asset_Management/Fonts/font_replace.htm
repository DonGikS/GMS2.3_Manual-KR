font_replace

이 기능을 사용하여 실행중인 시스템에 설치된 글꼴에서 게임의 글꼴을 게임으로 교체 할 수 있습니다. 대체되는 글꼴은 Asset Browser의 글꼴 일 수 없으며 font_add () 함수를 사용하여 이전에 추가 한 글꼴이어야합니다.

이러한 방식으로 글꼴을 교체 할 때 글꼴의 크기 (포인트)와 글꼴을 굵게 또는 기울임 꼴로 정의 할 수 있으며 포함 할 문자 범위를 정의 할 수도 있습니다. 기능이 실패하면 자동으로 실패하고 지정된 글꼴이 바뀌지 않습니다.

HTML5 대상 모듈의 경우이 함수 는 웹 글꼴 을 대체하는 데만 사용할 수 있으며 WebGL을 사용 하지 않는 경우에만 작동합니다. WebGL을 사용하는 경우 일반 글꼴 자원 (예 : 글꼴과 함께 draw_set_font () 사용)을 사용해야합니다 자산 브라우저에서) 또는 비트 맵 글꼴 ( font_add_sprite () 참조).

중대한! HTML5를 타겟팅 할 때 기울임 꼴 및 굵은 체 텍스트는 글꼴 자산 또는 웹 글꼴 자체에 이러한 문자가 포함 된 경우에만 지원됩니다. 기본 글꼴에서 합성되지 않습니다.

다음 표는 모든 브라우저에서 표준이며 문제없이 사용할 수있는 글꼴을 보여줍니다. 게임을 실행하는 컴퓨터 나 장치에 다른 글꼴이 있거나 없을 수 있으므로이 기능의 사용은 다음 표준 글꼴로 제한되어야합니다.

웹 이외의 다른 모든 대상에서이 기능을 사용하여 파일에서 글꼴을 바꿀 수 있습니다. 파일 은 GameMaker Studio 2의 포함 된 파일 기능을 사용하여 게임 번들 에 포함 되어야하며 * .ttf 형식의 글꼴 파일이어야하며 아시아 또는 아랍어와 같은 비표준 글꼴을 추가하는 데 유용합니다.

경고! : 게임에 * .ttf 형식의 글꼴을 포함 시키면 게임과 함께 배포 할 수있는 라이센스 가 있어야합니다 .

이런 방식으로 파일에서 글꼴을로드 할 때 글꼴의 크기 는 픽셀 단위 이며 첫 번째 값과 마지막 값은 무시되므로 글꼴의 모든 글리프가 추가됩니다. 이런 방식으로 추가 된 글꼴은 자체 텍스처 페이지에 할당되므로이 함수를 사용할 때는 그릴 때 텍스처 스왑 수가 증가하므로주의를 기울여야합니다. 글리프에는 글꼴에 대해 정의 된 경계 상자 외부에 그려진 부분이있을 수 있으므로 글꼴을 그릴 때 글꼴이 약간 더 크게 나타날 수도 있습니다. 특정 글꼴에 대해 단일 프레임으로 렌더링 할 수있는 약 200 개의 글리프 제한이 있다는 점도 알고 있어야합니다.

참고 : GameMaker Studio 2 : Studio에 글꼴을로드 할 때 더 이상 필요하지 않은 경우 글꼴을 다시 제거해야합니다 ( font_delete () 사용 ). 그렇지 않으면 메모리 누수로 인해 속도가 느려지고 게임이 충돌 할 수 있습니다.

 

통사론:

font_replace(ind, name, size, bold, italic, first, last);

논의 기술
산업 대체 할 (이전에 추가 된) 글꼴의 색인입니다.
이름 사용할 글꼴 이름 (예 : 'Arial') 또는 글꼴이 포함 된 * .ttf 파일 인 경우 파일 경로
크기 글꼴 크기-웹 글꼴, * .ttf 글꼴의 픽셀을 가리 킵니다.
굵게 글꼴이 굵은 체인지 (true) 아닌지 (false)
이탤릭체 글꼴이 기울임 체인지 (true) 아닌지 (false)
먼저 포함 할 첫 번째 문자 (확실하지 않은 경우 32로 이동)
마지막 포함 할 마지막 문자입니다 (확실하지 않은 경우 128로 이동).

 

보고:

Real

 

예:

font_replace(myfont, "Arial", 24, true, true, 32, 128);

이렇게하면 변수 "myfont"에 색인 된 글꼴이 24pt 크기의 새 글꼴로 바뀌고 "Arial"이 사용되며 굵고 기울임 꼴입니다. 글꼴 범위에는 대문자와 소문자, 숫자 및 모든 일반적인 문장 부호가 포함됩니다.
