오디오

GameMaker Studio 2에는 * .ogg , * .mp3 및 * .wav 사운드 형식을 기반으로하는 완벽한 오디오 엔진이 있습니다. 그런 다음 IDE에 추가 ​​된 이러한 유형의 사운드는 아래 표시된 기본 오디오 기능을 사용하여 게임에서 사용할 수 있습니다. 기본 음향 효과보다 더 복잡한 작업이나 단일 음악을 재생하는 경우 사운드 재생 방식을 수정할 수있는 고급 오디오 기능을 참조 할 수 있습니다. 또한 스트리밍 오디오, 오디오 배치, 3D 사운드 제공 및 오디오 그룹화에 특화된보다 전문화 된 기능들이 있습니다.

중대한! HTML5 대상에서 오디오를 사용할 때 모든 브라우저가 웹 오디오를 지원하는 것은 아니므로 실행시 프로젝트의 사운드가 재생되지 않을 수 있습니다. 다음 링크에서 Web Audio 지원에 대한 일반적인 아이디어를 얻을 수 있습니다. Web Audio를 사용할 수 있습니까? .

다음 기능은 가장 간단하고 간단한 방법으로 오디오를 처리하기위한 것입니다. 이미 터 사용을 피하고 이러한 사운드가 항상 청취자 위치 에서 생성되므로 사용자가 사운드를 생성하고 쉽게 음악을 재생할 수 있습니다 (아래 링크 된 섹션 참조) 리스너 에 대한 자세한 내용)은 일반적으로 리스너의 변경에 영향을받지 않습니다.

 

  • audio_exists
  • audio_get_name
  • audio_get_type
  • audio_play_sound
  • audio_play_sound_at
  • audio_pause_sound
  • audio_pause_all
  • audio_resume_sound
  • audio_resume_all
  • audio_stop_sound
  • audio_stop_all
  • audio_is_playing
  • audio_is_paused
  • audio_create_stream
  • audio_destroy_stream

 

다음 기능은 오디오 엔진의 작동 방식과이를 통해 재생되는 사운드가 청취자에 의해 "듣게"되는 방식을보다 잘 제어 할 수 있도록 설계되었습니다. 따라서 다음 기능을 사용하기 전에 나머지 GameMaker Studio 2 오디오 엔진의 작동 방식에 대한 실무 지식이 있어야합니다.

 

  • audio_sound_set_track_position
  • audio_sound_get_track_position
  • audio_sound_set_listener_mask
  • audio_sound_get_listener_mask
  • audio_sound_length
  • audio_sound_pitch
  • audio_sound_get_pitch
  • audio_sound_is_playable
  • audio_falloff_set_model
  • audio_sound_gain
  • audio_sound_get_gain

웹 오디오

HTML5 대상 플랫폼 용 게임을 만들 때 오디오 엔진에는 웹 오디오 지원이 필요하며, 이는 때때로 오디오가 원하는 시점이나 방식으로 재생되지 않는 경우가 있습니다. 웹 오디오 컨텍스트가 실행 중이 아니거나 게임이 실행 중일 때 실행이 중지 될 수 있기 때문입니다. 원인은 브라우저마다, 심지어는 같은 브라우저의 다른 버전 간에도 크게 다르므로 웹 오디오 컨텍스트 상태 변경시기를 감지하는 것이 매우 중요합니다 (예 : 컨텍스트가 변경되는시기를 감지하고 배경 음악과 같은 오디오 반복 / 일시 중지).

이 문제를 해결하기 위해 GameMaker Studio 2에는 다음 기능을 사용하여 Web Audio 컨텍스트 상태의 변화를 감지하는 두 가지 방법이 있습니다.

  • audio_system_is_available

 

웹 오디오 상태가 변경 될 때마다 트리거되는 비동기 시스템 이벤트를 사용할 수도 있습니다. 이 경우 오디오 이벤트 인 경우 " audio_system_status "문자열을 보유하는 " event_type "키로 채워진 내장 된 async_load DS 맵 을 얻게 됩니다. 이 키가 있으면 추가 " 상태 "키가 있으며 " 사용 가능 "또는 " 사용 불가능 "입니다. 이 이벤트는 모든 플랫폼에서 트리거되지만 HTML5를 제외한 모든 플랫폼에서는 오디오 엔진이 처음 초기화 될 때 Game Start에서 한 번만 트리거됩니다. 자세한 내용은 다음 섹션을 참조하십시오.

  • 비동기 시스템 이벤트
