일반 레이어 기능

GameMaker Studio 2 Room Editor를 사용 하면 방 에 다른 자산의 레이어 를 추가 할 수 있습니다. 타일 ​​맵 (타일 세트 사용), 배경, 스프라이트 또는 시퀀스 및 인스턴스 일 수 있지만 룸 편집기와 달리 레이어 당 단일 자산 유형으로 제한 되지 않으며 동일한 자산에 여러 자산 유형을 추가 할 수 있습니다. 예를 들어 배경과 타일 맵을 모두 같은 레이어에 둘 수 있습니다. 레이어에 추가 된 각 자산은 해당 레이어의 요소 로 간주됩니다.

이 섹션에서는 룸 레이어 작업시 사용되는 일반적인 기능과 인스턴스 레이어 요소 작업과 관련된 몇 가지 기능에 대해 간략하게 설명합니다.

 

  • layer_exists
  • layer_get_id
  • layer_get_depth
  • layer_get_id_at_depth
  • layer_get_name
  • layer_get_all
  • layer_get_all_elements
  • layer_get_element_layer
  • layer_get_element_type
  • layer_get_x
  • layer_get_y
  • layer_get_hspeed
  • layer_get_vspeed
  • layer_create
  • layer_destroy
  • layer_x
  • layer_y
  • layer_hspeed
  • layer_vspeed
  • layer_add_instance
  • layer_has_instance
  • layer_instance_get_instance
  • layer_destroy_instances
  • layer_element_move
  • layer_set_visible
  • layer_get_visible
  • layer_depth
  • layer_force_draw_depth
  • layer_is_draw_depth_forced
  • layer_get_forced_depth
  • layer_script_begin
  • layer_script_end
  • layer_shader
  • layer_get_script_begin
  • layer_get_script_end
  • layer_get_shader

 

또한 다른 방의 레이어를 타겟팅하는 것과 관련된 몇 가지 추가 기능이 있습니다. 현재 방이 아닌 방에서 무언가를 변경하고 싶거나 절차 상으로 방을 생성하거나 여러 가지를 원할 수도 있습니다. 이를 가능하게하기 위해 다음 기능을 사용합니다.

 

  • layer_set_target_room
  • layer_get_target_room
  • layer_reset_target_room

 

여기서 수행하는 작업은 적절한 기능을 사용하여 대상 룸을 설정 한 다음 레이어 기능이 정상이라고합니다. 이 레이어 기능은 이제 현재 룸이 아닌 대상 룸 에만 적용 됩니다 . 완료되면 reset 함수를 호출하여 코드 범위를 다시 현재 회의실로 되돌립니다.

경고! 다른 방을 대상으로하는 동안 instance_create_layer () 또는 instance_create_depth ()를 사용하여 인스턴스를 만들 수없고 layer_add_instance () 레이어 함수를 사용할 수없는 경우 를 제외하고 모든 일반 레이어 함수를 사용할 수 있습니다.

마지막으로 이것은 레이어의 인스턴스 에만 사용하기위한 것입니다 (레이어 자체에 다른 요소가있을 수 있지만 다음 기능의 영향을받지는 않지만). 모든 인스턴스를 비활성화하거나 다시 활성화하는 데 사용할 수있는 몇 가지 기능이 있습니다.

 

  • instance_deactivate_layer
  • instance_activate_layer
