xboxlive_read_player_leaderboard

이 함수는 주어진 사용자 이름에 대한 리더 보드 정보를 반환합니다. 리더 보드는 게임의 XDP 대시 보드에서 미리 생성되어 있어야하며 함수를 호출하면 관련 키 / 값 쌍으로 채워진 async_load 맵이 포함 된 콜백 소셜 비동기 이벤트 가 트리거됩니다. 이 DS 맵의 "id"키는 올바른 콜백을 식별하는 데 사용되며 (어떤 주어진 비동기 이벤트에 대해 하나 이상의 트리거 기능이있을 수 있음) IT는 여러 다른 키 /뿐만 아니라 상수 achievement_leaderboard_info 와 쌍을 이룰 것입니다 값 쌍. 지도의 정확한 내용은 다음과 같습니다.

  • " id "-이 함수의 경우 achievement_leaderboard_info
  • " leaderboardid "-제공자 대시 보드에 정의 된 리더 보드의 고유 ID입니다.
  • " numentries "-리더 보드에서받은 항목 수입니다.
  • " PlayerN "-플레이어의 이름입니다. 여기서 "N"은 순위표 항목 목록 내 위치에 해당하는 정수 값입니다.
  • " PlayeridN "-플레이어의 고유 사용자 ID "N"
  • " RankN "-순위표 내 플레이어 "N"의 순위입니다.
  • " ScoreN "-플레이어 "N"의 점수입니다.

이 기능을 사용하려면 필터 속성을 설정하기 위해 다음 상수 중 하나를 제공해야합니다.

 

일정한 기술
achievement_filter_all_players 모든 점수를 얻으십시오.
achievement_filter_friends_only 오름차순으로 친구 점수 만받습니다.
achievement_filter_favorites_only 오름차순으로 즐겨 찾기 점수 만 받기
achievement_filter_friends_alt 내림차순으로 친구 점수 만받습니다.
achievement_filter_favorites_alt 내림차순으로 즐겨 찾기 점수 만 가져옵니다.

 

 

통사론:

xboxlive_read_player_leaderboard(leaderboard_name, user_name, num_items, friend_filter);

논의 기술
leaderboard_name XDP에 설정된 리더 보드의 이름입니다.
사용자 이름 읽을 사용자의 이름입니다.
num_items 검색 할 항목 수입니다.
friend_filter 필터 상수 중 하나입니다 (위 설명 참조).

예:

var _user = xboxlive_user_for_pad(0);
var _uid = xboxlive_user_id_for_user(_user);
xboxlive_read_player_leaderboard("MyLeaderboard", _uid, 10, achievement_filter_all_players);

위 코드는 사용자 ID를 얻은 다음 해당 리더 보드를 사용하여 지정된 리더 보드의 처음 10 개 장소에 대한 모든 플레이어 데이터를 요청합니다.
