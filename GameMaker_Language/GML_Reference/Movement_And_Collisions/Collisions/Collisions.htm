충돌

움직임을 계획하거나 특정 행동을 결정할 때, 게임 세계의 특정 장소에서 다른 물체와 충돌이 있는지 확인하는 것이 종종 중요하며, 작업에 적합한 충돌을 선택하는 것이 가장 중요한 작업입니다. GameMaker Studio 2에는 프로젝트의 필요에 따라 충돌을 올바르게 처리하는 데 도움이되는 다양한 내장 기능이 있습니다.

중대한! 모든 충돌 함수는 입력 값에 관계없이 정수 위치를 사용하여 작동합니다. 정수가 아닌 값은 가장 가까운 정수로 반올림됩니다.

 

다음 함수는 인스턴스가 다른 인스턴스와의 충돌을 확인할 수있는 다양한 방법을 처리합니다.

 

  • place_empty
  • place_free
  • place_meeting
  • position_empty
  • position_meeting
  • position_change
  • position_destroy
  • instance_place
  • instance_place_list
  • instance_position
  • instance_position_list

 

고급 충돌 검사

이를 위해 다음 루틴을 사용할 수 있습니다 ( 이동 및 인스턴스 이동 관련 섹션에 자세히 설명 된 기타 기능). 이러한 충돌 함수는 모두 약간 다르게 작동하지만 여기서 설명 할 세 가지 공통 인수를 유지합니다.

  • obj 여기에서 함수가 충돌을 확인해야하는 객체를 지정합니다. 인스턴스 ID, 객체 유형 또는 특수 GM 키워드 all 이 될 수 있습니다. 선택한 개체 유형이 다른 개체의 지정된 부모 인 경우 해당 개체의 모든 자식 도이 검사에 포함됩니다.
  • prec . 이는 true 또는 false 일 수 있으며 충돌 검사의 정밀도와 관련이 있습니다. true로 설정되면 CPU에 대해 약간 더 복잡한 작업 인 정확한 충돌을 계산해야합니다 (단, 검사중인 객체의 마스크 속성에서 충돌 감지에 "정확한"표시가있는 경우에만). 조심스럽게. False 이면 검사 할 개체의 경계 상자를 기준으로 충돌이 계산됩니다.
  • notme . 코드를 실행하는 객체와 동일한 색인을 가진 객체와의 충돌을 확인해야 할 수 있으므로 코드가 자체적으로 충돌하는 것을 방지하기 위해이를 true로 설정할 수 있습니다. 코드를 실행하는 객체의 인스턴스를 구체적으로 찾고 있거나 중요하지 않은 경우 false 로 남겨 둘 수 있습니다.

기본 함수는 충돌이 발견 된 인스턴스의 ID 또는 충돌이없는 경우 특수 키워드 noone 을 반환하는 반면, 목록 함수는 충돌하는 인스턴스 수를 반환하고 미리 만들어진 DS 목록을 그들의 ID. 주어진 객체의 이러한 기능과 인스턴스에 의해 정의 된 영역과 여러 번 충돌이 발생하면 하나의 인스턴스 ID 만 반환 되며 충돌의 인스턴스 중 하나 일 수 있습니다.

고급 충돌을 처리하는 다음 기능이 있습니다.

 

  • 충돌 _ 원
  • conflict_circle_list
  • 충돌 _ 타원
  • conflict_ellipse_list
  • conflict_line
  • conflict_line_list
  • 충돌 _ 점
  • conflict_point_list
  • conflict_rectangle
  • conflict_rectangle_list

 

마스크없이 충돌 검사

위의 모든 기능은 충돌 검사 인스턴스 와 관련이 있으며 인스턴스에 대해 정의 된 충돌 마스크에 의존합니다. 그러나 포인트 또는 영역과 "충돌"을 확인해야하는 순간, 특히 인스턴스에 스프라이트가 할당되지 않았거나 마우스 등으로 작업 할 때 많은 시간이 있습니다. 따라서 GameMaker Studio 2도 이러한 상황에서 도움이되는 다음 기능을 제공합니다.

 

  • point_in_rectangle
  • point_in_triangle
  • point_in_circle
  • rectangle_in_rectangle
  • rectangle_in_triangle
  • rectangle_in_circle

 

물리 충돌

물리에서 빌드를 사용할 때 위의 충돌 함수는 물리 활성화 인스턴스에서 작동하지 않을 수 있습니다. 이는 이러한 인스턴스가 더 이상 대부분의 일반 내장 변수를 사용하지 않고 (물리 활성화 인스턴스에는 자체 변수 세트가 있음 ), 조명기 를 사용할 때 충돌 마스크 또는 경계 상자를 사용하지 않기 때문입니다. 대신에. 그러나 물리가 활성화 된 상태에서 충돌 고정물을 검사하는 특수 기능이 있습니다.

 

  • physics_test_overlap
