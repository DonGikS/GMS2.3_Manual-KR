숫자 함수

GameMaker Studio 2의 실수는 배정 밀도 부동 소수점 숫자 로 간주됩니다. 즉, 소수점 앞뒤에 고정 된 숫자가없는 소수점 또는 정수 입니다. 소수점 값이없는 정수 예를 들어 2는 정수이지만 2.01은 부동 소수점 실수입니다.

참고 : HTML5 대상에서 모든 실수는 두 배입니다.

정수와 부동 소수점 사이의 차이는 크로스 플랫폼 개발을 처리 할 때 Windows PC에서 수행되는 계산의 정밀도가 모바일 장치에서 수행 될 때의 동일한 계산의 정밀도와 동일 하지 않기 때문에 매우 중요합니다. 즉, 비교할 때 특히주의해야합니다.

if (image_index == 1)
    {
    do something...
    }

위의 예에서 image_speed 를 0.25로 설정 한 경우 (예 : 4 단계) image_index 값은 1이라고 가정 할 수 있지만 부동 소수점 수학이 작동하는 방식으로 인해 1.0000002와 같은 값일 수 있습니다. 따라서 평가는 사실 이 아닙니다. 이러한 유형의 오류는 디버깅하기가 매우 어려울 수 있으므로 항상 오류를 인식하고 미리 계획하고 값을 확인하는 다른 방법을 사용하거나 숫자를 정수를 확인하십시오 (부동 소수점 수학 및 이것이 왜 문제인지에 대한 자세한 내용 은 여기 를 참조하십시오 ). 예를 들어 위 코드는 다음과 같이 작성할 수 있습니다.

if (floor(image_index) == 1)
    {
    do something...
    }

YoYo Compiler 대상을 사용할 때 모든 표현식과 함수는 왼쪽에서 오른쪽으로 평가되는 반면 다른 모든 대상 플랫폼에서는 오른쪽에서 왼쪽으로 평가되므로 이는 예를 들어 다른 결과를 제공합니다. 플랫폼에서 :

val = max(num, ++num, num++);

참고 : 자세한 정보는 평가 순서 섹션을 참조하십시오.

GameMaker Studio 2에서 제공되는 특수 기능을 사용하여 부동 소수점 수학에 대한 엡실론 값을 설정할 수도 있습니다. 실수가 가장 가까운 부동 소수점 수로 반올림되면 엡실론 ( "기계 엡실론"이라고도 함)은 상대 오류의 상한을 형성하며 다음 함수를 사용하여 엡실론 값을 가져오고 설정할 수 있습니다.

 

  • math_set_epsilon
  • math_get_epsilon

 

이 함수는 모두 임의의 숫자와 값을 사용합니다.

참고 : 랜덤 기능을 사용할 때 GameMaker Studio 2는 게임을 시작할 때마다 동일한 랜덤 시드를 유지합니다. 랜덤 함수가 항상 처음에 같은 값을 반환한다는 보장으로 디버깅이 훨씬 쉬워 지지만, 원하지 않는 경우, 게임 시작시 랜덤을 사용하여 새로운 랜덤 시드를 먼저 설정해야합니다 ( ) 또는 random_set_seed () 입니다.

 

  • 고르다
  • 무작위
  • random_range
  • 이란
  • irandom_range
  • random_set_seed
  • random_get_seed
  • 무작위

 

이들은 모두 어떤 방식 으로든 값을 반올림하거나 다양한 주어진 값 중에서 단일 값을 선택하는 함수입니다.

 

  • 일주
  • 바닥
  • frac
  • 복근
  • 기호
  • 올림
  • 최대
  • 평균
  • 중앙값
  • 분
  • Lerp
  • 집게
