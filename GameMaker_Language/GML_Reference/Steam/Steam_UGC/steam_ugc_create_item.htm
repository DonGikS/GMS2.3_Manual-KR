steam_ugc_create_item

이 함수는 Workshop API를 준비하고 추가 할 항목에 대해 공개 된 파일 ID를 생성하는 데 사용됩니다. 업데이트를 위해 Steam 비동기 이벤트에 반환되는 고유 한 게시 된 ID 값을 사용해야하므로 업로드 할 항목으로 다른 작업을 수행하기 전에 함수 를 호출 해야합니다 . 이 함수를 사용하려면 게임에 Steam 앱 ID를 제공하고 file_type 인수에 다음 상수 중 하나를 사용해야합니다.

 

일정한 기술
ugc_filetype_community 이것은 업로드되어 커뮤니티의 모든 사람이 사용할 수있는 파일을 작성하는 데 사용됩니다.
ugc_filetype_microtrans 업로드 된 파일을 설명하는 데 사용되지만 게임에서 공식 콘텐츠로 추가하는 것을 고려하기위한 것입니다.

이 함수를 사용하면 항목의 생성을보고하기 위해 Steam 비동기 이벤트가 트리거 될 때 파싱 될 수있는 비동기 ID 값을 반환합니다. 이벤트는 async_load ds_map에 다음 키 / 맵 값을 포함합니다.

  • " id "-호출 함수가 리턴 한 비동기 ID
  • " result "-작업 결과 (실제 값). 이것은 GML 상수 ugc_result_success 또는 다른 실수입니다. 따라서이 상수를 확인하여 호출이 성공했는지, 그렇지 않은 경우 정상적으로 작동하지 않았는지 확인해야합니다. 반환 된 나머지 가능한 값은 Steam "EResult"값의 결과로 표시되며 가능한 89 개의 모든 값에 대한 SDK 헤더에 steamclientpublic.h 가 표시됩니다.
  • " event_type "-이 키는 " ugc_create_item "
  • " legal_agreement_required "-합니다 (참조 참 또는 거짓이 될 것이다 스팀 문서 자세한 내용을)
  • " published_file_id "-이 키는 항목에 대해 게시 된 고유 ID 인 int64를 보유합니다 ( int64 () 함수를 사용하여 후속 함수에서 사용할 때와 같이 캐스트해야 할 수도 있음)

 

통사론:

steam_ugc_create_item(consumer_app_id, file_type);

논의 기술
consumer_app_id Steam 게임의 고유 한 앱 ID입니다.
file_type 사용 가능한 파일 유형 상수 중 하나입니다 (아래에 나열 됨).

 

보고:

Async ID

 

확장 된 예 :

이 예제에서는 먼저 함수를 호출하고 비동기 ID 값을 변수에 저장합니다.

var app_id = steam_get_app_id();
new_item = steam_ugc_create_item(app_id, ugc_filetype_community);

그러면 Steam API에 요청을 보내 새로운 Worksop 항목을 생성하고 다음과 같이 처리 할 비동기 이벤트를 생성합니다.

var event_id = async_load[? "id"];
if event_id == new_item
    {
    var type = async_load[? "event_type"];
    if type == "ugc_create_item"
        {
        global.Publish_ID = async_load[? "published_file_id"];
        }
    }

위의 코드는 이벤트 유형을 확인하고 "ugc_create_item"인 경우 게시 된 파일 ID를 검색하여 나중에 참조 할 수 있도록 전역 변수에 저장합니다.
