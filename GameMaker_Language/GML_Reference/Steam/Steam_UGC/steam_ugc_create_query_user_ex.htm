steam_ugc_create_query_user_ex

이 함수는 UGC 데이터베이스를 쿼리하는 데 사용할 수 있습니다. 이 기능에는 사용자의 ID 값과 아이템을 소비 할 게임의 ID 및 / 또는 아이템을 생성 한 게임의 ID가 필요합니다. 또한 쿼리 할 데이터 유형, 쿼리 할 항목 유형 및 반환 된 항목이 정렬되는 순서 및 페이지 번호에 대해 다음 상수를 사용해야합니다. 쿼리는 최대 숫자를 리턴합니다. 50 개 항목 중

 

리스트 타입 상수 기술
ugc_list_Published Steam 설명서를 참조하십시오.
ugc_list_VotedOn -
ugc_list_VotedUp -
ugc_list_VotedDown -
ugc_list_WillVoteLater -
ugc_list_Favorited -
ugc_list_Subscribed -
ugc_list_UsedOrPlayed -
ugc_list_Followed -

 

일치 유형 상수 기술
ugc_match_Items Steam 설명서를 참조하십시오.
ugc_match_Items_Mtx -
ugc_match_Items_ReadyToUse -
ugc_match_Collections -
ugc_match_Artwork -
ugc_match_Videos -
ugc_match_Screenshots -
ugc_match_AllGuides -
ugc_match_WebGuides -
ugc_match_IntegratedGuides -
ugc_match_UsableInGame -
ugc_match_ControllerBindings -

 

정렬 순서 상수 기술
ugc_sortorder_CreationOrderDesc Steam 설명서를 참조하십시오.
ugc_sortorder_CreationOrderAsc -
ugc_sortorder_TitleAsc -
ugc_sortorder_LastUpdatedDesc -
ugc_sortorder_SubscriptionDateDesc -
ugc_sortorder_VoteScoreDesc -
ugc_sortorder_ForModeration -

이 함수는 다른 쿼리 함수에서 사용하기 위해 변수에 저장해야하는 고유 한 쿼리 핸들 값을 반환합니다. 이 함수는 쿼리 만 준비하지만 실제로는 전송하지 않습니다. steam_ugc_send_query () 함수를 호출해야합니다. 더 자세한 steam_ugc_query _ * () 함수를 사용하여 검색 요청을 실제로 보내기 전에 세분화 할 수 있습니다.

 

통사론:

steam_ugc_create_query_user_ex(list_type, match_type, sort_order, page, account_id, creator_app_id, consumer_app_id);

논의 기술
list_type 작성할 데이터 목록의 유형입니다 (아래에 나열된 상수 참조).
match_type 쿼리 할 UGC 항목의 유형입니다 (아래에 나열된 상수 참조).
sort_order 데이터를 정렬하는 방법 (아래에 나열된 상수 참조).
페이지 조회 할 페이지 번호입니다.
계정 ID Steam 계정 ID
creator_app_id 아이템 작성자 앱 ID
consumer_app_id 소비자 앱 ID

 

보고:

Query Handle

 

예:

query_handle = steam_ugc_create_query_user_ex(ugc_list_Published, ugc_match_Items, ugc_sortorder_TitleAsc, page, global.AccountID, 0, global.GameID);

위의 코드는 쿼리 요청을 생성하고 나중에 사용할 수 있도록 핸들에 변수를 저장합니다.
