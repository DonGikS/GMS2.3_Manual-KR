푸시 알림

모바일 장치에서는 일반적으로 언제든지 하나의 응용 프로그램 만 포 그라운드에서 활성화 될 수 있지만, 많은 게임과 응용 프로그램은 응용 프로그램이 포 그라운드에 있지 않을 때 사용자에게 관심있는 이벤트가 발생할 수있는 시간 기반 또는 상호 연결된 환경에서 작동합니다. 이 경우 로컬 및 원격 푸시 알림을 통해 이벤트 발생시 이러한 게임이 사용자에게 알릴 수 있습니다.

운영 체제가 로컬 또는 원격 푸시 알림을 전달하고 대상 응용 프로그램이 포 그라운드에서 실행되고 있지 않으면 배너 형식으로 사용자에게 알림을 표시합니다. 알림 경고가 있고 사용자가이를 누르면 게임이 시작되고 사용자가 지정한 콜백 문자열을 확인하고 해당 값을 기반으로 푸시를 해결할 수있는 비동기 푸시 이벤트 가 호출됩니다. 알림이 전달 될 때 애플리케이션이 포 그라운드에서 실행중인 경우 애플리케이션은 여전히 ​​비동기 푸시 이벤트를 수신하지만 사용자가 직접 작성하지 않으면 메시지가 표시되지 않습니다.

지원되는 플랫폼 간에는 몇 가지 차이점이 있습니다.

  • Android에서는 앱이 실행 중이 아니거나 백그라운드에서 수신 될 때 수신 된 모든 알림이 저장되고 다음에 앱이 실행될 때 비동기 이벤트에 의해 전달되는 데이터는 Android 게임 옵션 내에서 푸시 알림을 활성화해야합니다. 작업.
  • iOS의 경우 특정 알림에 대한 데이터는 알림을 선택한 경우 (또는 알림을받을 때 앱이 포 그라운드에서 실행중인 경우에만) 전달되며 iOS 게임 옵션 에서 푸시 알림을 활성화해야 작동합니다.

 

지역 알림

로컬 푸시 알림은 게임이 설치된 장치에 국한되며 백엔드 서버가 필요하지 않습니다. 당신은 단순히 알림 날짜와 시간을 설정하고 게임이 현재 실행되고 있지 않은 경우 사용자에게 표시됩니다. 이 알림 유형은 사용자가 게임을 다시 플레이하거나 "일일 보상"등을 할 수 있도록 "알림"을 설정하는 데 유용합니다. 사용자가 알림을 탭하면 게임이 시작되고 데이터가 전달됩니다. 그런 다음 게임에서 파싱하여 보상을 줄 수 있습니다.

참고 :이 기능은 iOS 및 Android 대상 모듈로 제한됩니다. Android의 경우 Google 푸시 알림 확장 프로그램을 설치해야합니다.

로컬 알림에 다음 기능을 사용할 수 있습니다.

 

  • push_local_notification
  • push_get_first_local_notification
  • push_get_next_local_notification
  • push_cancel_local_notification

 

 

원격 알림 메시지는 서버에서 장치 플랫폼 앱 스토어가 제공하는 서비스로 전송 된 다음 해당 메시지를 응용 프로그램이 설치된 모든 장치로 전달합니다. iOS 및 Android 대상 모듈의 GameMaker Studio 2에서 지원됩니다.

GameMaker Studio 2에는 원격 알림을 처리하는 기능이 없습니다.이 기능은 모두 서버에서 생성하고 해당 앱 관리자가 처리해야하기 때문입니다. 그러나 일단 올바르게 설정되면 GameMaker Studio 2 게임은 이러한 알림을 수신하며, 로컬 알림과 마찬가지로 비동기 푸시 이벤트 에서 처리 할 수 ​​있습니다.

원격 푸시 알림이 전달 될 것이라는 보장은 없으며 허용 된 데이터 페이로드는 상당히 작습니다. 이는 플랫폼마다 다르지만 iOS는 특히 제한적입니다. 애플 서비스는 가장 최근의 알림 만 제공합니다.이 알림은 페이로드 데이터가 비동기 이벤트로 전달되도록 수신자가 선택해야하며,이 알림의 최대 페이로드 크기는 256 바이트입니다. . 일반적으로 원격 푸시 알림은 예를 들어 서버에서 새 데이터를 사용할 수 있음을 나타냅니다.

참고 : 서버 측 구현은 전적으로 최종 사용자에게 달려 있으며, YoYo Games는 해당 기술 자료에서 제공하는 기본 설정 정보 이외의 다른 측면을 지원하지 않습니다.

사용 가능한 플랫폼 별 정보와 서버 설정 방법에 대한 자세한 내용은 다음 문서를 참조하십시오.
