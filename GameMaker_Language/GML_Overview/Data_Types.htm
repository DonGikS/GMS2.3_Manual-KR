데이터 타입

이전 섹션에서는 변수와 범위 지정 규칙에 대해 다루었지만 변수가 저장할 수있는 다른 데이터 유형 에 대해서는 거의 언급하지 않았습니다. 따라서이 섹션에서는 다양한 유형과 해당 유형에 대해 설명합니다.

계속하기 전에 "데이터 유형"의 의미를 간단히 설명하겠습니다. 변수를 작성할 때 정보를 보유하는 데 사용될 수 있으며 함수를 호출 할 때 리턴 된 정보를 저장할 수도 있습니다. 그러나이 정보는 다양한 "향미"로 제공 될 수 있습니다. 예를 들어, 실제 숫자 이거나 문자열 일 수 있습니다. 사용되는 이러한 다른 유형의 값을 데이터 유형 이라고 하며 GameMaker 언어를 사용 하는 경우 다음 중 하나 일 수 있습니다.

 

문자열 문자열

 

실수 실수

실수는 문자열이 아닌 모든 값이나 다른 가능한 데이터 유형입니다. 따라서 124, 45639.566546456, 0, -45.5 등은 모두 실수의 예입니다. 모든 실수는 배정 밀도 부동 소수점 값 또는 정수 값 (32 비트 정수 -int32- 또는 64 비트 정수 -int64 )으로 저장되며 컴파일러는 가능한 경우 최적화합니다 (예 : 0.0은 정수 값에 최적화 됨) 0). 정수가 아닌 값을 처리 할 때 부동 소수점 수학의 특성으로 인해 약간의 반올림 오류가 발생할 수 있습니다. 이 기능 및 기타 숫자 관련 기능에 대한 자세한 내용은 GML 참조-숫자 기능을 참조 하십시오.

참고 : HTML5 대상에서 모든 실수는 두 배입니다.

GameMaker Studio 2에서 생성 된 변수는 모두 배정 밀도 부동 소수점 숫자 또는 정수로 저장되지만 확장을 처리 할 때 다른 형식을 계속 사용할 수 있습니다. 확장 프로그램에서 GameMaker Studio 2로 전달 된 다음 적절한 is _ * () 함수를 사용하여 확인할 수 있습니다 (이 목록은 here 참조) .

16 진 값 16 진 값

GameMaker Studio 2는 16 진 리터럴 을 합법적 인 값으로 받아 들일 수 있습니다. 16 진수 값은 특히 색상으로 작업 할 때 일반적이지만 양의 정수 값이 필요한 모든 곳에서 사용할 수 있습니다. 16 진 값은 다음 두 가지 방법 중 하나로 형식화 될 수 있습니다. 여기서 < abcd > 는 실제 16 진 값입니다.

$<abcd>
0x< abcd>

예를 들어 다음 값은 표시된대로 16 진수로 표현할 수 있습니다.

11406 = $2c8e = 0x2c8e
16777215 = $ffffff = 0xffffff

 

부울 부울

부울은 단순히 true 또는 false 일 수있는 값입니다. 현재 GameMaker Studio 2는 0.5 이하의 실수를 false 값으로, 0.5보다 큰 실수를 true 로 해석 합니다 . 그러나 상수 1과 0 (또는 다른 실수)에서 참과 거짓을 확인해야한다는 것을 의미하지는 않습니다 . 상수 참 과 거짓 도 함께 제공되므로 코드에서 항상 문제가 발생하지 않도록 반드시 사용해야합니다. 부울 데이터 형식은 향후 업데이트에 추가됩니다.

다음 함수를 사용하여 실수를 내재적으로 부울 값으로 변환 할 수 있습니다.

포인터 포인터

포인터는 메모리 위치를 가리키는 "데이터 유형"입니다. 포인터에 대한 작업을 수행 할 수 없으며 다른 함수의 메모리에서 텍스처 또는 버퍼 주소를 얻는 것과 같은 매우 특정한 함수에만 사용됩니다. 포인터를 반환하는 함수의 예는 buffer_get_address () 또는 sprite_get_texture ()를 참조하십시오.

값이 포인터인지 확인하는 기능 (아래 " 데이터 유형 확인 "참조)과 값을 포인터로 변환하는 기능도 있습니다.

  • ptr
