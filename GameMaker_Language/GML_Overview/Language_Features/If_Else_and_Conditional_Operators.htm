if / else 및 조건부 연산자

대부분의 프로그래밍 언어의 기본 기능은 부울 true 또는 false 답변을 제공하는 간단한 질문을하는 기능이며, GML에서는 if 키워드를 사용하여 수행 됩니다 . 간단한 if 조건은 표현식 을 사용하며 표현식이 true로 해석되면 다음 기본 형식으로 하나 이상의 명령문을 수행합니다.

if (<expression>)
    {
    <statement>;
    <statement>;
    ...
    }

여기서식이 참으로 해결되면 무언가를 수행한다고 말하고 있습니다. 조건의 "then"부분은 암시 적이 지만 사용할 수있는 then 키워드가 있지만 (거의 항상 생략되지만) 다음과 같이 조건을 만들 수도 있습니다.

if (<expression>) then
    {
    <statement>;
    <statement>;
    ...
    }

if 및 then 외에도, 검사중인 표현식이 false로 평가되면 else 키워드를 사용하여 다른 작업을 수행 할 수도 있습니다. 이 " if ... then ... else ... "양식은 다음과 같습니다.

if (<expression>)
    {
    <statement>;
    }
else
    {
    <statement>;
    }

이 경우 표현식이 평가되고, false로 평가되면 else 이후의 명령문이 실행되고, 그렇지 않으면 초기 명령문이 실행됩니다 ( true ).

참고 : GameMaker 언어에서 0보다 작거나 같은 값은 false로 평가되고 0보다 큰 값은 true로 평가됩니다.

예를 들어 (실행됩니다 그렇지 않으면 첫 번째 문), 각 문에 대한 블록에서 새 줄을 취하는 경우에 문 주위 epxressions 및 중괄호 {} 주변에 항상 넣어 괄호에 좋은 습관이다 :

// This will work
if <expression> <statement>;

// Example:
if test == true variable = false else variable = true;

// This is better
if (<expression>)
    {
    <statement>
    }
else
    {
    <statement>
    }

// Example
if (test == true)
    {
    variable = false;
    }
else
    {
    variable = true;
    }

이것은 조금 더 장황하지만 코드에 모호함이 없으며 모든 플랫폼에서 항상 올바르게 컴파일된다는 것을 의미합니다. 그러나 식과 연산자 페이지의 섹션에 설명 된대로 초기 예는 그렇지 않을 수 있습니다.

주 :이 동일한 경우, 당신은 "=="연산자를 사용하여, 단지 지정을위한 "="하나를 사용해야 볼 두 값을 비교합니다. 현재 GameMaker는 이들을 상호 교환 가능한 것으로 취급하지만 향후 변경 될 수 있으며 비교 및 ​​할당에 올바른 연산자를 사용하면 코드가 더 깨끗하고 명확 해집니다.

if 사용의 적절한 예를 제공하려면 단계 이벤트에 배치 될 때 실에서 x = 200 위치로 인스턴스를 이동시키는 다음 코드를 고려하십시오.

if (x < 200)
    {
    x += 4;
    }
else
    {
    x = 200;
    }

if 에서 복합 검사를 수행 할 수도 있습니다. 즉, 동일한 명령문에서 다양한 값 또는 표현식을 검사하십시오. 이 검사는 다양한 조합 연산자 ( && and, || or, ^^ xor)를 사용할 수 있습니다. 이 작업을 수행하면 GameMaker Studio 2가 한 번에 하나씩 각각을 평가하고 평가 방법에 따라 나머지는 건너 뛸 수 있습니다. 예를 들면 다음과 같습니다.

if (keyboard_check_pressed(vk_enter)) && (instance_exists(obj_Player))
    {
    go = false;
    alarm[0] = room_speed;
    }

여기에서 우리는 검사 것, 그래서 && "와"연산자를 사용하여 확인하는 경우 true로 평가하고, 경우 첫 번째가 거짓 인 경우 다음 두 번째도 검사하지 않습니다에서 두 조건이 모두. 이를 확인하기 위해 표현을 결합 할 때 그래서, 당신은 성능을 위해 "저렴한"하나는 항상 첫 번째 평가되면 false로 더 epxensive 사람을 평가하지 않도록 처음이다 있는지 확인해야합니다, "단락"코드라고합니다.

조건부 연산 ( 삼항 연산이라고도 함)을 수행 할 수도 있습니다. 이는 기본 if 를 수행하는 "단축"방식입니다. 다음과 같은 구문이 있습니다.

variable = <condition> ? <statement1 (if true)> : <statement2 (if false)>

조건부 연산자 " ? "는 조건식이 true 또는 false로 평가되는지에 따라 다음 두 값 중 하나를 리턴합니다.

var temp_x = (x < (room_width / 2)) ? 32: (room_width - 32);

위의 코드는 "x"의 값을 룸 폭의 절반 값과 비교하여 확인한 다음 더 작 으면 " temp_x "를 32로 설정하고 그렇지 않으면 " temp_x "는 룸 폭 -32 가됩니다. 몇 가지 예가 더 있습니다. 사용 :

draw_text(x, y, "The fee is " + (global.Member ? "$2.00" : "$10.00"));
path_start(((global.level > 10) ? path_hard : path_easy;), 2, path_action_reverse, true);
(--hp <= 0) ? instance_destroy() : score += 10;

조건부 연산을 중첩 할 수는 있지만 각 연산을 괄호로 묶어야합니다. 예를 들면 다음과 같습니다.

var c = a ? "foo" : (b ? "bar" : "whee"); // Correct
var c = a ? "foo" : b ? "bar" : "whee";   // Will cause an error
