변수와 변수 범위

다른 프로그래밍 언어와 마찬가지로 GML 은 변수 를 대부분의 프로그래밍 작업의 기본 단위로 사용 합니다 . 변수는 나중에 (또는 즉시) 사용할 수 있도록 장치 메모리에 정보를 저장하는 데 사용되며 런타임 및 스크립트 함수에서 참조 할 수 있도록 이름이 지정됩니다. GML 의 변수는 실수 (예 : 100, 2.456575, -56 등), 문자열 (예 : "Hello world!"), 정수 (예 : 1, 556)와 같은 다양한 데이터 유형을 저장할 수 있습니다. , -7) 또는 부울 ( true 또는 false ) 및 기타 사항 :

var _num = 126.4545;
var _str = "Hello World";
new_num = _num * 100;
global.my_string = _str + " I said";

변수를 사용하여 함수에서 반환 된 값을 유지할 수도 있습니다. 예를 들면 다음과 같습니다.

var _id = instance_nearest(x, y, obj_Tree);
root = sqrt(1000);
global.str = string_upper("Hello World");

GML 에서 변수를 작성할 때 이름은 문자 또는 밑줄 기호 "_"로 시작해야하며 문자, 숫자 및 밑줄 기호 '_'는 최대 길이가 64 자일 수 있습니다. 따라서 유효한 변수는 fish , foo_bar , num1 또는 _str 이며 유효하지 않은 변수는 6fish , foo bar 또는 * num 입니다.

이제 많은 프로그래밍 언어에서 변수 "할당"을 작성해야 사용할 수 있습니다. 이것은 기본적으로 컴퓨터에 변수에 사용할 이름을 말하고 초기 값을 할당한다는 것을 의미합니다. 그런 다음 변수를 메모리에 저장하여 값을 저장하거나 그에 대한 작업을 수행합니다. 변수를 할당하는 형식은 다음과 같습니다.

<variable> = <expression>;

식은 단순한 값일 수 있지만 더 복잡 할 수도 있으므로 변수에 값을 할당하는 대신 + =를 사용하여 변수의 현재 값에 값을 추가 할 수도 있습니다. 예를 들면 다음과 같습니다.

a = 100;   // Assigning a simple value
b = 200;
c = 300;
a += b;    // Assigning with operation
a = b + c; // Assigning with expression

참고 : 게임 메이커 언어는 할당에 " : = "를 허용하지만 일반적으로 가장 일반적인 방법은 아닙니다.

<variable> := <expression>;

마찬가지로 -=를 사용하여 빼거나 * =를 사용하여 곱하거나 / =를 사용하여 나누거나 | = , & = 또는 ^ =를 사용하여 비트 연산자를 사용할 수 있습니다. 또한 더하거나 뺄 수 있습니다 하나 를 사용하여 값에서 ++ , - . 자세한 내용은 식 및 연산자 섹션을 참조하십시오.

다음 (또는 변형)을 수행 할 수 없습니다 .

a = b = c = 4;

대신 다음과 같이 수행해야합니다.

a = 4;
b = 4;
c = 4;

위에 표시된 변수 할당은 모두 인스턴스 변수이지만 GameMaker Studio 2로 프로그래밍 할 때 실제로 세 가지 다른 주요 변수 범주가 있으며 각 범주 에는 자체 범위 (작동 영역 또는 도달 범위 로 간주 될 수 있음)가 있습니다. 다양한 종류의 변수와 범위는 다음 페이지에 요약되어 있습니다.

  • 지역 변수
  • 인스턴스 변수
  • 글로벌 변수
  • 상수

또한 GameMaker Language에는 위에서 언급 한 범위 ( local 제외)를 가질 수있는 여러 가지 내장 변수가 있습니다. 이러한 변수는 기본적으로 게임 세계의 오브젝트와 룸의 일부로 포함되므로 특별합니다. 내장 된 전역 변수 중 일부는 위에서 언급 한 섹션에 나열되어 있으며 스프라이트, 룸, 객체 등에 대한 설명서의 다른 부분도 각 경우에 사용 가능한 내장 변수를 간략하게 설명합니다. 이러한 내장 인스턴스 변수의 예는 다음과 같습니다.

sprite_index
path_scale
speed

내장 전역 변수의 예는 다음과 같습니다.

view_xport
GM_version
room

대부분의 내장 변수는 다른 변수와 같이 변경하고 설정할 수 있으며 일부는 배열 일 수도 있습니다. 이미 기본값으로 초기화되므로 일반 변수처럼 변수를 만들도록 설정할 필요는 없습니다.
