지역 변수

로컬 변수는 특정 이벤트 나 함수에 대해서만 생성 한 다음 이벤트 나 함수가 완료되면 무시하는 변수입니다. 사용자 정의 함수에서 작성된 경우 로컬 변수는 함수에서만 사용할 수 있으며 함수가 완료되면 삭제됩니다. 왜 우리가 필요합니까? 변수는 메모리에서 공간을 차지하므로 한 번의 작업이나 함수에 대해서만 사용할 수 있습니다.이 경우 사용되는 짧은 시간 동안 만 메모리에 저장해야합니다. 이를 통해 코드베이스를 깨끗하고 깔끔하게 유지하고 메모리 공간을 실제로 필요한 곳에 최적화합니다. 지역 변수를 선언하려면 다음과 같이 var 연산자를 사용하십시오.

var _i, _num, _str;
_i = 0;
_num = 24.5;
_str = "Hello World";

// or

var _i = 0, _num = 24.5, _str = "Hello World";

// or

var _i = 0;
var _num = 24.5;
var _str = "Hello World";

이러한 방식으로 생성 된 모든 변수는 생성 된 이벤트 (또는 기능)가 끝날 때 "잊어"(즉, 메모리에서 제거)됩니다. 모든 var 선언 변수에 지정한 이름이 코드를 실행하는 객체 내에서 다른 인스턴스 변수와 일치하지 않도록주의하고 이벤트 또는 함수 외부에서 해당 변수에 저장된 값을 사용할 의도가 없는지 확인해야합니다. 당신은 그것을 선언합니다 (위의 예제에서 모든 var 선언 된 변수는 변수 이름 앞에 밑줄 "_"로 정의되었지만 필수는 아니며 코드에서 더 명확하게하기 위해 수행됩니다. 지역 변수와 함께). 즉, 로컬 변수 가 인스턴스 변수와 동일 해야 한다고 생각되는 상황에서 self 키워드 를 사용하면 인스턴스 변수를 개별적으로 식별 할 수 있습니다. 예를 들면 다음과 같습니다.

var hp = 10;
with (obj_Enemy)
{
self.hp -= hp;
}

로컬 변수는 프로그램에서, 특히 반복 횟수 를 계산하기위한 루프에서 또는 반복적으로 반복되지 않는 한 번의 조작에서 값을 여러 번 사용할 때 많이 사용됩니다. 다음은 몇 가지 예입니다.

var _i = 0;
repeat (10)
    {
    inventory[_i] = 0;
    _i += 1;
    }

위의 코드는 " _i "라는 로컬 변수를 만들고 모두 같은 줄에 0으로 설정합니다. 이전 버전의 것을 참고 GameMaker 먼저 지역 변수를 선언했다 다음 그들에게 값을 할당하지만,이 버전에서 당신이 선언 할 수 와 같은 시간에 그들에게 값을 할당합니다. 위의 코드는이 변수를 사용하여 배열 을 초기화합니다. " _i "변수는이 이외의 인스턴스에서 추가 기능에 사용되지 않으므로 범위 내에서 로컬 일 수 있습니다. 여기 또 다른 예가 있습니다 :

var _x, _y;
_x = x - 32 + irandom(64);
_y = y - 32 + irandom(64);
instance_create_layer(_x, _y, "Effects", obj_blood);

여기서는 로컬 변수 " _x "및 " _y "를 사용하여 인스턴스를 만드는 데 사용하는 임의의 임의 좌표를 저장했습니다. 이 예에서는 이러한 변수를 반드시 사용해야하는 것은 아니지만 가독성과 사용 편의성을 위해 사용합니다. 이 코드를 사용하는 것보다 훨씬 명확하고 분명합니다.

instance_create_layer(x - 32 + irandom(64), y - 32 + irandom(64), "Effects", obj_guts);

var로 선언 된 지역 변수에 대한 또 다른 점은 주목할 것입니다 ... 변수를 실행하는 이벤트 나 함수에 고유하기 때문에 다른 인스턴스에서도 코드를 통해 사용할 수 있습니다! 이는 " with () "구문을 사용하여 다른 인스턴스에서 이러한 변수를 사용하여 설정하고 변경할 수 있음을 의미합니다 (자세한 내용은 변수 범위 섹션 참조). 실제 코드 자체는 다음과 같습니다.

var num = instance_number(obj_Enemy);
with (obj_Enemy)
    {
    if num>10 instance_destroy();
    }

위의 코드는 var 선언 변수가 인스턴스 또는 게임 월드가 아닌 포함 된 이벤트 (또는 함수)에 로컬이기 때문에 작동하므로 객체에있는 한 모든 함수에서 사용할 수 있습니다 동일한 코드 블록.
