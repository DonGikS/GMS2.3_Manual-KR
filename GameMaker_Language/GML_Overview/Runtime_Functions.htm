런타임 함수

함수의 일반적인 정의는 다음과 같습니다.

함수에는 입력과 출력이 있으며 출력은 어떻게 든 입력과 관련이 있습니다.

GameMaker Studio 2에서 이것은 두 가지로 번역되지만 둘 다 같은 방식으로 작동합니다.

  • 런타임 기능 - 런타임 기능은 GML ( GameMaker Language)의 일부로 제공되는 기능입니다. 이것들은 언어에 내장되어 있으며 게임에서 일을하는데 사용될 수 있으며 많은 GML 런타임 기능이 있습니다.이 기능은 모두 매뉴얼의 언어 참조 섹션에 설명되어 있습니다.
  • 스크립트 기능 및 방법 - 스크립트 기능과 M은 변수 ethod 당신의 기본 빌딩 블록을 사용하여 직접 만든 것을 함수이다 GameMaker 언어 (GML)를, 이들은 자신의 코드의 일부로 실행 기능을 포함 할 수 있습니다.

위의 두 경우 모두 함수는 함수 이름 형식을 가지며 대괄호 () 사이의 입력 인수 와 쉼표로 구분됩니다 (함수에 입력 인수가없는 경우 대괄호 만 사용됨). 이 페이지는 런타임 기능 (즉, GameMaker 언어를 형성하기 위해 GameMaker에 내장 된 기능)에 중점을 두지 만 아래에 표시된 일반적인 규칙은 모든 기능 유형에 적용됩니다.

다음은 함수 구조의 개요입니다.

<function>(<arg0>, <arg1> ,... <argN>);

일부 함수는 값을 반환하고 expression에 사용될 수있는 반면, 다른 함수는 다음 두 가지 런타임 함수 예제에 설명 된대로 단순히 명령을 실행합니다.

// Destroy the calling instance
// This requires no arguments and returns nothing
instance_destroy();

// Get the distance from the current instance position to the mouse position
// This takes four arguments and returns a real value
dist = point_distance(x, y, mouse_x, mouse_y);

때로는 함수의 반환 값이 대입 에 사용하려는 값일 수도 있지만 대입 의 왼쪽으로 직접 함수를 사용할 수는 없습니다. 예를 들어, 인스턴스의 속도를 설정하는 다음 코드는 오류를 발생시킵니다.

instance_nearest(x, y, obj).speed = 0;

해당 코드 예제에서 표현식의 리턴 값은 정수 (가장 가까운 인스턴스의 고유 ID 값)이므로 이러한 방식으로 사용하고 필요한 인스턴스를 올바르게 처리하려면 괄호로 묶어야합니다 ( 기타 변수 주소 지정 참조) . 자세한 내용은 인스턴스) 를 참조하십시오. 위 코드는 다음과 같이 올바르게 작성됩니다.

(instance_nearest(x, y, obj)).speed = 0;

//or more correctly still

var inst = instance_nearest(x, y, obj);
inst.speed = 0;

매뉴얼의 언어 참조 섹션에는 사용 가능한 모든 런타임 기능이 나열되어 있으며 사용 가능한 방법의 예와 리턴 할 수있는 이벤트 또는 트리거 할 수있는 이벤트 등에 대한 기타 중요한 정보가 있습니다. 자세한 내용 사용자 정의 스크립트 함수 및 메소드에 대한 정보는 스크립트 함수 섹션 및 메소드 변수 섹션을 참조하십시오.
