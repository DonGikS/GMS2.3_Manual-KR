액션 블록 코드 생성

객체에 비헤이비어를 추가하기 위해 Action Toolbox 에서 사용할 수있는 다른 라이브러리의 Action 을 사용하여 코드를 구성 할 수 있습니다. 먼저 드래그 앤 드롭 프로젝트를 새로 만든 다음 새 오브젝트 를 만들어야 합니다 (필요한 경우 오브젝트에도 스프라이트 를 할당 할 수 있음). 새 객체에서는 추가 시작할 수 있습니다 이벤트를 하고 이벤트에 DnDâ "¢ 코드 작업을 추가 할 수 있습니다.

새 이벤트를 추가하면 지정된 이벤트에 대한 탭이있는 "코드"창이 열리고 (위 이미지 참조) 이제 오른쪽의 도구 상자에서 원하는 작업을 작업 블록 창으로 끌어 올 수 있습니다. 코드 창의 이제 어떤 동작도 추가 할 수 있지만 모든 동작이 작동하거나 프로젝트가 컴파일 될 것이라는 의미는 아닙니다. 일부 작업은 작동하기 위해 하나 이상의 변수 가 필요하지만 Draw 작업과 같은 다른 작업은 특정 이벤트에서 사용되는 경우에만 작동합니다. 어떤 것을 사용해야하는지 어떻게 알 수 있습니까? 글쎄, 일반적으로 논리를 사용하는 것의 문제입니다. 액션에 변수가 필요하고 아직 정의하지 않은 경우 변수를 만들기 위해 액션을 추가 할 때까지는 사용하지 않아야합니다.

도구 상자에서 작업을 기본 작업 블록 작업 공간으로 드래그하면 작업을 채우고 변경할 수있는 사용 가능한 인수 (매개 변수)가 표시되도록 확장됩니다. 아래 이미지에서 Toolbox 공통 라이브러리 에서 변수 할당 작업을 작업 블록 작업 공간으로 드래그했습니다.

새 작업이 코드 창의 왼쪽에 축약 형으로 표시되어 있음을 알 수 있습니다. 조치 개요라고하는이 조치 목록을 클릭하여 편집 할 해당 조치로 빠르게 이동할 수 있습니다. 생성 할 DnD ™ 코드의 흐름을 표시하기 위해 각각의 새로운 동작을 이전 동작에 "연쇄"하여 필요한 경우 이벤트에 동작을 계속 추가 할 수 있습니다. 추가 조치를 놓을 수있는 영역은 초기 조치 아래에 강조 표시되며 사용중인 조치에 따라 체인에서 추가 할 수있는 위치를 표시하기 위해 다른 영역이 강조 표시됩니다.

일부 작업은 메인 플로우에서 멀리 떨어진 별도의 체인에 코드를 배치합니다. If Variable 과 같은 것들은 메인 체인을 계속하기 전에 올바른 조건이 충족되면 발생하는 하위 액션 체인 을 만듭니다.

이와 같이 사이드 체인 블록을 가질 수있는 동작을 사용하는 경우 동작에는 추가 동작을 삭제하기 위해 두 영역이 강조 표시됩니다.

자동 완성 자동 완성

끌어서 놓기로 작업 할 때 변수와 표현식을 조치의 다른 입력 필드에 추가해야합니다. 그러나 이렇게하면 자동 완성 창이 나타나서 다음과 같은 도움이됩니다.

이 팝업 창에는 내장 된 모든 GML (GameMaker Language) 변수, 상수 및 함수는 물론 입력 한 문자의 초기 문자가 포함 된 모든 리소스가 나열됩니다. 직접 입력하지 않고도 조치에 필요한 자원 또는 변수를 빠르게 찾을 수 있습니다. 예를 들어, 모든 객실에 " rm_ "접두사가 붙은 경우 해당 객실을 입력하고 잠시 기다리면 모든 자산이 " rm_ "로 시작하는 자동 완성 창이 표시됩니다. 자동 완성 창에 표시되는 모든 내장 변수는 대부분의 GML 함수와 마찬가지로 작업에서 변수 또는 표현식이 필요한 모든 위치에서 사용할 수 있습니다.

 

액션 확장 액션 확장

Sometimes when using an action you will see a small plus icon 옆으로. 즉, 작업을 확장하여 추가 작업을 수행하거나 추가 인수를 취할 수 있습니다. 예를 들어, 임시 변수 선언 조치를 보면 다음 아이콘이 있음을 알 수 있습니다.

아이콘을 클릭하면 작업이 확장 되어 더 많은 변수를 선언 할 수있어 여러 변수를 동시에 더 쉽고 빠르게 정의 할 수 있습니다.

이 아이콘은 선택 조치와 같이 선택적 인수가 필요한 조치에 사용될 수 있으며,이를 통해 다양한 다른 값을 추가하여 리턴 할 수 있습니다.
