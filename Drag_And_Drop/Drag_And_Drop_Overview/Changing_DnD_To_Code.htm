DnD ™ 코드로 변경

Another feature of DnD™ is that you can convert it into GML (GameMaker Language) code, and in this way see what exactly is going on behind the scenes when you use actions. Before actually changing the actions into GML however, you can first Preview them by using the right mouse button 메뉴 옵션 라이브 미리보기 :

새 창이 열립니다 :

DnD ™ 액션을 추가, 변경 또는 제거하면 GML 라이브 미리보기가 업데이트되어 "장면 뒤에"생성되는 실제 코드를 표시합니다. 실시간 미리보기의 코드는 직접 편집 할 수 없지만 섹션을 선택하여 GML 스크립트 또는 코드 작업 (예 : 붙여 넣기)에 붙여 넣기 위해 복사 할 수 있습니다.

If you decide that you want to convert the DnD™ into code after seeing the preview, then this can be done again by clicking the right mouse button 조치가있는 이벤트 작업 공간에서 GML로 변환을 선택 하십시오 .

이 작업을 처음 수행하면 액션을 코드로 변환 할 수 있지만 나중에 다시 개별 액션으로 변환 할 수 없기 때문에 이것이 단방향 변환이라는 경고 메시지가 표시됩니다. 여기에서 "확인"을 클릭하면 변환이 수행됩니다.

결과 코드는 {} 을 사용하여 개별 동작을 구분하며 코드 내의 어떤 함수 또는 변수 선언과 어떤 동작이 관련되는지 명확하게 확인할 수 있습니다. DnD ™가 더 복잡한 경우 코드도 복잡하지만 동일한 일반 규칙이 적용되며 코드는 DnD ™를 작성한 것과 정확히 동일한 구조가됩니다. 때로는 코드에 사용되는 특정 값을 저장하기 위해 추가 로컬 (임시) 변수가 추가됩니다. 예를 들면 다음과 같습니다.

이 코드가 될 것입니다 :

여기서 코드는 우선 로컬 (임시) 변수를 작성하고이를 false로 설정 한 다음 인스턴스가 존재하는지 확인하고 로컬 변수를 함수 호출의 리턴으로 설정합니다. 그런 다음 로컬 변수를 검사하여 해당 변수가 참 인지 거짓인지 , 그리고 그것이 참 이면 나머지 코드가 실행되는지 확인합니다.

When learning to program using Drag and Drop, this can be an important tool in moving on to using GML at a later stage, but it's by no means obligatory and you can still make great games using DnD™! It's also worth noting that while the conversion process is one way, after converting actions to code, you can go back to using DnD™ again by using the right click 코드 편집기에서 " DnD로 변환 "을 선택 하십시오 . 이렇게하면 이전에 생성 된 코드가 코드 실행 작업 내에 배치되고 이전과 같이 DnD ™를 계속 사용할 수 있습니다.
