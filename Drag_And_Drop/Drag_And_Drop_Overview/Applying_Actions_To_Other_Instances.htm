다른 인스턴스에 작업 적용

DnD ™ 라이브러리의 대부분의 작업에는 다른 방법으로 작업을 적용 할 수있는 옵션이 있습니다. 이를 조치 범위 설정이라고하며 다음 중 하나 일 수 있습니다.

기본적으로 GameMaker Studio 2에 알리는 것은 어떤 인스턴스가 작업을 실행해야하는지입니다. 기본 작업 범위는 self 입니다. 즉, 객체가 실에서 인스턴스로 생성되면 해당 인스턴스가 작업 코드를 실행합니다. 그러나 이것이 항상 원하는 것은 아니며 일부 조치가 방의 다른 인스턴스 또는 모든 인스턴스에 영향을 줄 수 있습니다. 여기에서 활동 범위를 변경해야합니다.

조치 수행에 대한 다양한 범위가 아래에 나열되어 있지만, 이러한 방식으로 조치의 범위를 변경 하면 체인의 후속 조치가 아니라 해당 조치에만 새 범위가 적용됩니다. 조치 범위 변경을 여러 연쇄 조치에 적용하려면 먼저 적용 대상 ... 조치를 사용 하십시오 .

 

자기 자기

이것은 작업의 기본 범위이며 코드를 실행하는 인스턴스에서만 작업을 호출해야한다고 간단히 나타냅니다.

 

기타 기타

다른 범위에는 두 가지 주요 기능이 있으며 반환하는 값은 사용 위치 및 방법에 따라 다릅니다. Collision Event 에서 other 는 고유 인스턴스 ID 값 (모든 객체의 개별 인스턴스를 구별하는 데 사용되는 고유 값)을 반환하므로 예를 들어 "bullet"인스턴스를 생성하고 " player "인스턴스에서 다른 범위를 사용하여"player "개체에서 적중 지점을 제거한 다음 자체 범위로 돌아가"bullet "인스턴스를 파괴합니다.

충돌 이벤트 이외의 다른 설정은 범위가 지정된 동작 블록 내에서 호출되지 않는 한 아무도 설정되지 않은 것처럼 작동합니다. 이는 조치 그룹의 범위를 특정 오브젝트로 변경하면 해당 조치가 호출되는 동안 다른 범위는 처음에 조치 그룹을 호출 한 인스턴스의 인스턴스 ID를 리턴한다는 의미입니다. 예를 들어, 적용 대상 ... 조치 를 실행 한 후 다음 코드 블록에서 적용 범위에 속하는 인스턴스가 아닌 전체 이벤트 블록을 실행하는 인스턴스에 대한 조치를 수행하도록 범위를 기타 로 설정할 수 있습니다. 코드 블록. 아래 이미지는 예입니다.

모든 모든

모두를위한 액션의 범위를 정할 때, GameMaker Studio 2에게 현재 룸 내 모든 활성 인스턴스마다 해당 블록을 실행하라고 지시 합니다 . 예를 들어 객체 인스턴스 삭제 작업을 만들고 범위를 모두로 설정하면 룸의 모든 인스턴스가 생성 된 객체에 관계없이 사라집니다.

 

객체 객체

범위에 따라 개체에 작업을 제공 할 수도 있습니다. 이것이 의미하는 것은 주어진 객체의 모든 인스턴스가 호출되는 동시에 해당 액션을 실행한다는 것입니다. 예를 들어, 방에 100 개의 적 인스턴스가 있고 특정 지점을 향하도록 모두 설정하려는 경우. Set Point Direction을 호출하고 작업 범위를 "obj_Enemy"개체로 설정하면 해당 개체의 모든 인스턴스가 방향을 변경합니다.
