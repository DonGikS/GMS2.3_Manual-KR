마우스 오른쪽 버튼 메뉴 옵션

When working on your DnD™ code, you can click the right mouse button 조치, 작업 공간 또는 왼쪽에있는 단축 목록의 텍스트에서 오른쪽 마우스 단추 메뉴를여십시오.

이 메뉴에는 다음과 같은 옵션이 있습니다.

  • 잘라 내기 / 복사 / 붙여 넣기 : 작업 공간에서 선택한 작업을 잘라내거나 복사하거나 붙여 넣을 수 있습니다. 사용할 수 있습니다 + 행동 밖에서 드래그하거나 사용 / + 개별 조치에 대해 둘 이상을 선택하고 모든 조치에 명령을 적용하십시오.
  • 주석 / 주석 제거 :이 옵션을 사용하면 DnD ™ 코드 블록에서 주석을 추가하고 제거 할 수 있습니다. 주석을 선택하면 선택한 DnD ™ 블록의 오른쪽에 텍스트 편집기가 열리고, 무언가를 변경하라는 알림 또는 튜토리얼에 대한 유용한 주석이든, 심지어는 필요한 텍스트를 추가 할 수 있습니다. 팀원을위한 지침 : 주석을 추가 한 후에는 주석을 클릭하고 끌어서 수직 및 수평으로 배치 할 수 있습니다 (상위 노드의 오른쪽에만). DnD ™ 노드를 축소하면 주석이 축소되고 주석이 요약됩니다. 내용 (이 경우 재배치 할 수 없음). 주석이 더 이상 필요하지 않으면 주석의 모든 텍스트를 삭제하거나 메뉴에서 주석 제거 옵션을 사용할 수 있습니다. 노드를 복사 할 때 주석과 함께 주석이 복사되고 노드를 DnD ™에서 GML로 변환하면 주석이 보존되어 이미지에 표시된대로 생성 된 GML에 추가됩니다. 이하: 위에서 언급 한 것처럼 주석은 단일 노드에만 적용 할 수 있지만 간단히 줄 바꿈을 추가하여 확장하여 여러 노드를 포함하도록 주석을 확장 할 수 있습니다.이 경우 주석이 다른 위치와 겹치지 않도록 주석을 재배치해야 할 수도 있습니다. 노드 : 설명이 추가되면 RMB 메뉴에서 " 설명 "옵션을 다시 선택하거나 두 번 클릭 할 수 있습니다 편집 할 수 있습니다.
  • 비활성화 / 활성화 :이 명령을 사용하여 선택한 작업을 비활성화하거나 활성화 할 수 있습니다. 액션이 비활성화 되어도 작업 영역에는 여전히 표시되지만 게임을 컴파일 할 때는 실행되지 않습니다. 이러한 방식으로 테스트 및 디버깅을위한 조치를 추가 / 제거 할 수 있습니다. 아래 이미지는 동작이 비활성화 된 일부 DnD ™ 코드를 보여줍니다. "점수면"및 "다음 방으로 이동"동작이 비활성화되어 있습니다 (회색으로 표시되어 시각적으로 표시됨). 따라서 게임을 테스트하거나 컴파일 할 때 건너 뜁니다.
  • 중단 점 토글 :이 명령은 선택한 동작에 대해 중단 점을 켜거나 끕니다. 중단 점은 게임이 디버그 모드에서 실행될 때 디버거 가 중지해야하는 DnD ™ 코드의 한 지점입니다. 아래 이미지에 표시된대로 중단 점이 지정된 조치가 강조 표시됩니다 ( "건강 상태"조치가 중단 점으로 설정 됨).
  • 모든 중단 점 삭제 :이 명령을 사용하면 DnD ™ 코드에서 모든 중단 점을 제거 할 수 있습니다.
  • 모두 축소 / 모두 확장 : DnD ™ 작업을 축소하거나 확장 할 수 있습니다. 축소 된 작업은 단순히 작업 이름 만 표시하지만 내부 값은 표시하지 않으므로 작업 공간에 사용 가능한 공간이 최대화됩니다. 예를 들면 다음과 같습니다. 보시다시피, 작업은 훨씬 적은 공간을 차지하며 오른쪽에 제공된 간단한 설명에서 작업이 수행하는 작업을 계속 볼 수 있습니다. "모두 확장"옵션을 선택하여 작업 공간의 모든 노드에 대해이를 취소 할 수 있습니다.
  • 레이아웃 재설정 : DnD ™ 작업 공간을 재설정하여 확대 / 축소 수준을 100 %로 설정하고 현재 선택한 작업에 초점을 맞추고 축소 된 모든 작업을 확대합니다.
  • 실시간 미리보기 :이 옵션을 선택하면 DnD ™ to GML (GameMaker Language) 코드 미리보기 창이 활성화 또는 비활성화됩니다. 자세한 내용은 Change DnD ™ To Code 페이지에 자세히 설명되어 있습니다.
  • GML 로 변환 : 현재 이벤트의 모든 액션을 GML (GameMaker Language) 코드로 변환합니다. GML을 생성 된 DnD ™ 액션으로 다시 변환 할 수 없으므로이 점에 매우주의하십시오. 코드에서 이벤트를 DnD ™ 이벤트로 다시 변환 할 수 있지만 모든 코드는 단순히 코드 실행 작업으로 래핑됩니다. 자세한 내용은 Change DnD ™ To Code 페이지에 자세히 설명되어 있습니다.
  • 창 :이 섹션은 다음과 같은 하위 메뉴를 엽니 다. 사용 가능한 메뉴 항목은 다음과 같습니다. Go To Object : 작업 공간을 이벤트를 호출하는 객체로 이동시킵니다.
  • 상속 된 이벤트 열기 :이 옵션은 편집중인 오브젝트가 상위 오브젝트의 하위 인 경우에만 사용 가능하며 상위 오브젝트의 편집중인 이벤트를 열고 작업 공간을 해당 오브젝트로 이동합니다.
  • 단일 창 / 2 열 / 3 열 :이 옵션은 현재 worskspace를 1, 2 또는 3 개의 다른 "창"으로 분할하는 데 사용할 수 있습니다. 각 창은 스크립트 나 이벤트를 끌어서 놓을 수있는 별도의 작업 공간으로, 한 번에 2 개 이상의 DnD ™ 코드 블록을 편집하고 비교할 수 있습니다. 아래 이미지는 자신의 작업 공간 창에 표시된 두 개의 스크립트를 한 창에 표시합니다.

다음 두 옵션은 Rn 메뉴가 DnD ™ 노드에서 열린 경우에만 사용할 수 있습니다.

  • 이동 : 임의의 노드에서이 옵션을 선택하면 작업 공간이 이동하여 해당 노드를 중앙으로 울립니다.
  • 도움말 표시 ... :이 옵션을 선택하면 선택한 노드와 관련된 페이지의 새 작업 공간에서 매뉴얼이 열립니다. 다음을 사용하여 동일한 결과를 얻을 수도 있습니다 노드에서.
