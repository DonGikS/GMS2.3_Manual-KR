가상 키보드 표시

이 동작은 게임을 실행하는 장치에서 가상 키보드를 표시하는 데 사용할 수 있습니다. 이 조치를 호출하면 처음 세 개의 인수 각각에 대해 다음 상수 중 하나를 제공해야합니다.

  • 키보드 유형 :이 상수는 가상 키보드에서 사용할 수있는 키 세트를 설정하는 데 사용됩니다. 사용 가능한 키보드 유형은 다음과 같습니다. 키보드 타입 기술 kbv_type_default 현재 시스템의 기본 키보드 유형입니다. kbv_type_ascii ASCII 전용 키보드 kbv_type_url URL 입력에 최적화 된 일반 키보드. 일반적으로 ".com"또는 다른 도메인 키와 "/"및 "."기능이 있습니다. 키. kbv_type_email 이메일 입력에 최적화 된 일반 키보드. 일반적으로 "@"및 "." 문자. kbv_type_numbers 숫자 전용 키보드이며 일반적으로 숫자 패드로 표시됩니다. kbv_type_phone 전화 패드 키보드. 일반적으로 "*"및 "#"키는 숫자 만 사용합니다. kbv_type_phone_name 전화 번호와 이름을 모두 입력하는 데 최적화 된 키보드입니다. 일반적으로 ASCII 키보드와 비슷하지만 특수 문자 선택이 제한되어 있습니다.
  • 리턴 타입 :이 상수는 가상 키보드의 리턴 / 액션 키에 표시되는 것을 설정하는 데 사용됩니다. 사용 가능한 반품 유형은 다음과 같습니다. 반품 유형 기술 kbv_returnkey_default 현재 시스템의 기본 리턴 키 제목입니다. kbv_returnkey_go 리턴 키 제목을 "Go"로 설정합니다. kbv_returnkey_google 리턴 키 제목을 "Google"또는 경우에 따라 일반 검색 아이콘으로 설정합니다. kbv_returnkey_join 리턴 키 제목을 "Go"로 설정합니다. kbv_returnkey_next 리턴 키 제목을 "Next"로 설정합니다. kbv_returnkey_route 리턴 키 제목을 "Route"로 설정합니다. kbv_returnkey_search 리턴 키 제목을 "검색"으로 설정하거나 경우에 따라 일반 검색 아이콘으로 설정합니다. kbv_returnkey_send 리턴 키 제목을 "보내기"로 설정합니다. kbv_returnkey_yahoo 리턴 키 제목을 "Yahoo"로 설정하거나 경우에 따라 일반 검색 아이콘으로 설정합니다. kbv_returnkey_done 리턴 키 제목을 "완료"로 설정합니다. kbv_returnkey_continue 리턴 키 제목을 "계속"으로 설정하십시오. kbv_returnkey_emergency 리턴 키 제목을 "Emergency Call"로 설정합니다.
  • 자동 대문자 유형 :이 상수는 가상 키보드를 통해 입력 된 단어가 자동 대문자 로 표시되는 방법을 결정하는 데 사용됩니다. 사용 가능한 자동 자본화 유형은 다음과 같습니다. 자동 자본화 유형 기술 kbv_autocapitalize_none 자동 자본화가 비활성화되었습니다. kbv_autocapitalize_words 단어는 자동 대문자로 표시됩니다. kbv_autocapitalize_sentences 문장은 자동 대문자로 표시됩니다. kbv_autocapitalize_characters 모든 문자는 대문자로 표시됩니다.

마지막 주장은 예측 텍스트를 활성화 / 비활성화하는 것이며,이를 허용하려면 true 로 설정하고 그렇지 않으면 false 로 설정하지만 허용된다고해서 선호도에 따라 그대로 사용된다는 의미는 아닙니다. 장치에서 사용자의 예측 텍스트 모드 인 경우 가상 키보드는 일반적인 GameMaker Studio 2 키 누르기 이벤트를 생성하지 않습니다. 대신, 마지막으로 누른 문자 및 키보드 문자열 변수 만 업데이트합니다. 이는 키 누름 감지에 사용 된 내부 텍스트 필드의 변경이 실제 가상 키보드 키에서 발생했는지 또는 텍스트 제안에서 발생했는지 감지 할 수 없기 때문입니다. 이 경우 모든 종류의 원시 키 입력을 읽는 대신 키보드 키보드 입력을 읽으려고합니다.

사용자는 플랫폼 OS에 따라 서로 다른 기능을 가진 서로 다른 키보드를 얻을 수 있으며 대상마다 다음과 같은주의 사항이 있습니다.

  • UWP : UWP 구현은 return_key_type , autocapitalization_type 또는 predictive_text_enabled를 지원하지 않습니다. 이 파라미터를 함수에 전달하면 오류가 발생하지 않으며 키보드에는 아무런 영향을 미치지 않습니다.
  • Android / Amazon Fire : Android에서 키보드 유형 kbv_type_phone_name 은 지원되지 않으며 대신 ASCII 키보드를 표시하며 리턴 키 유형은 kbv_returnkey_go , kbv_returnkey_search , kbv_returnkey_next , kbv_returnkey_send 또는 kbv_returnkey_done 만 가능 합니다. 해당 플랫폼에서 다른 리턴 키 유형을 사용하는 경우 시스템은 기본적으로 아래에 나열된 대체 리턴 키를 사용합니다. 사용할 수없는 리턴 키 교체 키 kbv_returnkey_google kbv_returnkey_yahoo kbv_returnkey_search kbv_returnkey_join kbv_returnkey_route kbv_returnkey_emergency kbv_returnkey_go kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV / FireTV : 사용자 정의 리턴 키 유형은 ASCII 키보드에서 지원 되지 않습니다 . 기본 리턴 키는 항상 표시됩니다.
  • tvOS : 예측 텍스트 및 자동 대문자 화는 tvOS에서 지원되지 않으며, 시스템 이벤트 및 기능 keyboard_virtual_height () 에서 반환되는 키보드 높이는 전체 화면에 걸쳐 있고 키보드 사각형 치수 함수로 인해 기본적으로 화면 높이로 설정됩니다. tvOS에서 비활성화됩니다. 또한 가상 키보드를 열지 않으면 물리적 (블루투스) 키보드 이벤트가 브로드 캐스트되지 않습니다.

이 함수를 호출하면 async_load DS 맵 이 다음 키 / 값 쌍으로 채워지는 시스템 비동기 이벤트 가 생성됩니다.

  • " event_type "-가상 키보드 조치에 의해 트리거 될 때 " 가상 키보드 상태 "가됩니다.
  • " screen_height "-가상 키보드의 높이 (픽셀). 키보드가 보이지 않으면 0이됩니다.
  • " keyboard_status "- 키보드 의 현재 상태로, 다음 문자열 중 하나로 반환됩니다. "숨김"
  • "숨겨진"
  • "전시"
  • "명백한"

인수 :

 

논의 기술
keyboard_type 가상 키보드에서 사용 가능한 키 세트를 결정합니다.
return_key_type 가상 키보드의 리턴 / 액션 키에 표시되는 내용을 결정합니다.
자동 자본 유형 가상 키보드를 통해 입력 한 단어가 자동으로 표시되는 방법 / 경우를 결정합니다.
predictive_text_enabled 예측 텍스트 입력을 활성화 / 비활성화하려면 true / false 로 설정하십시오.
