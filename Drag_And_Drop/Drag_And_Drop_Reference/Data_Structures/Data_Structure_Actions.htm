데이터 구조 액션 라이브러리

게임에서는 종종 정보를 정확하고 순서대로 저장해야합니다. 예를 들어, 개인이 소지 한 품목 목록을 저장해야하거나 여전히 방문해야 할 장소의 그리드를 저장하고자 할 수 있습니다. 이제 여러 변수를 사용할 수 있지만 데이터 정렬 또는 특정 항목 검색과 같은 더 복잡한 작업을 수행하려면 실행 속도가 느리고 실행하기 어려울 수있는 많은 작업 블록 코드를 작성해야합니다. 디버그하고 작성하는 진짜 고통!

이를 해결하기 위해 GameMaker Studio 2에는 전문 기능을 통해 액세스 할 수있는 여러 내장 데이터 구조가 있습니다. 끌어서 놓기를 통해 사용할 수있는 4 가지 유형의 데이터 구조 ( GML 을 사용하는 경우 6 개가 있음)가 있으며, 각각 저장하려는 정보 유형 및 조작 방법에 따라 고유 한 장단점이 있습니다. 나중에-네 가지 유형은 다음과 같습니다.

기울기 목록은 정보가 추가 될 때 순차적으로 정보를 저장하는 데이터 구조입니다. 목록 색인은 0에서 시작하여 목록 끝에 추가 된 각 항목에 대해 올라가지 만 마지막 색인 이외의 위치에 데이터를 삽입 할 수도 있습니다. 스택 데이터 구조 (아래 설명)와 달리 언제든지 목록의 모든 인덱스에서 데이터에 액세스 할 수 있습니다.
지도 지도 데이터 구조는 키 와 값 쌍 을 저장할 수 있으므로 매우 유용합니다 . 예를 들어, 게임의 캐릭터는 다양한 아이템 (키)을 가질 수 있으며, 각 아이템마다 10 개의 체력 물약, 5 마나 물약을 가질 수있는 RPG와 같이 여러 아이템 (값)을 가질 수 있습니다. 100 골드 지도는 이러한 쌍을 모두 한 곳에 모으고지도에 쌍을 추가하고 일부 간단한 기능을 사용하여 특정 키에 해당하는 값을 검색 할 수 있습니다. 지도는 (인식 할 수있는) 방식으로 정렬되어 있지 않습니다. 즉, 특정 키를 찾으려면 전체를 반복해야 할 수도 있습니다 (매우 느림). 또한 동일한 두 개의 키를 보유 할 수있는 방법도 없습니다. 하나의 키에 두 개의 값을 지정할 수 있습니다.
그리드 격자는 너비와 높이를 정의하는 2 차원 값 배열입니다. 이렇게하면 그리드 내 총 그리드 "셀"수 (너비와 높이를 곱한 값)가 설정되고이 셀을 사용하여 다른 데이터 값을 보유 할 수 있습니다. 이는 허용되는 데이터 유형 중 하나 일 수 있습니다. 이 유형의 구조를 사용하면 x (열) 및 y (행) 위치를 지정하여 그리드의 셀 값을 설정하고 검색 할 수 있습니다 (행과 열 모두 0에서 시작하여 그리드 너비-1까지 올라갑니다) 그리드 높이 -1). 그리드 데이터 구조에 접근 할 때는 항상 셀 위치에 정수 값을 사용해야하며, 그렇지 않으면 정수가 아닌 모든 인덱스는 GameMaker Studio 2에 의해 바닥이됩니다 . 그것이 당신이 필요하지 않은 경우, 확인하려는 색인을 전달하기 전에 미리 반올림을 수행해야합니다.
스택 스택 데이터 구조는 소위 후입 선출 (LIFO) 구조입니다. 스택에서 값을 푸시하고 스택에서 값을 팝하여 다시 제거 할 수 있으며 스택에서 가장 최근에 푸시 된 값이 스택에서 가장 먼저 팝됩니다 (각각의 코인 스택을 생각하십시오) 추가 한 동전은 먼저 나머지 동전에 닿기 전에 먼저 제거해야합니다. 스택은 종종 처리 할 인터럽트가 있거나 재귀 함수가 있거나 심지어 게임의 기초 AI를 구성 할 때 사용됩니다.

 

기본적으로 모든 데이터 구조는 동일한 방식으로 작동합니다. 데이터 구조를 작성하고 색인 값을 변수에 저장합니다. 그런 다음이 색인을 사용하여 조작을 수행 할 수있는 모든 추가 함수 호출에서 데이터 구조를 참조하십시오. 마지막으로 완료되면 데이터 구조를 다시 파괴하여 메모리에서 제거합니다. 필요한만큼 많은 구조를 동시에 사용할 수 있으며 모든 구조는 문자열과 실제 값을 모두 저장할 수 있습니다.

참고 : 모든 동적 리소스와 마찬가지로 데이터 구조는 메모리를 차지하므로 더 이상 메모리 누수를 방지하기 위해 더 이상 필요하지 않은 경우 항상 파괴되어 게임 속도가 느려지고 게임이 중단됩니다.
