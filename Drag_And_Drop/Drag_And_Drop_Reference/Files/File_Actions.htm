파일 액션 라이브러리

파일 작업 은 두 가지 파일 형식 (버퍼 파일 및 Ini 파일)을 처리하며 기존 파일의 이름을 바꾸거나 복사하기위한 일반적인 파일 작업도 있습니다.

버퍼 파일 은 이전에 작성한 버퍼에서 데이터를 저장하여 작성 되며이 데이터는 나중에 언제라도 버퍼에 다시로드 될 수 있습니다. 버퍼에 대한 자세한 정보는 Buffer Actions 에서 확인할 수 있습니다.

Ini 파일 은 대부분의 플랫폼과 호환되는 작고 가벼운 파일입니다. 인터페이스 환경 설정, 로컬 높은 점수, 레벨 데이터 등과 같은 작은 정보를 저장하는 데 이상적이며 사용하기 매우 쉽습니다. 이러한 작업을 사용하기 위해 이전에 Ini 파일을 만들 필요는 없었으며 존재하지 않는 Ini 파일에서 읽을 수 있으며 기본 반환 값 (지정한 값) 만 가져 오지만 다음에서 만들 것을 권장합니다. 조치를 사용하기 전에 열고 수정하기위한 "기본"ini 파일. 이 기본 ini 파일은 파일을 닫으면 디스크에 기록되거나 자산 브라우저의 포함 된 파일 에 파일 을 포함 시킬 수 있으므로 Open Ini File 작업을 호출 한 다음 Ini File 닫기 를 호출하여 만들 수 있습니다. 자산 브라우저에 포함 된 파일을 기본 Ini로 사용 하는 경우 파일 시스템 작동 방식 에 대한 설명서 섹션도 읽어야 합니다 .

참고 : 현지화가있는 게임의 경우 외부 파일에서 악센트 부호가있는 문자를 읽을 수 있어야합니다. 즉, ini 파일을 UTF8 형식으로 먼저 만든 다음 포함 된 파일로 GameMaker Studio 2에 파일을 추가하여 게임 실행시 내보내고 ini에서 생성 한 기본 ANSI 형식 ini 파일 대신 사용해야합니다. 이전에 공급 된 것이 없을 때 작동합니다. 이러한 방식으로 모든 악센트 및 비 로마자 문자를 유지하면서 정확하게 읽고 쓸 수 있습니다.
