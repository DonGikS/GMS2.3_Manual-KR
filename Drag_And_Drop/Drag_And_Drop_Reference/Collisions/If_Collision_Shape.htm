충돌 모양 인 경우

이 동작은 주어진 모양 영역 내에서 또는 선을 따라 주어진 객체의 인스턴스와 충돌이 있는지 확인하고 확인하는 데 사용됩니다. 셰이프 (직사각형, 타원 또는 선), 확인할 객체를 제공 한 다음 사용중인 셰이프의 왼쪽 상단 및 오른쪽 하단 모서리에 대한 좌표를 제공하거나 라인의 경우 시작 및 시작 최종 위치. 이들은 게임 룸 내에서 절대 위치이거나 작업을 수행하는 인스턴스와 관련된 위치 일 수 있습니다. 아래 이미지에서 어떻게 작동하는지 확인할 수 있습니다.

당신이 볼 수 있듯이, 결과는 당신이 찾고있는 개체의 인스턴스 즉, 검사되는 인스턴스의 충돌 마스크에 따라 달라집니다 해야한다 sprite_index에 할당 된 스프라이트, 또는 mask_index에 할당 된 스프라이트가 있습니다. 그렇지 않으면 스프라이트 그리기 여부에 관계없이 이러한 기능에 의해 감지되지 않습니다. 타원 또는 직사각형 모양에 대해 설정된 경계 내에 완전히 속하는 인스턴스도 충돌을 반환한다는 점은 주목할 가치가 있습니다. 충돌이 감지되면 작업이 true로 평가합니다, 또는 그렇지 않은 경우는 false, 당신은 또한 확인할 수 있지만 하지 가 있는지 확인하는 플래그를 하지 주어진 모양의 영역과 충돌하고, 작업이 만 참으로 평가하는 경우 어떤 충돌 그렇지 않으면 false로 평가됩니다. 충돌 검사에서 호출 인스턴스를 제외하도록 자체 제외 옵션을 설정할 수도 있습니다.

다음으로 Return List 옵션을 체크 / 체크 해제 할 수 있습니다. 이를 확인하면 조치가 목록을 작성하고 채 웁니다. 모든 인스턴스가 모양과 충돌하는 데이터 구조를 사용하고 대상 변수를 DS 목록 ID로 설정하십시오. 이 옵션을 선택하지 않으면 충돌이 감지되지 않거나 충돌에있는 인스턴스 중 하나 의 고유 ID 값만 대상 변수를 0보다 작은 값으로 설정합니다 (여러 인스턴스가 충돌하는 경우에는 목록을 사용하지 않을 때 어떤 인스턴스 ID가 반환되는지 알 수 있습니다.

목록을 반환하도록 선택하고 대상 변수가 임시로 설정 되지 않은 경우 자유 대상 옵션을 선택할 수도 있습니다. true로 설정되면, 대상 변수에 포함 된 목록이 삭제되고 조치를 호출 할 때마다 새 목록이 작성되고 대상 변수로 리턴됩니다. 이런 방식으로 메모리 누수에 대해 걱정하지 않고 동일한 변수를 재사용 할 수 있습니다. 자유 대상 옵션을 선택하지 않으면 조치에서 변수를 다시 사용하기 전에 목록을 제거해야합니다. 이 옵션의 선택 여부에 관계없이 조치를 Return List 로 설정 한 경우 대상 변수는 항상 목록 ID를 포함 하므로 인스턴스가 손상되거나 인스턴스가 해제 될 때 해제되어야합니다. 룸 종료 (예 : 무료 데이터 구조 와 함께 클린업 이벤트를 사용할 수 있음) 동작). 미리 정의 된 변수 또는 표현식을 사용하도록 자유 대상 옵션을 설정할 수 있으므로 변수를 다른 위치 (또는 표현식에 대해 다른 결과를 가짐)를 설정하여 목록 데이터 구조가 해제되는시기를 제어 할 수 있습니다. true / false로 평가됩니다.

마지막으로, 리턴 된 값 또는 목록 ID를 보유하려는 대상 변수를 제공하십시오. 당신이 인으로 반환 된 값 보유 할 대상 변수 플래그 경우 온도 (로컬) 변수를, 다음 작업이 반환 값을 유지하기 위해이 변수를 만들 것입니다 만 이 경우에는 무료 대상 옵션은 아무것도하지 않고, 이벤트가 끝날 때까지 해당 옵션을 선택한 경우 DS 목록을 삭제해야합니다. 대상 변수가 제공되지 않고 리턴 목록 옵션이 선택된 경우 목록이 작성되지 않습니다.

중대한! 충돌은 즉 유효한 충돌 마스크를 가지고 해당 인스턴스에 등록됩니다 그들은 sprite_index에 할당 된 스프라이트, 또는 mask_index에 할당 된 스프라이트가 있습니다. 이 작업에서 확인중인 객체의 인스턴스에 충돌 마스크가없는 경우 해당 시점에 인스턴스가 그린 내용에 관계없이 충돌이 감지되지 않습니다.
