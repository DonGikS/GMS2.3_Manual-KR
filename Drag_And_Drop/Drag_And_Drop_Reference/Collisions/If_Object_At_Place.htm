객체가있는 경우

이 조치는 호출 인스턴스의 충돌 마스크를 기준으로 현재 인스턴스가 방의 한 지점에 배치 된 지정된 오브젝트의 하나 이상의 인스턴스와 충돌이 있는지 확인하는 데 사용됩니다. 충돌이 감지되면 작업은 true로 평가되고 그렇지 않으면 false로 평가 됩니다. 검사 할 개체와 인스턴스를 "배치"할 위치를 지정하여 검사를 수행합니다 (실내의 절대 위치 또는 인스턴스에 상대적인 위치 일 수 있음). 또한 Not 플래그를 검사하여 존재 여부를 확인할 수 있습니다 이다 없는 경우 작업이 만 참으로 평가되는 경우에 지정된 위치에서 충돌, 아니 , 그렇지 않으면 거짓으로 평가합니다, 충돌이 발견되지 않습니다.

객체와 위치를 지정한 후 Return List 옵션을 선택 / 선택 취소 할 수 있습니다. 이를 확인하면 조치가 목록을 작성하고 채 웁니다. 모든 인스턴스가 인스턴스와 충돌하는 데이터 구조를 사용하고 대상 변수를 DS 목록 ID로 설정하십시오. 이 옵션을 선택하지 않으면 충돌이 감지되지 않거나 충돌에있는 인스턴스 중 하나 의 고유 ID 값만 대상 변수를 0보다 작은 값으로 설정합니다 (여러 인스턴스가 충돌하는 경우에는 목록을 사용하지 않을 때 어떤 인스턴스 ID가 반환되는지 알 수 있습니다.

목록을 반환하도록 선택하고 대상 변수가 임시로 설정 되지 않은 경우 자유 대상 옵션을 선택할 수도 있습니다. true로 설정되면, 대상 변수에 포함 된 목록이 삭제되고 조치를 호출 할 때마다 새 목록이 작성되고 대상 변수로 리턴됩니다. 이런 방식으로 메모리 누수에 대해 걱정하지 않고 동일한 변수를 재사용 할 수 있습니다. 자유 대상 옵션을 선택하지 않으면 조치에서 변수를 다시 사용하기 전에 목록을 제거해야합니다. 이 옵션의 선택 여부에 관계없이 조치를 Return List 로 설정 한 경우 대상 변수는 항상 목록 ID를 포함 하므로 인스턴스가 손상되거나 인스턴스가 해제 될 때 해제되어야합니다. 룸 종료 (예 : 무료 데이터 구조 와 함께 클린업 이벤트를 사용할 수 있음) 동작). 미리 정의 된 변수 또는 표현식을 사용하도록 자유 대상 옵션을 설정할 수 있으므로 변수를 다른 위치 (또는 표현식에 대해 다른 결과를 가짐)를 설정하여 목록 데이터 구조가 해제되는시기를 제어 할 수 있습니다. true / false로 평가됩니다.

마지막으로, 리턴 된 값 또는 목록 ID를 보유하려는 대상 변수를 제공하십시오. 당신이 인으로 반환 된 값 보유 할 대상 변수 플래그 경우 온도 (로컬) 변수를, 다음 작업이 반환 값을 유지하기 위해이 변수를 만들 것입니다 만 이 경우에는 무료 대상 옵션은 아무것도하지 않고, 이벤트가 끝날 때까지 해당 옵션을 선택한 경우 DS 목록을 삭제해야합니다. 대상 변수가 제공되지 않고 리턴 목록 옵션이 선택된 경우 목록이 작성되지 않습니다.

중대한! 충돌은 즉 유효한 충돌 마스크를 가지고 해당 인스턴스에 등록됩니다 그들은 sprite_index에 할당 된 스프라이트, 또는 mask_index에 할당 된 스프라이트가 있습니다. 충돌의 인스턴스 중 하나 에 충돌 마스크가없는 경우 해당 시점에 인스턴스가 그리는 내용에 관계없이 충돌이 감지되지 않습니다.
