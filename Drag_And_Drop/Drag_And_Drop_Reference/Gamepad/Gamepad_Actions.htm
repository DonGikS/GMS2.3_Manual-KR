GamepadAction 라이브러리

GameMaker Studio 2에는 연결된 여러 게임 패드에서 아날로그 및 디지털 컨트롤을 모두 감지하는 데 사용할 수있는 여러 가지 전용 작업이 있습니다. 이러한 액션을 위해서는 게임 패드 인덱스 값을 지정해야합니다.이 값은 선택되는 게임 패드를 나타내는 숫자 (0부터 계산)입니다. 게임 패드가 장치에 연결되어 있거나 제거 된 경우 비동기 시스템 이벤트 가 트리거됩니다 (그러나이를 처리하려면 코드를 사용해야 할 수도 있음).

게임 패드 "슬롯"은 Xinput 게임 패드 전용 ( 예 : Xbox360 컨트롤러 및 호환) 슬롯 0-3을 포함하여 0부터 색인됩니다. 그러나 DirectInput 게임 패드 용 슬롯 4-11을 확인할 수도 있습니다. 즉,이 슬롯을 통해 연결된 다른 많은 컨트롤러 모델을 감지 할 수 있습니다.

게임 패드 액션으로 작업 할 때, GameMaker Studio 2가 다음 내장 상수 중 일부 또는 전부에 할당 할 axis , 버튼 및 / 또는 트리거 에서 입력을 가져올 수 있습니다 (일반적으로 "모자"는 비표준에서만 감지됩니다) 컨트롤러) :

 

일정한 기술
gp_face1 상단 버튼 1 (Xbox 360 컨트롤러의 "A" 와 PS 컨트롤러의 크로스 에 매핑 됨)
gp_face2 상단 단추 2 (Xbox 360 컨트롤러의 "B" 와 PS 컨트롤러의 원 에 매핑 됨)
gp_face3 상단 단추 3 (Xbox 360 컨트롤러의 "X" 와 PS 컨트롤러의 사각형 에 매핑 됨)
gp_face4 상단 단추 4 (Xbox 360 컨트롤러의 "Y" 와 PS 컨트롤러의 삼각형 에 매핑 됨)
gp_shoulderl 왼쪽 어깨 버튼
gp_shoulderlb 왼쪽 어깨 방아쇠
gp_shoulderr 오른쪽 어깨 버튼
gp_shoulderrb 오른쪽 어깨 방아쇠
gp_select 선택 버튼 (DS4 컨트롤러의 PS 버튼)
gp_start 시작 버튼 (PS4 컨트롤러의 "옵션"버튼)
gp_stickl 왼쪽 스틱을 눌렀습니다 (버튼으로)
gp_stickr 오른쪽 스틱을 눌렀습니다 (버튼으로)
gp_padu D 패드 업
gp_padd D 패드 다운
gp_padl 왼쪽 D 패드
gp_padr 오른쪽 D 패드
gp_axislh 왼쪽 스틱 수평 축 (아날로그)
gp_axislv 왼쪽 스틱 세로 축 (아날로그)
gp_axisrh 오른쪽 스틱 가로 축 (아날로그)
gp_axisrv 오른쪽 스틱 세로 축 (아날로그)

각 상수가 나타내는 컨트롤러 부분을 정확히 이해하려면 표준 Xinput 게임 패드의 다음 이미지를 참조하십시오.

다음 목록은 플랫폼 간의 현재 호환성을 보여줍니다 (향후 업데이트로 변경 될 수 있음).

  • Windows 는 한 번에 최대 11 개의 연결된 장치를 허용하여 완벽하게 지원됩니다. Windows에서 처음 4 개의 게임 패드 슬롯 (0-3)은 XInput dll을 사용하여 처리되므로 XBox 컨트롤러 만 100 % 호환 될 수 있으며 다른 컨트롤러 유형의 경우 나머지 게임 패드 슬롯 (4- 11).
  • macOS 는 한 번에 최대 4 개의 연결된 장치를 지원하며,이 장치는 Playstation3 또는 Xbox 360 유형 만 가능합니다. Mac 게임 옵션에서 " Mac AppStore 용 빌드 "옵션은 꺼져 있어야합니다. 패드 지원 작동합니다.
  • Ubuntu 는 GamePad 입력도 지원하지만 Ubuntu 리포지토리에서 추가 라이브러리를 설치해야 할 수도 있습니다. 명령 행 터미널을 열고 다음을 입력하면 조이스틱뿐만 아니라 조이스틱에 대한 GUI 지원 기능이 설치됩니다.

sudo apt-get install jstest-gtk
sudo apt-get install joystick.

  • HTML5 게임은 Safari를 제외한 대부분의 주요 브라우저에서 게임 패드를 지원 합니다 .
  • 게임 패드 지원은 iCade 캐비닛을 통해 iOS 로 확장됩니다. 왼쪽 축은 스틱 컨트롤러에 매핑됩니다 (입력은 아날로그가 아니라 디지털 임에도 불구하고). 4 개의 "얼굴"버튼은 캐비닛 전면 버튼에, 4 개의 어깨 버튼은 캐비닛 후면의 버튼에 매핑됩니다.
  • Android 내보내기는 NYKO 컨트롤러 및 일반 Bluetooth 컨트롤러 (OUYA 포함)를 지원하지만 활성화 된 경우에만 가능합니다. 즉, Android 게임 옵션 의 일반 섹션에서 iCade / Bluetooth 옵션을 선택해야 합니다. 완전히 작동하려면 API 레벨 12가 필요하며 GameMaker Studio 2는 실제로 연결되어 있는지 여부에 관계없이 장치가 연결된 Bluetooth 장치를 연결된 장치로 등록합니다. 따라서 "슬롯"을 할당하고 확인할 때이 점을 고려해야합니다.
  • PS4 에서 터치 패드 추적을 사용하려면 device_mouse_ * 의 기능을 사용해야합니다. 버튼 (이 작업에는 DnD ™ 작업이 없습니다).
