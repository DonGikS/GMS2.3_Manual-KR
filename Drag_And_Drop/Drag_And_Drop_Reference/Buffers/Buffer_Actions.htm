버퍼 조치 라이브러리

버퍼 는 데이터를 한 장소에서 다른 장소로 이동하거나 어떤 방식으로 조작하는 동안 데이터를 일시적으로 유지하는 데 사용되는 실제 메모리 영역입니다. 예를 들어 버퍼를 사용하여 게임 상태 데이터를 유지할 수 있습니다. GameMaker Studio 2에는 버퍼 생성, 수정 및로드 / 저장과 관련된 여러 가지 특정 작업이 있지만 사용하기 전에 다음 사항에 유의하십시오.

  • 버퍼가 생성되면 자동으로 지워지고 0으로 채워집니다.
  • 버퍼 함수의 "size"인수는 항상 버퍼의 크기 (바이트) 를 나타냅니다.
  • DnD ™ 버퍼는 항상 1 바이트로 정렬 됩니다. "정렬"은 데이터가 버퍼 내에 저장되는 방식을 나타내며 GML 버퍼 함수 를 사용할 때 사용되지만 드래그 앤 드롭의 경우에는 관련이 없습니다.
  • 버퍼의 데이터는 순차적 인 방식으로 읽고 씁니다 (즉, 버퍼 쓰기 작업을 세 번 호출 한 다음 첫 번째 데이터를 쓰고 앞으로 이동하고 두 번째 데이터를 오른쪽으로 이동 한 다음 마지막 데이터를 기록함). 버퍼에서 다음 읽기 / 쓰기가 발생하는 위치를 알려주는 "seek"값이라고합니다.
  • 모든 데이터는 버퍼에 기록되고 버퍼에서 문자열 로 리턴되므로 실수를 저장 한 경우 데이터 유형 조치 중 하나를 사용하여 문자열에서 값으로 변환해야합니다.

버퍼를 만드는 데 사용되는 메모리는 시스템 메모리이므로 게임에 포커스가없는 경우에도 버퍼에 저장된 모든 데이터는 안전해야하지만 앱을 닫거나 다시 시작하면 손실됩니다. 또한 버퍼를 파일로 저장 한 다음 파일 작업을 사용하여 게임에 다시로드 할 수도 있습니다.
