벡터

복잡한 게임을 만들 때 어느 시점에서 벡터 를 다루어야 할 것입니다. 물리학, 인공 지능, 삼각법 및 기타 여러 상황에서 사용되지만 벡터는 무엇입니까? 간단히 말하면, 벡터는 지시 량 입니다. 화살표가 0에서 시작하여 5로 끝나는 번호가 매겨진 선을 그리면 1과 같은 1 차원 벡터를 살펴보면서 시작하겠습니다. 이것은 5와 같은 벡터 "a"입니다. 우리는 5에서 시작하여 8에서 끝나는 또 다른 화살표를 그립니다. 우리는 3과 같은 벡터 "b"를가집니다.

벡터가 어디에서 시작하는지는 중요하지 않으며, 중요한 것은 벡터의 길이와 방향에 달려 있다는 것입니다. 따라서 벡터 "b"는 5에서 시작하고 3 단위의 길이이며 "오른쪽"을 가리 킵니다. 0에서 시작하여 3으로가는 벡터와 동일하게 만듭니다. 이제 두 벡터 "a"와 "b"를 끝까지 넣어서 벡터 "c"를 얻기 위해이 벡터들을 더할 수 있습니다. 8. 음수는 어떻습니까? 위의 이미지에서 "오른쪽"을 가리키는 벡터가 양수에 해당하는 경우 "왼쪽"을 가리키는 벡터가 음수에 해당하여 1 차원 벡터를 만들지 않음을 알 수 있습니다 부호있는 (+/-) 숫자 이상 이것은 벡터의 필수 개념을 설명합니다 : 위치가 아닌 길이 와 방향 (이 경우 "왼쪽"또는 "오른쪽") 만 계산합니다.

2 차원 벡터는 어떻습니까? 우리는 그것들을 "왼쪽"과 "오른쪽"뿐만 아니라 "위"와 "아래"로 구성한 것으로 생각할 수 있습니다.

이제는 아직 벡터가 아닙니다. 시작 좌표와 끝 좌표를 사용하여 벡터를 줄여야합니다. 벡터 "a"를 보면 시작 좌표가 [2,2]이고 끝 좌표가 [4,3]임을 알 수 있습니다. 따라서 벡터를 얻으려면 끝 좌표를 빼서 벡터를 줄여야합니다. 시작 좌표는 다음과 같습니다.

a = [(x2-x1), (y2-y1)] = [(4-2), (3-2)] = [2,1]

벡터 b에 대해서도 똑같이하겠습니다 :

b = [(-1.2 -(-3.2)) ,(2.1 - 1.1)] = [2 ,1]

뭔가 알아? 그 두 벡터는 같습니다! 이것은 벡터에 위치가없고 방향과 길이 만 있다는 것을 보여주는 또 다른 데모이며, 로컬 [0,0] 축 주위에서 서로에 대해 벡터를 그릴 수 있습니다.

이는 2D 벡터가 로컬 [0,0] 축에 상대적인 "x"및 "y"위치의 두 값으로 정의됨을 의미합니다. 그리고 3D 벡터는 어떻습니까? 글쎄, 그들은 "depth"라는 차원을 추가하여 지역 축 주위에 "x", "y"및 "z"위치로 계산합니다.

GameMaker Studio 2에서 사용되는 벡터의 좋은 예는 방 편집기의 phtsics gravity 속성입니다. 중력은 [0,0] 위치 주변의 벡터로 계산됩니다. 따라서 중력을 [x0, y10]으로 정의하면 10의 힘으로 중력이 떨어 집니다.

일반적으로 많은 상황에서 벡터를 사용할 수 있지만 때로는 각도를 다룰 때와 같이 값을 제한하려고하기 때문에 벡터를 정규화 하는 것입니다. 이것은 본질적으로 길이 n 의 벡터를 길이 1의 벡터로 변환하는 데 사용되는 수학적 트릭입니다. 이는 벡터 구성 요소가 0과 1 사이로 정규화 됨을 의미합니다. 이러한 벡터는 단위 벡터 라고도합니다.

정규화 된 벡터를 계산하려면 먼저 원래 벡터 구성 요소가 있어야하고이를 사용하여 벡터의 길이를 가져와야합니다. 그런 다음 각 벡터 구성 요소를이 길이로 나눠서 모든 좌표의 제곱의 합이 1 인 정규화 된 벡터를 형성하는 정규화 된 벡터 구성 요소를 얻습니다. 방법은 다음과 같습니다.

먼저 벡터의 좌표를 가져 와서 구성 요소를 얻습니다.

vx = (x2 - x1) = (7 - 1) = 6
vy = (y2 - y1) = (4 - 1) = 3

그런 다음이 값을 사용하여 벡터의 길이를 계산합니다.

len = sqr(vx2 + vy2); // = sqr(36 + 9) = sqr(45) = 6.708203932499369

이제 벡터 "a"의 정확한 길이를 얻었 으므로 두 벡터 성분 vx 와 vy 를 정규화하는 데 사용하겠습니다.

vx = (vx/len); // = (6 / 6.708203932499369) = 0.8944271909999159
vy = (vy/len); // = (3 / 6.708203932499369) = 0.4472135954999579

큰! 이제 벡터의 구성 요소를 정규화했습니다! 그러나 GameMaker Studio 2와 관련하여 실제로 어떤 용도로 사용됩니까? 실용적 예를 들어 보자 ...

플레이어가 적을 쏠 수있는 게임이 있고 총알 개체가 x와 y 축을 따라 각 단계를 따라 얼마나 움직여야하는지 알아야합니다.

이를 위해 플레이어와 적의 좌표를 사용하여 벡터 구성 요소와 길이를 얻은 다음 0과 1 사이의 값을 얻도록 정규화하고 최종적으로 각 단계에서 총알이 이동하는 속도를 곱합니다. 이 마지막 두 값은 저장하고 각 단계의 시작 x 및 y 좌표에 추가합니다. 복잡한 소리? 그렇지 않습니다. (간단 성을 위해 값이 소수점 이하 한 자리로 반올림되었습니다) :

px = 100;
py = 425;
ex = 356;
ey = 83;
bullet_speed = 5;

vx = (ex - px); // = 256
vy = (ey - py); // = -342

len = sqrt(vx2 + vy2); // = sqrt(65536 + 116964) = 427.2

vx = vx / len; // = 0.6
vy = vy / len; // = 0.8

speed_x = vx * bullet_speed; // = 3
speed_y = vy * bullet_speed; // = 4

따라서 목표물을 맞추려면 매 단계마다 총알 x 좌표에 3을, y 좌표에 4를 추가해야합니다.
