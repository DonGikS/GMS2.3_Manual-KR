러너 오류

코드 편집기에서 구문 검사를 한 다음 컴파일러가 코드에서 컴파일러 오류 를 검사하도록 한 후에도 문제가 발생할 수 있습니다. 대부분의 경우 다음과 같은 VM (가상 머신) 러너 오류 ( 런타임 예외 라고도 함)가 발생 합니다.

러너 오류는 코드 오류나 코드 편집기에서 감지 할 수없는 코드에 심각한 문제가 있음을 의미하므로 컴파일 오류보다 일반적으로 더 심각합니다. 따라서 모든 오류에주의를 기울여야합니다. 문제가 발생하면 팝업의 복사 단추를 사용하여 오류를 클립 보드에 복사 한 다음 나중에 참조 할 수 있도록 텍스트 파일에 붙여 넣을 수 있습니다.

이 오류의 구조는 다음과 같습니다.

  • 오류가 발생한 이벤트와 개체 (해당되는 경우)가 표시됩니다.
  • 실제 오류를 보여줍니다. 따라서 오류의 범위 와 오류 유형이 표시됩니다. 괄호 안의 첫 번째 값은 코드를 실행 한 인스턴스의 인스턴스 ID이거나 아래 표에 나열된 특수 값 중 하나 일 수 있습니다. 두 번째는 YYG가 버그를 식별하는 데 사용하는 내부 값이며 무시하십시오. (예 : 액세스하기 전에 존재하지 않음) 위의 이미지의 경우, 변수의 _image가 설정을되지 않았 음을 말하는 객체 obj_EA_DEMO_Control에.
  • 오류가 발생한 줄 번호와 위치를 나타내는 코드 스 니펫을 보여줍니다. 게임이 컴파일되는 방식으로 인해 항상 100 % 정확하지는 않지만 문제가있는 정확한 위치를 찾는 데 도움이됩니다.

위에서 언급 한 것처럼 특정 오류 메시지는 인스턴스 ID 값이 아니라 음수 값으로 범위를 식별합니다. 이러한 값은 오류의 정확한 특성과 가능한 다음 값과 관련하여 정확히 무엇을 나타내는 데 사용될 수 있습니다.

 

접두사 범위
-1 본인
-2 다른
-삼 모두
-4 아무도
-5 글로벌
-6 명시되지 않은
-7 현지

 

VM 러너에서 발생할 수있는 오류는 다음과 같습니다.

 

오류 메시지 조작 기술
DoSet 잘못된 비교 유형 데이터 타입 이것은 러너가 문자열과 실수와 같이 호환되지 않는 두 가지 데이터 유형을 비교하려고 시도했음을 나타냅니다.
DoConv 실행 오류 데이터 타입 이는 한 데이터 유형을 다른 데이터 유형으로 변환 할 때 발생하는 오류를 나타냅니다.
DoAdd 실행 오류 수학 더하기 ( + ) 표현식을 사용할 때 문제가 발생했습니다.
도물 실행 오류 수학 곱셈 ( * ) 표현식을 사용할 때 문제가 발생했습니다.
DoSub 실행 오류 수학 빼기 ( - ) 표현식을 사용할 때 문제가 발생했습니다.
DoSub 실행 엔진-문자열 유형에서 작동 할 수 없습니다 수학 잘못된 유형의 변수를 빼려고합니다 (예 : 실수에서 문자열을 빼기).
DoDiv 실행 오류 수학 나누기 ( / 또는 div ) 표현식을 사용할 때 문제가 발생했습니다.
DoDiv 실행 엔진-문자열 유형에서 작동 할 수 없습니다 수학 잘못된 유형의 변수를 나누려고합니다 (예 : 문자열을 실수로 나누기).
DoDiv 0으로 나누기 수학 0으로 나누려고합니다.
DoMod 실행 오류 수학 모듈로 ( mod ) 표현식을 사용할 때 문제가 발생했습니다.
DoMod 실행 엔진-문자열 유형에서 작동 할 수 없습니다 수학 잘못된 유형의 변수에 대해 modulo ( mod )를 사용하려고합니다 (예 : 실수로 문자열을 수정 하십시오).
DoAnd 실행 오류 비트 단위 비트 단위 "and"( & ) 표현식을 사용할 때 문제가 발생했습니다.
DoAnd 실행 엔진-문자열 유형에서 작동 할 수 없습니다 비트 단위 잘못된 유형의 변수에 비트 단위 "and"( & )를 사용하려고합니다 (예 : 실수로 문자열을 "and"하려고 시도).
문 실행 오류 비트 단위 비트 단위 "또는"( | ) 표현식을 사용할 때 문제가 발생했습니다.
문 실행 엔진-문자열 유형에서 작동 할 수 없습니다 비트 단위 잘못된 유형의 변수에 "또는"( | )을 사용하려고합니다 (예 : 실수로 문자열을 "또는"하려고 시도).
독소 실행 오류 비트 단위 비트 단위 "xor"( ^ ) 표현식을 사용할 때 문제가 발생했습니다.
독소 실행 엔진-문자열 유형에서 작동 할 수 없습니다 비트 단위 잘못된 유형의 변수에 "xor"( ^ )를 사용하려고합니다 (예 : 실수로 문자열을 "xor"하려고 함).
DoShl 실행 오류 비트 단위 비트 시프 팅시 값이 왼쪽 ( << )에 잘못되었습니다.
DoShl 실행 엔진-문자열 유형에서 작동 할 수 없습니다 비트 단위 비트 시프트 ( << )를 잘못된 유형의 변수로 남겨 두려고합니다 (예 : 문자열을 비트 시프트하려고 함).
DoShr 실행 오류 비트 단위 비트 시프트 값 ( >> )을 잘못 입력하면 문제가 발생했습니다.
DoShr 실행 엔진-문자열 유형에서 작동 할 수 없습니다 비트 단위 잘못된 유형의 변수를 비트 시프트 ( >> )하려고합니다 (예 : 문자열을 비트 시프트하려는 경우).
도네 그 실행 오류 부정하다 이 유형이 그러한 조작을 허용하지 않을 때 변수 유형을 음수로 바꾸려고합니다.
하지 마라 실행 오류 부정하다 이 유형이 그러한 조작을 허용하지 않을 때 변수 유형을 "하지"않으려 고합니다.
푸시 실행 오류-가변 인덱스 범위를 벗어남 (var) 스택 액세스되는 변수가 러너에 대해 설정된 범위를 벗어났습니다.
푸시 변수 가져 오기 (var) 스택 주어진 변수가 정의되지 않았거나 알 수 없습니다.
팝 범위를 벗어난 변수 색인 (var) 스택 액세스되는 변수가 러너에 대해 설정된 범위를 벗어났습니다.
팝 변수 가져 오기 (var) 스택 주어진 변수가 정의되지 않았거나 알 수 없습니다.
와 with with 문을 사용할 수 없습니다 와 " with "문 내에서 " global "을 변수로 사용하려고했습니다. 예 : with (global) {// do something; }
와 with with 문을 사용할 수 없습니다 와 " with "문 내에서 " local "을 변수로 사용하려고했습니다. 예 : with (local) {// do something; }
DoCall 실행 엔진 유형 오류 엔진 이것은 가상 머신 내에서 정의되지 않은 오류입니다. 이 경우 버그 보고서를 제출해야합니다 (이 작업을 수행하는 방법에 대한 자세한 내용은 도움말 메뉴 참조).
스택 오버플로 - 엔진 스택 오버플로는 호출 스택에서 너무 많은 메모리를 사용하고 게임에서 호출 스택에서 사용할 수있는 것보다 더 많은 공간을 사용하려고 할 때 (즉, 호출 스택의 경계를 넘어 메모리에 액세스하려고 할 때 발생합니다. 버퍼 오버 플로우) 스택이 오버 플로우되어 프로그램 충돌이 발생합니다. 컴퓨터와 GameMaker : Studio를 다시 시작하십시오. 오류가 지속되면 지원 센터에 연락하거나 버그를 제기하십시오 (위 설명 참조).
