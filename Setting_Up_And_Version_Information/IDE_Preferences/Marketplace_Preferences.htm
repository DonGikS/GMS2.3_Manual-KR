마켓 플레이스 환경 설정

마켓 플레이스 환경 설정 은 자산 업로드 또는 구매를위한 마켓 플레이스 인증서 를 설정하는 데 사용됩니다. 인증서 양식은 이미 인증서를 만들었을 때 다음 섹션으로 구성되어 있습니다 (인증서 작성 방법에 대한 자세한 내용은이 페이지 아래 부분 참조).

  • 인증서 파일 :이 섹션에서는 마켓 플레이스에서 자산 인증에 사용하는 * .pem 파일을 지정합니다. 이전에 작성된 인증서가있는 경우 여기에서 파일 브라우저 단추를 클릭하여 포럼에 추가 할 수 있지만, 인증서가없고 인증서를 작성하려면 여기에서 정보를 생략하고 대신 새 인증서 만들기 버튼 (아래에 설명)
  • 인증서 비밀번호 : 여기서 선택한 파일의 인증서 비밀번호를 입력해야합니다. 비밀번호는 * .pem 파일을 처음 작성할 때 작성되며 (아래 의 새 인증서 작성 참조) 작성된 인증서 파일에 돌이킬 수 없게 링크됩니다.
  • 확인 : 위의 필드를 채우면이 버튼을 클릭하여 비밀번호와 파일 선택이 올 바르고 함께 작동하는지 확인해야합니다.

대화 상자 확인 대화 상자 확인

여기에서 자산 브라우저에서 리소스로 작업 할 때 확인 대화 상자의 동작을 선택할 수 있습니다. 사용 가능한 옵션은 다음과 같습니다.

  • 오프라인 컨텐츠 삭제에 대한 자동 응답 : 웹에 연결하지 않고 라이브러리에서 마켓 플레이스 자산을 삭제하면 일반적으로 진행할 것인지 묻는 경고가 표시됩니다. 이 옵션을 "확인"으로 설정하면 "확인"버튼을 클릭 한 것처럼 메시지가 표시되지 않습니다. 기본값은 "메시지 표시"입니다.

 

 

새 인증서 작성

Marketplace를 사용하기 위해 인증서 파일을 이전에 작성하지 않은 경우이 양식의 맨 아래에있는 새 인증서 작성 단추를 클릭해야합니다. 그렇게하면 다음 정보를 입력하라는 새 창이 열립니다.

  • 게시자 이름 : 자산의 게시자 이름으로 사용될 이름입니다. 마켓 플레이스 대시 보드 내에서 필요한 경우 다른 게시자를 설정할 수 있으며 각 게시자마다 고유 한 * .pem 인증서 파일이 있어야합니다.
  • 인증서 비밀번호 : 인증서 파일을 만들려면 고유 한 비밀번호를 입력해야합니다. 나중에 변경할 수 없으므로 선택한 사항에주의하십시오. 위의 섹션에 인증서 파일에 대한 올바른 비밀번호가 없으면 기존 마켓 플레이스 자산을 업데이트하거나 새 자산을 업로드 할 수 없습니다.

이러한 세부 정보가 제공되면 인증서 생성 버튼을 클릭하여 * .pem 파일을 생성 할 수 있습니다. 파일의 저장 위치와 이름을 입력하라는 메시지가 표시되며 나중에 참조 할 수 있도록이를 기록해 두어야합니다. 이 작업을 완료하면 새 파일과 비밀번호가 위에 나열된 양식 필드에 자동으로 추가되고 확인 버튼을 클릭하여 인증서가 올바르게 생성되었는지 테스트 할 수 있습니다.

참고 : 실제 설정의 스크린 샷을 작성하고 * .pem 파일을 복사하는 것이 좋습니다. 나중에 참조하기 위해 또는 어느 시점에서 GameMaker Studio 2를 다시 설치해야하는 경우에 대비하여 둘 다 안전한 곳에 보관해야합니다. 업데이트 된 패키지에 원본과 동일한 인증서를 사용하여 서명하지 않으면 이미 업로드 된 자산을 업데이트 할 수 없습니다.
