일반 환경 설정

일반 기본 설정 은 GameMaker Studio 2 IDE 의 일반 동작을 정의하는 데 사용됩니다. 가장 일반적인 것들은 메인 페이지에 나열되어 있고 다른 것들은 여러 개의 하위 카테고리로 나뉘어져 있습니다.

  • IDE Language :이 옵션을 사용하면 제공된 옵션 목록에서 IDE 언어를 선택할 수 있습니다. 기본값은 영어입니다.
  • IDE Skin : IDE 에서 사용할 수있는 다른 스킨 중에서 선택할 수 있습니다. 기본적으로 GameMaker Studio 2에는 두 개의 스킨 (하나는 밝고 하나는 어두움)이 있으며 초기 설정은 어두운 스킨을 사용합니다.
  • GameMaker Studio 2 Editor 의 최대 프레임 속도 :이 옵션을 사용하면 GameMaker Studio 2 IDE가 업데이트 할 최대 프레임 속도를 설정하여 모니터의 업데이트 속도와 일치하도록 프레임 속도를 설정할 수 있습니다. 기본값은 60입니다.
  • 기본 오디오 장치 : 여기서 GameMaker Studio 2 IDE에서 사용할 기본 오디오 장치를 선택할 수 있습니다. 새로 고침 버튼을 클릭하여 목록을 새로 고침 할 수 있습니다 .
  • 자동 응답 종료 확인 :이 옵션을 사용하면 GameMaker Studio 2를 종료 할 때 확인 메시지를 표시할지 여부를 설정할 수 있습니다. 기본적으로 "메시지 표시"로 설정되어 있지만 마치 마치 메시지를 건너 뛰도록 설정할 수 있습니다 확인을 클릭하십시오.
  • 시스템 글꼴 변경 대화 상자에 대한 자동 응답 :이 옵션을 사용하면 시스템 글꼴을 추가, 제거 또는 편집 할 때 확인 메시지를 표시할지 여부를 설정할 수 있습니다. 기본적으로이 메시지는 "메시지 표시"로 설정되어 있지만 확인을 클릭 한 것처럼 메시지를 건너 뛰도록 설정할 수 있습니다.
  • 알림 활성화 :이 옵션을 선택하면 IDE가 새 런타임 등에 대한 알림을 표시 할 수 있습니다. 기본적으로 켜져 있습니다.
  • 알림 시간 초과 (ms) : 알림이 표시되는 시간 (밀리 초)을 설정합니다. 기본값은 6000입니다.
  • 저장 후 MS에서 FileWatcher 공차 설정 IDE에서 : 파일 저장 시간과 파일 감시자가 변경 사항을 확인하는 시간을 조정합니다. 프로젝트를 저장 중이고 파일 감시자가 무언가 변경되었다는 메시지가 잘못 표시되는 경우이 값을 조정하면 메시지가 더 이상 표시되지 않을 때까지 cvase를 지연시킬 수 있습니다. 기본값은 2000ms이지만 1ms에서 10,000ms까지 다양합니다.
  • 종료시 로그 아웃 :이 옵션을 선택하면 GameMaker 스튜디오 2를 닫을 때 다음 IDE가 자동으로 사용자 세션을 로그 아웃합니다. 이것은 체크 될 때 주로 교육 사용자를 대상으로합니다. 즉, 프로그램을 종료 할 때마다 사용자가 로그 아웃되어 다른 사용자가 사용할 수 있도록 세션 자리를 비 웁니다. 기본적으로 해제되어 있습니다.
  • DPI 재정의 사용 :이 옵션을 사용하면 기본 DPI 설정을 직접 재정의 할 수 있습니다. 먼저이 옵션을 활성화 한 다음 Raw DPi Override 값 (기본값은 96)을 설정하거나 기본 DPi 의 백분율 (기본값 100 %)을 설정해야합니다. 일단 변경되면 IDE를 다시 시작하라는 메시지가 표시됩니다. 현재 열려있는 프로젝트에서 수행중인 작업을 저장해야 할 수도 있습니다. DPi의 최소 백분율은 50 %이고 최대 500 %입니다. 정보 아이콘 위로 마우스를 가져 가면 사용중인 디스플레이에 권장되는 최대 설정을 볼 수 있습니다. 모니터의 권장 DPi 설정을 초과하면 DPi를 재설정 할 수 있다는 알림이 표시됩니다.
  • 시작시 시작 화면 표시 : 기본적으로 켜져 있으며 프로그램을 시작할 때 간단한 스플래시 화면을 표시하도록 GameMaker Studio 2에 지시합니다. 이 옵션을 선택 취소하면 스플래시 화면이 표시되지 않습니다.
  • 빈 탭이 바로 가기를 통해 닫히도록 허용 : 키보드 바로 가기를 사용하여 열려 있는 탭을 닫을 수 있습니다 / + W 이지만이 옵션을 비활성화하면 해당 바로 가기가 더 이상 작동하지 않습니다. 기본적으로 활성화되어 있습니다.
  • 파일을 IDE로 드래그 할 때 편집 할 새 리소스 열기 : 스프라이트 또는 사운드와 같은 특정 리소스를 GameMaker Studio 2 IDE의 메인 작업 공간으로 드래그 할 수 있으며 자산 브라우저에서 새로운 리소스로 감지 및 추가됩니다. 기본적으로이 경우 편집 할 수 있도록 해당 자원 편집기가 열리지 만이 옵션을 선택 취소하면 편집기가 열리지 않으므로 수동으로 열어야합니다.
  • 생성시 편집 할 새 리소스 열기 :이 옵션을 선택하면 자산 브라우저에서 새 리소스를 생성 할 때마다 해당 리소스 편집기가 자동으로 열립니다. 기본적으로 선택되어 있으며 선택 해제하면 새로 만든 모든 리소스를 수동으로 열어야합니다.
  • 변경된 파일 자동 재로드 :이 옵션을 선택하면 IDE가 파일을 자동으로 스캔하고 검사하여 GameMaker Studio 2 외부에서 변경된 파일 이 있는지 확인한 다음 파일이 있으면 다시로드합니다.
  • IDE 전환 애니메이션 비활성화 : GameMaker Studio 2 IDE에는 창 열기 등과 같은 여러 애니메이션 전환이 있지만 모든 사람의 취향에 맞지는 않으며 구형 또는 저전력 시스템에서 비활성화하면 성능이 향상 될 수 있습니다 비활성화 된 경우 여기서 비활성화 할 수 있으며 기본적으로 켜져 있습니다.
  • 피드백 설문 조사에 참여하기 :이 옵션을 선택하면 GameMaker Studio 2에 피드백 설문 조사에 참여하고 싶다는 메시지가 표시됩니다. 즉, 작업중인 메시지가 작업중인 영역에 대해 표시되고 평가를 요청하는 경우가 있습니다. 피드백. 이 옵션은 기본적으로 켜져 있습니다.

위의 옵션은 일반 환경 설정에서 사용할 수있는 기본 옵션이지만 아래 나열된 하위 범주에서 사용할 수있는 추가 옵션이 많이 있습니다.

자산 탐색기 자산 탐색기

객체에 스프라이트를 추가하거나 드래그 앤 드롭으로 글꼴을 사용하려는 경우 또는 여러 상황에서 자산 탐색기가 표시됩니다.

기본적으로 트리보기 (왼쪽)와 목록보기 (오른쪽)가 모두 표시되지만이 환경 설정을 사용하면 하나만 표시하도록 선택할 수 있습니다.

배경 배경

배경 환경 설정은 작업 공간 배경과 배경 및 표시 방법을 처리합니다. 옵션은 다음과 같습니다.

  • 배경 이미지 : 다른 작업 공간의 배경으로 사용될 이미지 파일입니다. 기본 이미지가 제공되며 기본적으로 활성화되어 있지만 여기에서 자신의 이미지를 선택할 수 있습니다. 선택할 수있는 이미지 유형은 * .png , * .gif , * .jpg , * .jpeg , * .tif , * .tiff , * .ico , * .bmp , * .2bp 또는 * .mdi입니다 . 그러나 초기 이미지 형식에 관계없이 모든 파일은 * .png 파일로 사용하기 위해 저장되므로 예를 들어 애니메이션 * .gif 파일은 단일 초기 프레임으로 만 렌더링됩니다.
  • 이미지 스케일링 : 다른 작업 공간 내에서 배경 이미지의 스케일을 조정하는 방법을 제어합니다. 선택 사항은 다음과 같습니다. 스트레치 -이미지는 작업 영역에 맞게 스트레치 되며 보이는 도크에 따라 스케일이 변경됩니다.
  • 타일 -이미지가 작업 공간을 덮도록 바둑판 식으로 배열됩니다
  • 최적 맞춤 -이미지가 1 : 1 맞춤이 아닌 경우 가장자리 주위에 막대가 추가되어 작업 공간에 비례하여 크기가 조정됩니다.
  • 데스크탑 스트레치 -배경이 전체 작업 공간을 덮고 도크를 열거 나 닫을 때 크기가 변경되지 않고 이동합니다.
  • 화면 늘이기 -배경은 창 크기의 영역을 덮고 도크를 열거 나 닫을 때 이미지의 일부를 표시하도록 자르기 / 확장됩니다.
  • 작업 공간에 배경 이미지 표시 :이 옵션은 선택한 배경 이미지의 표시 여부를 제어합니다. 기본적으로이 설정은 켜져 있습니다.

    컴파일 컴파일

    컴파일 환경 설정은 게임을 컴파일하거나 패키지를 만들 때 게임을 컴파일 할 때 출력 과정과 관련된 특정 옵션을 처리합니다. 옵션은 다음과 같습니다.

    • 감지시 구문 오류 창 자동 열기 : 컴파일시 구문 오류 알림을 활성화 또는 비활성화합니다. 기본적으로 해제되어 있습니다.
    • 컴파일 중에 컴파일 창 표시 : 컴파일 문서가 닫혀 있어도 컴파일 컴파일이 열리더라도 프로젝트를 컴파일 할 때 기본적으로 컴파일 창이 표시됩니다. 체크를 해제하면이 동작이 꺼지고 컴파일 창이 열리면 켜집니다.
    • 컴파일 출력에서 ​​컴파일러 명령 표시 : 기본적으로 선택되어 있으며 컴파일러가 컴파일 창에 모든 관련 명령을 표시하도록합니다. 이 옵션을 선택 취소하면 출력이 작아지고 정보가 줄어 듭니다.
    • 자세한 컴파일러 출력 표시 :이 옵션을 선택하면 컴파일 프로세스와 관련된 추가 명령이 컴파일러에 표시됩니다. 일반적으로 이것은 디버깅 할 때만 필요하며 문제가있는 경우 지원 센터에서 요청할 수 있습니다. 기본적으로 꺼져 있습니다 (권장 설정).
    • 드라이브 경로에 Subst 사용 :이 옵션을 선택하면 GameMaker Studio 2가 전체 파일 경로를 대체 할 가상 드라이브를 생성하고 더 이상 필요하지 않은 경우 다시 제거합니다. 이 가상 드라이브는 OS에 대해 파일 경로가 너무 길어서 (특히 Android 용으로 컴파일하거나 확장을 사용할 때) 잠재적 인 문제를 우회하므로이 기능을 활성화하는 것이 좋습니다. 그러나 GameMaker Studio 2에서 이와 같은 가상 드라이브 생성에 문제가있는 경우 옵션을 비활성화 할 수 있습니다. 기본적으로 활성화되어 있습니다.
    • 빌드 완료시 탐색기에 패키지 표시 :이를 체크하면 GameMaker Studio 2가 최종 실행 패키지가 생성 된 위치에서 OS 탐색기 창을 엽니 다. 기본적으로 켜져 있습니다.
    • 컴파일 시작시 출력 창 지우기 : 기본적으로 컴파일러 창은 각 빌드 시작시 컴파일 출력 창에서 모든 이전 메시지를 지우지 만,이 옵션을 선택 취소하면 이전 컴파일러 로그가 유지됩니다. 기본적으로 설정되어 있으며 큰 컴파일러 로그로 인해 컴파일 시간이 느려질 수 있으므로 권장되는 구성입니다.
    • 최대 병렬 컴파일 작업 :이 값은 프로젝트 컴파일 전용의 최대 CPU 코어 수를 설정합니다. 기본값은 8이며 일반적으로이 값을 변경할 필요는 없지만 랩톱 또는 저가형 컴퓨터에있는 경우이 값을 낮추고 싶을 수 있습니다. 기본값은 8로 설정되어 있습니다.

    확인 확인

    대화 확인 확인 환경 설정은 GameMaker Studio 2 IDE에서 특정 작업을 수행 할 때 나타나는 일반적인 확인 팝업 중 일부를 처리합니다. 옵션은 다음과 같습니다.

    • 키보드 단축키를 통해 작업 공간을 닫는 것에 대한 자동 응답 : 위에서 언급 한 키보드 단축키를 사용하여 열린 탭을 닫을 수 있지만 때로는 빈 탭을 실수로 닫을 수도 있습니다. 이를 방지하기 위해이 옵션을 "메시지 표시"(기본 옵션)로 설정하여 빈 탭을 닫을 것인지 묻습니다. 다른 설정은 "아니오"입니다. 바로 가기를 사용하는 경우 아무 작업도 수행하지 않거나 메시지없이 탭을 닫으려면 "확인"을 누르십시오.
    • 실행중인 빌드 재시작에 대한 자동 응답 : 빌드 를 재시작하고 다른 이전 빌드가 여전히 실행중인 경우 현재 빌드를 먼저 중지할지 묻는 메시지가 표시됩니다. 이 메시지를 체크하지 않으면 해당 메시지가 억제되고 GameMaker Studio 2가 "확인"을 응답으로 클릭했다고 가정합니다. 이 옵션은 기본적으로 "메시지 표시"로 설정되어 있습니다.
    • YYC 디버깅 시도시 경고에 대한 자동 응답 : YYC 대상은 기본 코드로 컴파일되므로 내장 디버거를 사용하여 디버깅 할 수 없습니다. YYC 대상에서 디버거를 사용하려고하면이를 알려주는 경고 메시지가 표시되고 대신 VM을 사용하여 프로젝트가 컴파일된다는 사실이 표시됩니다. 이 옵션은 기본적으로 "메시지 표시"로 설정되어 있지만 "확인"으로 설정하면이 경고가 표시되지 않습니다.
    • 프로젝트 레이아웃 재설정에 대한 자동 응답 : 레이아웃 메뉴 에서 프로젝트 레이아웃을 재설정하도록 선택하면 기본적으로 열려있는 리소스 창을 닫는 방법에 대한 경고 메시지가 표시됩니다. 그러나이 메시지를 표시하지 않고 "확인"을 선택하여 레이아웃을 자동으로 재설정하거나 "아니요"를 선택하여 아무 것도 설정하지 마십시오. 기본값은 "메시지 표시"입니다.

    디버깅 디버깅

    여기에 기본 설정은 GameMaker Studio 2 디버거 와 관련이 있으며 다음 옵션이 있습니다 :

    • 디버거 종료에 대한 자동 응답 :이 옵션을 사용하면 디버거를 닫을 때 자동 응답을 설정할 수 있습니다. "메시지 표시"로 설정하면 디버거를 닫을 때 대화 상자가 표시되며 "예"또는 "아니요"로 설정하면이 대화 상자가 표시되지 않고 표시된대로 작업을 수행합니다.

    • 디버거 포트 : 디버그 모듈이 사용할 포트 번호를 설정합니다. 기본값은 6509입니다.
    • 디버거 리소스 뷰에 이벤트 아이콘 표시 : 디버거 리소스 뷰에 아이콘을 표시 할지 여부를 설정합니다. 기본적으로 선택되어 있습니다.

    입력 입력

    입력 환경 설정은 IDE 내에서 마우스 및 키보드 입력의 특정 측면을 제어합니다. 사용 가능한 옵션은 다음과 같습니다.

    • 느린 더블 클릭 시간 (ms) : IDE의 일부 영역에서는 리소스 이름을 바꿀 때와 같이 제대로 작동하려면 느린 더블 클릭이 필요합니다 (빨리 더블 클릭하면 리소스 편집기가 열리고 느린 더블 클릭하면 이름을 바꿀 수 있음) ). 이 옵션을 사용하면 기본값 500ms로 감지 속도를 설정할 수 있습니다.
    • 스크롤 잠금 시간 (ms) :이 옵션은 GameMaker Studio 2가 겹치는 스크롤 창 사이에서 대기해야하는 시간을 처리합니다. 기본적으로 "A"창에서 스크롤 할 때 커서가 "B"창의 다른 스크롤 가능 영역 위로 마우스를 가져갈 수 있으므로이 환경 설정은 "A"스크롤을 중지 한 후 " B "이므로 실수로 다른 패널을 스크롤하지 않습니다. 기본값은 500ms입니다.
    • 패닝시 화면 주위에 커서 줄 바꿈 :이 옵션을 활성화하면 작업 영역을 패닝 할 때 커서가 화면 경계를 둘러 싸게됩니다. 기본값은 활성화입니다.
    • 스크롤 속도 : 다른 작업 공간 내에서 스크롤 속도를 제어합니다. 기본값은 1입니다.
    • 바탕 화면 최대화를 해제하기위한 탭 드래그 거리 (px) : IDE 내에서 작업 공간 탭을 기본 위치에서 멀어지게 드래그 할 때 해당 시간에 하나의 작업 공간 만 활성화되어 있으면 GameMaker Studio 2 창이 최대화되지 않고 마우스 (여러 개의 작업 공간이 활성화 된 경우 작업 공간을 보유하기 위해 IDE의 새 인스턴스가 작성됩니다). 이 설정은 작업 영역이 인식하기 전에 탭을 얼마나 멀리 드래그해야하는지 제어하며 기본값은 50px입니다.
    • 창 내용과 상호 작용하기위한 최소 확대 / 축소 수준 (%) :이 옵션은 마우스가 더 이상 작업 공간 창과 상호 작용할 수없는 확대 / 축소 지점을 제어합니다 (예 : 노드 추가, 창 끌기 등). 기본값은 40입니다.
    • 마우스 휠 줌에는 Ctrl 키가 필요합니다 .이 옵션을 선택하면 모든 편집기 (및 기본 작업 공간)에서 / + 마우스 휠 을 선택 해제하면 마우스 휠 MMB 아이콘만으로 확대 / 축소가 활성화됩니다.
    • 수동 대신 상속 된 이벤트를 열려면 Ctrl + MMB :이 항목을 선택하면 / + 마우스 가운데 버튼 객체에서 매뉴얼 대신 객체 상속 이벤트 (있는 경우)를 엽니 다.
    • 줄 넘김을 방지하기 위해 텍스트 선택에 바이어스 추가 :이 옵션을 선택하면 텍스트 창에서 한 줄을 선택하는 동안 생각되는 줄 높이가 채워져 세로 방향의 부정확성이 줄어 듭니다. 기본적으로 활성화되어 있습니다.
    • 모든 장치에서 랩탑 모드 활성화 : 이것을 체크하면 IDE에 랩탑 모드 버튼이 표시되어 GameMaker Studio 2를 실행하는 장치가 랩탑인지 여부에 관계없이 필요에 따라 스위치를 켜거나 끌 수 있습니다. 기본적으로 선택되어 있지 않습니다.

    출력 창 출력 창

    출력 창 환경 설정은 IDE의 다양한 출력 창에 대해 다른 색상 및 서식 옵션을 제어합니다. 각기 다른 창에는 아래와 같은 옵션 세트가 있습니다.

    • 색상 : 창에 쓰여지는 텍스트의 기본 색상입니다.
    • 글꼴 이름 : 사용할 글꼴 이름입니다 (사용 가능한 글꼴은 시스템에 설치된 글꼴에 따라 다름).
    • 스타일 : 굵은 체 또는 이탤릭체 와 같이 사용할 글꼴 스타일 (일부 창에서 동일한 스타일을 사용할 수있는 것은 아니며 사용 된 글꼴에 따라 다름).
    • 크기 : 주어진 출력 창에 글꼴을 설정할 크기입니다.
    • 앤티 앨리어싱 : 여기에서 출력 창 글꼴 렌더링에 앤티 앨리어싱 모드를 설정할 수 있습니다. 기본값 사용 을 선택하여 IDE가 결정하도록하거나 해당 옵션 중 하나를 선택하여 강제로 켜 거나 끌 수 있습니다.

    경로 경로

    경로 환경 설정은 GameMaker Studio 2 IDE의 요소와 필요한 외부 편집기에 필요한 다른 파일 경로를 제어합니다. IDE에는 다음과 같은 옵션이 있습니다.

    • 임시 디렉토리 : 모든 임시 폴더를 저장하는 위치입니다. 기본적으로 이것은 \\ Users \\ <Username> \\ AppData \\ Local \\ Temp \\ GameMakerStudio2 입니다.
    • IDE cache directory : IDE 캐시 저장 위치. 기본적으로 이것은 \\ Users \\ <Username> \\ AppData \\ Roaming \\ GameMakerStudio2-Dev \\ Cache 입니다.
    • 자산 캐시 디렉토리 : 각 프로젝트의 자산 캐시를 저장하는 위치입니다. 기본적으로 이것은 \\ Users \\ <Username> \\ AppData \\ Roaming \\ GameMakerStudio2-Dev \\ Cache 입니다.
    • "내 프로젝트"위치 : GameMaker Studio 2가 처음에 새 프로젝트를 생성 할 위치입니다. 기본적으로 이것은 \\ Users \\ <Username> \\ Documents \\ GameMakerStudio2 입니다.
    • 닫을 때 임시 디렉토리 자동 삭제 : 체크하면 GameMaker Studio 2가 컴파일 등을 위해 프로젝트별로 생성 한 Temp 폴더를 자동으로 강제로 삭제합니다.이 설정은 기본적으로 켜져 있으며, 체크를 해제하면 해제됩니다 (즉, 나중에 임시 파일을 수동으로 제거해야합니다).
    • 닫을 때 자동으로 자산 캐시 삭제 :이 옵션을 선택하면 GameMaker Studio 2를 종료 할 때 자산 컴파일러 캐시 폴더가 제거됩니다. 이것은 기본적으로 해제되어 있으며 활성화하면 프로젝트를로드하고 실행할 때마다 캐시가 게임의 크기에 따라 시간이 걸릴 수 있습니다.
    • 닫을 때 IDE 캐시 자동 삭제 :이 옵션을 선택하면 GameMaker Studio 2를 종료 할 때 IDE 컴파일러 캐시 폴더가 제거됩니다. 기본적으로 해제되어 있으며,이를 활성화하면 GameMaker Studio 2를 시작할 때마다 캐시가 필요합니다. 재건 될 것입니다.
    • 임시 폴더 삭제 :이 버튼을 클릭하면 프로젝트의 임시 폴더가 삭제됩니다.
    • 자산 캐시 삭제 :이 버튼을 클릭하면 프로젝트의 컴파일러 자산 캐시가 삭제됩니다.
    • IDE 캐시 삭제 :이 버튼을 클릭하면 IDE 캐시 가 삭제됩니다

    외부 편집기의 경로를 설정하기위한 다음 옵션이 있습니다.

    • SWF 파일의 외부 편집기 / 뷰어 경로 : SWF 형식 스프라이트로 작업하는 경우 원하는 뷰어 / 편집기의 경로로 설정할 수 있으며 스프라이트 편집기에서 이미지 편집 버튼을 클릭하면 해당 파일이 열립니다. 프로그램. 여기서 기본값은 경로가없는 것입니다.
    • Spine 파일의 외부 편집기 / 뷰어 경로 : Spine 형식 스프라이트로 작업하는 경우 원하는 뷰어 / 편집기의 경로로 설정할 수 있으며 Sprite 편집기에서 이미지 편집 버튼을 클릭하면 해당 파일이 열립니다. 프로그램. 여기서 기본값은 경로가없는 것입니다.

    절전 절전

    여기의 절전 설정은 GameMaker Studio 2 IDE를 생성하는 데 사용되는 CPU 전력 및 GPU 전력의 양에 영향을 미칩니다 (실제로 프로그램이 실행되는 기본 프레임 속도가 낮아져 전력 소비가 줄어 듭니다). 이 중 일부는 File Watcher 백그라운드 프로세스에만 해당됩니다. File Watcher는 외부 변경 사항이 있는지 프로젝트 폴더를 스캔합니다 (예 : Notepad ++에서 gml 스크립트를 변경 한 경우 파일 워처는이 변경 사항을 감지하여 다시로드 할 것인지 또는 폐기 할 것인지 묻습니다). 발생하면 성능 및 전력 소비에 심각한 영향을 줄 수 있습니다.

    옵션은 다음과 같습니다.

    • AC 전원 프레임 속도 (%) : 전원 입력에 연결된 경우 기본 IDE 프레임 속도 수정 자. 기본값은 100 %입니다.
    • 배터리 전력 프레임 속도 (%) : 배터리를 사용하여 기계에 전원을 공급할 때 기본 IDE 프레임 속도 수정 자입니다. 기본값은 35 %입니다.
    • 배경 프레임 속도 (%) : GameMaker Studio 2의 초점이 맞지 않을 때 (전원 공급 장치에 관계없이 ) 기본 IDE 프레임 속도 수정 자. 기본값은 15 %입니다.
    • 파일 감시자 빈도 (ms) : 데스크톱 (또는 도킹 된 랩톱)에서 파일 감시자가 실행할 빈도입니다. 기본값은 2000입니다.
    • 랩톱 파일 감시자 빈도 (ms) : 랩톱 모드에서 파일 감시자가 실행할 빈도입니다. 기본값은 5000입니다.

    • 프로젝트로드시 자동으로 열기 : 활성화하면 새 프로젝트를로드 할 때 최근 Windows 목록이 표시됩니다. 기본값은 비활성화입니다.
    • 포커스 된 창을 맨 위로 이동 :이 옵션을 활성화하고 최근 창 목록에서 항목을 클릭하면 선택한 항목이 목록의 맨 위로 이동합니다. 기본적으로이 기능은 비활성화되어 있습니다.
    • 현재 열려 있는 창만 표시 :이 옵션을 활성화하면 최근 Windows 목록이 더 이상 작업 공간에서 열리지 않은 항목을 제거합니다 (예 : 창을 닫으면 목록에서도 제거됨). 기본적으로이 기능은 비활성화되어 있습니다.
    • 표시 할 최대 창 : 최근 창 목록에 표시 될 이전에 열린 창의 최대 수를 설정합니다. 기본값은 10입니다.

    자료 자료

    여기 환경 설정에는 프로젝트에서 특정 리소스 를 그리는 방법에 대한 옵션이 포함되어 있습니다.

    • 새 프로젝트의 초기 기본 그리기 색상 : GameMaker Studio 2 기능 또는 DnD ™ 액션을 사용하여 색상을 지정하지 않고 글꼴을 사용하거나 도형을 그릴 때 여기에 설정된 색상이 사용됩니다. 기본 색상은 흰색입니다 ( ARGB : $ FFFFFFFF).
    • 새 프로젝트의 초기 기본 창 색상 :이 옵션은 HTML5 대상의 HTML 페이지 배경색을 설정하고 모든 대상 (HTML5 포함)의 방 외부에 표시되는 색상을 설정하는 데 사용됩니다. 기본 색상은 검은 색입니다 (ARGB : $ 000000FF).

     

    작업 공간 작업 공간

    여기에있는 환경 설정은 다른 GameMaker Studio 2 작업 공간 내에서의 이동 방법에 영향을줍니다. 옵션은 다음과 같습니다.

    • 작업 공간 키보드 탐색 정도 범위 : 작업 공간 내에서 서로 다른 객체와 리소스에 대해 여러 개의 창을 열 수 있으므로 그 사이를 빠르게 이동하려면 키보드 단축키를 사용할 수 있습니다 / + + <화살표 키> 사이를 이동합니다. 작업 공간 내의 윈도우가 그리드로 정렬되지 않은 경우, IDE는 가능한 8 개의 방향 중 하나를 누를 때 어떤 윈도우를 의미하는지 선택해야하며,이 옵션을 사용하면 사용되는 "모양 각도"를 설정할 수 있습니다. 기본값은 40 °입니다.
    • 작업 공간 체인 열 패딩 (px) : 작업 공간 요소는 서로 "체인"되어있는 경우가 많습니다 (예 : 코드 편집기가 이벤트 편집기에 연결되어 있고 개체 편집기에 연결되어 있음).이 옵션을 사용하면 다음과 같은 수평 거리를 설정할 수 있습니다. 두 개의 연결된 창문. 기본값은 30px입니다.
    • 작업 공간 체인 행 패딩 (px) : 위에서 언급했듯이 작업 공간 요소는 종종 서로 "체인"되어 있으며이 옵션을 사용하면 두 체인 윈도우 사이의 수직 거리를 설정할 수 있습니다. 기본값은 20px입니다.
    • 작업 공간 체인 렌더링 세그먼트 수 : 렌더 체인의 최대 세그먼트 수를 설정할 수 있습니다. 이 값을 낮추면 저가형 시스템의 성능에 도움이 될 수 있습니다. 기본값은 20입니다.
    • 작업 공간 체인 링크 색상 : 작업 공간 요소 사이에 체인을 표시하는 데 사용되는 색상입니다. 기본 색상은 ARGB $ 99DD00입니다.
    • 작업 공간 키보드 탐색으로 확대 / 축소가 재설정됩니다 . 위에서 언급 한 것처럼 키보드 단축키를 사용하여 작업 공간 내에서 창을 건너 뛸 수 있습니다. 작업 공간을 사용하여 작업 공간을 확대하거나 축소 할 수도 있습니다. / + 마우스 휠 . 이 옵션을 설정하면 작업 공간을 확대 한 상태에서 바로 가기를 사용하여 다른 창으로 건너 뛰면 작업 공간이 1 : 1보기로 돌아가고 선택된 창을 중심으로합니다. 이 옵션은 기본적으로 해제되어 있습니다.
    • 작업 공간 체인이 겹칠 수 있습니다 :이 옵션을 사용하면 작업 공간 체인이 교차하고 겹칠 수 있습니다. 기본적으로 해제되어 있습니다.
