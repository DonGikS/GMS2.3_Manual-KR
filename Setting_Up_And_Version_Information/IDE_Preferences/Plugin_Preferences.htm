플러그인 환경 설정

플러그인 환경 설정 은 GameMaker Studio 2 IDE에 추가 ​​된 추가 플러그인을 제어하는 ​​데 사용됩니다. 기본적으로 여기에는 GameMaker Studio 2 프로젝트를 Git 과 통합하기위한 소스 제어 플러그인 전용 섹션이 있습니다.

소스 컨트롤 (Git) 소스 컨트롤 (Git)

환경 설정이 절 거래는 당신이 망할 놈 얻을 세트에 필요하다고 SCM은 GameMaker 스튜디오 2. 첫 번째로 작업하려면 플러그인이 G eneral 옵션의 세트를 가지고 :

  • 소스 제어 출력 창 자동 열기 :이 옵션을 선택하면 무언가 변경 될 때 소스 제어 출력 창이 자동으로 표시됩니다. 이것은 기본적으로 켜져 있습니다.
  • Verbose Logging : Git 출력 로그에 대한 자세한 로깅을 활성화합니다. 기본적으로 해제되어 있습니다.

다음은 채워야하는 다음 필드가있는 사용자 ID 섹션입니다.

  • 커밋 할 사용자 이름 : 여기에 Git 커밋에 사용할 사용자 이름을 입력해야합니다 (Git 리포지토리를 설정할 때 지정한 사용자 이름과 동일해야 함).
  • 다음으로 커밋 할 이메일 :이 Git ID와 연관된 이메일입니다.

사용자 ID를 설정 한 후 새 사용자 / 패스 인증 추가 라는 단추를 클릭하여 인증을위한 새 사용자 이름 / 비밀번호 조합을 추가 할 수 있습니다. 그러면 다른 필드가 채워져 다음과 같이 채워집니다.

해당 레이블이있는 단추를 클릭하여 새 SSH 키 쌍 인증 을 추가 할 수도 있습니다. 이렇게하면 다음과 같은 창이 열립니다.

이 버튼 아래에서 다음 옵션을 사용하여 SCM 용 병합 도구 및 Diff 도구 를 설정할 수 있습니다.

  • 병합 도구 경로 : Git 병합 도구의 전체 파일 경로를 지정합니다.
  • 병합 도구 옵션 :이 필드에서 병합 도구를 사용할 때마다 실행할 추가 명령을 추가 할 수 있습니다.
  • Diff Tool 경로 : Git diff 도구의 전체 파일 경로를 지정합니다.
  • Diff Tool 옵션 :이 필드에서 Diff Tool을 사용할 때마다 실행할 추가 명령을 추가 할 수 있습니다.

마지막으로, 설정 후 마지막 옵션이 있습니다.

  • 소스 제어 자동 활성화 :이 옵션을 선택하면 모든 새 프로젝트에 대해 소스 제어가 자동으로 활성화됩니다. 기본적으로 해제되어 있습니다.
