아마존 화재 환경 설정

Amazon Fire 환경 설정은 아래 나열된 여러 섹션으로 나뉩니다. Amazon Fire 대상은 Android를 기반으로하고 동일한 SDK 요구 사항을 공유하므로이 중 다수는 Android 도구를 나타냅니다.

기본 설정의 상단 에는 Android의 설정 을 동기화하는 옵션 이 있습니다 . Amazon Fire는 Android와 거의 동일한 빌드 도구를 사용하기 때문에 모바일 라이센스가 있고 이미 Android 내보내기를 설정하고 사용한 사람들을위한 것입니다. 이 버튼을 클릭하면 Android에서 적절한 설정을 모두 가져온 다음 설정이 올바른지 확인하십시오.

동기화 버튼에서 JDK 메모리 힙 의 최대 크기를 설정할 수 있습니다 (기본값은 2GB). 다음으로 GameMaker Studio 2가 최종 실행 패키지를 빌드하고 게임을 테스트하는 데 필요한 다양한 도구를 찾을 위치를 알 수 있도록 다양한 경로 를 설정해야합니다. 이것은 Android SDK , Android NDK 및 Java JDK의 세 부분으로 나뉩니다. 이러한 경로는 자동으로 채워졌지만 문제가있는 경우 오른쪽의 버튼을 클릭하여 파일 탐색기를 열고 각 요소의 위치를 ​​수동으로 선택할 수 있습니다. 각 SDK 요소에 필요한 버전에 대한 자세한 내용은 여기를 참조하십시오 . 경로를 적용 할 때 경로를 확인해야하며 메시지에 " 발견 "이 표시되거나 오류가 표시됩니다 (이 경우 모든 정보를 수정하여 올바른지 확인해야 함).

아래에는 다양한 패키징 옵션이 있습니다.

  • 항상 APK 전체 설치 : 플래그를 지정하면 GameMaker Studio 2가 새 버전을 설치하기 전에 테스트 장치에서 게임의 모든 이전 파일을 비롯하여 장치에서 모든 관련 파일 (예 : * .ini 파일)을 제거합니다. 게임.
  • 패키지에 설치 :이 옵션이 표시되면 최종 실행 파일 * .apk 패키지를 컴파일하면 현재 선택된 대상 장치에 자동으로 설치됩니다.

마지막으로 KeyStore에 옵니다. 이 파일은 모든 Amazon Fire 앱에 "서명"하는 데 사용되는 파일이므로 모든 세부 정보를 정확하게 입력하고 향후 모든 Amazon Fire 앱을 생성하고 업데이트하는 데이 파일이 필요하다는 것을 명심하십시오. 설정 한 후에 는 사용 된 설정의 스크린 샷을 작성하고 최종 파일을 백업하는 것이 좋습니다 . 이 파일을 잃어 버리면 상점에 업로드 된 기존 게임을 업데이트 할 수 없습니다 .

참고 : 동일한 키 저장소를 Android 및 Amazon Fire 대상 모두에 사용할 수 있습니다.

새 키 저장소 파일을 작성하려면 다음 세부 사항이 필요합니다.

  • Filename : KeyStore 파일의 이름입니다. 참고 : 이것은 귀하의 이름 또는 회사 이름이 아니라 생성 될 파일의 ​​이름입니다! 이전 프로젝트의 키 스토어 파일이 이미 있다면 여기에서 파일 브라우저 버튼을 클릭하고 GameMaker Studio 2를 이전에 생성 된 키 스토어 파일로 보내거나 가져 오기 버튼을 클릭 할 수 있습니다. 이 경우에도 필요한 정보 (당신이 먼저 파일을 생성 할 때 예전 그대로)의 나머지 부분을 작성해야하지만 당신은 생성 키 스토어 버튼을 클릭하지 않아야합니다! 그러면 새 키 저장소 파일이 생성되어 이전 파일을 덮어 씁니다.
  • 공통 이름 : 일반적으로 귀하의 이름입니다.
  • 비밀번호 : 키 저장소 파일의 보안 비밀번호 (6 자 이상이어야 함)
  • 별명 : KeyStore 파일에 사용할 수있는 다른 이름이며 위에 입력 한 이름과 달라야합니다.
  • Alias ​​Password : KeyStore 파일의 별칭 보안 암호는 6 자 이상이어야합니다.
  • 조직 구성 단위 : 소속 된 회사 내 부서.
  • 조직 : 회사 이름.
  • 위치 : 거주하는 도시 또는 도시의 이름입니다.
  • 국가 : 거주 국가 의 표준 두 문자 코드 입니다.

이 정보가 채워지고 만족 스러우면 GameMaker Studio 2 용 키 스토어 생성 버튼을 눌러 필요한 키 스토어 파일을 생성하십시오. 이 작업에는 다소 시간이 걸릴 수 있지만 완료되면 일반적으로 다시 변경할 필요가 없습니다. 가져 오기 버튼을 클릭하여 이전에 생성 된 키 저장소 파일을 가져오고 (위의 파일 이름 섹션 참조) 키 해시 생성 버튼을 클릭하여 키 저장소 SHA1 해시를 검색 할 수도 있습니다. 이 키를 누르면 일부 웹 사이트에서 올바른 기능 (예 : Facebook 또는 Sha1 해시 용 Google Play)에 필요한 키 스토어 파일에 직접 연결된 두 개의 특수 보안 해시 키가 생성됩니다. 필요한 경우 버튼을 누른 다음 생성 된 해시 키를 복사하여 붙여 넣으십시오.
