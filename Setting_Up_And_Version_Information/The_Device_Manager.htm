장치 관리자

클릭하면 장치 관리자 에 액세스 연필 버튼에 대상 목록 내 :

이 버튼을 클릭하면 컴파일하고 테스트 할 대상 플랫폼에 대해 다른 장치를 구성 할 수있는 장치 관리자 가 열립니다 (사용 가능한 플랫폼은 보유하고있는 라이센스에 따라 다르지만 모든 장치가 표시되지는 않습니다). 아래에서 각 대상 플랫폼에 대한 장치 구성에 대한 세부 사항을 찾을 수 있습니다.

안드로이드 안드로이드

여기에서 Add New Device (새 장치 추가) 버튼을 클릭하여 새 Android 장치를 목록에 수동으로 추가하거나, 장치 감지 단추를 클릭하여 GameMaker Studio 2가 유선 연결을 통해 연결된 모든 장치를 자동으로 감지하도록 할 수 있습니다. AVD 실행이라고 표시된 버튼을 클릭하여 Android 가상 장치 를 설정할 수도 있습니다. 새 AVD를 생성 할 수있는 Android 가상 장치 관리자가 열립니다 (자세한 내용은 여기 참조). 생성 한 후에는 AVD 관리자에서 실행 한 다음 GameMaker Studio 2에서 장치 검색 버튼을 클릭하면 자동으로 장치 목록에 추가됩니다.

장치 (물리적 또는 가상)가 추가되면 연결 테스트 버튼을 사용하여 연결을 확인하거나 장치 제거 버튼을 클릭하여 목록에서 다시 연결 을 제거 할 수 있습니다.

HTML5 HTML5

HTML5 장치 관리자는 실행시 타겟팅 할 수 있도록 다른 브라우저를 장치 목록에 추가하기위한 것입니다. 우선, 브라우저 검색 버튼을 클릭하면 컴퓨터에 설치된 각 브라우저에 대한 장치가 생성됩니다. 브라우저가 목록에 없다면, 새 브라우저 추가 버튼을 클릭 한 다음 장치 이름을 지정하고 GameMaker Studio 2를 브라우저 실행 파일 경로로 지정하십시오.

사용 가능한 추가 필드는 다음 옵션을 사용하여 브라우저가 프로젝트를 실행하는 방법을 제어하는 ​​데 도움이되도록 설정할 수 있습니다.

  • 브라우저 명령 행 인수 : 여기에서 브라우저 경로에 추가 할 명령 행 인수를 추가 할 수 있습니다. 다음은 인기있는 주요 브라우저에 대한 링크에서 이러한 목록을 찾을 수 있습니다. 크롬 (Chrome)
  • Firefox
  • 인터넷 익스플로러
  • 웹 서버 주소 : 대상 웹 서버 의 주소 (기본값은 127.0.0.1)입니다. 다른 API의 사양에 따라이를 변경해야 할 수도 있습니다. 예를 들어 Facebook은 http : // localhost 로 설정 한 다음 앱 콘솔에 허용 목록을 표시해야합니다.
  • 웹 서버 포트 : 웹 서버 의 포트 번호 를 설정합니다 (기본값은 51264).
  • 허용 된 웹 서버 IP / CIDR IP 범위 : 필요에 따라 허용 된 웹 서버, 호스트 이름 및 CIDR IP 범위를 설정하십시오.

    iOS iOS

    iOS 장비를 설정하기 전에 Mac 장비를 설정하고 연결을 테스트해야합니다 (자세한 내용은 아래 참조). 완료되면 창 상단의 드롭 다운 메뉴에서 대상으로 설정할 Mac을 선택한 다음 장치 감지 버튼을 클릭하여 GameMaker Studio 2가 시뮬레이터를 포함하여 연결된 iOS 장치를 자동으로 찾도록 할 수 있습니다. 연결되면 장치 세부 정보가이 창의 목록에 표시되며 장치 이름을 편집하여보다 사용자 친화적으로 만들 수 있습니다 (이 변경 사항은 장치가 대상 관리자에 표시되는 방식에만 영향을 미침).

    tvOS tvOS

    tvOS 장치를 설정하기 전에 Mac 장치를 설정하고 연결을 테스트해야합니다 (자세한 내용은 아래 참조). 완료되면 창 상단의 드롭 다운 메뉴에서 대상으로 지정할 Mac을 선택한 다음 장치 감지 버튼을 클릭하여 GameMaker Studio 2가 시뮬레이터를 포함하여 연결된 tvOS 장치를 자동으로 찾도록 할 수 있습니다. 연결되면 장치 세부 정보가이 창의 목록에 표시되며 장치 이름을 편집하여보다 사용자 친화적으로 만들 수 있습니다 (이 변경 사항은 장치가 대상 관리자에 표시되는 방식에만 영향을 미침).

    macOS macOS

    Mac 장비를 관리자에 추가하려면 먼저 장비 추가를 클릭하고 연결을 활성화하기 위해 채워야하는 빈 필드를 생성하는 새 장비 추가 라는 버튼을 클릭해야합니다.

    먼저 대상 목록에서 장치를 식별 한 다음 Mac의 호스트 이름 (또는 IP)을 지정할 수 있도록 장치의 이름 을 지정 해야합니다. 그런 다음 연결하려는 사용자의 사용자 이름 과 사용한 암호 를 제공해야합니다. 원하는 경우, Mac의 설치 폴더 를 프로젝트의 빌드 파일을 저장하기 위해 GameMaker Studio 2를 원하는 위치로 변경할 수도 있습니다.

    모두 완료되면 연결 테스트 버튼을 클릭하여 GameMaker Studio 2가 Mac과 통신 할 수 있는지 확인할 수 있습니다.

    참고 : iOS 용으로 개발하는 경우 먼저이 작업을 수행하고 개발에 사용하려는 Mac에 연결되어 있어야합니다.

    참고 : 대상 Mac에서 원격 로그인 을 활성화 해야합니다 . 로그인 하려는 사용자의 환경 설정 -> 공유 및 원격 로그인 사용 으로 이동하여 수행 할 수 있습니다.

    우분투 (Linux) 우분투 (Linux)

    관리자에 Ubuntu (Linux) 장치를 추가하려면 먼저 새 장치 추가 라는 단추를 클릭하여 장치 를 추가하고 연결을 활성화하기 위해 채워야하는 빈 필드를 만들어야합니다.

    먼저 대상 목록에서 장치를 식별 할 수 있도록 장치 이름 을 지정한 다음 Ubuntu 시스템 (또는 VM)의 호스트 이름 (또는 IP)을 제공해야합니다. 그런 다음 연결하려는 사용자의 사용자 이름 과 사용한 암호 를 제공해야합니다. 또한 프로젝트 설치에 사용할 설치 폴더 를 Ubuntu 시스템에 제공 해야합니다 . 완료되면 연결 테스트 버튼을 클릭하여 GameMaker Studio 2가 Ubuntu 컴퓨터와 통신 할 수 있는지 확인할 수 있습니다.

    Windows UWP Windows UWP

    Windows 10 UWP 대상 장치를 추가하려면 개발자 모드를 활성화하고 장치 에서 장치 포털 을 설정해야합니다. 이 작업을 수행하지 않으면 GameMaker Studio 2가 장치와 통신하거나 UWP 앱을 제대로 만들 수 없습니다.

    개발자 모드를 활성화하는 가장 쉬운 방법은 Windows 검색에 " 개발자 모드 "를 입력 한 다음 표시 될 " 설정 "항목을 클릭하는 것입니다. 다음 창이 열립니다 :

    개발자 모드 옵션 (Windows에서 일부 추가 패키지를 다운로드하여 설치해야 할 수도 있음)을 선택한 다음 그 아래 옵션 인 장치 포털 사용 을 선택해야 합니다 . 장치 포털은 GameMaker Studio 2가 Windows 10 PC와 통신 할 수있게 해주는 기능이며, 활성화되면 사용자 이름 과 암호 를 제공해야합니다.

    사용자 이름과 비밀번호는 포털에 고유해야하며 Microsoft 계정 또는 기타 Windows 자격 증명에 사용 된 이름이 아니어야합니다. 이 정보를 제공했으면 적용 이라고 표시된 버튼을 클릭 한 다음 설정 관리자를 닫고 GameMaker Studio 2로 돌아갑니다.

    장치 포털에 대한 자세한 내용은이 Microsoft 문서 : 데스크탑 장치 포털을 참조하십시오.

    이제 GameMaker Studio 2에서 장치 관리자 를 열고 사용할 장치를 설정할 수 있습니다 :

    Windows 10 UWP 호환 장치를 관리자에 추가 하려면 먼저 새 장치 추가 라는 단추를 클릭하여 장치 를 추가하고 연결을 활성화하기 위해 채워야하는 빈 필드를 만들어야합니다.

    먼저 대상 목록에서 장치를 식별 할 수 있도록 장치 이름 을 지정한 다음 호스트 를 지정해야합니다. 네트워크에서 게임을 호스팅 할 Windows 10 장치의 IP 주소이며 " : 50443 으로 접미사로 지정해야합니다 . "를 눌러 포트 번호를 설정하십시오. 그런 다음 장치 포털에서 설정 한 사용자 이름 및 비밀번호 를 추가하십시오 (앞서 언급 한대로). UWP Windows 10 Desktop에는 필요하지 않으므로 PIN을 비워 둘 수 있습니다.

    다른 UWP 출력 (데스크탑, XboxOne)을 다루는보다 구체적인 설정 정보는 YYG 헬프 데스크 : UWP 설정을 참조하십시오.

     

     

    다른 모든 장치 창에는 하단에 장치 지우기 옵션이 있으며, 클릭하면 목록에서 연결된 모든 장치가 제거됩니다.

    장치를 설정 한 후에는 GameMaker Studio 2 IDE 상단의 대상 옵션에서 장치 중 하나에 연결할 수 있으며 게임을 테스트 / 빌드하면 프로젝트가 해당 장치로 푸시됩니다.
