구성

크로스 플랫폼 개발 요구와 다양한 클라이언트 또는 장치 또는 플랫폼에 대해 하나의 제품을 사용하고 사용자 정의해야 할 필요성으로 인해 GameMaker Studio 2에는 구성 을 단순화하는 구성 이 포함되어 있습니다 . 이 시스템을 사용하면 서로 다른 스플래시 화면, 아이콘 및 포함 된 파일 간을 빠르게 전환하고 언제든지 동일한 기본 코드에서 대상 플랫폼 또는 클라이언트와 관련된 파일 만 테스트하거나 내보낼 수 있습니다.

클라이언트와 플랫폼의 다양한 요구를 처리하기 위해 Asset Browser 의 Extras 메뉴에서 액세스 할 수있는 Configuration Editor 를 사용할 수 있습니다.

GameMaker Studio 2로 만든 모든 게임은 "기본"구성으로 시작합니다. "기본"구성은 처음 열 때 편집기에 나열됩니다.

구성을 설정하려면 먼저 프로젝트의 구성 가능한 모든 자산을 구성하여이 "기본"구성을 준비하는 것이 좋습니다. 이렇게하려면 다음 옵션 목록을 살펴보고 기본적으로 필요한 옵션으로 설정하십시오.

  • 확장 옵션 : 다양한 "Copy to ..."필드를 구성에 필요한 플랫폼으로 설정하십시오. 여기에 생성 된 "Proxy"파일은 "Target"플랫폼도 구성에 연결됩니다.
  • Sprite Editor : 스프라이트를 추가 할 텍스처 그룹을 설정합니다.
  • Tileset Editor : 타일셋을 추가 할 텍스처 그룹을 설정합니다.
  • 폰트 에디터 : 폰트를 추가 할 텍스처 그룹을 설정합니다.
  • Texture Group Manager : 여기에서 텍스처 그룹이 속한 부모 그룹을 포함하여 구성 기반에 대한 속성을 설정할 수 있습니다.
  • 오디오 그룹 관리자 : 다른 오디오 그룹에 할당 된 사운드는 여기에서 구성 가능합니다.
  • 포함 된 파일 : "복사 위치 ..."필드를 구성에 필요한 플랫폼으로 설정하십시오.

Once you have set up the default configuration, you can then add a new one to the Configuration Editor by right clicking 구성 폴더에서 구성 추가를 선택하거나 창의 맨 아래에있는 " 작성 "단추를 클릭하십시오. 새 구성은 기본 구성의 "자식"으로 표시됩니다. 모든 사용자 정의 구성은 기본 구성의 자식입니다. 적절한 이름을 지정할 수 있습니다.

To edit the config, you must either double click 그것에 마우스 오른쪽 버튼을 사용 메뉴 옵션 " Set Config "는 편집기에서 구성을 선택하고 일단 완료되면 계속해서 사용자 정의 할 수 있습니다. 이 작업은 위에 나열된 옵션을 통해 새로 생성 된 구성에 대해 원하는 설정으로 변경하면됩니다. GameMaker Studio 2는 설정을 위해이 값을 자동으로 저장하며, RMB를 사용하여 언제든지 설정을 변경할 수 있습니다 메뉴 옵션 또는 더블 클릭 .

편집기를 작성할 때 강조 표시된 구성에 따라 기본 구성의 하위 또는 사용자 정의 구성의 하위로 추가 구성을 추가 할 수 있습니다. 아래 이미지 예에서 3 개의 사용자 정의 구성이 정의되어 있으며,이 중 2 개는 기본 구성의 자식이고 다른 하나는 사용자 정의 구성의 자식입니다.

프로젝트를 컴파일하거나 테스트 할 때 주어진 대상 플랫폼에서 주어진 구성을 사용할 수 있다는 점에 주목할 가치가 있습니다. 대상 관리자 의 구성 섹션에서 필요한 구성을 선택하면됩니다.

또한 오른쪽 상단의 메뉴 버튼 (위 이미지에서 강조 표시됨)을 클릭하여 대상 관리자에서 임의의 시간에 구성 편집기를 열 수 있습니다.

이러한 방식으로 사용되도록 구성을 설정하면 테스트를 위해 또는 최종 자산 패키지를 생성하기 위해 대상으로 컴파일 할 때마다 선택한 구성이 사용됩니다. 대상 창에 대한 자세한 내용은 컴파일 섹션을 참조하십시오.
