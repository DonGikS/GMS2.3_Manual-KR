안드로이드 게임 옵션

이 섹션에서는 Android 게임 프로젝트를 컴파일하는 방법을 제어 할 수있는 다양한 옵션에 대해 간략하게 설명합니다. 다른 섹션은 다음과 같습니다.

 

일반

일반 섹션의 첫 번째 부분은 제품 정보 및 빌드 설정 전용입니다. 당신은 프로젝트가 도메인, 회사 및 제품 구성 이름, 패키지 정보를 표시 포함하는 제품의 정보를 입력해야합니다. 패키지 정보는 AZ 및 숫자 0-9의 문자로만 구성 될 수 있으며 기호 나 기타 특수 문자가 전혀없고 대문자도 없어야합니다. 그런 다음이 세부 사항을 사용하여 " com.company.myappname "형식의 패키지 ID를 생성하십시오.

참고 : Amazon Fire 대상 게임 옵션을 이미 설정 한 경우 상단의 " Amazon Fire에서 동기화 "버튼을 클릭하여 관련 정보를 Amazon Fire 대상에서 Android 대상으로 전송할 수 있습니다.

그런 다음 Build , Target , Minimum 및 Compile SDK 버전을 설정해야합니다.

참고 : " 지원 라이브러리 "버전에 사용할 옵션도 있습니다. GameMaker Studio v2.2.4 이상에서는 더 이상 필요하지 않으며 여기에 사용 된 값은 무시됩니다. 이 설정은 2.2.4 이전의 런타임이 필요한 레거시 프로젝트에서 사용하도록 유지됩니다.

빌드 도구 (및 필요한 경우 지원 라이브러리)는 GameMaker Studio 2가 Android / Amazon Fire 대상 용 게임을 빌드 할 때 사용되며 Android Studio를 사용하여 설치 한 도구에 따라 적절한 값으로 설정해야합니다.

  • Target SDK 는 여기에 지정한 버전에서 앱을 테스트했음을 나타냅니다. 이는 단순히 OS 기능 측면에서 앱을 처리하는 방법에 대한 아이디어를 Android OS에 제공하기위한 것입니다. 실질적인 목적으로 대부분의 앱은 Compile SDK 설정에서 사용되는 최신 버전의 API로 Target SDK를 설정하려고합니다. 이렇게하면 최신 Android 기기에서 앱이 최대한 멋지게 보입니다. 2019 년 11 월부터 Google은 API 레벨 28을 필수 목표로 설정했습니다.
  • 최소 SDK 버전은 프로젝트를 실행할 최소 API 레벨입니다. 2019 년 11 월부터 Google은 API 수준 16을 필요한 최소 수준으로 만들었지 만 확장을 추가하면 더 높은 최소 API 수준이 필요할 수 있습니다.
  • Compile SDK 버전은 프로젝트가 컴파일되는 API 버전입니다. 즉, 해당 버전의 API에 포함 된 Android API 기능을 사용할 수 있습니다. 예를 들어 API 16 기능을 사용하려고하지만 Compile SDK 15를 설정하면 컴파일 오류가 발생합니다. 그러나 Compile SDK를 28로 설정하면 API 26 장치 (및 다른 모든 이전 버전에서도)에서 앱을 계속 실행할 수 있습니다.

SDK 값을 선택한 API 수준으로 미리 채우기 버튼을 클릭하여 사전 정의 된 대상 설정 중 하나를 선택할 수 있습니다.이 버튼을 클릭하면 선택할 수있는 다른 API 목록이 열립니다. 이들 중 하나를 선택하면 모든 SDK 파일이 적절한 값으로 채워져 필요에 따라 사용하거나 편집 할 수 있습니다.

참고 : 프로젝트가 올바르게 컴파일 되려면 Android SDK Manager 또는 Android Studio를 통해 적절한 API를 설치해야합니다.

일반 섹션의 두 번째 부분은 다음과 같은 옵션을 사용하여 빌드 할 CPU 아키텍처를 다룹니다 (모두 선택하거나 필요한 옵션 만 선택할 수 있지만 각 옵션은 컴파일 된 apk의 최종 크기에 추가됨) :

  • ARMV5 용 빌드 (기본적으로 선택되어 있음)
  • ARMV7 용 빌드 (기본적으로 선택되어 있음)
  • x86 용 빌드 (기본적으로 선택되어 있음)
  • Mips 용 빌드 (기본적으로 선택되어 있음)
  • ARM64를위한 빌드
  • x86_64 용 빌드

설정 후 최상의 사용자 경험을 위해 게임을 실행할 수있는 다른 방향을 선택할 수 있습니다. 사용 가능한 옵션은 다음과 같습니다.

  • 초상화
  • 세로로 뒤집힌
  • 경치
  • 가로 채기

마지막으로 다음과 같은 기타 옵션이 있습니다.

  • Bluetooth / iCade 지원 활성화 : 게임 프로젝트에 Bluetooth 또는 iCade 게임 패드에 대한 지원이 추가됩니다. 이것은 기본적으로 켜져 있습니다.
  • Lint Code Analysis 실행 :이 옵션이 플래그로 지정되면 빌드 도구는 빌드시 프로젝트에서 Lint 코드 분석을 실행하여 프로젝트가 컴파일 될 때 더 자세한 오류 확인을 제공하고 출력 창에 더 많은 정보를 표시합니다 (LOG 파일도 생성되며, 출력 창 에서 컴파일이 끝나면 파일 경로가 표시됩니다). 린트 분석에 의해 오류로 표시되는 모든 것이 반드시 게임에 문제가되는 것은 아니며,이 기능을 활성화하면 게임이 꺼져 있어도 더 이상 컴파일되지 않고 컴파일 및 실행됩니다. 기본적으로 이것은 디버깅 전용이며 옵션은 기본적으로 해제되어 있습니다.
  • 설치 위치 : 여기서 게임의 기본 설치 위치를 설정할 수 있습니다. 이것은 하나이 경우 게임이 자신의 장치에서 사용자가 선택한 위치에 설치되며, 자동으로 설정 될 수있다, 또는 항상 장치의 외부 저장 장치에 게임을 설치하려고 할 외부를 선호하도록 설정할 수 있습니다 .

여기에서 Android Sleep Margin을 설정할 수도 있습니다. 이 옵션은 특정 시스템에서 게임을 실행할 때 끊김을 줄이는 것과 관련이 있습니다. 기본적으로 게임이 룸 속도보다 빠르게 실행되는 경우 GameMaker Studio 2가 남은 시간 동안 "절전"상태가되지만이 절전 모드는 상당히 부정확 할 수 있으며 종종 필요 이상으로 절전 모드를 종료 할 수 있습니다. 말더듬을 유발합니다. 이 문제를 해결하려면 더 적은 시간 동안 잠을 자고 나머지 시간 동안 꽉 조여서 더 정확하게 만들 수 있습니다. 루프에 앉아있는 문제는 CPU 사용량이 증가한다는 것입니다. CPU 온도가 상승하고 장치가 더 많은 배터리를 사용하게합니다. 기본적으로이 값은 4로 설정되어 있으며 99.99 %의 경우에는 문제가 없지만 저급 장치 또는 백그라운드 프로세스가 많은 장치의 경우 이상적인 솔루션이 아니며 값이 아닐 수 있습니다 5 ~ 10이 필요할 수 있습니다. 이것은 매우 장치 별 구성이며 빌드 장치에서 작동하는 것은 다른 사용자에게 적합하지 않을 수 있으므로 의심스러운 경우 4로 설정하십시오.

 

적응 형 아이콘

Android 8.0 (API 레벨 26)에는 게임에 적합한 적응 형 실행기 아이콘이 도입 되었으며이 아이콘 은 다양한 장치 모델에서 다양한 모양을 표시 할 수 있습니다. 이 문제를 해결하려면 아이콘이 다른 크기로 표시 될 수 있도록 여러 개의 전경 및 배경 이미지를 제공해야합니다. 이 아이콘 크기는 다음과 같습니다.

  • 81x81 픽셀 (LDPI)
  • 108x108 픽셀 MDPI)
  • 162x162 픽셀 (HDPI)
  • 216x216 픽셀 (XHDPI)
  • 324x324 픽셀 (XXHDPI)
  • 432x432 픽셀 (XXXHDPI)

각 전경 및 배경 이미지를 개별적으로 제공하거나 파일에서 아이콘 생성을 선택할 수 있습니다. 즉, GameMaker Studio 2는 기본 (XXXHDPI) 파일에서 필요한 아이콘 이미지를 생성합니다. 전경과 배경에 대해 각각 하나의 이미지를 제공 한 다음 생성이라고 표시된 버튼을 클릭하면 아이콘이 만들어집니다. 이미지는 모두 * .png 형식 이어야 합니다.

적응 형 아이콘은 특정 장치에서 마스킹 될 수 있으며, 적응 형 아이콘 미리보기 메뉴에서 이러한 마스크와 함께 어떻게 표시되는지 확인할 수도 있습니다.

이것은 선택된 마스크로 아이콘 이 어떻게 보이는지에 대한 미리보기이며, 설정은 아이콘이 게임이 실행되는 장치 및 OS에 따라 아이콘을 내보내고 표시하는 방법에 영향을 미치지 않습니다. 또한이 아이콘은 Project Image Generator 도구를 사용할 때 수행 된 이미지 생성의 일부로 생성 되지 않습니다 .

 

제도법

다음 옵션을 사용하여 게임 표시 방법과 관련된 다음 세부 정보를 변경할 수 있습니다.

  • 픽셀 간 색상 보간 : 기본적으로 "매끄러운"픽셀 인 선형 보간을 켭니다. 선명한 픽셀 그래픽의 경우 꺼져 있어야하지만 멋진 알파 블렌드와 매끄러운 가장자리 그래픽이있는 경우 그대로 유지하는 것이 좋습니다. 이것은 기본적으로 켜져 있습니다.
  • Screen Color Depth : 렌더링의 색 농도를 16 비트 또는 24 비트로 설정하는 데 사용할 수 있습니다. 이는 24 비트로 설정된 경우 일부 구형 디바이스와의 호환성에 영향을 미치며 필요한 불확실성 을 증가시킵니다 (이것은 AmazonFire 디바이스의 공유 메모리입니다).
  • 장치 지원 :이 옵션은 전용 GPU가있는 장치 만 지원할지 또는 모든 장치를 지원 할지를 설정합니다. 기본적으로 게임에 알파 블렌딩, 첨가제 블렌딩, 표면 등과 같은 고급 드로잉 기능이있는 경우 GPU가있는 장치로 제한해야합니다. 그렇지 않으면 게임에 부정적인 의견이 나고 등급이 떨어질 위험이 있습니다. 구형 장치의 그래픽 오류 또는 충돌.
  • 크기 조정 : 여기에서 화면에 맞게 가장자리 주위에 "패딩"을 추가하여 장치 화면 내에서 가로 세로 비율을 유지하여 그리기 캔버스의 배율을 조정하거나 장치 화면에 맞게 그리기 캔버스를 확장하도록 선택할 수 있습니다.

마지막으로 텍스처 페이지 의 크기를 설정하는 옵션이 있습니다. 기본 (가장 호환되는) 크기는 2048x2048이지만 256x256에서 최대 8192x8192 사이에서 선택할 수 있습니다. 또한이 플랫폼에 대한 텍스처 페이지를 생성 한 다음 미리보기 로 표시된 단추가 있으며, 창을 열어서 모양을 볼 수 있습니다. 텍스처 페이지가 어떻게 구성되어 있는지 확인하고 텍스처 페이지가 필요 이상으로 커지는 것을 방지하려는 경우 매우 유용 할 수 있습니다. 텍스처 페이지에 대한 자세한 내용은 여기 를 참조 하십시오 .

참고 : 텍스처 페이지의 크기가 클수록 게임이 다른 브라우저 및 장치와 호환되지 않습니다.

 

아이콘

이 섹션에서는 프로젝트가 다른 상점 페이지 및 장치에 필요한 다양한 아이콘 이미지를 추가 할 수 있습니다. 모든 아이콘 이미지는 주어진 적절한 크기의 24 비트 * .png 파일로 생성되어야합니다.

GameMaker Studio 2에는 게임이 컴파일되는 모든 다른 대상 플랫폼에 필요한 모든 이미지를 자동으로 만드는 데 사용할 수있는 프로젝트 이미지 생성기 도구가 있습니다. 이 도구를 사용하는 경우 필요한 이미지가되도록 만들어진 이미지를 수정해야합니다. 이 도구는 이미지뿐만 아니라 아이콘도 생성합니다 (아래 섹션 참조).

 

이미지

여기에서 전체 화면을 채우도록 설정되지 않은 스플래시 화면의 배경으로 사용될 테두리 색 을 설정할 수 있습니다. 게임이 장치에로드되는 동안 표시되는 가로 및 세로 모드 모두에 대해 최종 게임 파일에 스플래시 화면 을 추가 할 수도 있습니다. 이 화면은 24 비트 * .png 파일 이어야 하며 게임의 첫 번째 방 (또는보기)과 동일한 크기를 권장합니다. 특정 시간 동안 스플래시 화면을 표시해야하는 경우 여기에서 0-10 초 범위로 설정할 수도 있습니다 (기본 시간 0은 자산로드 기간 동안 만 표시됨을 의미 함). 다음 옵션에서 스플래시 화면을 표시하는 방법을 설정할 수도 있습니다.

  • 가로 세로 비율 유지 : 장치 화면과 비교하여 이미지의 가로 세로 비율을 유지하여 필요한 경우 가장자리 주위에 "패딩"을 추가합니다.
  • 자르기 맞춤 : 장치 화면에서 이미지를 1 : 1로 유지하고 필요에 따라 가장자리를 자르거나 채 웁니다.
  • 늘이기 : 이미지를 늘려 장치 화면을 채 웁니다 (표시된 이미지가 왜곡 될 수 있음).

마지막으로 24 비트 * .png 인 Android TV 장치에서 게임의 배너 이미지 를 설정할 수 있습니다.

 

포장

Android 게임에 Google Play를 사용하는 기능 중 하나는 게임 라이센스 를 부여 하는 기능입니다. 즉, 마켓을 통해 게임을 다운로드 한 사람 만 게임을 플레이 할 수 있으므로 게임의 불법 복제물이 배포되는 것을 방지 할 수 있도록 Google에서 게임을 제어합니다.

게임에서이 기능을 활성화하려면 먼저 Google 라이센싱 활성화 상자에 플래그를 지정한 다음 퍼블릭 키 를 Google 라이센싱 공개 키 상자에 복사해야합니다. 이 라이센스는 Google Play 개발자 페이지 로 이동 한 다음 편집 할 게임을 선택하면 찾을 수 있습니다. 게임 프로필 페이지에서 " 서비스 및 API "섹션까지 아래로 스크롤하면이 기능을 활성화하는 데 필요한 공개 키가 있습니다.

활성화되면 게임을 소유 할 권한이없는 사람이 게임을 시도하면 해당 장치에서 라이센스를 찾을 수 없다는 팝업 메시지 상자가 표시되고 다시 시도 할 수있는 옵션이 제공됩니다. 다시 테스트하거나 게임을 종료하십시오.

라이센싱의 또 다른 기능은 게임에서 APK 확장 을 사용할 수 있다는 것입니다. 이는 Google Play에서 부과 한 최대 50MB의 애플리케이션 크기를 우회하는 방법입니다 (Google Play 만 해당). 작은 * .apk 파일과 더 큰 * .zip 파일의 두 부분으로 게임을 생성합니다.이 파일은 모든 게임 자산을 저장하며 둘 다 상점에 업로드됩니다. 사용자가 처음 게임을 실행하면 zip이 다운로드되고 압축이 풀립니다. 이 옵션에 대한 자세한 내용은 여기를 참조 하십시오 .

마지막으로 최종 앱 패키지를 Android TV 게임으로 준비 할 수 있습니다 (게임이 Android TV 지침을 준수하는 경우에만이 옵션을 선택해야 함).

 

권한

이 탭에서 게임이 Android Manifest 내에서 요청할 수있는 일부 권한을 변경할 수 있습니다. GameMaker Studio 2는 게임이 사용하는 기능에 따라 필요에 따라 권한을 자동으로 할당하지만, 사용자 지정 확장 프로그램을 사용할 때와 같이 올바르게 할당되지 않은 게임에 대한 권한이 필요한 경우 필요한 권한 상자.

아래 목록에는 사용 가능한 권한이 요약되어 있습니다.

  • WRITE_EXTERNAL_STORAGE : 게임이 외부 저장소에 쓸 수 있도록합니다.
  • READ_PHONE_STATE : 전화 상태에 대한 읽기 전용 액세스를 허용합니다.
  • ACCESS_NETWORK_STATE : 게임이 네트워크에 대한 정보에 액세스하도록 허용합니다.
  • 인터넷 : 게임이 네트워크 소켓을 열도록 허용합니다.
  • 블루투스 : 게임이 페어링 된 블루투스 장치에 연결되도록합니다.
  • RECORD_AUDIO : 게임에서 마이크의 오디오 입력을 녹음 할 수 있습니다.

참고 : 이것들이 필요한지 확실하지 않다면, 기본적으로 체크하지 않은 상태로두고 GameMaker Studio 2가 게임 권한을 처리하도록해야합니다.

안드로이드 권한 secton 또한 당신이 AndroidManifest.xml에있는 안드로이드 응용 프로그램 태그에 권한을 주입 할 수있는 영역이있다. 사용 가능한 태그의 전체 목록은 다음 Android 설명서에서 찾을 수 있습니다.

  • 안드로이드 개발자 : 애플리케이션 태그

대상 API 28 이상에서는 다음 태그를 추가하지 않는 한 " http "- " https "를 사용하는 외부 웹 호출을 더 이상 사용할 수 없습니다.

사회적인

이 섹션은 Android 게임에서 다양한 소셜 기능을 활성화하기위한 것입니다. 다음과 같은 옵션을 선택할 수 있습니다.

  • Facebook : Facebook 기능이 필요한 경우이 확인란을 선택한 다음 Facebook 앱 ID 및 Facebook 앱 이름을 제공해야 합니다. 모든 Facebook 기능은 확장을 통해 액세스되며, Facebook 사용 상자를 처음 선택하면 마켓 플레이스 에서 Facebook 확장을 다운로드하여 설치하라는 메시지가 표시됩니다.
  • Google – 클라우드 절약 사용 :이를 선택하면 Google 클라우드 절약이 활성화되고 마켓 플레이스 에서 관련 확장을 다운로드하여 설치하라는 메시지가 표시됩니다. 비동기 함수 페이지에서 클라우드 절약 섹션을 참조하십시오.
