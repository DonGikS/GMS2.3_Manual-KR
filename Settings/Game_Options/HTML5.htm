HTML5 게임 옵션

이 섹션에서는 HTML5 게임 프로젝트를 컴파일하는 방법을 제어 할 수있는 다양한 옵션에 대해 간략하게 설명합니다. 다른 섹션은 다음과 같습니다.

 

일반

이 섹션에는 게임에 대한 다음 정보를 입력 할 필드가 있습니다.

  • 브라우저 제목 : 여기서 브라우저에서 게임 탭의 제목 표시 줄에 사용될 텍스트를 제공 할 수 있습니다.
  • 버전 : 프로젝트의 버전 번호입니다.
  • 폴더 이름 : HTML5 게임 파일이 저장 될 폴더의 이름입니다 (기본값은 " html5game ").
  • 출력 이름 : 출력 HTML 파일 이름이며 기본 " index.html "이외의 다른 이름으로 변경할 수 있습니다.

 

게임 정보 필드 아래에는 다음과 같은 옵션도 있습니다.

  • Output Debug To Console : 모든 디버그 출력을 브라우저의 JS 콘솔로 보냅니다. 기본적으로 해제되어 있습니다.
  • Display Cursor : 게임 캔버스 위에있을 때 마우스 커서를 비활성화합니다. 이것은 기본적으로 켜져 있습니다.
  • "외부 서버 실행"경고 표시 : 게임을 로컬에서 테스트하는 경우 (예 : 서버가 아닌 컴퓨터에서 색인 파일을 클릭하면 권장되지 않음) 일반적으로 다음과 같이 작동하지 않는다는 경고가 표시됩니다. 로컬 파일의 브라우저 보안으로 인해 예상 될 수 있습니다. 이 옵션을 선택 취소하여이 경고를 표시하지 않을 수 있습니다. 기본적으로 켜져 있습니다.

일반 설정 아래에는 다음과 같은 고급 옵션이 있습니다.

  • index.html로 포함 된 파일 : GameMaker Studio 2가 기본적으로 생성하는 사전 정의 된 파일 대신 사용할 사용자 정의 "index.html"파일을 추가하기위한 것입니다. 웹 사이트 디자인을 구현했거나 모든 HTML5 빌드에 특정 기능을 포함하려는 경우 매우 유용합니다. 사용할 파일 은 게임 프로젝트에 포함 파일 로 포함되어야합니다.
  • 로딩 바 확장 : GameMaker Studio 2 프로젝트에 확장 으로 추가 된 기본 로딩 바를 변경합니다.
  • Prepend output .js : 게임에 필요한 추가 JavaScript로 * .js 출력을 앞에 추가 할 수 있습니다. 예를 들어 Chrome 스토어 앱에서 window.localStorage = undefined; 이 옵션을 사용하는 출력 JS에.

 

해석학

이 섹션에서는 게임에 분석을 추가하여 특정 타사 제품을 사용하여 게임 및 판매 또는 플레이를 추적 할 수 있습니다. Flurry Analytics 또는 Google Analytics 를 사용하여 게임을 추적할지 여부를 지정할 수 있습니다. 이러한 기능의 특성으로 인해 한 번에 하나만 활성화 할 수 있으며 Flurry Analytics 사용 또는 Google Analytics 사용으로 플래그를 지정한 후에는 해당 영역에 지정된 고유 ID를 추가해야합니다.

분석 데이터를 전송하는 데 사용할 수있는 기능에 대한 자세한 내용은 여기 를 참조 하십시오 .

 

제도법

여기에서 게임 표시 방법과 관련된 다음 세부 정보를 변경할 수 있습니다 (일부 브라우저에서는 동일한 방식으로 게임이 표시되지 않으므로 가능한 많은 브라우저에서 이러한 옵션을 테스트하여 원하는 효과가 있는지 확인하십시오) :

  • 전체 화면 전환 허용 :이 옵션을 선택하면 사용자가 게임을 창에서 전체 화면 모드로 전환 할 수 있습니다 / Mac에서는 + " F ", Windows에서는 F10 이것은 기본적으로 켜져 있습니다.
  • 픽셀 간 색상 보간 : 기본적으로 "매끄러운"픽셀 인 선형 보간을 켭니다. 선명한 픽셀 그래픽의 경우 꺼져 있어야하지만 멋진 알파 블렌드와 매끄러운 가장자리 그래픽이있는 경우 그대로 유지하는 것이 좋습니다. 이것은 기본적으로 켜져 있습니다.
  • 브라우저 중앙에 게임 센터 : 체크하면 게임을 실행하는 브라우저 내에서 게임 화면이 자동으로 중앙에 위치합니다. 기본적으로 해제되어 있습니다.
  • 내장 파티클 사용 :이 옵션은 내장 파티클 시스템에 필요한 스프라이트를 게임과 함께 내 보냅니다. 이 스프라이트는 개별 텍스처 페이지에 저장되므로이 플랫폼에서 파티클을 생성하는 가장 효율적인 방법이 아니므로 (WebGL을 사용하지 않는 한)이 옵션을 선택 취소하고 나만의 파티클 스프라이트를 사용하는 것이 좋습니다. 게임의 메인 텍스처 페이지에 모두 저장됩니다. 이것은 기본적으로 켜져 있습니다.
  • 내장 글꼴 사용 :이 옵션은 게임 파일의 나머지 부분과 함께 내장 글꼴로 GameMaker Studio 2를 내 보냅니다. 텍스처 페이지의 크기가 약간 증가하고 약간의 추가 메모리와 저장 공간이 필요하므로 게임에서 내장 Arial 글꼴을 사용하지 않는 경우 비활성화해야합니다. 기본적으로 켜져 있습니다.
  • WebGL : WebGL 지원을 끄거나, 선택적으로 (자동 감지) 또는 게임 요구 사항으로 만들지 여부를 선택하십시오. 모든 상황에서 성능을 테스트하려면이 기능을 비활성화 한 다음 일반적으로 게시를 위해 자동 검색하도록 설정해야합니다. 요구 사항을 만들면 게임의 브라우저 유형과의 호환성이 상당히 떨어질 수 있습니다. 기본적으로 자동 감지로 설정되어 있습니다.
  • 스케일링 : 브라우저 내에서 가로 세로 비율 을 유지 하도록 드로우 캔버스를 자동으로 스케일링하도록 게임을 구성하거나 풀 스케일로 실행하도록 선택할 수 있습니다. 풀 스케일 옵션은 브라우저에서 게임을 전체 화면으로 표시하지 않고 게임의 첫 번째 방에서 정의한대로 캔버스 크기에 맞게 그린 것을 늘립니다. 기본적으로 종횡비를 유지하도록 설정되어 있습니다.

마지막으로 텍스처 페이지 의 크기를 설정하는 옵션이 있습니다. 기본 (가장 호환되는) 크기는 2048x2048이지만 256x256에서 최대 8192x8192 사이에서 선택할 수 있습니다. 또한이 플랫폼에 대한 텍스처 페이지를 생성 한 다음 창을 열어서 모양을 볼 수있는 미리보기 버튼이 있습니다. 텍스처 페이지가 어떻게 구성되어 있는지 확인하고 텍스처 페이지가 필요 이상으로 커지는 것을 방지하려는 경우 매우 유용 할 수 있습니다. 텍스처 페이지에 대한 자세한 내용은 여기 를 참조 하십시오 .

참고 : 텍스처 페이지의 크기가 클수록 게임이 다른 브라우저 및 장치와 호환되지 않습니다.

 

이미지

이 섹션에서는 Favicon 및 스플래시 화면을 정의 할 수 있습니다. 아이콘은 * .ico 파일로 작성되어야하며 크기는 16x16, 32x32, 48x48 또는 64x64 픽셀이고 색상 깊이는 8 비트, 24 비트 또는 32 비트 일 수 있습니다. 스플래시 화면은 게임을 유지하려는 캔버스의 크기 여야하며 스플래시 화면 사용 옵션을 선택하면 로딩 막대를 재정의합니다 (예 : 로딩 막대 또는 스플래시 화면을 가질 수 있지만 둘다는 아님) . 더 작거나 큰 이미지는 허용되지만 크기가 맞도록 조정됩니다. 스플래시 화면은 * .png 이미지 여야합니다.

GameMaker Studio 2에는 게임이 컴파일되는 모든 다른 대상 플랫폼에 필요한 모든 이미지를 자동으로 만드는 데 사용할 수있는 프로젝트 이미지 생성기 도구가 있습니다. 이 도구를 사용하는 경우 필요한 이미지가되도록 만들어진 이미지를 수정해야합니다.

 

사회적인

소셜 탭은 게임에서 Facebook 통신을 설정하는 데 사용됩니다. Facebook 기능이 필요한 경우 "Facebook 사용"확인란을 클릭 한 다음 Facebook 앱 ID 및 Facebook 앱 표시 이름을 제공해야 합니다. 모든 Facebook 기능은 확장 프로그램을 통해 액세스되며, Facebook 사용 상자를 처음 선택하면 마켓 플레이스 에서 Facebook 확장 프로그램을 다운로드하여 설치하라는 메시지가 표시됩니다 (다운로드하기 전에 YoYo 계정에 로그인 해야 할 수도 있음) .
