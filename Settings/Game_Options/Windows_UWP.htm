Windows UWP 게임 옵션

이 섹션에서는 UWP (유니버설 Windows 플랫폼) 게임 프로젝트를 컴파일하는 방법을 제어 할 수있는 다양한 옵션에 대해 간략하게 설명합니다. 다른 섹션은 다음과 같습니다.

 

일반

UWP 옵션의 일반 섹션에서는 제품 정보를 설정할 수 있습니다.

  • 표시 이름 : 사용자에게 표시되는 앱의 이름을 지정합니다 (이 문자열은 지역화 가능합니다).
  • 패키지 이름 : 생성 한 패키지 이름으로 게임용으로 만든 개발자 키와 일치해야합니다 (아래 패키징 섹션 참조).
  • 패키지 표시 이름 : 패키지 의 "짧은 이름"이며 표시 이름과 동일해야합니다.
  • 게시자 표시 이름 : 회사 또는 게임 게시자로 연결하려는 사람의 이름입니다.
  • 버전 : 게임의 버전 번호입니다.

설정 후 최상의 사용자 경험을 위해 게임을 실행할 수있는 다른 방향 을 선택할 수 있습니다. 사용 가능한 옵션은 다음과 같습니다.

  • 경치
  • 초상화
  • 가로 채기
  • 세로로 뒤집힌

기본적으로 이러한 모든 항목이 선택되므로 필요하지 않은 항목은 선택 해제해야합니다.

방향 옵션을 설정 한 후 다음 옵션에서 UWP 앱에 필요한 다른 권한 을 알릴 수 있습니다.

  • 인터넷 권한 활성화 : 게임에 인터넷 액세스 권한을 부여합니다. 기본적으로 해제되어 있습니다.
  • 마이크 사용 권한 : 게임을 장치의 마이크에 액세스 할 수있는 것으로 표시합니다. 기본적으로 해제되어 있습니다.
  • 인앱 구매 샌드 박스 : 샌드 박스 인앱 구매를 사용하는 것으로 게임에 플래그를 지정합니다. 기본적으로 해제되어 있습니다.

마지막으로 대상 및 최소 Windows 플랫폼 버전을 설정하기위한 두 가지 옵션이 있습니다. 일반적으로이를 만져보고 싶지는 않지만 기본값을 그대로 두어야합니다. 그러나 빌드 문제가있는 경우, 특히 기본적으로 Windows SDK 세트를 설치할 수없는 경우이를 변경해야합니다. 당신이 설치 한 것에-그런 다음 다른 값으로 설정할 수 있습니다. 설정할 값을 잘 모르는 경우 Visual Studio에서 새 UWP 프로젝트를 만들고 값을 설정하는 것을 살펴보십시오.

 

제도법

그래픽 옵션은 게임에서 대상 UWP 장치의 그래픽 카드를 사용하는 방법을 결정하기 위해 구성해야하는 옵션입니다. 수정할 수있는 옵션은 다음과 같습니다.

  • 픽셀 간 색상 보간 : 기본적으로 "매끄러운"픽셀 인 선형 보간을 켭니다. 선명한 픽셀 그래픽의 경우 꺼져 있어야하지만 알파 블렌드와 가장자리가 매끄럽고 멋진 그래픽이 있으면 기본적으로 꺼져 있습니다.
  • 커서 표시 :이 옵션이 활성화되면 기본 OS 커서가 게임에 표시됩니다. 이 기능을 비활성화하면 마우스가 게임 창 위에있는 동안 게임 프로젝트의 일부로 마우스 커서를 만들지 않으면 커서가 보이지 않습니다 (기본적으로 켜져 있음).
  • 전체 화면 시작 :이 옵션을 선택하면 게임이 전체 화면 모드에서 시작되고 그렇지 않으면 창이 시작됩니다. 기본적으로 해제되어 있습니다.
  • 전체 화면 전환 허용 :이 확인란을 선택하면 표준 Ubuntu OS 바로 가기를 사용하여 전체 화면에서 창으로 전환 한 후 다시 전환 할 수 있습니다. 기본적으로 해제되어 있습니다.
  • 크기 조정 : 가로 세로 비율을 유지하거나 (16 : 4에서 4 : 3 회의실이 "편지함"으로) 또는 전체 크기로 전체 화면에 맞게 이미지를 늘릴 수 있습니다. 경고! 응용 프로그램 화면을 끄면 다시 켜질 때까지 Windows 게임 옵션에 설정된 모든 크기 조정 옵션이 비활성화됩니다. 자세한 내용은 응용 프로그램 표면을 참조하십시오.

마지막으로 텍스처 페이지 의 크기를 설정하는 옵션이 있습니다. 기본 (가장 호환되는) 크기는 2048x2048이지만 256x256에서 최대 8192x8192 사이에서 선택할 수 있습니다. 또한이 플랫폼에 대한 텍스처 페이지를 생성 한 다음 창을 열어서 모양을 볼 수있는 미리보기 버튼이 있습니다. 텍스처 페이지가 어떻게 구성되어 있는지 확인하고 텍스처 페이지가 필요 이상으로 커지는 것을 방지하려는 경우 매우 유용 할 수 있습니다. 텍스처 페이지에 대한 자세한 내용은 여기 를 참조 하십시오 .

참고 : 텍스처 페이지의 크기가 클수록 게임 사양이 낮은 PC에서 게임 호환성이 떨어집니다.

 

이미지

이미지 섹션의 첫 번째 부분은 게임 로고 를 정의하기위한 것입니다. 이것은 본질적으로 게임이 UWP 생태계 내에서 사용할 아이콘이며,이 이미지는 투명한 .png 파일 일 수 있으며,이 경우이 섹션에 지정된 배경색 위에 그려 질 것입니다. 이 배경색은 앱 소유 대화 상자의 버튼 색과 장치에 저장의 앱 설명 페이지에도 적용됩니다.

로고를 설정 한 후 게임의 스플래시 화면 을 설정할 수 있습니다. 게임을 시작할 때 표시되는 화면이며 620 x 300 픽셀보다 클 수 없으며 로고와 같이 투명한 .png 파일이 될 수 있습니다.이 경우 선택한 배경색 이 사용됩니다 .

마지막으로 UWP에서 실행할 수있는 다른 장치에서 게임에서 사용할 다양한 타일 을 설정할 수 있습니다. 다음을 설정할 수 있습니다 (모두 * .png 형식 파일 이어야 함 ).

  • 작은 타일 이미지는 검색 창 및 UI의 다른 영역에서 앱 이름 옆에 나타나는 로고 이미지입니다. 44 x 44 픽셀이어야합니다.
  • 중간 타일 이미지는 71x71 픽셀 이미지 여야합니다.
  • 다음 타일 이미지는 스토어 로고 타일 이며 50 x 50 픽셀이어야합니다.
  • 넓은 이미지 타일 은 타일이 넓은 형식 일 때 표시되며 310 x 150 픽셀이어야합니다. 이 이미지가 제공되지 않으면 타일은 정사각형 형식으로 만 표시 될 수 있으며 넓은 템플릿 유형을 기반으로하는 알림을 수락 할 수 없습니다. 사용자가 타일 사용 형식을 결정할 수 있으므로 넓은 이미지 타일을 포함하는 것이 좋습니다. 또한 넓은 이미지 타일이 제공되면 처음에 타일이 넓은 형식으로 나타날 수 있습니다.
  • 큰 이미지 타일 은 큰 라이브 타일 형식이며 310 x 310 픽셀이어야합니다. 이것은 데스크톱 OS에서만 사용됩니다.

GameMaker Studio 2에는 게임이 컴파일되는 모든 다른 대상 플랫폼에 필요한 모든 이미지를 자동으로 만드는 데 사용할 수있는 프로젝트 이미지 생성기 도구가 있습니다. 이 도구를 사용하는 경우 필요한 이미지가되도록 만들어진 이미지를 수정해야합니다.

 

포장

GameMaker Studio 2는 Windows UWP 개발을위한 기본 개발자 키 를 제공하지만 직접 생성하여 여기를 지정하는 것이 좋습니다. 이렇게하려면 Visual Studio에서 빈 프로젝트를 만든 다음 빈 프로젝트에 포함 할 자동 생성 된 * .pfx 파일의 복사본을 안전한 곳에 저장하십시오 (개발자 키임). 그런 다음 * .pfx 파일을 여기에 연결해야합니다.

파일에 연결 한 후 관련 설치 버튼을 클릭 한 다음 열리는 명령 프롬프트의 지침을 따라야합니다. 이렇게하지 않으면 Run / Debug 버튼을 사용하거나 F5를 누르면 UWP 플랫폼에서 게임을 테스트 할 수 없습니다. 그러나 Create Executable 을 사용하면 * 인 경우 인증 설치 프로세스가 자동으로 시작됩니다 . pfx 는 아직 등록되지 않았습니다.

참고 : 앱을 스토어에 업로드 할 때는 타이틀에 맞는 * .pfx 를 가져와야 합니다. 이 작업은 Visual Studio에서 앱을 Windows 스토어와 연결하여 Visual Studio가 앱에 대해 새 * .pfx 를 생성하도록합니다. 또한 Visual Studio에서 모든 앱을 Windows 스토어와 연결할 수 있습니다. 즉, 빈 프로젝트를 Windows 스토어 앱과 연결하고 생성 된 * .pfx 를 안전한 위치에 복사 한 다음 UWP 내에서 사용할 수 있습니다. 게임 옵션. 이 작업이 수행되는 방법에 대한 지침은 YoYo Games Knowledge Base 에서 찾을 수 있습니다.

이 섹션에서는 서명 인증서를 생성하는 데 사용 된 게시자 이름도 지정해야합니다. 입력은 항상 " CN = "으로 시작해야합니다 (예 : CN = YoYoGames) .

 

엑스 박스 라이브

UWP 게임 옵션의 Xbox Live 섹션은 ​​Xbox One 본체에서 사용할 UWP 프로젝트를 활성화 할 수있는 곳입니다. Xbox 용 프로젝트를 활성화하면 프로젝트가 Xbox One에서 실행될 때 다음을 수행 할 수 있습니다.

  • 앱 내에서 Xbox Live에 로그인
  • 프로젝트에서 Xbox Live 사용자 ID를 사용하여 (예 : 게이머 태그 또는 게이머 사진 표시)

그러나 이러한 작업을 수행하기 전에 Xbox Live 사용 확인란을 클릭했는지 확인해야하며 Microsoft Creators Program에 속한 경우 Creators Program 앱 옵션도 확인해야합니다. 크리에이터 프로그램의 일부인 경우 타이틀 ID 및 서비스 구성 ID 를 추가해야합니다 (자세한 내용은 Microsoft 에서 찾을 수 있음).

현재 프로젝트에서 Xbox Live 기능을 테스트하려면 UWP Xbox 출력에서 ​​패키징 정보 (위 참조) 대신 Windows 스토어 서명 인증서를 사용해야하므로 실행 대신 실행 파일 만들기 를 사용해야합니다. GameMaker Studio 2와 함께 제공되는 임시 버전으로 Run을 눌렀을 때 사용됩니다.

XBox Live를 활성화 한 후에는 프로젝트에서 XBox Live 기능을 사용할 수 있지만, 가장 기본적인 기능은 제작자 프로그램에 참여 해야하며 고급 기능을 위해서는 ID @ 가 필요하다는 점에 유의하십시오. XBox 계정 ..
