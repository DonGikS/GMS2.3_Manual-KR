Windows 게임 옵션

이 섹션에서는 Windows 게임 프로젝트를 컴파일하는 방법을 제어 할 수있는 다양한 옵션에 대해 간략하게 설명합니다. 다른 섹션은 다음과 같습니다.

 

일반

Windows 게임의 일반 페이지는 게임이 설치되어 실행될 때 Windows가 표시하는 데 필요한 정보를 제공해야합니다. 이 정보는 제품 정보 섹션에 있으며 완전히 기입해야합니다.

이 창의 옵션 섹션에서는 " % localappdata % "또는 " % appdata % "폴더에 게임 패키지를 설치할 위치를 선택하고 exe를 출력 폴더에 복사 할지 여부를 선택할 수 있습니다. 일반적으로이 옵션은 필요하지 않지만 서로 DLL이 게임과 통신하는 다양한 DLL이있는 사용자에게 제공되며, 활성화되면 테스트 할 때 게임 폴더를 테스트 폴더에 배치하지 않고 출력 폴더에 유지합니다. Temp 폴더

다음으로 Windows Sleep Margin을 설정할 수 있습니다. 이 옵션은 특정 시스템에서 게임을 실행할 때 끊김을 줄이는 것과 관련이 있습니다. 기본적으로 게임이 룸 속도보다 빠르게 실행되는 경우 GameMaker Studio 2가 남은 시간 동안 "절전"상태가되지만이 절전 모드는 상당히 부정확 할 수 있으며 종종 필요 이상으로 절전 모드를 종료 할 수 있습니다. 말더듬을 유발합니다. 이 문제를 해결하려면 더 적은 시간 동안 잠을 자고 남은 시간 동안 더 정확하게 잠그면 더 정확 해집니다.-루프에 앉아있는 문제는 CPU 사용량이 증가한다는 것입니다. CPU 온도가 상승하고 팬이 더 빨리 작동합니다. 기본적으로이 값은 10으로 설정되어 있으며 99.99 %의 경우에는 문제가 없지만 저급 컴퓨터 나 백그라운드 프로세스가 많은 컴퓨터의 경우 이상적인 솔루션이 아닐 수 있습니다. 매우 높은 수준의 장비도 필요) 1 또는 15 또는 그 이상의 값이 필요할 수 있습니다. 이것은 시스템 고유의 구성이므로 빌드 머신에서 작동하는 것은 다른 사용자에게는 적합하지 않을 수 있습니다. 의심스러운 경우 10으로 설정하십시오.

다음 옵션은 게임에서 Steam 을 활성화 하는 것입니다. 이 옵션을 활성화하면 Steamworks SDK가 설치되어 있고 ( 여기 참조) 게임의 일반 옵션 에 Steam 앱 ID 를 추가했는지 확인하십시오. Windows 및 IDE의 Steam 버전에서 GameMaker Studio 2를 실행 중이고 Steam 용 게임을 개발하는 경우 Steam 용 대체 실행기 사용 옵션도 확인해야합니다. 이 옵션의 기능은 대체 실행기 프로그램을 사용하여 테스트 할 때 게임 자체 응용 프로그램 ID 대신 GameMaker Studio 2의 응용 프로그램 ID로 Steam에 표시되는 것을 방지하기 위해 게임을 시작하는 것입니다. Steam 용으로 개발할 때와 게임 시작에 문제가있는 경우에만이 기능을 활성화해야합니다.

마지막으로 파일 시스템 샌드 박스 비활성화 옵션이 있습니다. 이를 체크하면 파일을 저장하고로드하기위한 GameMaker Studio 2 샌드 박싱이 비활성화되어 게임이 게임을 실행하는 시스템의 어느 곳에서나 파일에 액세스 할 수 있습니다. 모든 시스템의 파일에 대해 GameMaker Studio 2 샌드 박스를 유지하는 것이 좋으며이 옵션을 선택하면 OS 수준 권한에 따라 일부 저장 /로드 영역을 사용하지 못할 수도 있습니다. 샌드 박스에 대한 자세한 내용은 GameMaker Studio 2 파일 시스템 페이지를 참조하십시오.

 

제도법

그래픽 옵션은 게임에서 대상 Windows PC의 그래픽 카드를 사용하는 방법을 결정하기 위해 구성해야하는 옵션입니다. 수정할 수있는 옵션은 다음과 같습니다.

  • 커서 표시 : 이것이 체크되면 일반 윈도우 커서가 표시되며, 체크 해제하면 게임 코드에서 커서를 만들지 않으면 커서가 표시되지 않습니다. 이것은 기본적으로 켜져 있습니다.
  • 전체 화면 시작 :이 옵션을 선택하면 게임이 전체 화면 모드에서 시작되고 그렇지 않으면 창이 시작됩니다. 기본적으로 해제되어 있습니다.
  • 전체 화면 전환 허용 :이 확인란을 선택하면 표준 Windows 바로 가기를 사용하여 전체 화면에서 창으로 전환 한 후 다시 전환 할 수 있습니다. 기본적으로 해제되어 있습니다.
  • 픽셀 간 색상 보간 : 기본적으로 "매끄러운"픽셀 인 선형 보간을 켭니다. 선명한 픽셀 그래픽의 경우 꺼져 있어야하지만 멋진 알파 블렌드와 매끄러운 가장자리 그래픽이있는 경우 더 나은 상태로 유지하는 것이 좋습니다.
  • 동기화를 사용하여 테어 링 방지 : v-sync를 켜거나 끕니다. 룸 속도가 120 인 게임의 플레이어에 새로 고침 빈도가 60 인 모니터가있는 경우이 옵션을 켜면 게임 속도도 60으로 고정됩니다. 기본적으로 해제되어 있습니다.
  • 창 크기 조정 허용 :이 옵션을 선택하면 사용자가 게임 창의 크기를 변경할 수 있습니다 (이 작업을하려면 Borderless Window 옵션이 꺼져 있어야합니다). 이 옵션은 기본적으로 해제되어 있습니다.
  • 경계선없는 창 :이 항목을 선택하면 경계선없는 창으로 게임이 실행되어 일반 최소화, 제거 및 버튼과 게임 제목이 제거됩니다. 기본적으로 해제되어 있습니다.
  • 크기 조정 : 가로 세로 비율을 유지하거나 (16 : 4에서 4 : 3 회의실이 "편지함"으로) 또는 전체 크기로 전체 화면에 맞게 이미지를 늘릴 수 있습니다. 경고! 응용 프로그램 화면을 끄면 다시 켜질 때까지 Windows 게임 옵션에 설정된 모든 크기 조정 옵션이 비활성화됩니다. 자세한 내용은 응용 프로그램 표면을 참조하십시오.

마지막으로 텍스처 페이지 의 크기를 설정하는 옵션이 있습니다. 기본 (가장 호환되는) 크기는 2048x2048이지만 256x256에서 최대 8192x8192 사이에서 선택할 수 있습니다. 또한이 플랫폼에 대한 텍스처 페이지를 생성 한 다음 창을 열어서 모양을 볼 수있는 미리보기 버튼이 있습니다. 텍스처 페이지가 어떻게 구성되어 있는지 확인하고 텍스처 페이지가 필요 이상으로 커지는 것을 방지하려는 경우 매우 유용 할 수 있습니다. 텍스처 페이지에 대한 자세한 내용은 여기 를 참조 하십시오 .

참고 : 텍스처 페이지의 크기가 클수록 게임 사양이 낮은 PC에서 게임 호환성이 떨어집니다.

 

이미지

이미지 섹션은 게임에 필요한 이미지를 제공하는 곳입니다. Windows의 경우 아이콘 파일 ( .ico 형식이어야 함)과 시작 화면 ( .png , .bmp , .jpg 또는 .gif 형식)을 제공해야합니다. 게임이로드되는 동안 스플래시 화면이 표시되며 스플래시 화면 사용 옵션 (기본적으로 해제되어 있음)을 선택하여 활성화 또는 비활성화 할 수 있습니다.

GameMaker Studio 2에는 게임이 컴파일되는 모든 다른 대상 플랫폼에 필요한 모든 이미지를 자동으로 만드는 데 사용할 수있는 프로젝트 이미지 생성기 도구가 있습니다. 이 도구를 사용하는 경우 필요한 이미지가되도록 만들어진 이미지를 수정해야합니다.

 

인스톨러

설치 프로그램을 사용하여 Windows 게임을 만들 수 있으며 여기에서 GameMaker Studio 2를 만들 때 사용할 다양한 설정 및 그래픽을 정의 할 수 있습니다. 먼저 설치 프로그램의 모양을 정의하는 그래픽 옵션이 있습니다.

  • 완료 : 설치 프로그램의 "완료"페이지에 나타나는 그래픽. * .bmp 형식이어야하고 164x314px로 측정되어야합니다.
  • 헤더 : 설치 프로그램이 실행되는 동안 나타나는 그래픽. * .bmp 형식이어야하며 크기는 150x57px입니다.

특정 라이센스 계약 파일 (예 : GameMaker Studio 2와 함께 제공됨) 및 사용자 지정 NSI 스크립트 파일 (자세한 내용은 여기 참조 )을 가리 키거나 적절한 버튼을 클릭하여 GameMaker Studio와 함께 제공되는 기본 파일로 돌아갈 수도 있습니다 2.
