텍스처 그룹

텍스처 그룹 관리자 는 IDE의 도구 메뉴 에서 사용할 수 있습니다. 여기에서 Texture Groups를 추가 , 삭제 및 이름 변경할 수 있습니다 . GameMaker Studio 2를 사용하면 각 그래픽 자산 (스프라이트, 타일셋 및 글꼴)을 다른 텍스처 그룹에 할당하여 게임에서 특정 시점에 하드웨어가 수행하는 텍스처 스왑 수를 최적화하고 시도 할 수 있습니다. 이를 위해서는이 창에서 필요한 텍스처 그룹을 정의한 다음 적절한 각 자산 유형을 살펴보고 드롭 다운 메뉴에서 소속 그룹을 선택해야합니다. 또는 마우스 오른쪽 버튼으로 텍스처 그룹에 자산을 추가 할 수 있습니다 자산 브라우저의 메뉴. 타일셋에 대해 아래와 같이 하나 이상의 자산 (또는 전체 자산 그룹 폴더)을 선택한 다음 RMB 메뉴를 사용하여 자산을 그룹에 추가 할 수 있습니다.

단일 텍스처 그룹은 여러 텍스처 페이지를 가질 수 있으며 동등 하지 않다는 것을 인식하는 것이 중요합니다. 게임에 각 레벨마다 고유 한 그래픽이 많은 경우 각 레벨에 대해 단일 텍스처 그룹 을 정의 할 수 있지만 그래픽에는 그룹당 두 개 이상의 텍스처 페이지가 필요합니다.

By default all images in your game will be added to the "Default" texture group, which always exists. However you can click the Add New button to create a new texture group, and if you select the name of the group you can change it something appropriate. To start with the group will show "this group is empty" for each section of the group (Sprites, tilesets and fonts), and so you then need to go through your graphic assets and add them. You can add a resource using the Add Resource button at the bottom, or by selecting the texture group from the resource editor for the image, tile set, or font. Texture groups other than the "Default" one can also be deleted at any time, which will move their contents into the "Default" group (note that if you delete a group by mistake you can use / + " Z ")를 실행 취소합니다.

Once you have defined texture groups - and assigned graphic assets to them - you will see them listed on the left of the main Texture Group editor window, and clicking on any of the sprites in this list will show them in the image preview window on the right. This is particularly helpful when working on large projects and you need to be able to see which images have been assigned to which group. Note that if the images assigned to the group have multiple sub-images (ie: they are animated), then only the first sub-image is shown in the preview window. You can also right click 리소스에서 작은 메뉴를 열면 다른 텍스처 그룹으로 이동하거나 해당 자산의 편집기를 열 수있는 옵션이 제공됩니다.

자산이 목록에 별도의 것으로 표시 되더라도 모든 스프라이트, 타일셋 및 글꼴이 동일한 텍스처 페이지에 추가되고 각 자원 유형에 대해 개별 페이지를 얻지 못한다는 점에 주목할 가치가 있습니다. 이 규칙에 대한 유일한 예외는 스프라이트 자산이 " 텍스처 분리 페이지 "용으로 플래그가 지정된 경우입니다.이 경우 자체 텍스처 페이지가 제공됩니다 (에 대한 Sprite Editor의 "텍스처 설정"섹션 참조). 자세한 내용은).

텍스처 페이지 및 생성 방법에 대한 자세한 내용은 다음 페이지를 참조하십시오.

  • 텍스처 페이지

텍스처 그룹 편집기의 그룹 설정 은 다음 옵션을 사용하여 주어진 그룹에 대한 텍스처 페이지를 만드는 방법을 정확하게 결정하는 데 사용됩니다.

  • 스케일링 허용 -플래그를 해제하면 GameMaker Studio 2가 어떤 상황에서도 텍스처 그룹을 스케일하지 않습니다. 이 기능은 특히 유니버설 앱을 다룰 때 유용합니다. 글꼴과 같은 특정 리소스를 자체 그룹에 배치하여 앱을 특정 기기에서 실행할 때 사용할 수 있기 때문입니다. 예를 들어, 글꼴은 레티 나 디스플레이에서 멋지게 보일 수 있지만, 일반 디스플레이에서 그리도록 축소 될 때 끔찍해 보일 수 있으므로 특별한 비 스케일링 그룹에 할당 할 수 있습니다. iPhone 용으로 개발하는 경우 텍스처 스케일링으로 인해 이미지가 흐려져 앱이 거부되었으므로이 옵션을 선택해야합니다. 이는 기본적으로 같은 위치에 있었 네요.
  • 자동 자르기 -이미지를 GameMaker Studio 2에 추가 할 때 이미지 주위에 "공간"이 있으면 (즉, 100 % 투명도) 텍스처 페이지에 추가되고 "자동 자르기"가 표시 될 때 자동으로 자릅니다. 그러나 예를 들어 프리미티브 또는 이와 유사한 것을 위해 반복되지 않는 텍스처로 이미지를 사용하는 경우 텍스처 매핑을위한 UV 좌표가 올바르게 표시되지 않습니다. 이 경우이 옵션의 플래그를 해제하면 스프라이트가 텍스처 페이지에 "있는 그대로"추가되어 가장자리 주위에 투명 필름이 유지됩니다. 이는 기본적으로 같은 위치에 있었 네요.
  • 밉맵 생성 -이 옵션을 플래그하면 (기본적으로 해제되어 있음) GameMaker Studio 2에 원본 텍스처 페이지의 크기를 조정하고 텍스처 파일 내에서 여러 해상도로 필터링해야합니다. 이 기능은 일반적으로 꺼져 있지만 카메라를 사용하여 게임 룸의 넓은 영역을 확대 또는 축소하거나 3D 게임을 작업하는 경우이 기능 을 활성화 한 다음 Mipmapping 기능 을 사용하여 게임은 다양한 밉맵 레벨을 처리합니다. 이 확인란에 플래그를 지정하면 "테두리 크기"값이 8로 설정됩니다 (원래 8보다 작게 설정된 경우). 밉맵을 사용할 때 텍스처 페이지의 이미지 사이에 더 큰 경계선이 필요하기 때문에 더 높은 밉 레벨에서 함께 번지기 시작하기 때문입니다. 기본값 인 8을 사용하면 하위 이미지가 함께 번지기 전에 이미지를 원래 크기의 약 8 분의 1로 축소 할 수 있지만 필요에 따라 필요한 값으로 변경할 수 있습니다. 스프라이트에 " 별도 텍스처 페이지 "옵션을 사용하는 경우 여기에서 각 텍스처를 살펴보고 밉맵을 생성하기 위해 플래그를 지정할 필요는 없지만 대신 일반 게임 옵션으로 이동하여 " 생성 "옵션에 플래그를 지정할 수 있습니다. 별도의 텍스처 페이지에 대한 밉맵 ".
  • 테두리 크기 -테두리 크기는 텍스처 그룹에 대해 생성 된 페이지에서 모든 자산의 가장자리 주위의 겹침 (또는 "픽셀 배가") 양을 변경하여 이미지 크기를 조정할 때 사이에 간격 (또는 이음새)이 보이지 않도록합니다. 그들. 기본값은 2 픽셀이지만보기를 크게 확대 또는 축소하는 극단적 인 경우이 값을 늘려야 할 수 있습니다. Tile Set Editor (타일 세트 편집기) 에는 타일 의 출력 테두리를 설정하는 옵션이 있으며 (필요한 이유에 대한 설명도 제공함) 여기에서 테두리 설정과 별개입니다. 기본적으로 타일은 각 출력 테두리로 생성됩니다 임시 텍스처에 바둑판 식 배열을 한 다음 여기에 바깥 쪽 가장자리 주위에 테두리가있는 텍스처 페이지에 추가합니다. 또한 밉맵을 활성화하면 텍스처 페이지의 기본 테두리 크기가 자동으로 8로 설정되어 더 높은 밉 레벨에서 텍스처 간 "번짐"을 방지 할 수 있지만 변경할 수 있습니다.
  • Parent Texture Group- 여기에서 기존 그룹 중 하나를 현재 선택된 그룹의 "부모"그룹으로 선택할 수 있습니다. "자식"그룹은 부모 텍스처 그룹에 대한 모든 변경 사항을 상속하며 주로 크로스 플랫폼 게임 작업시 리소스 관리를위한 것입니다. 부모가있는 텍스처 그룹은 여전히 ​​그 자체로 텍스처 그룹이며 다른 그룹과 동일한 텍스처 페이지를 생성하지만 부모를 할당하면 다른 플랫폼에서 큰 청크를 제외 할 수 있습니다. 예를 들어, Android 플레이어 스프라이트를 "player"그룹에 추가하고 Android 적 스프라이트를 "enemy"그룹에 추가 한 다음 둘 다 "android"상위 그룹을 사용할 수 있습니다. 그런 다음 Windows 또는 기타 플랫폼 빌드에서 "android"상위 그룹을 제외하면 하위 그룹도 제외됩니다.

해당 설정 외에 창의 Use Group On ... 섹션에서 해당 상자를 선택하거나 선택 취소하여 주어진 텍스처 그룹을 특정 플랫폼으로 내보내거나 내 보내지 않도록 선택할 수도 있습니다. <Default> 텍스쳐 그룹은 항상 모든 플랫폼으로 내보내 지며, 체크 / 체크 해제 할 수 없습니다 (커스텀 텍스쳐 그룹 만 허용).

텍스처 그룹도 구성 설정에 연결됩니다. 이것이 의미하는 바는 게임의 각 개별 구성에 대한 텍스처 그룹을 정의한 다음 스프라이트, 타일 세트 또는 글꼴 리소스 할당을 이러한 다른 그룹에 설정할 수 있으며 다른 구성으로 변경하면 "기억"됩니다. 텍스처 그룹 창에서 선택한 내보내기 옵션도 마찬가지이므로, 텍스처 그룹을 구성별로 특정 플랫폼으로 내보내도록 설정할 수 있습니다.

예를 들어, iPad 구성을하고 2048x2048px 텍스처 페이지 ( 게임 옵션을 통해 설정)를 만들 수 있습니다. 그런 다음 모든 그래픽 자산을 적절한 텍스처 그룹에 할당하고 완료되면 다른 구성 (예 : HTML5의 경우)으로 변경하고 텍스처 그룹 창에서 512x512px 인 새 텍스처 그룹을 생성하여 이 새로운 그룹에 자산. 페이지 하단에는 텍스처 그룹 설정을 한 구성에서 다른 구성으로 복사 할 수있는 버튼이 있습니다.

마지막으로 텍스처 그룹은 비트 맵 이미지에만 관련되며 비트 맵이 아닌 스프라이트는 텍스처 그룹에 할당되지 않습니다. 그들 중 일부는 이미 고유 한 텍스처 페이지 (골격 애니메이션 스프라이트)를 가지고 있지만 다른 일부는 vector 로 구성되어 있으므로 텍스처가 필요하지 않습니다. 자세한 내용은 다음 섹션을 참조하십시오.

  • 비트 맵이 아닌 스프라이트
