자산 브라우저

게임의 핵심은 기본적으로 IDE 의 오른쪽에있는 Asset Browser 에 추가 된 자산에서 생성됩니다. 게임 룸, 스프라이트 , 오브젝트, 경로 및 기타 여러 가지를 포함하여 게임 실행에 필요한 모든 것을 추가 할 수 있습니다. GameMaker Studio 2의 기본 게임을 실행하려면 룸이 필요합니다 (새 프로젝트는 항상 룸 자산이 이미 생성되어 생성됨). 일반적으로 하나 이상의 오브젝트와 스프라이트입니다.

자산 브라우저는 게임의 자산을 생성 및 구성하고 플랫폼 별 설정 및 구성과 같은 특정 측면을 수정하는 곳입니다. 다음은 각기 다른 섹션에 대한 설명입니다.

검색 창 검색 창

검색 창은 일부 텍스트를 입력하고이를 사용하여 아래의 주요 자산 목록에 표시된 내용을 필터링 할 수있는 곳입니다. 예를 들어 "room"을 입력하면 이름에 "room"이 포함 된 모든 자산이 표시됩니다. 자산 목록을 기본보기로 다시 설정하려면 검색 창에서 텍스트를 삭제하십시오.

 

자산 추가 자산 추가

자산 추가 버튼을 사용하여 자산 목록에 새 자산을 추가하도록 선택할 수 있습니다. 이것을 클릭하면 다음과 같은 창이 열립니다.

You can choose which of the two views above to use by clicking either the 목록 아이콘 또는 그리드 아이콘을 클릭하면 다른 프로젝트에서 기존 자산을 가져 오려면 가져 오기 아이콘을 클릭하십시오. 아무것도 가져 오지 않으려면 만들려는 자산을 선택하고 맨 아래에있는 생성 버튼을 클릭하십시오. 창 하단에서 생성 할 번호를 설정 한 다음 생성을 클릭하면 선택한 자산의 번호가 자산 브라우저에 추가되므로 여기에서 단일 자산 만 추가 할 수 있습니다. 이 방법으로 생성 된 모든 자산은 폴더 외부의 자산 목록 하단에 추가되며 현재 작업 공간에서 편집 할 수 있도록 열립니다.

여기에 추가 할 수있는 자산은 매뉴얼의이 섹션에서 더 아래에 나열되어 있습니다.

 

필터 필터

자산 필터 버튼을 클릭하면 다음 창이 열립니다.

여기서 자산 브라우저에 나열된 자산을 필터링하고 표시하는 방법을 선택할 수 있습니다. A-Z 또는 Z-A에서 주문할 수 있으며 폴더를 먼저 주문한 다음 폴더 내의 자산을 주문하거나 모든 것을 주문할 수 있습니다. Custom Order 필터를 선택하면 하나 이상의 자산을 선택하고 자산 브라우저 트리에서 위 또는 아래로 끌어 필요한대로 배치 할 수 있습니다 ( "폴더 우선"옵션은 비활성화 됨).

필터링 할 자산 유형을 하나 이상 선택할 수도 있으며 해당 유형의 프로젝트에있는 자산 만 자산 목록에 표시됩니다. "모든 유형"을 선택하면 필터가 재설정됩니다. 할당 한 태그를 사용하여 필터링 할 수도 있습니다 (태그는 설명서의이 섹션에서 자세히 설명합니다). "모든 유형"이외의 필터를 선택한 경우 필터 단추의 색상이 변경되어 필터링이 활성화되었음을 시각적으로 나타냅니다.

빠른 액세스 빠른 액세스

자산 브라우저의이 섹션은 중요하거나 자주 사용되는 자산에 빠르게 액세스 할 수있는 다양한 방법을 제공하는 데 사용되며 다음 섹션으로 나뉩니다.

  • 최근 : 여기에서 가장 최근에 사용한 자산 목록을 찾을 수 있습니다. 기본적으로 최대 10 개의 자산이 여기에 나열되지만 자산 브라우저 기본 설정 에서 변경할 수 있습니다.
  • 즐겨 찾기 : 자산 목록에 자산을 추가 한 후에는 언제든지 자주 액세스하는 자산이있을 수 있으므로 이러한 자산을 "즐겨 찾기"로 플래그 지정하여 자산을이 목록에 추가하면됩니다. 빨리 찾을 수 있습니다. 자산을 즐겨 찾기 로 플래그 지정하려면 마우스 오른쪽 버튼을 사용하십시오 팝업 메뉴에서 즐겨 찾기를 선택하십시오. 마우스 오른쪽 버튼을 사용할 수 있습니다 다시 모든 자산을 unfavourite하고, 메인 즐겨 찾기 폴더에있는 버튼을 사용하는 경우, 당신은 모든 것을 제거하기 위해 모두 지우기를 선택할 수 있습니다.
  • 객실 순서 : 여기에서 객실의 주문 순서를 변경할 수 있습니다. 모든 프로젝트를 실행하려면 하나 이상의 객실이 필요하며 목록 맨 위에있는 객실은 "홈"객실이며 가장 먼저 입력 된 객실이됩니다. 게임이 처음 시작됩니다 ( "홈"아이콘으로 표시됩니다) ). 객실 주문을 빠르게 변경하려면 클릭 방을 목록 위나 아래로 끈 다음 마우스 단추를 놓아 새 위치에 놓습니다. "홈"아이콘을 클릭하면 룸 관리자 가 열리고 마우스 오른쪽 버튼을 사용하면 방에 당신은 아이 방을 만들 수 있습니다. 객실 주문 및 아동 실에 대한 자세한 내용은 회의실 관리자 페이지를 참조하십시오.
  • 저장된 필터 : 필터링 옵션을 사용할 때 나중에 참조하기 위해 선택한 내용을 저장하고 싶을 수 있습니다. 이렇게하려면이 페이지의 추가 설명에 따라 필터를 설정 한 다음 마우스 오른쪽 버튼을 클릭하십시오. 이 옵션에서 필터 저장을 선택 하십시오 . 필터가이 목록에 저장되며 필요한 경우 이름을 바꿀 수 있습니다. 필터를 사용하려면 두 번 클릭하십시오. 그것에 마우스 오른쪽 버튼을 사용 메뉴 옵션 필터 적용 . RMB에서 저장된 필터를 삭제 하고 이름을 바꿀 수도 있습니다 메뉴.
  • 태그 : 자산 브라우저의 모든 자산에 사용자 정의 태그를 할당 할 수 있습니다. 그러면 여기에 나열되며 자산 목록에 표시된 자산을 필터링하는 데 사용할 수 있으며 두 번 클릭 할 수 있습니다 모든 태그 (또는 마우스 오른쪽 버튼 사용) 태그별로 필터링 옵션을 선택하여 해당 태그를 자산 목록 필터에 적용하십시오. 적용된 모든 태그는 정보 섹션의 자산 브라우저 하단에 표시됩니다. 태그에 대한 자세한 내용은이 페이지의 아래 부분을 참조하십시오.
  • 게임 옵션 : 프로젝트의 다양한 게임 옵션 에 빠르게 액세스 할 수 있습니다. 이들은 다른 대상 플랫폼에 대한 설정 및 컴파일과 관련된 프로젝트의 다양한 측면을 제어합니다 (사용 가능한 게임 옵션은 사용중인 라이센스 유형에 따라 다릅니다). 게임 옵션 섹션에서 나열된 각 옵션에 대한 자세한 정보를 찾을 수 있습니다.

 

자산 목록 자산 목록

자산 브라우저의이 섹션에는 게임에서 사용중인 모든 다른 자산이 나열됩니다. 기본적으로 새 프로젝트를 만들 때 추가 할 수있는 다양한 종류의 자산으로 레이블이 지정된 수많은 폴더 그룹이 채워지지만 이름을 바꾸거나 제거 할 수 있으며 동일한 자산을 저장할 필요가 없습니다. 각 그룹에서 친절합니다. 예를 들어, 각 유형의 자산이 아닌 게임 레벨별로 자산 그룹을 중단하는 것이 더 유용하므로 "Level 1", "Level 2"등의 폴더 그룹을 만들 수 있습니다. 각 레벨에 대해 서로 다른 모든 자산을 동일한 폴더에 저장합니다. 물건을 정리하는 방법은 당신에게 달려 있습니다!

목록에 새 자산을 추가하려면 자산 브라우저 상단의 자산 추가 버튼을 사용하거나 마우스 오른쪽 버튼을 열 수 있습니다 자산 목록의 아무 곳이나 메뉴를 열고 생성 할 리소스를 선택하십시오.

자산이 생성되면 작업 공간에서 자산이 열리고 게임에 적합한 이름으로 이름을 바꿀 수 있지만 자산 이름은 영숫자 일 수 있으며 숫자로 시작할 수 없으며 추가 문자로 밑줄 "_". 오른쪽 버튼 메뉴를 사용하여 그룹 생성 옵션을 사용하여 자산을 포함 할 새 폴더 그룹을 생성 할 수 있으며 원하는 이름을 지정할 수 있습니다. 자산 또는 폴더 그룹의 이름을 바꾸려면 천천히 두 번 클릭하면됩니다. 그들에게 또는 인민폐를 사용 메뉴 옵션 이름 바꾸기 .

An important feature of the asset browser is the ability to colour assets so they are more immediately recognisable within the list. You can do this by clicking the right mouse button 왼쪽의 자산 목록 노드에서 색상 선택기 팔레트에서 색상을 선택하십시오. 폴더에서이 작업을 수행하면 폴더 내에 포함 된 모든 자산에도 색상이 지정됩니다.

이와 같은 노드에서 색상 선택기 팔레트를 열 때 마우스 오른쪽 버튼을 클릭 할 수 있습니다. 색상 견본을 사용하여 사용자 정의 색상을 생성 할 수 있습니다.

Individual assets can also be coloured independently of the group colour by using the RMB  자산 메모 또는 RMB 하단의 팔레트를 사용하여 메뉴 ( 애셋 브라우저 환경 설정 에서 비활성화 할 수는 있지만)를 사용할 수 있습니다. 또는 / 마우스 왼쪽 버튼과 함께 여러 자산을 선택하고 한 번에 모든 자산에 색상을 할당하려면 :

물건을 색칠하는 방법은 당신에게 달려 있습니다-자산 유형, 사용 또는 다른 기준에 따라 달라질 수 있습니다!

One final thing to note with the asset list is that you can select folders and assets then click 드래그하여 자산 목록의 다른 폴더 그룹에서 또는 다른 폴더 그룹으로 이동하거나, 필요하지 않은 경우 폴더 그룹을 일부 또는 전부 삭제하여 어셋 목록을 깔끔하게 유지하고 필요한 요소 만 표시 할 수 있습니다 프로젝트를 위해.

태그

위의 여러 섹션에서 언급했듯이 자산 브라우저를 사용하면 개별 자산에 태그를 추가하고 폴더를 그룹화 할 수 있습니다. 태그는 자산 브라우저에 나열된 자산을 구성하고 필터링 할뿐만 아니라 일반적으로 게임을 프로그래밍 할 때 할당 된 태그를 코드에 사용할 수 있으므로 매우 강력한 도구입니다.

To create a tag you need to use the RMB 폴더 그룹 또는 자산의 메뉴에서 태그 편집 옵션을 선택하십시오. 사용하려는 태그를 입력 할 수있는 태그 편집기 가 나타나면를 눌러 적용하십시오.

단일 항목에 여러 태그를 할당 할 수 있으며 또는 / 마우스 왼쪽 버튼과 함께 여러 자산을 선택하고 한 번에 모든 자산에 태그를 할당합니다. 폴더 그룹에 태그를 추가하면 폴더 내의 모든 자산에 태그가 적용됩니다. 추가 된 모든 태그는 빠른 액세스 메뉴 및 필터 옵션 (위 섹션에서 설명한대로)에 표시되므로 자산 목록을 신속하게 필터링하고 선택한 태그가있는 항목 만 표시하는 데 사용할 수 있습니다 .

태그가 활성 필터의 일부인 동안 생성 된 자산에는 해당 태그가 자동으로 부여됩니다 . 즉, "수준 1"태그 아래의 모든 자산을보고있는 경우 (예를 들어)이 필터를 활성화하여 생성 된 모든 새 자산에는 생성시 "수준 1"태그가 추가됩니다.

마지막으로, 프로젝트 코드에서 태그를 사용하여 자산을 식별 한 다음 그에 따라 행동 할 수 있으므로 태그 또는 태그 그룹을 기반으로 동작을 코딩 할 수있는 강력한 도구입니다. 예를 들어, " Die On Touch "태그가있는 프로젝트가 있다고 가정하십시오. 프로젝트를 프로그래밍 할 때 적절한이 태그와 코드 기능을 확인할 수 있으며 나중에 아티스트와 레벨 디자이너는 모든 유형의 새 자산을 생성하고 " Die On Touch "태그로 태그를 지정한 다음 레벨을 디자인 할 때 자산을 사용할 수 있습니다 . 이 태그를 사용하도록 이미 프로그래밍 된 코드는 이러한 새 자산을 자동으로 포함합니다.

프로젝트 코드에서 태그 사용에 대한 자세한 내용은 자산 및 태그 섹션을 참조하십시오.

애니메이션 곡선 애니메이션 곡선

애니메이션 곡선은 시간이 지남에 따라 값이 어떻게 변하는지를 나타내는 하나 이상의 곡선을 포함하는 자원이며 선형 보간 또는 catmull-rom 보간 을 사용하여 곡선의 값 사이를 이동할 수 있습니다. 설정 한 값은 세로 축에서 -1과 1 사이 일 수 있으며 가로 축을 따라 지속 시간은 항상 0에서 1까지 정규화되므로 게임 코드에서 승수를 사용하여 다른 시간 범위를 쉽게 타겟팅 할 수 있습니다. 애니메이션 곡선은 "채널"로 구성되며 각 채널에는 고유 한 곡선 설정이있을 수 있습니다. 예를 들어 x / y 위치에 두 개의 채널이있는 공간 위치를 설명하거나 4 개의 채널이있는 컬러 형식. 애니메이션 커브 애셋은 시퀀스로 작업 할 때 사용할 수 있으며 코드를 사용하여 액세스 할 수 있으므로 게임을 만들 때 강력한 도구가됩니다.

애니메이션 곡선 섹션에서 애니메이션 곡선 추가에 대해 자세히 알아볼 수 있습니다.

개체 개체

객체는 게임의 측면을 제어하고 특정 작업을 수행하는 데 사용하는 특별한 리소스입니다. 대부분 게임 룸에서 볼 수 있도록 스프라이트가 연결되어 있지만 사용자와 관련된 작업이나 타이밍 등을 수행하기 위해 "장면 뒤"컨트롤러로 사용되는 경우가 있습니다. 행동을 취하고 특정 이벤트와 서로에 반응 할 수 있으며 게임에서 볼 수있는 대부분의 것은 사물과 그 상호 작용을 기반으로합니다. 실제로 게임 기반에 오브젝트를 배치하는 것이 아니라 기본적으로 오브젝트 자원의 복제본 (또는 원하는 경우 복제) 인 이러한 오브젝트의 인스턴스 를 배치하기 때문에 "기반"이라고합니다. 인스턴스와 객체가 같지 않고 각각에 영향을 줄 수있는 고유 한 기능 세트가 있으므로 기억해야 할 것은 매우 중요합니다.

모든 객체에는 스프라이트와 같이 객체를 물리학 사용 여부 또는 다른 객체의 "자식"객체인지 여부와 같이 객체 편집기에서 지정할 수있는 속성 세트가 있습니다. 또한 동작, 스크립트 및 코드에서 사용할 수있는 일련의 내장 변수가 있습니다. 이 내장 변수는 게임 룸에 배치 될 때 인스턴스의 위치, 애니메이션 속도, 이동 방향 및 기타 여러 가지를 정의하는 데 사용됩니다. 시간이 지남에 따라 오브젝트의 동작을 제어하기 위해 각 게임 프레임은 일련의 이벤트로 분할되므로 특정 이벤트에 코드 또는 조치를 추가 할 수 있으며 이벤트가 트리거 될 때만 실행되며 이벤트는 다음과 같은 것으로 트리거 될 수 있습니다. 마우스 버튼을 누르거나 다른 인스턴스와 충돌합니다.

객체는 게임의 빌딩 블록과 매우 실질적으로 일치하며, 룸과 함께 생성하는 모든 프로젝트의 핵심을 형성합니다.

게임을 만들 때 어느 시점에서 레벨을 통과하는 경로를 따라 가려면 객체 중 하나의 인스턴스가 필요할 수 있습니다. 이제는 방 안에 여러 개의 위치를 ​​만들고 그 사이에서 인스턴스를 이동 시키면됩니다. 그러나 설정하기가 쉽지 않고 다른 것들에 대해 변경, 테스트 및 적응하기가 어렵습니다. 이때 경로 리소스를 만들고 싶을 때입니다.

스크립트 스크립트

스크립트 리소스는 단순히 하나 이상의 함수를 작성하기위한 컨테이너입니다. 예를 들어, 모든 적의 물체가 죽으면 금, 폭발, 피 같은 소리를 내고 소리를내는 등 여러 가지 다른 인스턴스를 만들려고한다고 가정 해 봅시다. 이제 필요한 모든 코드를 필요한 각 객체에 추가 할 수 있지만 일반적으로 스크립트 리소스에서 단일 함수를 만든 다음 호출하는 것이 좋습니다. 이것은 새로운 스크립트 기능을 호출하기 위해 객체에 한 줄의 코드 만 추가하면되고, 무언가를 변경하려면 스크립트에서 한 번만 변경하면 모든 객체가 자동으로 새로운 코드를 실행합니다. 모든 개체를 거치고 여러 곳에서 동일한 것을 변경해야합니다.

The RMB 스크립트 메뉴에는 다음과 같은 옵션과이 페이지 위쪽에 나열된 옵션이 있습니다.

  • DnD ™ / GML 로 변환-선택한 스크립트를 DnD ™ 또는 GML로 변환합니다. 스크립트가 GML 인 경우 스크립트는 코드 실행 작업 내에 배치되고 DnD ™는 Gnn 코드로 직접 변환됩니다 ( DnD ™ 코드 변경 섹션 참조).

스크립트 편집기 섹션에서 스크립트 추가에 대한 자세한 내용을 확인할 수 있습니다.

 

시퀀스 시퀀스

시퀀스 자산은 본질적으로 시간이 지남에 따라 동적 애니메이션을 수행하는 다른 리소스의 모음입니다. 여기에는 스프라이트, 인스턴스, 사운드 및 기타 것들이 포함될 수 있으며 시퀀스의 특정 트랙에 자산을 할당하여 시퀀스 자체 내에서 시간이 지남에 따라 시작 또는 중지하도록 색상을 이동하거나 변경하도록 설정할 수 있습니다. 이러한 트랙에는 다른 속성이 적용되어 정적이거나 시간이 지남에 따라 변경 될 수 있습니다. 시퀀스의 모든 편집은 다음으로 구성된 시퀀스 편집기 에서 수행됩니다.

  • 캔버스 -시퀀스를 구성하는 자산을 배치하는 위치
  • 트랙 패널 -자산을 배치하여 자산 트랙을 생성하고 파라미터 트랙을 생성하여 시간이 지남에 따라 자산을 조작
  • 도프 시트 -각 트랙의 키 프레임을 추가 / 제거 / 편집하고 시간이 지남에 따라 키 프레임 동작을 제어하는 ​​위치입니다.

시퀀스 내에 시퀀스를 포함하여 복잡한 애니메이션과 효과를 만들 수 있으며 코드를 통해 시퀀스에 액세스하고 변경하여 게임에서 표시 및 사용되는 방식을 가능한 한 많이 제어 할 수 있습니다.

의 섹션에서 시퀀스 추가에 대한 자세한 내용은 시퀀스 편집기 .

 

셰이더 셰이더

셰이더는 게임이 화면에 렌더링하는 그래픽을 조작하는 데 사용할 수있는 매우 강력한 도구로, 예를 들어 스프라이트에 미묘한 색상 색조를 추가하는 등 전체 화면 왜곡 효과에 이르기까지 매우 빠른 효과를 허용합니다. . 기본적으로 그래픽 카드 자체에서 직접 실행되는 두 부분으로 구성된 프로그램으로 GPU가 모든 작업을 수행하고 게임 코드의 CPU 사이클을 확보하므로 매우 빠릅니다. 풀 쉐이더는 버텍스 쉐이더 프로그램과 프래그먼트 쉐이더 프로그램 (픽셀 쉐이더라고도 함)으로 구성됩니다. 이 작은 프로그램은 그래픽 카드가 화면에 렌더링하는 것을 조작하기 위해 함께 작동합니다. 그러면 디스플레이 버퍼에 실제로 렌더링되는 위치, 색상 및 알파 값을 실시간으로 조작 할 수 있습니다.

GameMaker Studio 2는 다음 쉐이더 언어를 지원합니다 :

쉐이더 언어 대상 플랫폼
GLSL ES 모든 대상 플랫폼
GLSL 맥과 우분투 (Linux)
HLSL11 윈도우, UWP, XboxOne
PSSL 플레이 스테이션 4

   
인민폐 셰이더 메뉴에는 다음과 같은 옵션과이 페이지 위쪽에 나열된 옵션이 있습니다.

  • Shader Type- 셰이더 리소스에 사용되는 셰이더 언어의 유형을 설정할 수 있습니다.

셰이더 편집기 섹션에서 셰이더 추가에 대한 자세한 내용을 확인할 수 있습니다.

 

소리 소리

사운드는 게임 플레이에 깊이를 더하고 사운드 효과 형태로 피드백을 제공하고 음악 형태로 분위기를 추가하는 데있어 모든 게임에서 중요한 부분입니다. GameMaker Studio 2는 WAV , MP3 및 OGG 형식 파일을 지원합니다.

일반적으로 WAV 파일은 일반적으로 큰 파일이지만 재생을 위해 어떤 유형의 디코딩도 필요하지 않기 때문에 즉시 재생되는 짧은 사운드 효과에 사용됩니다. MP3 및 OGG 파일은 배경 음악이나 오랜 시간 동안 재생되거나 파일 크기가 다소 큰 효과에 사용해야합니다. 이러한 파일은 WAV 파일보다 훨씬 작지만 재생하기 전에 디코딩해야하기 때문에 이와 관련된 CPU 오버 헤드 가 있습니다.

The RMB  소리 메뉴에는 다음과 같은 옵션과이 페이지의 위쪽에 나열된 옵션이 있습니다.

  • 열기에서 사운드 믹서 - 이것은 모든 선택 소리 열립니다 사운드 믹서 .
  • 오디오 그룹 할당 -이 옵션을 선택하면 사용 가능한 오디오 그룹 목록이 표시되고 필요한 경우 하나를 선택하여 사운드를 추가하거나 새로 만들 수 있습니다. 자산 브라우저에서 여러 사운드를 선택하거나 사운드 그룹 폴더를 선택한 경우 모든 사운드가 선택한 오디오 그룹에 추가됩니다. 오디오 그룹에 대한 자세한 내용은 여기 를 참조 하십시오 .

사운드 편집기 섹션에서 사운드 추가에 대한 자세한 내용을 확인할 수 있습니다.

타임 라인 타임 라인

대부분의 게임에서는 특정 순간에 특정 일이 발생하기를 원할 것입니다. 이제 인스턴스에서 알람 이벤트 를 사용하여이 작업을 수행 할 수 있지만 상황이 너무 복잡하면 더 이상 작동하지 않습니다. 특히 12 개의 알람으로 제한되어 있기 때문입니다. 이것이 우리에게 타임 라인 자료가있는 이유입니다. 타임 라인에서 게임 시간의 특정 순간에 어떤 동작을 수행해야하는지 지정하고, 코드뿐만 아니라 다른 이벤트에서 오브젝트에 대해 사용 가능한 모든 동작을 사용할 수 있습니다. 타임 라인을 생성 한 후에는이를 객체에 할당 할 수 있으며 해당 객체의 인스턴스는 룸 내에 배치하거나 생성 할 때 지정된 순간에 작업 및 코드를 실행합니다.
