컴파일

Compiling your game can mean one of two things: compiling it for testing, or compiling it to create an executable package or a specific target platform. Compiling your game for testing can be done by simply pressing the Play button  IDE 상단에 지정된 타겟을 사용하여 테스트 할 게임이 시작됩니다. 디버그 버튼을 사용하여 테스트하여 디버그 모드 에서 게임을 실행할 수도 있습니다. . 게임이 실행되지만 디버그 창이 열립니다. 여기서 게임 성능을 모니터링하고 문제를 확인할 수 있습니다 (자세한 내용은 디버깅 섹션 참조).

By default GameMaker Studio 2 will run and debug using the built in Virtual Machine (VM), which is more or less the same as running on the desktop OS being used. However GameMaker Studio 2 is a cross platform engine and you can test, debug and compile executable packages of your projects on a number of different target platforms (the exact platforms available will depend on the details of your licence). To change the current target platform you can click 타겟 버튼에서 대상 창 을 열려면 다음과 같이 표시됩니다 (정확한 세부 정보는 라이센스 유형에 따라 다름).

상단의 대상 버튼 옆에는 현재 사용중인 플랫폼과 특정 설정을 알려주는 현재 설정이 있으며 나머지 창의 사용 가능한 모든 대상에 대한 세부 정보 및 옵션이 표시됩니다. 대신 사용하도록 선택하십시오. 이 창의 각 섹션은 다음과 같습니다.

작업자 작업자

대상 관리자의 첫 번째 열은 사용 가능한 원격 작업자 전용입니다. 우선 여기에는 GameMaker Studio 2가 설치된 머신 인 "로컬"머신이 하나만 있지만 원격 작업자가 사용하도록 구성 할 수도 있습니다. 원격 작업자가 구성되면 여기 목록에서 원격 작업자를 선택하고 GameMaker Studio 2가 설치된 호스트 컴퓨터를 사용하지 않고 프로젝트를 빌드 할 수 있습니다.

원격 작업자를 설정하고 사용하는 방법에 대한 자세한 내용은 아래 섹션을 참조하십시오.

플랫폼 플랫폼

This section lists all the available target platforms. The contents of this list will vary depending on the licence that you have, but will always have at least the "Test" target. To select a target, simply click 그 위에. 그러면 선택한 플랫폼에 따라 다른 세부 정보가 표시되도록 나머지 옵션 창을 업데이트합니다.

 

출력 출력

각 대상 플랫폼은 하나 이상의 출력 형식을 가질 수 있으며 주요 형식은 다음과 같습니다.

  • VM : VM (Virtual Machine) 대상은 각 플랫폼에 일반 러너 를 사용한 다음 게임 코드를 해석합니다. 일반적으로이 옵션은 빠른 빌드 시간으로 인해 테스트에 사용되지만 YYC 옵션 (사용 가능한 경우)을 사용하는 것과 동일한 성능 향상을 제공하지는 않습니다. 이 기능을 사용하면 성능이 문제가되지 않는 소규모 게임이나 게임을위한 실행 패키지를 컴파일 할 수 있습니다.
  • YYC : YYC (YoYo Compiler)는 일반 GameMaker Studio 2 출력을 가져 와서 대상 플랫폼을위한 기본 코드로 컴파일하여 불필요한 기능을 "제거"하고 더 작고 성능이 향상된 실행 패키지를 만들기 위해 다양한 기타 최적화 기술을 수행합니다. 이를 통해 특히 논리가 많은 게임에서 게임 성능을 최소 2 ~ 3 배 향상시킬 수 있으며 규모가 크거나 CPU를 많이 사용하는 프로젝트에 이상적입니다. 컴파일 시간이 오래 걸릴 수 있으므로 대상 플랫폼에 대한 최종 전체 자산 패키지를 빌드하기 전에 항상 컴파일러 캐시를 비워야합니다. YYC 대상에는 선택한 플랫폼에 추가 도구를 설치해야 할 수 있습니다. 그렇지 않으면 작동하지 않습니다. YoYo Games 도움말 센터 및이 매뉴얼에 자세히 설명 된 개별 대상 기본 설정 페이지에서 이에 대한 자세한 정보를 찾을 수 있습니다.
  • JavaScript : JavaScript 대상은 HTML5 대상과 같은 일부 대상에서만 사용할 수 있으며 게임을 순수한 JavaScript로 컴파일하도록 설정합니다.

 

장치 장치

Certain platforms (like iOS or Android) permit you to associate one or more devices with GameMaker Studio 2 so that games can selectively compile to them. Initially, the device list will be empty and you need to click 연필 아이콘 장치 편집기 를 열려면

여기에 새로운 장치를 추가하고 GameMaker Studio 2가 연결될 수있는 모든 장치에 대한 연결을 테스트하도록 할 수 있습니다. 이 창의 정확한 내용은 플랫폼 별 정보에 따라 다릅니다 (해당 플랫폼에 대한 자세한 내용은 장치 관리자 섹션 참조). 기기를 찾거나 추가하면 Android의 다음 이미지 예와 같이이 창에 표시됩니다.

장치 설정 및 문제 해결에 대한 정확한 절차 및 요구 사항은 GameMaker Studio 2 기술 자료 의 해당 섹션에서 찾을 수 있습니다.

 

구성 구성

섹션에서 설명한대로 구성 , 당신은 CONFIGS로 게임을 컴파일하는 특정 정보를 저장할 수 있습니다. Targets 창의이 섹션에서는 GameMaker Studio 2가 특정 대상 플랫폼에 대한 특정 구성을 자동으로 선택할 수 있습니다.

 

 

최종 실행 가능 패키지 작성

Compiling your game project to any target platform will require that you have set up the correct build tools (see here) and also filled in the appropriate Platform Preferences. Once you have done that you can simply click the Create Executable button  IDE에서 컴파일러 빌드를 시작하거나 빌드 메뉴 에서 실행 파일 작성을 선택 하십시오 . 옵션을 선택하면 파일 탐색기 창이 열리고 저장 및 저장 프로세스를 시작하기 전에 저장 을 클릭하기 전에 게임 실행 파일에 사용할 최종 이름을 지정할 수 있습니다. 이 작업을 완료하면 필요한 파일이 생성되어 원하는대로 배포 할 수 있습니다.

중대한! 무료 30 일 평가판 라이센스가 있는 경우 최종 실행 파일을 컴파일 할 수 없으며 유료 라이센스 중 하나로 업그레이드하여 배포 용 게임을 컴파일해야합니다.

NOTE: Before doing a final build of your project for release, you should always clear the Asset Compiler Cache (using the "broom" icon IDE 상단). GameMaker Studio 2는 많은 게임 파일을 캐시하여 컴파일 속도를 최소로 유지합니다.

각 대상 옵션은 아래 나열된 플랫폼 별 형식으로 저장됩니다.

  • Windows- 일반 Windows OS 용 컴파일은 먼저 설치 프로그램 또는 Zip 패키지를 생성하도록 요청합니다. 여기서 설치 프로그램은 게임을 설치할 단일 실행 파일이고 Zip 파일은 단일 * .zip 형식입니다. 모든 게임 파일이 저장된 파일 (게임을 실행하려면 파일을 추출해야합니다). 패키징 옵션 기억 상자를 선택 하면 GameMaker Studio 2는 향후 모든 컴파일에 대한 선택을 기억합니다 ( Windows 환경 설정 에서 재설정하거나 변경할 수 있음). YoYo Games 도움말 센터 에서 자세한 내용을 확인할 수 있습니다.
  • Ubuntu (Linux) -Ubuntu (Linux) 앱은 * .zip 파일로 컴파일 할 수 있습니다. YoYo Games 도움말 센터 에서 자세한 내용을 확인할 수 있습니다.
  • HTML5 - HTML5 빌드를 선택한 경우 게임 파일이 포함 된 폴더와 함께 index.html 파일 (기본 이름이지만 플랫폼 기본 설정에서도 고유 한 이름을 지정할 수 있음)이 만들어져 저장됩니다. 지정된 위치 게임이 작동하려면이 두 가지를 모두 서버에 업로드해야 합니다 . index.html 은 다른 배경색 또는 다른 위치 등으로 게임을 표시하도록 사용자 정의 할 수 있지만 HTML에 대한 지식이 필요하며 빌드 할 때 사용자 정의 색인 파일을 지정할 수도 있습니다 패키지 ( HTML5 게임 옵션 참조). YoYo Games 도움말 센터 에서 자세한 내용을 확인할 수 있습니다.
  • Android -Android 디바이스의 경우 게임 저장을 위해 표시되는 창에서 * .apk 또는 * .aab (Android App Bundle) 파일을 빌드하도록 선택할 수 있습니다. 선택한 파일 유형은 타겟팅하려는 상점에 따라 다르며 * .aab 파일은 Google Play에 필요하지만 * .apk 파일은 다른 상점에서 사용할 수 있습니다. YoYo Games 도움말 센터 에서 자세한 내용을 확인할 수 있습니다.
  • iOS - iOS로 컴파일하면 xarchive 파일이 생성되어 Xtools에서 최종 iOS 패키지를 만드는 데 사용됩니다. iOS 용으로 컴파일하려면 OSX 이상을 실행하는 Apple Mac 컴퓨터와 관련 인증서 및 권한이 필요 합니다. YoYo Games 도움말 센터 에서 자세한 내용을 확인할 수 있습니다.
  • macOS -macOS 대상은 나중에 Mac App Store에 업로드할지 여부에 따라 * .app 파일 또는 * .pkg 파일을 빌드합니다. iOS와 마찬가지로 OSX 이상을 실행하는 Apple Mac 컴퓨터와 관련 인증서 및 권한이 필요 합니다. YoYo Games 도움말 센터 에서 자세한 내용을 확인할 수 있습니다.
  • Windows UWP -Windows UWP의 경우 GameMaker Studio 2는 * .appx 패키지를 생성 한 다음 Windows 스토어에 업로드 할 수 있습니다. Create Executable 버튼을 클릭하면 GameMaker Studio 2에 생성하려는 패키지 유형 (ARM, x64 또는 x86 아키텍처 일 수 있음)을 알려주는 메시지가 표시되며 게임에서 장치를 대상으로하는 대상을 선택해야합니다 다음에서 지원됩니다 : Store Upload for Store Upload 확인란을 선택하면 생성 된 최종 패키지는 * .appxupload 파일이며,이 기사 에서 설명하는 것처럼 Microsoft가 스토어에 앱을 제출하는 데 사용해야하는 파일입니다. 또한 GameMaker Studio 2가 향후 모든 컴파일에 대한 선택을 기억하도록 UWP 환경 설정에서 기본 패키징 옵션을 설정할 수 있습니다 (이 옵션 창은 표시되지 않음). YoYo Games 도움말 센터 에서 UWP 플랫폼 설정 및 컴파일에 대한 자세한 내용을 확인할 수 있습니다.

실행 가능한 자산 패키지를 만든 후에는 파일을 다른 사람에게 제공하거나 웹 사이트에 배치하여 다운로드하거나 개별 배포를위한 다른 호스팅 서비스 또는 온라인 상점 (Google Play, iTunes 또는 일반 유통 및 소매를위한 MS Store).

GameMaker Studio 2로 제작 한 게임은 판매를 포함하여 원하는 방식으로 자유롭게 배포 할 수 있습니다. 물론 이것은 스프라이트, 이미지 및 사운드를 배포 또는 판매 할 수 있으며 모든 자산에 대한 법적 권리가 있다고 가정하고 게임이 YoYo Games EULA를 준수한다고 가정합니다. GameMaker Studio 2 .
