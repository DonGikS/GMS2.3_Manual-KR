디버깅

게임 메이커 언어 (GML) 또는 드래그 앤 드롭 (DnD ™)을 사용하든 게임을 프로그래밍 할 때 실수를하기 쉽습니다. 잘못된 변수를 사용하거나, 잘못된 인수를 전달하거나, 잘못된 함수를 사용하면 우리가 만드는 가장 일반적인 오류 중 일부일뿐입니다. 이러한 실수는 스크립트 / 액션 편집기에 포함 된 구문 검사기에서 항상 감지되는 것은 아닙니다. 즉, 이러한 실수는 실제로 게임을 실행하거나 컴파일 할 때만 분명해지며, 때로는 무엇이 잘못되었는지 정확히 알기가 어려울 수 있습니다. 따라서 선택한 장치에서 게임이 어떻게 수행되고 있는지, 실내에 어떤 인스턴스가 있는지, 어떤 전역 변수가 만들어 졌는지, 그리고 기타 다양한 세부 정보를 확인하는 것이 매우 중요합니다. 이러한 사항을 확인하는 것을 " 코드 디버깅 "이라고하며, 발견 할 수있는 오류나 문제는 " 버그 "라고합니다.

이제 게임에 소위 "디버그 코드"를 추가하여 필요한 정보를 화면에 표시하거나 오류가있을 때 메시지를 표시하도록 할 수 있습니다. 이것은 종종 많은 작업이며 문제가 처음에 어디에 있는지 확실하지 않은 경우 버그를 추적하는 데 시간이 많이 걸리는 시행 착오가 될 수 있습니다. 이를 위해 GameMaker Studio 2에는 디버그 모드라는 특수 테스트 모드가 있으며, 디버그 모듈 을 시작하는 포괄적 인 오류보고 시스템도 있으며, 게임 관련 문제를 추적하는 데 도움이되는 메시지를 컴파일 링 에 표시합니다.

아래 섹션에서는 게임을 디버깅 할 수있는 다양한 방법을 설명합니다.

 

오류보고 오류보고

프로젝트를 컴파일하고 실행하는 동안 오류가 발생하면 아래와 같이 컴파일 오류 출력 창에 메시지가 표시됩니다.

대부분의 컴파일러 오류 메시지는 오류가 발생한 개체를 나타내는 메시지와 줄 번호와 함께 이벤트, 마지막으로 오류 자체에 대한 간단한 설명과 함께 동일한 개요를 따릅니다. 오류보고에 의해 제공되는 이러한 유형의 정보는 완성 된 게임의 충돌을 유발할 구문 및 언어 관련 버그를 감지하고보고하지만 매우 미묘한 버그는보고하지 않으며 피드백을 제공하지 않으므로 매우 중요합니다. 성능에. 이러한 유형의 경우 디버그 모드 (아래에 설명)를 사용해야합니다.

이러한 컴파일 오류 외에도 구문 오류가 발생할 수 있습니다. 이는 DnD ™ 액션 또는 코드를 추가 할 때 감지되며 아래 이미지와 유사한 구문 오류 출력 창 에 표시됩니다.

컴파일러 오류와 마찬가지로 구문 오류 창은 개체, 이벤트 및 문제를 일으키는 특정 줄로 안내하며 종종 오류에 대한 간단한 설명과 함께 표시됩니다.

 

디버그 모드 디버그 모드

If you need to check things more carefully, you can run the game in Debug Mode by clicking the debug run button . 그렇게하면 게임과 함께 디버그 모듈 이 열립니다. 이 모듈을 사용하면 글로벌 퍼포먼스부터 개별 인스턴스의 개별 변수 값까지 게임에서 모든 것이 작동하는 방식을 모니터링 할 수 있습니다. 다음 페이지에서 디버그 모듈과 그 가능성 및 사용 방법에 대한 추가 정보를 찾을 수 있습니다.

  • 디버거

디버그 모듈을 사용할 때 코드 또는 DnD ™에 중단 점 을 추가 할 수도 있습니다. 중단 점은 게임 루프에서 디버그 모듈을 일시 중지하고 한 번에 한 줄씩 실제 게임 코드를 단계별로 안내하는 지점입니다. "F9"키를 사용하여 중단 점을 추가 할 수 있으며, 필요에 따라이를 활성화 / 비활성화 할 수있는 출력 창에 표시됩니다.

 

추가 도구 추가 도구

컴파일 된 게임을 디버깅하기위한 추가 도구로서 Windows에서 최종 게임 패키지를 실행할 때 사용할 수있는 특정 명령 줄 매개 변수 가 있습니다. 작동 방식에 대한 정보는 여기 를 참조 하십시오 .

게임을 테스트 할 때 디버그 오버레이를 생성 할 수 있습니다. 게임을 실행하는 동안 게임 성능에 대한 추가 디버그 정보가 표시되며, 자신의 텍스트를 제공 할 수있는 컴파일 콘솔에 디버그 메시지 를 출력 할 수도 있습니다 게임의 동작을 디버그하는 데 도움이됩니다. 자세한 내용은 다음 페이지를 참조하십시오.

  • show_debug_overlay
  • show_debug_message
  • debug_event
