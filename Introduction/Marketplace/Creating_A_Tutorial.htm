튜토리얼 만들기
이 페이지를 전체 내용으로 보려면 여기를 클릭하십시오

마켓 플레이스 자습서 만들기

Creating a tutorial is similar to creating a regular Marketplace asset package, only now you have to add in specific included files as well as include an initial tutorial page (and an optional video page). The Tutorial workspace will look something like this:일반 Marketplace Assets에 대해 위에서 설명한대로 게시자를 선택한 다음 튜토리얼에 이름과 고유 ID (역 URL 형식)를 제공 한 다음 "유형"드롭 다운에서 튜토리얼을 선택해야합니다. 그러면 기본 HTML 파일을 추가하여 자습서를 시작할 수있는 추가 옵션과 비디오 파일 및 URL (옵션)이 제공됩니다. 비디오 파일은 여기 에서 얻을 수있는 나머지 튜토리얼과 마찬가지로 마켓 플레이스에서 제공 한 템플릿을 사용해야하며, 사용자가 튜토리얼을로드하면 비디오는 IDE 내에서 별도의 프레임으로로드됩니다. 시작 페이지와 비디오 페이지에 제공 한 페이지는 포함 된 파일 내의 상대 경로이며, 탐색기를 열어 GameMaker Studio 2가 필드를 채우는 기본 페이지 이외의 다른 것을 선택할 수 있습니다.

Actually creating your tutorial will require you to add Included Files into your tutorial project, and these have to be added in a very specific way for the IDE to properly detect which files are to be used for the tutorial. The basic folder structure is shown in the image below:

따라서 두 개의 기본 폴더가 있습니다.

  • TutorialContent : 모든 튜토리얼 HTML 파일, 스타일링을위한 CSS 및 필요한 이미지 (리소스에 필요한 이미지가 아니라 튜토리얼 내에 표시되는 이미지)와 필요한 경우 비디오 템플릿 . 예를 들어, 자습서를 장으로 분할하려는 경우 하위 폴더도있을 수 있지만 자습서의 시작 페이지로 사용할 기본 HTML 파일이 하나 이상 있어야합니다.
  • TutorialResources :이 폴더에는 이미지 및 사운드 자산 용 하위 폴더가 두 개 더 있습니다. 튜토리얼에서 사용자가 스프라이트를 추가해야하고 (예를 들어) 튜토리얼에 포함 시키려면 이미지 리소스를 이미지 폴더에 배치합니다. 이 폴더는 튜토리얼 프로젝트로 가져 와서 사용자가 스프라이트 편집기를 열고 이미지를 추가 할 때 탐색기 창의 기본 경로가이 위치로되어 필요한 자산을 쉽게 찾을 수 있습니다.

튜토리얼이 IDE 내에서 작동 하려면이 폴더 구조 와 스타일링을위한 CSS 파일이있는 기본 HTML 시작 페이지가 있어야합니다. 튜토리얼 프로젝트 내에서 비디오의 일부로 건너 뛰거나 특정 섹션이나 기능에 대한 매뉴얼을 열 수있는 많은 특수 링크가 있으며,이 링크는 모두 마켓 플레이스의 공식 튜토리얼 템플릿에 있습니다.

중대한! 고유 한 자습서를 만들기 전에 마켓 플레이스에서 자습서 템플릿 을 다운로드 하는 것이 좋습니다. 여기에는 튜토리얼에서 버튼과 메뉴를 다시 작성하기위한 CSS 및 이미지뿐만 아니라 HTML의 모든 중요한 부분에 대한 예제가 포함되어 있으므로 컨텐츠를 표시하는 방법보다는 걱정할 필요가 없습니다.

당신이 포함 된 파일을 설정하고 필요한 정보를 제공 한 후에는 편집 및 아이콘, 텍스트, 스크린 샷 등을 추가 할 수 있도록이 완료되면 목록 자산에 작업 공간을 열고, 당신은 확인을 클릭합니다 수 있으며, 튜토리얼은 마켓 플레이스에 업로드됩니다 ... 튜토리얼 자산이 게시되면 사용자는 시작 페이지 의 튜토리얼 섹션에서 다운로드하여 설치할 수 있습니다.
