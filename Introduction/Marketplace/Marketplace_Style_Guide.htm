마켓 플레이스 스타일 가이드

이 페이지는 YoYo Games Marketplace에 자산 및 서비스 를 작성하는 사람들을위한 것입니다. 다음은 자산 목록을 설정할 때 자산을 설명하고 브랜딩 할 때 도움이되는 간단한 안내서입니다.

이 지침을 반드시 준수 할 필요는 없지만, 최종 사용자에게 자산을 더 바람직 하게 만들며 Marketplace의 홈 페이지에 자산을 표시 할 가능성이 높습니다. 자산 자체의 내용에 대한보다 일반적인 지침은 이 안내서를 참조하십시오 .

표시 이름 표시 이름

표시 이름은 자산에 고유해야하며, 본인이 상표 소유자가 아닌 한 상표권 단어를 포함하지 않아야합니다. 표시 이름은 제품을 적절하게 설명해야하지만 기본 상점 페이지에 표시된 상자에는 약 11 자만 입력 할 수 있습니다. 이에 대한 자세한 내용은 아래 "아이콘"섹션을 참조하십시오.

설명 설명

자산에 대한 설명은 자산의 용도와 자산 자체에이 정보를 포함하지 않은 경우 자산이 어떻게 사용되는지를 적절하게 설명해야합니다. 설명은 반드시 길어야 할 필요는 없지만 표시 이름을 반복하는 것보다 조금 더 적은 한 문장은 많은 고객을 유치하지 않을 것입니다! 자산 팩이 그래픽 기반 인 경우 자산 팩에 포함 된 이미지 유형을 나열 할 수 있습니다. 자산이 음악 앨범 인 경우 트랙 목록을 제공 할 수 있습니다. 자산이 코드 기반 인 경우 자산 유형 등을 보여주는 짧은 코드 블록을 추가하는 데 도움이됩니다.

다음 마크 업을 사용하여 올바르게 형식을 지정하면 설명이 더 눈길을 끌 것입니다.

  • 기울임 꼴 텍스트 : 기울임 꼴 텍스트 를 만들려면 텍스트를 한 쌍의 별표 (*) 사이에 배치하십시오. 예를 들어 다음과 같습니다. *이 텍스트는 기울임 꼴입니다 *
  • 굵은 체 텍스트 : 굵은 체 텍스트 를 만들려면 다음과 같이 두 쌍의 별표 (*) 사이에 텍스트를 배치하십시오. **이 텍스트는 굵게 표시됩니다 **
  • 굵은 기울임 꼴 텍스트 : 굵은 기울임 꼴 텍스트 를 만들려면 다음과 같이 세 쌍의 별표 (*) 사이에 텍스트를 배치하십시오. ***이 텍스트는 굵고 기울임 꼴입니다 ***
  • URL 참조 : URL 참조 를 만들려면 대괄호 [] 사이에 링크를 원하는 곳에 텍스트를 배치하고 다음과 같이 사이에 명확한 간격을두고 링크를 참조하십시오. 웹 페이지 [link]에 대한 [link] : http : //yoyogames.com 팁 : 페이지를 깔끔하게 만들려면 페이지 하단에 참조를 두는 것이 좋습니다.
  • 글 머리 기호 목록 : 글 머리 기호 목록 을 만들려면 목록 위와 아래에 줄 바꿈이 있는지 확인한 다음 별표 (*) 또는 하이픈 (-)으로 각 줄을 시작하십시오. * 목록 항목 하나 * 목록 항목 2 * 목록 항목 3
  • 번호 매기기 목록 : 번호 매기기 목록 을 만들려면 목록 위와 아래에 줄 바꿈이 있는지 확인한 다음 연속 번호로 각 줄을 시작하십시오 (예 : 1. 목록 항목 1 2. 목록 항목 2 3. 목록 항목 3).
  • 코드 블록 : 들여 쓰기 된 코드 블록을 만들려면 코드 위와 아래에 줄 바꿈이 있는지 확인한 다음 4 개의 공백으로 각 줄을 시작하십시오. 예 : Lorem ipsum 내 코드를 살펴보십시오. game_maker_function (arguments); yet_another_function (더, 인수, 여기);

 

키워드 키워드

자산의 키워드를 결정할 때는 사람들이 자산과 관련하여 일반적으로 검색하는 단어와 자산을 고유하게 만드는 단어를 혼합하여 사용하십시오. 예를 들어, 플랫폼 엔진에 적합한 키워드 세트는 Platform , Jump 및 Gravity 및 Zombie , Space 또는 Tiger 와 같은 고유 한 자산을 설명하는 단어 일 수 있습니다. 또한 사용자가 특별히 디자인 한 자산을 찾는 경우 자산이 표시되도록 게시자 이름을 나타내는 모든 자산에 하나의 단일 태그를 포함 할 수도 있습니다.

기능 그래픽 기능 그래픽

마켓 플레이스 홈 페이지의 메인 쇼릴에서 에셋을 표시하려면 피처 그래픽이 반드시 있어야합니다. 웹 사이트 자체에서 기능 그래픽에 제목이나 설명을 표시하지 않으므로 항상 자산의 제목과 기능 그래픽에 간단한 설명을 포함하는 것이 좋습니다. 예를 들어, 셰이더를 만들면 자산이 셰이더, 스크립트 또는 그림에 사용 된 스프라이트인지 여부를 사용자가 알지 못할 수 있으므로 기능 그래픽으로 일반 스크린 샷을 사용하는 것만으로는 충분하지 않을 수 있습니다. 텍스트 나 중요한 아이콘을 그래픽에 추가 할 때 페이지의 다른 항목을 명심해야합니다. 아래는 지형지 물 그래픽에 중요한 텍스트와 이미지를 포함하기에 알맞은 안전 지대를 나타내는 이미지입니다.

아이콘 아이콘

게임의 아이콘은 모든 카테고리 목록에서 볼 수 있으므로 매우 중요하며 주요 마켓 플레이스 페이지를 탐색하는 사람들에게 자산을 알리는 유일한 방법입니다. 아이콘은 각 자산마다 고유해야합니다. 모든 단일 자산에 게시자 로고 아이콘을 재사용하면 잠재 소비자를 한눈에 혼동 할뿐 아니라 각 개별 자산의 기능을 실제로 광고하지는 않습니다. 또한 자산의 표시 제목이 마켓 플레이스 목록에서 약 11 자로 잘 리므로 자산 이름이 너무 길면 이미지에 다시 포함시켜 사용자가 전체 텍스트를 볼 수 있습니다. 자산 아이콘의 좋은 예는 아래를 참조하십시오.

스크린 샷

스크린 샷은 특히 자산이 그래픽 기반이 아닌 경우 많은 사람들이 어려움을 겪고 있습니다. 그러나 스크린 샷은 코드 기반 인 경우에도 자산을 브랜딩하는 데 중요합니다. 아래는 마켓 플레이스 자산에 대한 스크린 샷을 만드는 몇 가지 "해야 할 일"과 "하지 말아야 할 일"입니다.

해야 할 일 :

  • 비주얼에 무거운 자산이 있다면 모형 또는 스크린 샷을 포함 시키십시오.
  • IDE가 스크린 샷의 주요 초점이 아닌 한, 자산의 스크린 샷을 GameMaker : Studio IDE에 자유롭게 포함하십시오. 이 스크린 샷에서 다른 프로그램과 아이콘이 GameMaker와 함께 표시되지 않도록주의하십시오 (예 : PC의 작업 표시 줄).
  • 원하는 경우 회사 로고와 자산 사용의 주요 이점을 설명하는 텍스트를 포함하십시오.

하지 마십시오 :

  • 스크린 샷은 필수, 또는 사실에 대해 불평 스크린 샷을하지 마십시오 "블랭크"라고 메시지, "스크린 샷", "장소 보유자"또는 자신의 스크린 샷 가치가 없어 다른 내용. 자산이 시각적으로 무겁지 않은 경우 스크린 샷을 사용하여 제품에 대한 자세한 내용을 설명하십시오. 필요한 경우 멋진 배경이나 게시자 로고 사본을 사용하여 설명을 이미지에 복사하여 붙여 넣으십시오. 자산과 관련된 모든 것이 농담이나 빈 화면보다 낫습니다.
  • 요요 게임 로고, 유유 게임 이름, GameMaker 이름이나 스크린 샷에있는 다른 유유 게임 공식 이미지 나 워드 마크를 사용하지 마십시오. 상점을 사용하는 사람들은 YoYo Games 로고를 포함하여 자산이 GameMaker와 함께 작동한다는 사실을 이미 알고 있으며 자산 브랜드를 혼동 할 수 있습니다. 이 점은 기능 그래픽 및 자산 아이콘과 같은 다른 이미지에도 적용됩니다. 그러나 위에서 설명한 것처럼 애셋에 필요한 경우 GameMaker IDE 자체가 실제로 표시 될 수 있습니다.
  • 같은 당신의 스크린 샷에서 기존의 게임에서 스프라이트로 저작권이있는 이미지를 포함하지 마십시오. 자산이 플랫폼 엔진이지만 예술에 능숙하지 않은 경우 기존 게임의 스프라이트를 자리 표시 자로 사용하려는 유혹을받지 마십시오. 귀하의 자산이 다른 사람이나 회사의 권리를 침해 한 것으로 신고 된 경우 귀하의 자산은 상점에서 인출 될 수 있습니다.

비디오 비디오

비디오 예고편은 자산을 마케팅하는 좋은 방법입니다. 자산이 사운드 파일을 주요 판매 지점으로 포함하는 경우 특히 중요합니다. 이는 자산을 다운로드하기 전에 사용자가 사운드를 미리 볼 수있는 유일한 방법이기 때문입니다. 저작물이 사운드 기반 인 경우 동영상 예고편의 오디오에 여러 사운드 클립 또는 알맞은 음악 덩어리를 포함해야합니다.

자산이 코드 기반 인 경우 자신의 GameMaker 사본에서 15 초 정도의 게임 영상을 녹화하는 것만으로도 자산이 실제로 작동하는 것을 보는 것이 도움이 될 수 있습니다.

최고의 비디오는 와이드 스크린이며 최소 1280 x 720 픽셀이지만 자산이 공정하게 표현되는 한 그다지 중요하지 않습니다.

스크린 샷에 대해 위에서 언급 한 브랜딩 가이드 라인은 비디오에도 적용되며, 허가가없는 한 비디오에 GameMaker 로고 또는 다른 사람의 지적 재산을 포함해서는 안됩니다.

위의 지침을 따르면 자산의 인기 나 특징이 보장되지는 않지만 확실히 많은 도움이 될 수 있으며 자산을 바람직하게 만드는 데 아무런 해가 없습니다! 이 기사가 도움이 되었기를 바랍니다. 궁금한 점이 있으면 언제든지 티켓을 제출하거나 소셜 미디어를 통해 문의하십시오.
