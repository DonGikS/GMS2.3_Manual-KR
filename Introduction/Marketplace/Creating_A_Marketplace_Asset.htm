마켓 플레이스 자산 생성

When you select the Create Package option from the IDE Marketplace Menu you will be taken to the following workspace:여기에서 먼저 자산 패키지를 게시하려는 게시자 ID ( 게시자 대시 보드 에서 설정해야 함)를 선택한 다음 표시 이름 과 버전 번호를 제공해야 합니다. 그러면 사용 된 표시 이름과 게시자 계정을 기반으로 패키지 ID가 생성되지만 역방향 URL 형식을 유지하는 한 원하는 경우 다른 것으로 편집 할 수 있습니다. 그런 다음 자산 패키지 유형을 선택해야합니다.

  • 자산 : 마켓 플레이스에서 판매하려는 자산 (스프라이트, 스크립트, 객체 등) 모음입니다.
  • 데모 : 이것은 개념이나 게임 아이디어를 보여주는 데모입니다 .
  • 튜토리얼 : 업로드되는 자산은 GameMaker Studio 2 IDE 내에서 실행되도록 설계된 튜토리얼입니다 (자세한 내용은 이 페이지 참조).

그런 다음 오른쪽 섹션을 사용하여 프로젝트에서 자산 패키지에 추가 할 자산을 선택할 수 있습니다. 모든 자산을 추가하거나 일부 자산 만 추가하고 실수로 추가 한 자산을 제거 할 수 있습니다. GameMaker Studio 2에서 이전에 업로드 할 자산을 생성했으며 현재 프로젝트에 업로드하는 데 필요한 모든 것이 포함되어 있다고 가정합니다.

생각해 내다! 자산 패키지를 작성하거나 사용하려면 이전에 사용자 프로파일 인증서를 작성해야합니다. 여기 에 설명 된 마켓 플레이스 환경 설정에서이를 수행 할 수 있습니다.

당신이 자산의 선택에 만족하고 마켓 플레이스에 패키지를 업로드 할 때, 단순히 하단에있는 좋아요 버튼을 클릭하고 자산 패키지를 생성하고 마켓 플레이스에 업로드됩니다. 업로드가 완료되면 편집 할 자산 페이지의 브라우저보기와 함께 새 작업 공간이 열립니다. 설정 방법에 대한 자세한 내용은 다음 페이지를 참조하십시오.

  • 자산 리스팅

Marketplace Asset을 업데이트해야하는 경우, 원래 패키지를 작성할 때와 거의 동일하게 수행됩니다. 이제 사용 가능한 패키지 드롭 다운 목록에서 업데이트 할 패키지를 지정해야합니다.

업데이트 할 때 패키지 ID를 변경할 수 없으며 표시된 버전 번호 는 현재 버전 번호이며 이를 적절하게 증가시켜야합니다. 그렇지 않으면 업데이트가 실패합니다 (두 개의 자산을 가질 수 없기 때문에) 버전 번호가 동일한 패키지). 자산의 변경 로그에 대한 텍스트를 추가 할 수있는 필드도 있으며, 자산과 함께 업로드되어 마켓 플레이스 페이지에 표시됩니다.
