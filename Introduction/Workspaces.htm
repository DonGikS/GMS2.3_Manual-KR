작업 공간

로그인하고 새 프로젝트를 시작하면 GameMaker Studio 2가 IDE에 도킹 된 기본 창이있는 초기 작업 공간 으로 이동합니다. 일반적으로 작업 공간은 작업하는 동안 게임의 여러 자산을 구성 할 수있는 영역입니다.

As you can see, the initial workspace is on a tab at the top of the screen (and you can rename it by double clicking the tab), but you can create further workspaces for the project by clicking the  단일 프로젝트에 대해 여러 작업 공간을 제공합니다. 예를 들어, 플레이어와 여러 개의 적 대상 사이의 상호 작용을 수행하고있을 수 있으므로 플레이어는 자신의 작업 공간에 있고 다른 대상에는 다른 작업 공간에 있고 둘 다 사용하는 스크립트를 표시하기 위해 다른 작업 공간에 플레이어를 두어야합니다.

Another important feature of workspaces is that you can click 탭에서-마우스 버튼을 계속 누르고 있습니다. down-메인 IDE 창에서 개별 창으로 끌어다 놓기 때문에 여러 디스플레이를 사용하는 경우 작업을 매우 쉽게 구성 할 수 있습니다. 또한 탭을 다시 첫 번째 창으로 끌어서 보조 작업 공간 창을 기본 작업 공간 창으로 다시 병합합니다. 이 작업을 수행 할 때 두 개의 IDE 인스턴스가 실행중인 것처럼 보이지만 둘 다 동일한 프로젝트를위한 것이며 GameMaker Studio 2의 두 인스턴스를 구체적으로 열지 않는 한 하나의 프로젝트를 다른 프로젝트에 가질 수 없습니다.

GameMaker Studio 2를 처음 시작하면 작업 공간이 이미 IDE에 " 도킹 "될 두 개의 창으로 채워집니다. 아래에 간단히 설명되어 있습니다.

  • 출력 — 출력 창 위의 이미지에서도 볼 수 있습니다. 이 창에는 Source Control , Searching 및 Debugging 과 관련된 여러 개의 하위 탭이 있으며 초기 탭은 콘솔 / 컴파일러 출력용이며 테스트를 위해 게임을 컴파일하거나 게임을 만들 때 발생하는 상황을 보여줍니다 배포를위한 최종 패키지. 또한 런타임에 프로젝트에서 보내도록 선택한 디버그 메시지가 표시되고 일반 기본 설정 에서 다른 수량의 정보를 표시하도록 구성 할 수 있습니다. 출력 창을 닫고 복구하려는 경우 Windows 메뉴를 사용할 수 있습니다.
  • 자산 -화면 오른쪽에서 자산 브라우저를 찾을 수 있습니다. 여기에서 게임에서 사용하는 다양한 자산을 생성 및 편집 할 수있을뿐만 아니라 구성, 게임 옵션, 객실 주문 및 기타 항목을 생성 및 변경할 수 있습니다. 마우스 오른쪽 버튼을 클릭하여 자산을 생성합니다 자산 폴더 (또는 그 밖의 주요 자산 브라우저보기에서), 또는이 방법으로 같은 여러 자산을 만들 수 있습니다 상단에있는 플러스 아이콘을 클릭하고 자산 작성 창 (주에서 자산을 선택하여 생성 및 선택 맨 아래에 값을 설정하여 한 번에 입력하십시오. 이 옵션 중 하나를 선택하면 이름 필드가 강조 표시되어 지정된 유형의 빈 자원이 새로 만들어져 즉시 이름을 지정할 수 있습니다. 애셋 브라우저가 기본적으로 IDE에 도킹되어있는 동안 클릭하면 해당 브라우저를 꺼내고 자체 창으로 이동할 수 있습니다. 상단의 "자산 브라우저"텍스트에서 드래그하십시오. IDE의 측면이나 하단으로 드래그하여 언제든지 다시 도킹 할 수 있습니다. 자산 브라우저 창을 닫고 복구하려는 경우 Windows 메뉴를 사용할 수 있습니다. 사용 가능한 자산에 대한 자세한 내용은 편집자 섹션을 참조하십시오.

 

 

작업 공간 RMB 메뉴

While within any workspace you can call up the Right Button Menu at any time by right-clicking  다음 메뉴가 열립니다 :

자산 을 클릭하면 생성 할 수있는 자산 브라우저 의 모든 자산 목록이 열리고 그 중 하나를 선택하면 해당 자산 자원이 생성되고 해당 자원의 편집기 창에 작업 공간이 초점을 맞 춥니 다. Windows 옵션을 클릭하면 작업 공간 내에서 현재 열려있는 모든 창의 목록이 표시되며 해당 창에 작업 공간이 집중되도록 창을 선택할 수 있습니다. 마지막으로 다음과 같은 창을 여는 이동 옵션이 있습니다.

This window permits you to search though every asset, Game Option or Preference in your project for the search term you input. For example, typing "icon" as the search term will show all items in your project that have icon somewhere in their name, eg: if a sprite is spr_Icon_Play, then it will show up here, as will the object " obj_Iconoclast", and you'll even see the Game Options for the different icons like " Icon for hdpi". Clicking on any of the items shown on filtered list will open it in the current workspace or focus the workspace on it if it is already open. Note that you can also use the keyboard shortcut / + T 를 눌러 이동 창을 열고 나열된 항목 수는 기본적으로 최대 16으로 설정되지만 일반 환경 설정 의 이동 섹션에서 변경할 수 있습니다.

작업 공간 탐색에 대한 자세한 내용은 IDE 입력 및 탐색 섹션을 참조하십시오.

 

 

도킹

We mentioned briefly above that you can dock many windows into the current workspace window and you can un-dock those that are already docked. If you click  예를 들어 상단의 자산 브라우저 탭에서 현재 작업 공간의 왼쪽으로 끌어다 놓으면 자유 부동 창이됩니다.

이 대화는 사실이며 특정 창을 작업 영역의 측면 (또는 아래쪽)으로 드래그하면 고정되어 기본적으로 IDE 창 오버레이의 일부가되고 더 이상 독립적이지 않습니다. 이 작업을 수행 할 때 창을 도킹하려는 영역으로 드래그하면 창을 도킹 할 수 있음을 나타 내기 위해 강조 표시됩니다.

이러한 방식으로 작업 흐름과 작업 방식에 맞는 개인화 된 IDE 환경을 만들 수 있습니다. 도킹 된 항목의 측면에있는 단추 (이 페이지 상단의 이미지에서 강조 표시됨)를 클릭하여 모든 도킹 된 창을 개별적으로 숨기거나 숨기기를 해제 할 수 있습니다. 또는 빠른 도킹을 사용하여 도킹 된 모든 항목을 숨기거나 숨기기 해제 할 수 있습니다. 단추 IDE의 맨 위에 있으며, IDE는 세션 간 도킹 된 창을 기억합니다.

참고 : 회의실 편집기와 같이 편집기를 변경하면 일부 창이 도크에서 사라집니다. 그러나 해당 편집기로 돌아 오면 Dock에서 다시 열립니다.
