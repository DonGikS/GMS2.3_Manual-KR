빠른 시작 가이드

빠른 시작 안내서에 오신 것을 환영합니다! 이 매뉴얼 섹션은 GameMaker Studio 2의 가장 중요한 측면을 익히기 위해 고안되었으며 가능한 빨리 게임을 만들 수있는 기본 사항을 보여줍니다. 이 가이드는 필수 IDE 기능을 다루고 프로그래밍에 대한 간단한 소개를 제공하므로 GameMaker Language (GML) 또는 시각적 스크립팅 언어 Drag and Drop ™을 사용하여 제품을 최대한 빨리 활용할 수 있습니다.

GameMaker Studio 2 워크 플로우를 설명하고 무언가를 직접 만들 수 있도록하기 위해, 본 가이드는 GameMaker Studio 2의 기본 사항과 프로그램의 기본 사항, 일반적인 자산 생성 및 움직임과 점수와 같은 기본 사항을 코딩하십시오.

GameMaker Studio 2로 게임을 만드는 것은 개념적으로 다음과 같은 일반적인 단계에 따라 매우 간단하고 간단합니다.

  • 이미지 자산 (생성 스프라이트 , 타일 세트 , 폰트 들)을하고는 자산 브라우저에 추가 할 수 있습니다.
  • 게임에서 사용하려는 것을 나타내는 객체 자산을 만들고 필요한 이미지 자산을 할당하십시오.
  • 특정 이벤트 에 응답하여 작업을 수행하도록 이러한 개체를 프로그래밍하십시오.
  • 프로그램을 수행 할 방 에 이러한 개체의 인스턴스 를 추가하십시오.

다음 페이지에서 위에 나열된 각 항목의 세부 사항을 살펴볼 것이지만 이것이 게임을 만드는 데 필요한 일반적인 작업 흐름입니다. 여기서주의해야 할 중요한 점은 게임의 주요 "빌딩 블록"이 항상 스프라이트, 오브젝트 및 룸이지만이 보조 아이템에 기능이나 행동을 추가하는 데 사용할 수있는 여러 보조 자산이 있다는 것입니다 (예 : 쉐이더 또는 타임 라인 ).

이제 이러한 각 사항에 대해 자세히 설명하기 전에 GameMaker Studio 2가 위의 빌딩 블록을 게임으로 전환하는 방법에 대해 간략하게 설명하겠습니다. 기본적으로 모든 것은 Game Loop 라고하며, 초당 프레임 수로 측정 된 속도를 가지며, 하나의 "프레임"은 GameMaker에서 Step 이라고합니다. 초당 기본 프레임 (또는 FPS )은 60입니다. 즉, GameMaker Studio 2를 통과하는 1 초마다 게임 루프를 60 회 실행하고 각 단계 (루프, 프레임)마다 많은 이벤트가 발생하고 게임이 진행됩니다.

이를 이해하기 위해 게임 룸에서 움직일 캐릭터가 있다고 상상해 봅시다. 해당 캐릭터에 대한 스프라이트 를 생성 한 다음 객체 를 생성하고 스프라이트를 할당하고 객체에 Step Event 라는 이벤트를 추가합니다. 이 경우 오른쪽으로 2 픽셀 씩 이동하도록 지시합니다. 그런 다음 해당 객체의 인스턴스 를 게임 룸에 배치하고 게임을 실행할 때 인스턴스는 1 단계마다 2 픽셀 씩 왼쪽으로 이동하기 시작합니다 (단계 이벤트는 게임 단계마다 한 번씩 실행 됨). 오른쪽으로 120 픽셀 이동했습니다.

위의 예에서 객체가 왼쪽으로 이동하도록하는 방법에 대해 이야기하지만 어떻게해야합니까? 글쎄, 그건 우리가 중 하나를 사용하여 쓰기 프로그램을 사용하여 이루어집니다 끌어 Dropâ "¢ (시각 스크립트 언어) 또는 GML을 합니다 (G는 M의 아 케르의 L의 anguage을 AME). 어느 쪽을 사용 하느냐는 자신에게 달려 있고 원하는 방식이지만 자신을 사용하여 게임을 프로그래밍하고 있습니다. 다음 섹션에서는 이에 대해 자세히 설명하지만 계속하기 전에 위에서 논의한 내용 중 일부를 설명하고 다음 섹션을 준비하는 다음 비디오 자습서를 살펴볼 수 있습니다.
