객체와 인스턴스

지금까지 본 애셋으로 게임 애셋 브라우저에 멋진 이미지와 사운드를 추가 할 수 있지만 아직 게임 자체에서 아무것도하지 않습니다! 이를 위해 GameMaker Studio 2에서 가장 중요한 자산 인 Object 자산이 필요합니다. 오브젝트는 게임의 측면을 제어하고 특정 작업을 수행하는 데 사용하는 특수 자산입니다. 대부분의 경우 게임 룸에서 볼 수 있도록 스프라이트가 연결되어 있지만 때때로 트랙 통계와 같은 작업을 수행하거나 컷 장면의 타이밍을 설정하는 "장면" 컨트롤러 오브젝트로 사용됩니다. 등 ... 객체에 동작을 부여하고 특정 이벤트 및 서로에 반응 할 수 있으며 게임에서 볼 수있는 대부분의 객체는 객체 및 사용자와의 상호 작용을 기반으로합니다.

이제 Instances 에 대해 이야기하지 않고 Objects에 대해 이야기 할 수 없습니다. GameMaker Studio 2로 제작 된 게임에서 게임에서 볼 수있는 캐릭터, 몬스터, 공, 벽 등은 실제로 인스턴스 로 간주됩니다. 인스턴스는 기본적으로 자산 브라우저에서 찾은 다음 방에 배치되는 객체의 "복사"입니다 (객체를 템플릿으로 생각합니다). 따라서 인스턴스에 영향을 주거나 변경하는 것에 대해 이야기 할 때, 방에있는 객체의 특정 사본 하나가 영향을받는 반면 나머지는 영향을받지 않는다는 것을 의미하지만, 객체에 영향을 주거나 변경하는 것에 대해서는 그 시점에서 생성 된 모든 인스턴스에도 반영됩니다. 따라서 객체는 인스턴스의 템플릿이며 인스턴스는 게임을 만들기 위해 방에 배치하는 것 입니다.

새 객체 자산을 만들려면 다른 모든 자산과 동일하며 마우스 오른쪽 버튼을 클릭하기 만하면됩니다 Objects asset 폴더에서 Create를 선택하거나 Asset Browser의 아무 곳이나 마우스 오른쪽 버튼으로 클릭하고 Create Object를 선택합니다.

이것은 개체 편집기 창이며 현재 개체 속성을 표시하며 가장 먼저해야 할 일은 " obj_ "또는 다른 식별자와 같은 접두사를 사용하여 개체 이름을 지정하는 것입니다. 일반적으로 객체가 무언가를 그리고 다른 객체의 인스턴스와 충돌을 등록하려면 스프라이트가 할당되어 있어야합니다. 단추. 그러면 사용할 스프라이트를 선택할 수있는 자산 탐색기 가 열립니다 (애셋 브라우저에서 스프라이트를 드래그하여 오브젝트 편집기에 놓아 할당 할 수도 있습니다).

아직 적절한 스프라이트가 없으면 새 스프라이트 버튼에 표준 스프라이트 생성 옵션을 통해 자산을 생성 한 것처럼 새 스프라이트 자산을 생성하고 원하는대로 편집 할 수 있습니다. 이제는 생성을 시작할 때 연 오브젝트에 자동으로 할당됩니다. 스프라이트를 선택하면 스프라이트 편집 또는 이미지 편집 버튼을 눌러 스프라이트 자체의 이미지 또는 스프라이트 자산을 편집합니다. 애셋 브라우저에서 스프라이트를 찾은 다음 편집하라고 표시하는 것보다 빠를 수 있습니다.

스프라이트를 추가하거나 만든 후에는 표시 여부와 같은 다른 객체 속성을 변경할 수 있습니다 ( 예 : 충돌 마스크 종류). 간단한 게임의 경우 여기에서 아무것도 변경할 필요가 없으며 이제 객체에 이벤트 를 추가 할 수 있습니다.

이벤트는 게임 루프에서 신중하게 결정된 순간입니다. 이벤트는 프로그래밍 된 내용에 따라 발생합니다. GameMaker Studio 2는 이러한 이벤트의주기와 함께 작동합니다. 방이 시작된 순간부터 완료 될 때까지 일련의 이벤트가 실행되는 모든 단계에서 실행되는 게임 루프가 있으며 GML 코드 또는 DnD를 배치하도록 선택할 수 있습니다. ¢ 이러한 이벤트에 응답하는 인스턴스에서의 조치 ( 단계 는 게임 속도 설정에 의해 결정되는 게임 시간의 순간입니다 ( "프레임"이라고도 함)).

위에서 다른 모든 이벤트 범주를 볼 수 있으며, 그 중 일부에는 다른 하위 범주가 포함되어 있습니다. 여기서는 모든 이벤트를 진행하지는 않지만 가장 중요한 다섯 가지 이벤트를 간략하게 설명합니다.

이벤트 만들기 이 이벤트는 객체의 인스턴스가 처음 생성 될 때 발생하며, 룸에 들어갈 때 룸 편집기를 통해 룸에 배치 된 인스턴스 내에서 가장 먼저 발생합니다. 즉,이 이벤트는 변수를 초기화하고, 타임 라인을 시작하고, 경로를 설정하는 등의 이상적인 장소이며, 일반적으로 인스턴스가 방에 처음 나타날 때만 수행하면됩니다.
스텝 이벤트 GameMaker Studio 2는 게임 시간을 게임 속도로 단계별로 나눠서 초당 몇 단계가 필요한지를 정의합니다. 단계는 기본적으로 게임이 실행되는 동안 필요에 따라 모든 이벤트를 확인하고 트리거하여 지속적으로 실행되는 루프이므로 단계 이벤트는 인스턴스가 존재하는 동안 게임의 모든 단계마다 확인되는 이벤트입니다. . 단계 이벤트 범주에는 인스턴스 내에서 타이밍을 "미세 조정"하기위한 세 개의 하위 범주가 있지만 일반적으로 기본 단계 이벤트를 가장 많이 사용합니다.
충돌 이벤트 분명히 게임을 만들 때 객체의 두 개 이상의 인스턴스가 충돌 한 시점과 충돌 이벤트가 발생하는 시점을 아는 것이 매우 중요합니다. 이것은 객체에 놓은 다음 충돌을 확인해야하는 다른 객체에 대해 지정하는 이벤트입니다. 이벤트를 추가하고 확인할 객체를 선택하면 GML 코드 또는 DnD ™를 추가하여 충돌을 해결할 수 있습니다.
이벤트 추첨 추첨 이벤트 카테고리에는 여러 가지 추첨 이벤트가 있지만 가장 많이 사용하는 추첨 이벤트는 맨 위에있는 "그리기"입니다. 이 일반적인 드로우 이벤트는 스텝 이벤트와 같이 모든 게임 프레임에서 호출되지만 두 가지 방식으로 작동합니다. 첫 번째는 오브젝트에이 이벤트를 추가하지 않을 때입니다 ... 지정된 Draw 이벤트가 없으면 GameMaker Studio 2는 기본적으로 오브젝트의 다른 곳에 적용된 변환을 사용하여 오브젝트에 지정된 스프라이트를 그리도록 설정합니다 객체 (크기 또는 알파 블렌딩 변경과 같은)와 같으므로 이벤트 목록에 Draw Event가 나열되어 있지 않더라도 인스턴스가 룸에 배치 될 때 여전히 실행 중이며 화면에 계속 그려집니다. 두 번째 방법은 이벤트 자체를 추가하고 GML 또는 DnD ™를 이벤트에 추가하는 것입니다. 이벤트에 무언가를 추가하면 GameMaker Studio 2에 " 이 개체의 인스턴스에 대해 그리는 것을 제어하고 싶습니다 "라고 알려주며 기본 그리기를 완전히 무시 합니다 . 즉, 스프라이트가 할당 된 오브젝트를 가질 수 있고 draw 이벤트를 설정하여 텍스트를 그리면 GameMaker Studio 2에 텍스트를 표시하지 않아도 스프라이트가 그려지지 않습니다.
이벤트 파괴 이 이벤트는 인스턴스가 파괴 될 때 가장 마지막으로 실행되는 이벤트입니다. 객체에 동작을 추가 할 때 종종 간과되지만, 적을 죽일 때 폭발 또는 파티클 효과를 생성하거나 방의 다른 부분에서 객체의 새로운 인스턴스를 다시 생성하는 등 매우 유용 할 수 있습니다. 또는 점수에 점수를 추가하기 위해.
