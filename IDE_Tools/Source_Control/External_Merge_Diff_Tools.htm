외부 병합 / 확산 도구

다른 사람과 프로젝트를 작업 할 때는 여러 사람이 동일한 파일을 편집하고 대부분의 소스 제어 시스템이 충돌 한 파일을 병합하려고 시도하지만 변경 내용이있을 때 항상 충돌이 발생할 가능성이 있습니다. 같은 장소에서해야 할 일을 아는 데 약간의 도움이 필요합니다. 여기에서 병합 도구 를 사용하여 변경 사항을 표시하고 결정하도록합니다.

각 병합 도구가 다르기 때문에 병합 도구 선택 이라는 단추를 클릭하여 충돌 창에서 설정할 수있는 파일을 지정하는 데 사용할 수있는 4 개의 매크로가 있습니다. :

해당 버튼을 클릭하면 Git 플러그인에서 환경 설정 창이 열립니다 :

여기서는 다음과 같은 옵션을 사용하여 선택한 병합 도구에 대한 경로를 제공 한 다음 수행 할 작업을 결정할 때 사용할 병합 도구 옵션 매크로 (또는 매크로)를 제공합니다.

  • $ {scm_mine} : 버전 경로
  • $ {scm_thers} : 버전 경로
  • $ {scm_base} : 충돌하는 변경이 발생하기 전에 버전의 경로
  • $ {scm_merged} : 병합 도구를 저장할 경로

설정에 대한 도움으로 특정 병합 도구에 대한 경로를 제공하면 GameMaker Studio 2는 선택한 도구에 따라 시작하기 위해 몇 가지 기본값을 채 웁니다.

  • KDiff3 : $ {scm_theirs} $ {scm_mine} -o $ {scm_merged}
  • Meld : $ {scm_mine} $ {scm_merged} $ {scm_theirs}
  • 메모장 ++ : $ {scm_merged}
  • TortoiseMerge : / base : $ {scm_base} / theirs : $ {scm_theirs} / mine : $ {scm_mine} / merged : $ {scm_merged}

병합 도구가 없거나 사용하지 않으려는 경우 IDE에서 스크립트를 열고 수정을 시도 할 수 있습니다.

그래서 절에 명시된 바와 같이 우리의 예를 충돌 다시 충돌 ... 우리는 마스터에서 풀을 수행하고 파일 중 하나는 갈등한다. 그러면 병합 도구를 구성 했으므로 충돌 한 스크립트를 클릭하고 이제 병합을 클릭 할 수있는 소스 제어 충돌 창이 열립니다. 병합을 클릭하면 충돌을 처리하고 제공된 매크로를 사용하여 필요한 변경을 수행 할 수있는 도구가 열립니다. 병합 도구를 마치고 닫으면 GameMaker Studio 2 IDE에 다음 확인 창이 표시됩니다.

병합 도구를 사용하여 방금 수행 한대로이 프롬프트에 "예"라고 답한 다음 병합 조작의 끝을 나타 내기 위해이 변경 사항을 커밋해야합니다 ( 소스 제어 > 커밋 변경 ). 커밋 창에 이제 병합이 진행 중임을 표시하고 스테이지되지 않은 파일도 표시합니다.

이 스테이지되지 않은 파일은 .base , .ours 및 .theirs 가 추가 된 스크립트 파일입니다. 이 모든 것을 안전하게 삭제할 수 있습니다 상단에 다음 + 맨 아래에 모두 선택하십시오. 그런 다음 선택한 파일을 마우스 오른쪽 버튼으로 클릭하고 경로 삭제를 선택하여 제거하십시오. 또한 커밋 메시지가 미리 작성되어 병합되어 있고 충돌이 있음을 나타냅니다 (우리는 원하는대로 추가하거나 편집 할 수 있음).

커밋 을 클릭하십시오.

다른 파일에 문제가있을 수 있기 때문에 원격 또는 공유 저장소를 사용할 때의 워크 플로우는 다음과 같아야합니다.

Make Changes > Save > Commit > Pull > Merge > Push

저장소에 커밋을 수행 할 때 사용할 Diff Tool 도 설정해야합니다. 이것은 병합 도구와 동일한 환경 설정에서 설정됩니다.

Diff 도구는 병합 도구와 거의 같은 방식으로 구성되지만 $ {scm_base} 및 $ {scm_theirs} 옵션 만 사용합니다.이 경우 $ {scm_base} 는 수정되지 않은 파일을 나타내고 $ {scm_theirs} 는 현재를 나타냅니다 파일 상태. 도구 옵션은 병합 도구와 마찬가지로 Meld , KDiff3 및 TortoiseMerge 의 기본 옵션으로 미리 채워져 있어야합니다. GameMaker Studio 2는 단일 파일 diff 패치의 출력을 지원하지 않으므로 Notepad ++의 기본값은 없습니다.

Diff 도구를 사용하여 현재 버전과 이전 버전 사이의 변경 사항을 보려면 커밋 창을 연 다음 파일을 두 번 클릭하십시오. 도구가 올바르게 설정되면 도구가 시작되고 파일 사이의 변경 사항을 볼 수 있습니다.
