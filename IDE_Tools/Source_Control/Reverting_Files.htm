파일 되돌리기

이전 섹션에서는 리포지토리 를 생성 하고 커밋 하는 방법을 보았지만 실수를해서 이전 커밋으로 "롤백"하려는 경우 어떻게됩니까? 고의적 인 실수를 저지르고 어떻게 처리 할 수 ​​있는지 봅시다 ...

우선 새 리소스를 생성합니다 (이 예제에서는 스크립트 리소스를 생성 함). 소스 컨트롤에 관한 한 아직 존재하지 않는 상태 아이콘에는 상태 아이콘이 표시되지 않습니다. 스크립트 리소스에서 다음과 같은 간단한 코드 줄을 추가하십시오.

show_debug_message("Hello World");

If you now close the code editor, the script will save and the modified file icon 자산 브라우저 에서 옆에 나타납니다. 커밋 창을 열고 새로운 리소스가 자동으로 추가됨에 따라 4 가지 단계적 변경이 있음을 알 수 있습니다.

준비된 변경 사항은 다음과 같습니다.

  • .yyp 프로젝트 파일
  • 뷰 파일
  • GML 스크립트 자체
  • 그것을 참조하는 .yy 파일

앞서 설명한 것처럼 커밋 메시지를 입력 한 다음 커밋 버튼을 클릭해야합니다. 이제 스크립트로 돌아가서 코드 줄을 다음과 같이 변경합니다 (예 :).

show_debug_message("Hello World, how are you?");

When we close the code editor, again a red icon 스크립트 옆에 표시되지만 코드에 실수가 있음을 깨달았으며 이전 버전으로 롤백하여 문제를 해결하려고합니다 (예를 들어 스크립트를 열고 쉽게 수정할 수 있음) 매우 간단하지만 일반적으로 실현 가능한 접근 방식이 아닌 대규모 프로젝트에서는). 우리가해야 할 일은 변경 사항을 되 돌리는 것입니다.

변경 사항을 되돌리려면 먼저 커밋 창을 다시 열어야하므로 소스 컨트롤 > 커밋 변경으로 다시 한 번 이동합니다. 변경된 파일을 다시 커밋 할 준비가 되었으나 원하는 단계가 아니므로 모두 스테이지 해제 버튼을 클릭하여 준비 영역에서 가져옵니다.

If we now right click 변경된 파일에서 경로 되돌리기 옵션이있는 컨텍스트 메뉴가 나타납니다.

이 옵션을 선택하면 파일을 이전 상태로 되돌립니다.

IMPORTANT! When you revert you will get a message asking you to reload or save the current project due to the file checker detecting changes on disk:
저장하면 되 돌린 파일을 다시로드하는 대신 메모리가 저장되므로 다시로드를 누르고 저장하지 않아야합니다.

그것은 무엇인가를 편집하고 이미 커밋을 수행하지 않고 다시 변경하고 싶을 때 유용하지만, 하나 이상의 변경을 커밋하고 이전 커밋으로 롤백하려는 경우는 어떻습니까? 자, 스크립트를 다시 편집하여 다음과 같이 말하겠습니다.

show_debug_message("This is a bad idea");

이제 우리는 소스 제어하에 그것을 얻기 위해 이전과 같이 커밋합니다. 그래서 이것은 나쁜 생각이었고 우리는 롤백하기를 원합니다. 먼저 소스 제어 > 히스토리보기 창을 열어야합니다. 여기서는 초기 커밋, 스크립트 커밋, "나쁜 생각"으로 편집 한 스크립트 커밋으로 시작하여이 창에있는 모든 이전 커밋 목록을 볼 수 있습니다.

이제 두 가지 선택이 있습니다. 우리는 전체 수정본을 되돌릴 수 있습니다-선택된 커밋의 파일로 변경된 모든 파일을 되돌 리거나 경로를 되돌 리도록 선택하면 단일 파일을 되돌릴 수 있습니다. 첫 번째 옵션으로 시작해 봅시다 :

  • 마우스 오른쪽 버튼으로 클릭 커밋 히스토리 분할 창 ( "Hello World"커밋)의 두 번째 커밋.
  • 되돌리려면이 개정을 선택한 다음 경고 메시지에 다시로드를 선택합니다.

You should see that the script now has a red status icon 을 열면 다음과 같은 결과가 나타납니다.

그런 다음 프로젝트에 대한 커밋을 수행 할 수 있으며 다시 한 번 다시 돌아올 것입니다. 그러나 파일에 대한 해당 작업을 취소하고 다음을 수행하여 "나쁜 생각"을 다시 수행 할 수도 있습니다.

  • 커밋 변경 창을 엽니 다.
  • "Hello World"코드가 포함 된 스크립트를 준비 해제하십시오.
  • 마우스 오른쪽 버튼으로 클릭 경로 되돌리기 .

이제 우리는 다시 프로젝트에 잘못된 파일을 가지고 있습니다! 글쎄, 우리는 모든 것을 이전 버전으로 되 돌리는 대신 파일 경로 만 되돌릴 수있는 기회를 가질 수 있습니다. 이를 위해 다음을 수행해야합니다.

  • 보기 히스토리 창을여십시오.
  • 두 번째 커밋 ( "Hello World"커밋)을 클릭하십시오.
  • 마우스 오른쪽 버튼으로 클릭 파일 창의 스크립트 GML에서 경로 되돌리기를 선택합니다.

"Hello World"코드를 다시 가져 와서 수정 된 파일로 소스 컨트롤을 다시 업데이트하기 위해 새로 커밋 할 수 있습니다.
