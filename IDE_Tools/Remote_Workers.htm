원격 근무자

게임을 컴파일 할 때는 일반적으로 GameMaker Studio 2가 설치된 컴퓨터 인 "로컬"워커를 사용합니다. 그러나 경우에 따라 프로젝트를 빌드하기 위해 다른 컴퓨터를 사용해야 할 수도 있습니다. 예를 들어, Mac에서 작업 중이고 Windows 프로젝트를 빌드하려는 경우 원격 작업자 를 구성하고 Windows PC에서 빌드를 수행하려고합니다. 기본적으로 원격 작업자를 사용하는 경우 GameMaker Studio 2는 실행중인 (로컬) 컴퓨터에서 모든 필요한 파일을 원격 작업자 컴퓨터로 보낸 다음 해당 컴퓨터를 사용하여 프로젝트의 최종 실행 파일을 빌드합니다 ( 원격 시스템에 연결하거나 나중에 로컬 시스템에 반환). 그러나이 기능을 사용하려면 빌드 머신에 특정 도구를 설치하고 일부 설정을 수행해야합니다. 여기에서 설명합니다.

중대한! 로컬 Windows PC와 원격 Windows PC를 사용하거나 로컬 Mac PC와 원격 Windows PC간에 원격 작업자를 설정할 수 있지만 현재 로컬 Windows (또는 Mac) PC와 원격 Mac PC를 설정할 수는 없습니다. : Mac 원격 작업자는없고 Windows 작업자 만있을 수 있습니다.

초기 설정 초기 설정

원격 작업자를 설정하려면 먼저 원격 작업자 빌드 도구 응용 프로그램 을 사용할 PC에 설치해야합니다. 로컬 컴퓨터에서 GameMaker Studio 2의 빌드 메뉴 로 이동하여 " 원격 작업자 설치 프로그램 표시 "옵션을 선택하면이 도구를 찾을 수 있습니다.

또는 Target Manager 에서 "eye"아이콘을 클릭하고 다음과 같은 방식으로 Remote Worker 설치 관리자 를 얻을 수 있습니다.

설치 프로그램은 " GameMakerStudio-Remote-Installer-XXXXXX.exe "와 같은 이름으로 표시되며 원격 작업자로 사용하려는 PC로 복사해야합니다 (GameMaker Studio 2를 설치할 필요는 없습니다. 원격 작업자,이 프로그램 만 해당).

빌드 PC에 복사 한 후 화면의 지시에 따라 설치하고 설치 프로그램이 완료되면 OS에서 방화벽을 통해 네트워크에 액세스하라는 메시지가 표시 될 수 있습니다. 액세스 권한을 부여하면 설치 프로그램이 웹 브라우저에서 페이지를 열고 다음 단계에서 필요할 때이 페이지에 표시된 IP 주소 와 포트 번호를 기록해 두어야합니다.

이제 GameMaker Studio 2가 설치된 머신으로 돌아가서 Target Manager 버튼을 클릭해야합니다 대상 관리자 창을 열고 작업자 섹션으로 이동하여 작업자 추가 버튼을 클릭하십시오. . 그러면 다음 창이 열립니다.

여기에서 원격 작업자의 이름을 지정해야합니다 (대상 관리자의 작업자 목록 에 표시됨). 그리고 호스트 이름 섹션에서 이전에 표시하고 언급 한 포트와 함께 주소를 추가하십시오. 작업자 기계.

모든 것이 올바르게 진행되면 원격 작업자 시스템에 성공적으로 연결할 수 있다는 메시지가 나타납니다. 원격 작업자를 설치하고 기본 IDE 컴퓨터에 연결 한 후에는 프로젝트를 컴파일하기 전에 각 플랫폼에 대한 로컬 빌드 설정 (자세한 내용은 섹션 참조)을 설정해야합니다.

 

원격 설정 원격 설정

설정 페이지에는 원격 작업자 응용 프로그램의 다양한 경로 및 빌드 설정이 표시됩니다. 설정 맨 위에는 원격 작업자 응용 프로그램에서 사용되는 다른 경로가 있으며 다른 작업을 수행하기 전에 필요한 경우 경로를 수정하고 대체 경로로 설정해야합니다 (일반적으로 기본 경로는 양호해야 함). 다음 방법 중 하나를 사용하여 원격 작업자 설정 페이지에 액세스 할 수 있습니다.

  • 원격 컴퓨터의 브라우저에서 열리는 원격 작업자 페이지 상단의 "설정"링크를 클릭하십시오.
  • 원격 작업자 트레이 아이콘에서 RMB를 사용하여 원격 컴퓨터의 브라우저에서 설정을 엽니 다.
  • 로컬 머신에서 Target Manager를 열고 "cog"아이콘을 클릭하여 로컬 머신의 창에서 작업자 설정을 엽니 다.

프로젝트 빌드 경로를 설정할 수있는 다양한 옵션을보고 설정할 수있는 설정 페이지가 표시됩니다.

로컬 빌드 설정 로컬 빌드 설정

원격 작업자 응용 프로그램의 경로를 확인한 후 필요한 경우 다른 로컬 빌드 설정 을 설정할 수 있습니다. 플랫폼마다 다른 빌드 도구를 빌드 PC에 설치해야하며 원격 작업자는이 도구를 사용하여 설정 페이지를 통해 게임을 빌드 할 수 있어야합니다. 여기에서 첫 번째 설정은 일반 설정입니다.

여기 도움말 서버와 디버거 사용에 대한 포트의 포트를 설정할 수 있습니다. 디버그 모듈을 사용할 때 연결 문제가있는 경우 변경할 수 있지만 일반적으로 기본값을 그대로 두어야합니다.

이제 원격 작업자가 빌드 할 수있는 플랫폼에 대해 다른 옵션을 설정할 수 있습니다.

Windows 대상 플랫폼에 액세스 할 수있는 경우 가상 머신 (VM) 옵션을 사용하여 Windows PC에서 원격 작업자를 사용하여 즉시 구축 할 수 있어야합니다. 그러나 YoYo Compiler (YYC) 를 사용하려면 빌드 PC에서 Visual Studio의 경로를 설정해야합니다. 프로젝트를 컴파일하기 위해 Windows PC를 설정하는 방법에 대한 자세한 내용은 다음 헬프 데스크 기사를 참조하십시오.

  • Windows 용 설정

컴파일 컴파일

원격 작업자를 설정하고 로컬 빌드 설정을 완료하면 (위 섹션 참조) 게임 컴파일에 사용할 수 있습니다. 이를 위해서는 대상 관리자 목록에서 원격 작업자를 선택하고 테스트 / 컴파일 할 플랫폼과 다른 옵션 (컴파일 유형 또는 구성 등)을 선택한 다음 재생을 누르는 경우입니다. 또는 실행 파일 작성 .

NOTE: The available platforms in the Target Manager will change depending on the capabilities of the Remote Worker computer, and not all targets will be available. Also note that there is no "Test" target for Remote Workers.

단순히 프로젝트를 재생하는 경우 원격 작업자 시스템이이를 컴파일하여 실행하지만 실행 파일을 작성하는 경우 원격 작업자가 프로젝트를 컴파일 한 다음 * .zip 파일을 로컬 시스템에 리턴하여 저장합니다.

디버그 버튼도 없습니다. 또는 청소 버튼 원격 작업자를 사용할 때 사용할 수 있습니다. 또한 현재 한 번에 하나의 원격 빌드 만 수행 할 수 있습니다.

원격 작업자가 원격 컴퓨터에 설치되어 실행되면 작업 표시 줄에 아이콘으로 표시되며 마우스 오른쪽 단추를 클릭하여 다음 옵션을 표시 할 수 있습니다.

  • 열기 : 기본 브라우저에서 Remote Worker 응용 프로그램을 엽니 다.
  • 로그 표시 : 브라우저에서 원격 작업자의 로그 파일을 엽니 다.
  • 네트워크 설정 : 여기에서 원격 작업자 컴퓨터로 빌드를 보낼 수있는 컴퓨터를 필터링 할 수 있습니다. 각 필터는 CIDR 스타일 필터이거나 허용 할 시스템의 호스트 이름 일 수 있습니다. 여러 대의 컴퓨터를 하나의 원격 작업자에 연결하거나 연결 유형 (예 : 유선 및 Wi-Fi)을 혼합하려는 경우 여기에서 IP 허용 범위를 편집해야 할 수 있습니다.
  • 설정 편집 : 원격 작업자 설정 메뉴를 엽니 다 (위의 설정 섹션에서 설명).
  • 종료 : 원격 작업자 앱을 종료합니다. 따라서 원격 작업자 앱을 다시 시작할 때까지 더 이상 로컬 컴퓨터에서 앱에 연결하거나 컴파일 할 수 없습니다.
