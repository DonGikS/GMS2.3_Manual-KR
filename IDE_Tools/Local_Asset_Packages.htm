로컬 자산 패키지

도구 메뉴 에는 로컬 자산 패키지 를 생성하고 가져 오기위한 두 가지 옵션이 있습니다. 로컬 자산 패키지는 기존 프로젝트에 쉽게 설치할 수있는 * .yymp 파일 형식을 사용하여 저장된 리소스 그룹입니다. 이는 YoYo Games Marketplace 를 사용하지 않고도 스크립트 라이브러리, 스플래시 화면 소개, 메뉴 시스템 등과 같이 일반적으로 사용되는 리소스를위한 패키지 라이브러리를 직접 만들 수 있다는 의미입니다.

 

자산 패키지 생성

자산 패키지를 작성하려면 먼저 프로젝트에 저장하려는 자원을 작성한 후 도구 메뉴에서 로컬 패키지 작성 옵션을 선택해야합니다. 다음 창이 열립니다 :

이 창에서 선택적인 게시자 이름 과 필수 표시 이름 및 패키지 ID를 제공 할 수 있습니다. 패키지 ID는 패키지가 저장 될 파일 이름을 생성하는 데 사용되며 생성시 * .yymp 파일 확장자가 자동으로 추가됩니다. 패키지의 버전 번호 를 설정하고 서명 여부를 선택할 수도 있습니다 (기본적으로 해제되어 있음). 패키지에 서명하려면 서명 인증서를 작성해야합니다 ( Marketplace 기본 설정 에서이를 수행 할 수 있음). 서명 된 자산은 생성 후 변조 된 경우 가져올 수 없습니다.

Once you have filled in the details of the package, you can then go ahead and select the assets that you want to include from the "Project" section on the right (using the left mouse button 또는의 조합 / 과 마우스 왼쪽 버튼과 함께). 선택한 자원은 모두 추가 또는 모두 추가 단추를 사용하여 " 자산 패키지 "보기로 이동하고 모두 제거 및 모두 제거 단추를 사용하여 다시 제거 할 수 있습니다.

리소스 선택에 만족하고 필요한 세부 사항을 입력 한 후 Ok 버튼을 클릭하면 패키지를 저장할 위치를 선택할 수있는 파일 탐색기가 열립니다. 일단 선택되면 패키지가 생성되고 OS 파일 탐색기에서 패키지 위치를 열 수있는 옵션과 함께 확인 창이 표시됩니다.

 

자산 패키지 가져 오기

로컬 자산 패키지를 가져 오는 방법에는 두 가지가 있습니다.

  • 첫 번째는 * .yymp 파일을 탐색기 창에서 GameMaker Studio 2 IDE로 드래그하는 것입니다.
  • 두 번째는 도구 메뉴 에서 " 로컬 패키지 가져 오기 "옵션을 선택하는 것입니다.

도구 메뉴를 사용하면 탐색기 창이 열리고 패키지 위치를 찾은 다음 가져 오기 위해 선택할 수 있습니다. 어느 쪽이든 GameMaker Studio 2에서 다음 창을 엽니 다.

왼쪽에는 패키지의 내용이 있고 오른쪽에는 가져 오려는 부분이 있습니다. 패키지에서 하나 이상의 리소스를 선택할 수 있습니다 (마우스 왼쪽 버튼 사용) 또는의 조합 / 과 마우스 왼쪽 버튼과 함께)를 누른 다음 추가 버튼을 사용하여 가져 오기 위해 추가하거나 모두 추가 버튼을 사용하여 가져올 모든 것을 추가하십시오.

모든 준비가 끝나면 가져 오기 버튼을 클릭하여 선택한 리소스를 프로젝트에 추가 할 수 있습니다. 또한 모든 자원을 새 프로젝트로 가져 오기 옵션이 있습니다. 그러면 새 프로젝트가 만들어지고 (저장할 위치를 지정하라는 메시지가 표시됨) 패키지 자원을 추가 할 수 있습니다.

GameMaker Studio 2.3 이전 버전을 사용하여 생성 한 자산 패키지는 2.3 업데이트로 프로젝트 파일 형식이 변경되었으므로 변환해야합니다. 즉, 자산을 다른 프로젝트로 자주 가져 오는 경우 전체 프로젝트를 가져 와서 모든 것을 변환 한 다음 새 패키지로 새 위치로 내보내는 것이 좋습니다.
