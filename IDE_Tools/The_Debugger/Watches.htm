시계

디버거의 전체 아이디어는 게임의 모든 세부 사항을 "감시"할 수 있도록하는 것입니다. 이것은 기본적으로 IDE 하단에 도킹 된 출력 창 에서 열리는 "감시 창"을 사용하여 수행됩니다. 각기 다른 탭을 클릭하면 포함 된 시계를 볼 수 있으며, 일부 시계는 실시간으로 업데이트 할 수 있습니다 (실시간 디버깅시) (디버거 도구 모음에서 활성화 됨) 다른 사용자는 디버거가 게임을 일시 중지해야합니다.

임의의 창에서 변수 를 볼 때 여러 데이터 유형 중 하나 일 수 있습니다. 그러나 GameMaker Studio 2가 특정 항목을 저장하는 방식으로 인해 디버거는 변수가 무엇을 보유하고 있는지 정확히 알지 못합니다. 이는 데이터 구조 또는 인스턴스의 내부 ID 값이 정수 이므로 디버거는 변수에 저장되는 정수 값이 데이터 구조 또는 인스턴스에 대한 것인지 아니면 단순히 정수 값인지 항상 알 수 없기 때문입니다. 따라서 마우스 오른쪽 버튼을 클릭하면 변수 값에 상관없이이 변수가 될 수있는 사용 가능한 데이터 유형을 나열하는 팝업 창이 표시됩니다.

문자열과 배열은이 옵션을 허용하지 않으며, 데이터 구조가 인스턴스 변수에 저장되어 있어도 ID 번호는 전역 적이므로 4 개의 인스턴스가 있고 각 인스턴스가 ds_list를 생성하면 (예 :) 각 목록의 ID는 0에서 3까지이며 모두 0이 아닙니다.

데이터 유형을 선택하면 변수는 작은 항목을 클릭하여 구조의 항목을 표시합니다. 이름 옆에. 그러면 구조 내용이 확장되고 두 번 클릭하여 표시된 변수의 값 중 일부를 변경할 수도 있습니다. 가치에 새로운 가치를 부여합니다.

참고 : 이런 식으로 배열이나 데이터 구조를 재 할당 할 수는 없지만 일부 내용을 변경할 수 있습니다 (맵의 "키"값 또는 우선 순위 대기열의 우선 순위 값은 변경할 수 없음).

아래에는 각기 다른 감시 창 목록과 용도 및 사용 방법에 대한 설명이 나와 있습니다.

지역 주민 지역 주민

Locals 창에는 현재 실행중인 현재 이벤트 (또는 스크립트)의 이름, 현재 스크립트에 대한 VM (가상 머신) 코드의 오프셋이 포함 된 현재 단계 특성이 표시됩니다 ( "PC"는 "프로그램 카운터"의 약어 임) ), 코드 블록을 실행하는 "self"인스턴스 및 이벤트의 "other"인스턴스 이 마지막 부분은 이벤트가 충돌 이벤트이거나 코드에 " with "가있는 경우에만 유효합니다. 인스턴스가 충돌하는 경우 "other"는 충돌중인 다른 인스턴스 (및 해당 변수)를 표시하지만 with 인 경우 "Self"인스턴스는 코드를 실행중인 인스턴스가되고 "Other"는 실제로 with 가 포함 된 인스턴스 다른 모든 경우에 "other"는 "Self"와 동일한 데이터를 표시합니다.

"Self"및 "Other"인스턴스는 다음을 클릭하여 확장 할 수 있습니다. 단계별로 코드 블록에 대해 현재 선언 된 모든 로컬 및 인스턴스 변수의 목록을 봅니다. 마우스 오른쪽 버튼을 클릭하면 값에서 데이터 구조의 세부 사항을 보도록 유형 을 설정할 수 있습니다.

글로벌 글로벌

Globals 창에는 현재 선언 된 모든 전역 변수와 해당 값의 목록이 표시됩니다. 마우스 오른쪽 버튼을 클릭하면 값에서 데이터 구조의 세부 사항을 보도록 유형 을 설정할 수 있습니다.

시계 시계

이 창은 추적하려는 특정 변수를 추가하기위한 것입니다. 이것은 전역 또는 인스턴스 범위 변수, 내장 변수 또는 배열 또는 데이터 구조 일 수 있습니다. 단계별로 진행중인 인스턴스 코드에 감시 변수가있는 경우 여기에 표시됩니다. 즉, Locals 또는 Globals 창에서 값을 검색하지 않고도 값을 쉽게 추적 할 수 있습니다. 마우스 오른쪽 버튼을 클릭하면 값에서 유형을 설정하거나 삭제하거나 창에서 모든 시계를 지울 수 있습니다.

인스턴스 인스턴스

이름에서 알 수 있듯이이 창에는 모든 인스턴스 변수, 인스턴스 이벤트의 로컬 변수, 내장 변수 및 소스 창에서 현재 단계별로 진행중인 인스턴스와 관련된 구조체가 표시됩니다 (인스턴스가 "other"키워드를 사용하여 참조되는 경우) "또는 with 문을 사용하면이 창에 해당 인스턴스 속성이 대신 표시됩니다). 마우스 오른쪽 버튼을 클릭하면 값에서 유형을 설정하거나 삭제하거나 창에서 모든 시계를 지울 수 있습니다.

모든 인스턴스 모든 인스턴스

현재 회의실에있는 모든 인스턴스를 확인해야하는 경우이 창을 선택하면 해당 인스턴스가 표시됩니다. 내장 변수, 인스턴스 변수 또는 구조체 의 상태를 볼 수 있으며 마우스 오른쪽 버튼을 클릭 할 수 있습니다 유형 을 설정하는 값.

선택된 인스턴스 선택된 인스턴스

이 창은 게임에서 인스턴스를 선택할 때 사용됩니다. 게임을 일시 정지 한 다음 게임 창에서 보이는 인스턴스를 클릭하면됩니다. 마우스 커서 아래에있는 인스턴스는 선택한 인스턴스 창에 해당 정보가 표시되며, 여기에는 변수, 인스턴스 변수가 내장되어 있으며 인스턴스 창과 동일한 구조가 표시됩니다.

렌더링 상태 렌더링 상태

현재 프레임의 상태에 대한 모든 정보를 사용할 수이 시계 쇼 당신은되고 렌더링 그리기 알파 또는 채우기 색상처럼. 실시간으로 업데이트 할 수 없으므로 프레임 세부 정보를 업데이트하고 얻으려면 게임을 일시 중지해야합니다.

표면 / 텍스처 표면 / 텍스처

이 시계를 사용하면 게임에있는 모든 표면 (응용 프로그램 표면 포함)에 정확히 무엇이 그려지고 있는지, 메모리에있는 텍스처 페이지 를 볼 수 있습니다. 시계 하단의 메뉴에서 텍스처 또는 표면을 선택한 다음 텍스처 또는 표면 ID를 마우스로 가리켜 팝업 창에 표시 할 수 있습니다.

디버거는 업데이트되기 전에 일시 중지 되어야합니다.

프로필

이 창에서 테스트중인 게임의 현재 시스템 프로필 을 볼 수 있으며 전체 이벤트에서 개별 함수 호출에 이르기까지 게임의 모든 항목을 확인하여 수행 방법을 확인하고 문제 영역이나 병목 현상을 확인할 수 있습니다. 프로파일 링에 대한 자세한 내용은 다음 페이지를 참조하십시오.

버퍼 버퍼

게임에서 버퍼를 초기화 한 경우이 창에 버퍼 데이터 가 표시 됩니다 . 게임에서 생성 된 첫 번째 버퍼에 대해 0에서 시작하고 연속 된 각 버퍼에 대해 1 씩 증가하는 버퍼 ID 값을 설정하여 생성 된 버퍼를 선택할 수 있습니다.

마우스 오른쪽 버튼을 클릭하면 데이터 창에서 또는 데이터 표시 단추를 클릭하면 데이터 표시 방법을 선택하고 정렬 및 유형 을 설정하고 열 선택 상자에서 데이터를 배치하는 방법을 설정할 수도 있습니다. 여기에 기본 레이아웃 목록에서 선택할 수있는 드롭 다운 창이 있습니다.
