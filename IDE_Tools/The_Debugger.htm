디버거

The Debugger is a powerful tool for checking your game, and is especially useful for tracking down errors and bugs in your code, as well as checking that scripts are run when they should be, and that variables and arrays contain the values you expect etc... You can start the Debugger by running your game from GameMaker Studio 2 IDE using the Debug button IDE 상단에서 또는 Build 메뉴 로 이동하여 " Debug "를 선택 하십시오 .

참고 :이 모듈에 대한 보안 예외를 작성하라는 메시지가 Windows 방화벽 (또는 실행중인 다른 방화벽 프로그램)에 의해 표시 될 수 있습니다. 그렇지 않으면 올바르게 작동하지 않습니다 . 이것은 디버거가 작동하는 방식 때문이며 모바일 플랫폼을 디버깅하는 데 필수적입니다.

디버그 모드에서 게임을 실행하면 IDE에 디버거에 사용할 수있는 다양한 도구가 표시됩니다.

위의 이미지는 디버거의 기본 레이아웃을 보여 주지만 IDE의 다른 측면과 마찬가지로 필요에 따라 창을 닫거나 필요에 따라 문서 및 창 크기를 변경하여 필요에 맞게이를 사용자 지정할 수 있습니다. IDE 맨 위에있는 디버거 컨텍스트 메뉴에서 언제든지 디버거 레이아웃을 재설정하거나 닫힌 창을 다시 열 수 있습니다.

아래는 디버거의 각 섹션에 대한 설명입니다.

그래프보기 그래프보기

그래프보기는 프로젝트에서 시간이 지남에 따라 메모리와 프레임 속도를 사용하는 방법을 그래픽으로 표시합니다.

그래프는 현재 및 평균 메모리 사용량과 시간에 따른 현재 및 평균 FPS를 보여 주며, / 마우스 휠과 함께 그래프 요소 중 하나에 마우스 커서를 놓으면 선택한 점에서 해당 요소의 값이 표시됩니다.

그래프보기에는 특정 디버그 이벤트가 그래프 길이에 따라 점으로 표시됩니다. 이러한 이벤트는 게임 실행 중 특정 시점에 GameMaker Studio 2에 의해 자동으로 생성되어 특정 중요 이벤트를 표시하지만 show_debug_message () 및 debug_event () 함수를 사용하여 자체 그래프 요소를 추가 할 수도 있습니다. 시스템 이벤트는 코드를 통해 트리거되는 이벤트와 다른 색으로 표시됩니다.

디버그에서 게임을 실행하고 이러한 이벤트를 마우스로 이동하면 게임의 해당 시점에서 트리거 된 모든 이벤트의 목록이 표시되며,이를 클릭하여 수정을 위해 별도의 창으로 목록을 가져 오는 옵션이 있습니다. " 새 창에서 열려면 클릭하십시오 "텍스트 :

 

리소스 뷰 리소스 뷰

The Resource View shows you a list of all the resources being used in the project that can contain code or DnD™ actions. This section works much like the Asset Browser and you can expand the items in it to show more information, for example, the different events for a specific object. You can then double click 코드보기 창에서 코드를 열려면 모든 리소스에서 값을 확인하고 중단 점을 추가 또는 제거 할 수 있습니다.

 

콜 스택 콜 스택

이 창에는 현재 진행중인 이벤트 호출 스택과 단계별 코드 번호가 표시됩니다. 콜 스택은 본질적으로 "마지막 선착순"데이터 구조로, GameMaker Studio 2 러너 모두에서 모든 게임 코드의 실행 순서를 보여줍니다. 디버깅은 VM (가상 머신)을 사용하여 수행되기 때문입니다. 게임을 위해 작성했습니다. 여기에 정보가 표시되기 전에 디버거를 일시 중지 해야합니다.

 

툴바 툴바

툴바에는 메모리 사용, 게임의 현재 (실제) FPS 및 디버거가 현재 연결되어 있는지 여부를 표시하는 표시기와 같은 기본 정보와 함께 디버거 컨트롤이 있습니다 (실행중인 인스턴스에 연결되면 표시기는 녹색으로 표시됨) 프로젝트의 빨간색과 그렇지 않으면 빨간색). 사용 가능한 컨트롤은 다음과 같습니다.

게임이 일시 정지 된 후 다시 시작됩니다 (키보드 단축키 " f5 ").
게임이 일시 정지됩니다.
게임을 다시 시작합니다.
게임을 중지합니다.
코드 블록 또는 스크립트를 입력하십시오 (키보드 단축키 " f11 ").
코드 블록 또는 스크립트를 건너 뜁니다 (키보드 단축키 " f10 ").
코드 블록 또는 스크립트에서 나가기 (키보드 단축키 + " f11 ").
디버그 정보에 대한 실시간 업데이트를 켜거나 끕니다 (기본적으로 해제되어 있음).

처음 세 개의 버튼은 게임을 제어하는 ​​데 사용되므로 언제든지 게임을 시작 , 일시 중지 , 다시 시작 또는 중지 할 수 있습니다. 게임이 일시 정지되면 (일시 정지 버튼 또는 중단 점에서 ) 3 단계 버튼을 사용하여 코드를 단계별로 실행할 수 있습니다.

코드를 통한 "스테핑 (stepping)"에 대해 이야기 할 때, 단순히 게임에 현재 코드 라인을 실행하도록 지시 한 후 다음 코드에 "스텝 (step)"하여 전체 게임을 한 줄의 코드로 실행할 수 있습니다. 당신이 원하는 시간. 일반적으로 당신은 단순히이 작업을 수행하는 버튼에서 단계를 사용합니다,하지만 당신은 또한 코드의 스테핑에 대한 코드를 통해 스테핑 하나 하나가 있습니다. 이 단계를 클릭하면 전체 스크립트 또는 코드 블록이 단일 단계로 실행되고 이후에 다음 줄에서 중지되기 때문에 큰 스크립트를 실행할 때 유용합니다. 범위가 변경되면 다음 단계에서 중지합니다.

실시간 업데이트 의 최종 아이콘을 사용하면 디버그 창에서 실시간으로 데이터를 업데이트 할 수 있으므로 변수가 변경되거나 인스턴스가 생성되고 파괴되는 방식을 확인할 수 있습니다. 이는 게임 성능에 영향을 줄 수 있으며 모든 정보가 실시간으로 제공되는 것은 아닙니다. 예를 들어 렌더 상태 시계 ( 시계 섹션 참조)는 실시간으로 볼 수 없습니다.

 

코드 뷰 코드 뷰

The code view is where we can see and review different scripts or event code from our project as it runs. By default this starts empty, but you can double click 리소스보기의 모든 자산에서 여기를 엽니 다. 여러 개의 다른 이벤트 나 스크립트도 열 수 있으며 상단의 탭으로 코드 뷰 작업 공간에 추가됩니다.

While any code window is open in the Code View, you can left click 거터 (줄 번호가있는)에서 중단 점 을 추가하거나 제거 합니다 ( 중단 점에 대한 자세한 내용은이 페이지 아래 부분 참조).

You can also use the right mouse button 컨텍스트 메뉴 를 열면 중단 점을 추가하거나 제거하고 선택한 코드를 클립 보드에 복사 할 수 있습니다.

 

시계 시계

이 섹션은 게임의 다양한 측면을 "감시"하여보다 심층적이고 자세한 디버깅을 허용합니다. 이에 대해서는 다음 페이지에서 자세히 설명합니다.

  • 시계


게임이 실행되는 동안 기본적으로 변수 값과 기타 세부 사항으로 디버그 창을 업데이트하지 않으므로 먼저 일시 정지 버튼을 사용해야합니다. 로컬, 글로벌 및 기타 데이터를 사용할 수 있기 전에 디버거에서 게임을 일시 중지하거나 상단의 아이콘에서 실시간 업데이트를 활성화 . 이 규칙의 유일한 예외는 프로필 보기입니다. 실시간보기와 독립적으로 활성화되면 자체 켜기 / 끄기 버튼이있어 실시간으로 업데이트됩니다. 다음 페이지에서이보기에 대한 전체 정보를 찾을 수 있습니다.

  • 프로파일 러

Another important part of debugging your game is the use of Breakpoints. A breakpoint is a place in your project code that you wish it to stop at so you can see what's going on. With the Debugger running, you can add or remove breakpoints at any time and in any code or script block by left clicking 거터 (줄 번호가있는 곳)에서 또는 마우스 오른쪽 버튼을 사용할 수 있습니다 상황에 맞는 메뉴 를 열면 중단 점을 추가하거나 제거 할 수 있습니다.

GameMaker Studio 2 IDE에서 중단 점을 설정할 수도 있습니다. 프로젝트를 작성할 때 f9를 사용하거나 마우스 오른쪽 버튼을 사용하여 언제든지 프로젝트를 추가하거나 제거 할 수 있습니다. 개체 창 (DnD ™ 사용자의 경우) 또는 코드 편집기 창의 행에서 " 토글 중단 점 토글 "을 선택합니다. 이제 게임을 디버그 모드에서 실행하면 정의한 지점에서 일시 중지됩니다.

IDE에서 설정 한 중단 점은 프로젝트와 함께 저장되며 다시 제거 될 때까지 다양한 실행에 걸쳐 지속되지만 디버거에서 코드에 중단 점을 추가하면 모듈을 닫거나 새로운 테스트 실행이 시작된 후 중단 점을 잃게됩니다 . 모든 중단 점은 출력 창에 나타나 활성화 또는 비활성화하거나 제거 할 수 있습니다.
