확장 편집기

확장 패키지 를 사용하면 GameMaker Language (GML)에 여러 가지 추가 기능과 상수 를 추가하거나 사람간에 게임 자산 또는 전체 프로젝트를 쉽게 전송할 수 있습니다. 확장 기능이 추가 기능을 추가하는 경우 사용되는 기능은 게임을 외부 SDK 또는 다른 언어로 작성된 소스 코드 (예 : Android 용 Java) 및 기본 GML에 게임을 연결할 수 있습니다.

일반적으로 타사 확장 프로그램은 GameMaker 마켓 플레이스 에서 제공되며 가입 한 확장 프로그램은 마켓 플레이스 라이브러리 창에 표시되며 여기에서 설치할 수 있습니다. 마켓 플레이스에서 확장 패키지를 얻지 못한 경우 다시 마우스 오른쪽 버튼을 클릭하여 프로젝트에 추가 할 수 있습니다 애셋 브라우저의 어느 곳에서나 Add Existing (기존 추가 )을 선택한 다음 확장 위치를 ​​찾아 선택하십시오.

이전 버전의 GameMaker Studio로 * .GMEZ를 생성 한 경우 마우스 오른쪽 버튼을 선택하여이를 추가 할 수 있습니다 메뉴 옵션 가져 오기 확장 .

참고 : 레거시 형식 * .GEX 확장은 더 이상 지원되지 않으며 GameMaker Studio 2 프로젝트로 가져올 수 없습니다.

자산 브라우저에 확장을 추가 한 후에는 이름과 버전 번호를 지정하고 내보낼 플랫폼을 설정해야합니다. 선택한 플랫폼을 선택 취소하면 GameMaker Studio 2가 해당 플랫폼에 대한 실행 파일의 일부로 확장을 내 보내지 않도록 할 수 있습니다. 또한 확장 특성이 대상 플랫폼과 일치하는지 확인하고 다음을 기반으로 확장 파일의 내보내기 대상을 편집하십시오.

  • 모든 대상 : GML 확장 이는 GML (GameMaker Language)을 사용하여 만든 확장이며 모든 대상 플랫폼과 호환됩니다.
  • Windows (Standard, UWP, XBoxOne) : Windows DLL 파일 (Dynamic Link Libraries)을 사용할 수 있습니다.
  • macOS : macOS 대상은 확장에 표준 DyLib 파일을 사용할 수 있습니다.
  • Ubuntu (Linux) : Ubuntu 대상의 경우 확장 프로그램에서 SO 파일을 사용해야합니다.
  • HTML5 :이 플랫폼은 JavaScript 확장을 사용할 수 있습니다.
  • Android :이 플랫폼에는 Java 또는 Android Framework SDK 로 작성된 확장이 필요합니다.
  • iOS / tvOS :이 플랫폼의 확장 기능은 Objective C로 작성되었으며 * .mm 및 * .h 파일이 필요하거나 프레임 워크를 사용하는 경우 사전 컴파일 된 * .a 네이티브 파일입니다.
  • PlayStation : PlayStation 3 또는 4의 경우 확장 프로그램은 PRX 파일이어야하며 각 대상은 대상 사양 (PS3 또는 PS4)을 준수하기 위해 서로 다른 PRX 가 필요합니다. PSVita에서 작업하는 경우 확장자는 대신 SUPRX 여야 합니다.

단일 확장명에는 다양한 관련 파일이있을 수 있으므로 여러 플랫폼에 대한 단일 확장명을 만들 수 있습니다.이 경우 기본 확장 편집기 (예 : 옆에있는 확인란을 선택)에서 호환되는 모든 플랫폼을 지정한 다음 확장의 각 부분에 대한 개별 파일 속성은 해당 파일을 내보낼 플랫폼을 설정하십시오.

특정 확장 파일 또는 자리 표시 자의 대상 플랫폼을 변경하려면 먼저 확장 속성을 열어야합니다 (두 번 클릭). 자산 브라우저의 확장 또는 사용 Open Properties )를 선택한 다음 확장 등록 정보에서 한 번 편집하려는 오른쪽에서 리소스 파일을 선택하십시오.

참고 : macOS 또는 Ubuntu 대상으로 내보내는 경우 확장명에 대한 파일 권한을 유지하기 위해 "zip 파일로 압축 해제"옵션을 설정할 수 있습니다.

익스텐션을 설치 한 후에는 GameMaker 언어에서 사용되는 내장 함수 및 상수와 마찬가지로 함수 및 상수를 사용할 수 있습니다. 게임이 크로스 플랫폼 (예 : Windows 및 Android 대상 모두에서 실행되도록 설계되어있는 경우) 인 경우 OS 상수 및 함수 를 사용하여 확장 관련 기능을 호출하기 전에 플랫폼을 확인하여 문제를 방지해야합니다.

switch(os_type)
    {
    case os_android:
        call_android_extension_function(x, y, z);
        break;
    case os_windows:
        call_windows_extension_function(x, y, z);
        break;
    }

플랫폼마다 확장명이 함수 이름을 공유 할 수 있으므로 Windows 및 Android 용 메시지를 표시하기 위해 고유 한 파일을 가진 확장명이있는 경우 확장 파일은 동일한 기능 이름을 공유하는 한 동일한 기능 이름을 공유 할 수 있습니다. 둘 다 해당 플랫폼으로 만 내보내도록 설정되었습니다.
