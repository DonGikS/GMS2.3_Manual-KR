스크립트 편집기

이 섹션에서는 게임을 실행하는 데 필요한 모든 코드를 입력하는 스크립트 편집기 ( 텍스트 편집기 라고도 함)를 다룹니다. 이 편집기는 커스텀 스크립트 자산 생성, 객체 이벤트 코딩, 룸 생성 코드 추가 및 기타 여러 가지 용도로 사용됩니다. 따라서이 섹션에서는 스크립트 생성 방법이 아니라 스크립트 편집기 작동 방식에 대해 자세히 설명합니다. 에셋이나 드래그 앤 드롭을 사용하여 스크립트를 만드는 방법. 이를 위해 다음 섹션을 참조하십시오.

  • GML 스크립트
  • DnD ™ 스크립트

스크립트 편집기는 GameMaker 언어 코드를 작성하는 곳으로 GameMaker Studio 2에서 사용하는 내장 프로그래밍 언어입니다 (자세한 내용은 GML 참조 섹션 참조). GameMaker Studio 2에 익숙해지고 최대한 활용하고 싶다면이 언어를 사용하는 법을 배우는 것이 좋습니다.이 언어는 게임을 만들 때 가능성을 크게 넓히기 때문입니다.

마우스 오른쪽 버튼을 클릭하여 스크립트 자산의 이름을 지정할 수 있습니다 애셋 브라우저 에서 이름 바꾸기를 선택하지만 스크립트 이름은 함수의 스크립팅 규칙을 준수해야하므로 영숫자 만 가능하고 문자로 시작해야하며 포함 할 수있는 유일한 기호는 " _ "하단 막대 기호. 다른 위치의 스크립트 편집기는 이름을 바꿀 수 없으며 예를 들어 개체 이벤트와 같이 편집중인 항목에 고유 한 이름을 갖습니다.

스크립트 편집기 창을 열면 다음과 같은 옵션과 레이아웃이 있습니다.

스크립트 / 이벤트 탭 스크립트 / 이벤트 탭

스크립트 편집기는 상단에 탭이있는 창에서 열리므로 한 창에 여러 개의 스크립트 또는 이벤트가있을 수 있습니다 (이 동작은 환경 설정 에서 변경하여 각 스크립트 또는 이벤트에 새 창을 제공 할 수 있음). 클릭 할 수 있습니다 탭에서 드래그하여 탭 순서를 변경하거나 원하는 경우 현재 창에서 끌어서 작업 공간에 배치하여 해당 스크립트에 대한 새 창을 만들거나 다른 창에 추가 할 수 있습니다 ) 및 스크립트 편집기를 최대화하여 새 작업 공간을 만들 수도 있습니다. IDE 창 밖으로 스크립트 탭을 끌어다 놓으면이 스크립트 리소스를 보유하기 위해 새 IDE가 생성되며 기본 창처럼 사용할 수 있습니다.

스크립트 편집기에서 오브젝트 이벤트의 코드를 편집하고 스크립트 편집기를 최대화하거나 별도의 창 또는 작업 공간에있는 경우 마우스 오른쪽 단추를 클릭하십시오. 편집기 메뉴에는 몇 가지 추가 옵션이 있습니다.

  • 작업 공간으로 복원 : 스크립트 편집기를 최대화 / 작업 공간 상태에서 해제하여 작업 공간의 객체에 다시 연결합니다.
  • Go to Object : 코드가있는 객체가있는 작업 공간으로 이동하여 객체 에 초점을 맞 춥니 다.
  • 이벤트 추가 / 열기 : 현재 스크립트가 속한 객체에 새 이벤트를 추가 할 수 있으며 스크립트 편집기에서 추가 된 이벤트에 대한 새 코드 탭이 열립니다. 선택한 이벤트에 이미 코드가있는 경우 새 탭에서 열립니다.

나머지 마우스 오른쪽 버튼 클릭에 대한 정보 메뉴 옵션은 아래의 편집 섹션을 참조하십시오.

 

거터 거터

거 터는 코드의 줄 번호를 표시하고 코드 상태에 대한 특정 정보를 전달하는 데 사용됩니다. 가장 중요한 것은 오타 를 만들거나 코드를 잘못 구성하면 GameMaker Studio 2 IDE 가 코드 라인에 빨간색 느낌표를 표시하여 문제를 알려줍니다 구문 오류를 표시합니다. 그런 다음 기호 위로 마우스를 이동하여 문제가 무엇인지 간단히 설명 할 수 있습니다.

다음 섹션에서 구문 오류 (및 코드를 테스트하거나 컴파일 할 때 발생하는 일반 오류)에 대한 자세한 정보를 찾을 수 있습니다.

  • 오류보고

거 터는 또한 중단 점이 추가 된 스크립트의 모든 행을 표시합니다. 중단 점은 단순히 스크립트에서 스크립트가 도달하면 디버거 가 게임 실행을 일시 중지하도록하는 장소입니다. " f9 "를 누르거나 마우스 오른쪽 단추를 사용하여 스크립트 또는 오브젝트 이벤트의 임의 행에서 중단 점을 토글 할 수 있습니다. 메뉴. 중단 점을 추가하면 거터에 다음과 같이 표시됩니다.

마지막으로 거터에는 책갈피 된 항목도 표시됩니다. 코드 줄을 북마크에 추가하려면 간단히 / + + 숫자 (0에서 9까지)를 사용하면 IDE를 사용 하여이 코드 줄로 건너 뛸 수 있습니다. / + 숫자 (0에서 9까지). 북마크를 삭제하려면 코드 줄에 북마크를 다시 추가 할 때와 동일한 작업을 수행하면됩니다. 북마크가 추가되면 다음과 같이 표시됩니다.

 

검색 / 바꾸기 검색 / 바꾸기

스크립트 편집기에서 작업하는 동안 / + " F "를 사용하여 검색 상자를 열거 나 / 검색 및 바꾸기를 열려면 + " H ":

여기에서 입력 한 키워드에 대한 로컬 검색을 수행 할 수 있고 검색어를 입력 한 후 검색 창의 오른쪽 상단에있는 화살표를 사용하여 스크립트에서 한 단어에서 다음 단어로 건너 뛸 수 있습니다. 다음 버튼을 토글하여 검색 조작 방법을 변경할 수 있습니다.

이것을 토글하면 검색 기능은 전체 입력 문자열과 일치하는 문자열 만 강조 표시합니다. 예를 들어, " random "을 검색하지 않으면 irandom () 또는 randomise () 와 같이이 문자열을 포함하는 모든 단어가 표시되고 on으로 전환하면 random () 함수 만 표시됩니다.
이 옵션을 토글하면 GameMaker Studio 2에 검색 문자열의 내용뿐만 아니라 경우도 확인하도록 지시합니다. 예를 들어, " spr_Dog "라는 스프라이트가 있고이 토글을 끄고 " dog "를 검색하면 스프라이트 문자열이 강조 표시되지만 토글이 켜져 있으면 " Dog "가 no이므로 더 이상 " 개 "와 같은 것으로 간주됩니다.

다음을 사용하여 검색 창을 연 경우 / + " F "이면 교체 기능이 보이지 않으므로 교체 버튼을 클릭해야합니다. 그것을 열 수 있습니다. 열린 후에는 다음 버튼을 사용하여 주어진 검색 문자열을 바꾸는 데 사용할 문자열을 입력 할 수 있습니다.

이를 클릭하면 스크립트에서 찾은 다음 검색 문자열이 지정된 바꾸기 문자열로 바뀝니다. "다음"용어는 현재 커서 위치 다음에 나오는 용어로 간주되며 검색 창의 오른쪽 상단에있는 화살표 버튼을 사용하여 다른 단어로 건너 뛸 수 있습니다.
이것을 클릭하면 주어진 replace 문자열을 사용하여 스크립트 내에서 검색 문자열의 모든 예제를 대체합니다.

전역 검색 (예 : 현재 스크립트 또는 이벤트가 아닌 전체 프로젝트 검색)을 수행하려는 경우을 누를 수 있습니다. / + + "F".
