폰트 에디터

게임에서 텍스트를 그리려면이 텍스트는 표준 Arial 12 포인트 글꼴 로 그려 지지만 더 흥미 롭거나 독창적 인 텍스트를 만들려면 다른 글꼴을 사용해야합니다. 따라서 컴퓨터에있는 다른 글꼴을 사용하려면 GameMaker Studio 2에서 글꼴 자산 을 만들어야합니다. 자산 브라우저에서 " 글꼴 만들기 "를 선택 하십시오 . 생성하는 각 글꼴 리소스에 대해 컴퓨터에서 특정 글꼴 유형을 지정하면 게임에서 동작 또는 코드를 사용하여 그리기위한 글꼴을 설정할 수 있습니다.

글꼴 리소스를 처음 만들면 다음 옵션이 포함 된 글꼴 편집기 창이 열립니다.

세부 사항 세부 사항

여기에서 글꼴에 고유 한 이름 (알파벳 및 "_"아래 막대 기호 만)을 지정하고 사용할 글꼴을 선택할 수 있습니다. 폰트는 머신에 설치해야하며 라이센스 문제로 인해 GameMaker Studio 2는 기본적으로 프로젝트 파일과 함께 실제 폰트 파일을 저장하지 않지만 대신 폰트는 일련의 비트 맵 글리프로 텍스처 페이지에 렌더링 됩니다. 텍스처 페이지는 텍스처 그룹에 할당되며 여기에서 그룹 옵션에서도 선택할 수 있습니다.

Marketplace 자산을 만들 때 또는 다른 이유로 프로젝트와 함께 실제 글꼴 파일 ( * .ttf 만)을 배포하려는 경우 프로젝트에 복사 옵션을 선택할 수 있습니다. 글꼴 파일을 선택할 수있는 파일 탐색기가 열리고 프로젝트와 함께 번들로 제공되는 파일에 추가됩니다.

중대한! 프로젝트에서 재배포하거나 사용할 수있는 법적으로 허용되고 허가 된 글꼴 만 사용해야 합니다 .

 

프리셋 프리셋

이 섹션에서는 크기 (포인트 단위-기본값은 12pt)와 글꼴을 굵게 또는 기울임 꼴로 그릴 것인지 또는 앤티 앨리어싱을 적용 할 것인지를 설정할 수 있습니다. 오른쪽의 미리보기 창은 지정한 크기와 변형으로 게임에서 보이는 텍스트를 대략적으로 보여줍니다.

범위 제어 범위 제어

글꼴 리소스를 만들 때 글꼴에 대한 모든 단일 글리프를 포함하는 경우는 거의 없습니다. 이것이 GameMaker Studio 2가 기본적으로 선택한 글꼴의 32에서 127까지의 문자 만 사용하도록하는 이유입니다.이 경우 특정 문장 부호와 함께 숫자와 작은 글자 및 대문자가 표시됩니다. 그러나 게임 최적화에 사용되는 문자 범위를 변경할 수 있습니다. 예를 들어 특정 글꼴의 숫자 만 필요한 경우 숫자 만 선택하십시오. 이렇게하려면 먼저 삭제 버튼을 클릭하여 현재 범위를 지우십시오 (클릭하여이 범위를 선택해야 할 수도 있음) 미리보기 창에서 추가 한 다음 추가 버튼을 클릭하여 새 범위를 추가하십시오. 그러면 다음과 같은 연결 창이 열립니다.

이 창에는 글꼴의 표준 범위를 설정하는 데 도움이되는 몇 가지 단추가 있습니다.

-32에서 127까지의 일반 범위-0에서 255까지의 ASCII 범위-10 자리 만 포함하는 숫자 범위-모두 대문자와 소문자 만 포함하는 문자 범위

문자 범위 입력 상자에 설정하려는 범위의 첫 번째 및 마지막 문자 색인을 입력하여 다른 범위를 사용할 수 있습니다 (문자가 범위 내에 있지 않은 경우 공백으로 대체 됨). 이 버튼과 입력 상자 외에도 특히 유용한 두 가지 추가 옵션이 있습니다.

  • 코드에서 :이 버튼을 클릭 한 다음 확인을 클릭하면 GameMaker Studio 2가 자동으로 게임 코드를 구문 분석 한 다음 게임의 모든 텍스트를 포함하는 다양한 문자 범위를 만듭니다. 모든 문자열 ( ""또는 ''내)을 검색하므로 문자 범위에 파일 이름이 포함될 수도 있습니다.
  • 파일에서 : 이것은 위에서 설명한 "코드에서"옵션과 유사하게 작동합니다. 파일을 클릭하면 파일을 제공하라는 메시지가 표시되고 완료되면 확인을 클릭해야합니다. GameMaker Studio 2는 파일을 파싱하고 그 안에있는 텍스트의 문자 범위를 생성합니다.

미리보기 창에 직접 텍스트 글리프를 추가하거나 삭제하여 텍스트 범위를 수동으로 편집 할 수 있습니다.

 

텍스처 컨트롤 텍스처 컨트롤

위에서 언급 한 것처럼 ( Font Details 섹션에서) 기본적으로 모든 글꼴 글리프는 텍스처 그룹의 일부로 텍스처 페이지에 렌더링 되어 그래픽 리소스로 변환되며 게임이 컴파일되면 글꼴이 임시에 추가됩니다 텍스처를 사용하여 필요한 모든 글꼴 글리프가있는 단일 이미지를 만든 다음 내보내기 위해 기본 텍스처 그룹에 추가합니다. 여기에있는 버튼은 글꼴 텍스처 생성 방법을 조정하거나 새 글꼴 텍스처를 재생성하는 데 사용됩니다.

옵션 버튼은 고급 글꼴 텍스처 옵션을 엽니 다 :

이러한 옵션은 때때로 게임에서 글꼴이 표시되는 방식을 완전히 바꿀 수 있으며 모든 글꼴에 동일한 설정이 필요한 것은 아닙니다. 다음 옵션은 글꼴에 대해 가능한 최고의 사용자 정의를 제공하도록 설계되었습니다.

  • 힌트 : 글꼴 힌트 (명령이라고도 함)는 수학 명령을 사용하여 윤곽선 글꼴의 표시를 조정하여 래스터 화 된 그리드와 정렬되도록합니다. 낮은 화면 해상도에서 힌트는 명확하고 읽기 쉬운 텍스트를 생성하는 데 중요합니다. 여기서 최상의 힌트를 얻으려면 다양한 힌트 알고리즘 중에서 선택할 수 있습니다. 기본값은 "정상"입니다.
  • 글리프 옵션 :이 옵션은 다른 작업을 활성화 / 비활성화하여 글리프가로드되는 방식을 변경하는 데 사용할 수 있습니다. 이것들은: 힌트 사용 안함 : 글꼴 기본 힌트 및 자동 힌트를 사용하지 않습니다 .
  • 자동 힌더 비활성화 : 자동 힌더 를 비활성화하고 대신 글꼴 기본 힌트 를 사용합니다 (있는 경우). 이는 "자동 장애 선호"옵션보다 우선합니다.
  • Auto-Hinter 선호 : GameMaker Studio 2는 글꼴에있을 수있는 기본 힌트보다 auto-hinter를 선호합니다. 이는 "자동 장애 사용 안 함"및 "힌트 사용 안 함"옵션에 의해 재정의됩니다.
  • 워프 :이 그리드 정렬을 개선하기 위해 약간 워프 문양을 것이다 활성화. 이를 위해서는 자동 장애 및 "일반"힌트 모드가 필요합니다.
  • 스케일 없음 :이 옵션을 선택하면 글리프가 글꼴 단위로 유지되고 요청 된 크기는 무시됩니다. 이는 힌트가 비활성화되었음을 의미합니다.
    • The default state for all the above options is disabled.
  • Interpreter : GameMaker Studio 2가 폰트 렌더링에 사용해야하는 트루 타입 버전을 선택할 수 있습니다. 기본값은 TrueType v40 입니다.
  • 반올림 :이 옵션은 소수점 이하 자릿수를 정수로 변환 할 때 반올림 방법을 설정합니다. 기본값은 "바닥"입니다.

재생성 버튼을 사용하여 글꼴 질감을 재생성 할 수 있습니다. 기존 글꼴에 대한 변경 사항을 업데이트하거나 새 글꼴을 만들었고 미리보기 창에 표시되는 모양을 보려면이 작업이 필요합니다. 오른쪽 버튼에서 글꼴을 재생성 할 수도 있습니다. 자산 브라우저의 글꼴 메뉴.

새 글꼴을 처음 만들 때 편집기에서 텍스처를 재생성하지 않으면 게임을 처음 실행하거나 테스트 할 때 텍스처가 만들어 지지만 이것이 유일한 시간이라는 점에 유의해야합니다. 자동으로 생성됩니다. 즉, 글꼴에 텍스처가 생성되면 글꼴 리소스를 변경하면 게임에서 변경 사항을 볼 수 있도록 글꼴을 재생성해야합니다. 그렇지 않으면 "오래된"글꼴 텍스처가 계속 사용됩니다.

GameMaker Studio 2는 생성 된 폰트 텍스처를 나머지 프로젝트 파일과 함께 PNG 파일로 렌더링하며, 프로젝트를로드하고 기본 폰트를 찾지 못하면 저장된 텍스처를 사용하여 텍스트가 올바르게 표시됩니다. 그러나 글꼴이 없으면 텍스처 그룹 편집기 에서 이름, 글꼴 패밀리 및 텍스처 그룹 이외의 속성을 더 이상 변경할 수 없습니다.

미리보기 모드 미리보기 모드

이 섹션에서는 미리보기 창에 선택한 글꼴의 일부 샘플 텍스트 또는 글꼴에 대해 생성 된 렌더링 된 텍스처 이미지가 표시되도록 선택할 수 있습니다.

 

미리 보기 미리 보기

미리보기 창 에서 글꼴의 미리보기를 텍스트 샘플 또는 텍스처로 볼 수 있습니다. 미리보기 모드 가 샘플 로 설정되면 미리보기 창에 직접 테스트하려는 텍스트를 입력하여 모양을 확인할 수 있으며 게임 미리보기 를 선택하면 글꼴에 대해 생성 된 렌더링 된 텍스처 이미지가 표시됩니다. 이미지가 없으면 재생성 버튼을 클릭하여 텍스처를 생성해야합니다.

텍스처 이미지는 샘플과 다를 수 있습니다. 텍스처가 명시 적으로 지시되지 않는 한 텍스처가 재생성되지 않기 때문입니다. 즉, 처음에 폰트 텍스처를 생성 한 후에는, 텍스처를 재생성하지 않으면 폰트 리소스에 대한 변경 사항이 게임으로 전달되지 않습니다.
