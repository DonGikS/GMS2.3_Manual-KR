편집 코드

스크립트 편집기를 사용하기 시작하면 올바른 GML 구문 에 따라 스크립트 저작물이나 이벤트 등에 필요한 코드를 입력하는 것입니다. 이렇게하는 동안 자동 완성 창에서 도움을받을 수 있습니다.

이 창은 코드를 작성하는 동안 (보통 약간의 일시 정지 후 환경 설정에서 설정할 수 있음) 팝업되며 알파벳 순서로 "최적의"답변을 선택할 수 있습니다.

While using the code editor you can click 마우스를 끌어 코드 행을 선택하거나 길게 누르십시오. 그런 다음 클릭 드래그하여 코드 열을 선택하십시오. 이 방법으로 선택한 모든 것은 다음의 표준 키보드 단축키를 사용하여 복사 , 잘라 내기 및 붙여 넣기 를 할 수 있습니다 / + " C ", / + " X "및 / + " V ".

다양한 방식으로 코드에 주석을 달 수 있다는 점은 주목할 가치가 있습니다. 코드를 주석 처리하는 것은 생산성을 유지하는 데있어 매우 중요합니다. 코드를 명확하게 유지하고 코드 부분을 분리하여 각 부분의 기능과 그 이유를 명확하게 알 수 있습니다. 또한 팀에서 일하는 경우 다른 회원들에게 메모를 남길 수 있으며 그들이 한 일이나 특정 코드 섹션이 무엇인지 정확히 알 수 있음을 의미합니다. 솔로 개발자라도 코드에 주석을 남기는 것이 중요합니다. 특히 프로젝트를 떠나서 나중에 다시 방문해야하는 경우에 특히 그렇습니다.

Commenting your code is simple, as you just have to use "//" before any line of text or you can comment out whole paragraphs by using "/* text here */". There are even two keyboard shortcuts to comment and un-comment selected lines of code: / + " K "로 댓글을 달고 / + 주석을 해제하려면 + " K ". 다음 이미지는 이러한 메소드와 JS Doc 주석 (자세한 설명)을 사용하여 작성된 코드를 보여줍니다.

또한 스크립트에 포함 된 함수의 속성을 정의하기 위해 특정 JSDoc 스타일 주석을 추가하여 코드에서 함수를 사용할 때 인수 및 기타 세부 정보와 함께 자동 완성으로 표시되도록 할 수도 있습니다. 자세한 내용은 JSDoc 스크립트 주석 섹션에서 이에 대한 자세한 내용을 찾을 수 있습니다.

기본적으로 작성된 모든 코드에는 각 일치하는 중괄호 {} 옆에 코드 접기 아이콘이 있습니다. 이것들은 작은 더하기 + 또는 빼기 - 아이콘의 형태를 취하며, 이것을 클릭하면 일치하는 괄호 안에 코드가 "접혀져"시각적 공간을 덜 차지하게됩니다.

특수한 #region 태그를 사용하여 코드 폴딩 영역을 수동으로 설정하여 코드 섹션을 표시 할 수도 있습니다. 이것이 작동하는 방식은 #region 태그 (설명 텍스트와 함께)를 코드 섹션 상단에 추가 한 다음 해당 #endregion 태그를 섹션 하단에 추가하면 코드를 "접을"수 있다는 것입니다. 즉, 편집기에서 볼 수있는 코드 줄을 한 번에 줄일 수 있도록 숨기십시오.

실제 코드는 다음과 같습니다.

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

알다시피, 스크립트 텍스트의 일부는 채색되어 있습니다. 스크립트 편집기는 기존 객체, 내장 변수 및 함수 등에 대해 "인식"하고 그에 따라 색상을 지정합니다. 컬러 코딩은 이름을 잘못 입력하거나 키워드를 변수로 사용했거나 구문 오류가있는 경우 즉시 확인할 수 있으므로 실수를 피하는 데 도움이됩니다. 그러나 색상 코딩이 마음에 들지 않으면 환경 설정 에서 색상을 켜거나 끄고 ( F10 을 사용하여 편집기에서 켜거나 끌 수 있음) 사용되는 색상을 사용자 정의 할 수 있습니다.

You can right-click 편집기의 아무 곳이나 다음 메뉴를 엽니 다.

코드 줄 잘라 내기, 복사 및 붙여 넣기를 수행 할 것으로 예상되는 일반 기능 외에도 선택한 텍스트에 주석을 달거나 주석을 해제하고 들여 쓰기를 추가 또는 제거하고 코드 창을 1, 2 또는 3으로 분할하는 옵션도 있습니다 별도의 창 (편집중인 코드에 아무것도 정의하지 않은 경우 #regions 와 관련된 옵션을 사용할 수 없습니다).

분할 창으로 분할하는 마지막 옵션은 실제 창을 다른 부분으로 분할 한 다음 스크립트 또는 이벤트를 끌어서 비교 및 ​​편집을 위해 다른 창이나 이벤트와 함께 표시 할 수 있습니다.

이 기능은 다른 창이나 탭 사이를 끊임없이 전환하지 않고도 두 스크립트를 비교하거나 동시에 연결된 스크립트를 볼 때 매우 유용합니다.

이 메뉴의 다른 옵션은 디버거 와 함께 사용할 중단 점을 토글하고 GML 코드를 DnD ™로 또는 그 반대로 변환하는 것입니다. 코드를 DnD ™로 변환하는 경우 각 코드 줄에 대해 개별 작업을 생성 하지 않고 단일 코드 작업을 사용하여 이전에 이벤트 나 스크립트에 있었던 모든 항목을 보유합니다.
