트랙 패널

시퀀스 작업은 트랙 패널에 추가 된 자산 및 매개 변수에 의해 관리됩니다. 다음 중 한 가지 방법으로 편집기에 트랙을 추가 할 수 있습니다.

  • 애셋을 시퀀스 캔버스 로 드래그
  • 에셋을 트랙 패널 자체로 드래그
  • "자산 추가"를 클릭하여 트랙 패널 하단에있는

이 마지막 방법은 추가 할 자산을 선택할 수있는 자산 탐색기를 엽니 다.

선택한 방법에 관계없이 자산에 대한 새로운 자산 트랙 이 만들어집니다. 이 트랙은 시퀀스에서 자산의 기본 트랙이며 재생 헤드 위치의 Dope Sheet에 자산에 대해 해당 자산 키 트랙이 생성됩니다.

끝에있는 색상 막대를 클릭하여 추적 패널에서 자산 시트의 자산 키 색상을 설정할 수 있습니다. 그러면 도프 시트의 키 트랙에 사용할 색상을 선택할 수있는 색상 피커 가 열립니다.

트랙 편집기의 각 자산에는 매개 변수 트랙 세트가 추가 될 수도 있습니다. 이 트랙은 시간이 지남에 따라 트랙에 대해 수정 될 수있는 다양한 속성 (매개 변수)을 제어합니다. 이 페이지의 아래쪽에있는 " 매개 변수 트랙 추가 "섹션에서 사용 가능한 다양한 매개 변수에 대해 자세히 설명하지만 여기에 사용 가능한 항목에 대한 정보를 제공하는 간단한 목록이 있습니다.

  • 모든 자산 : 위치 및 회전 .
  • 소리 : 피치 및 볼륨 .
  • 순서 : 원점 및 척도 .
  • 스프라이트 및 객체 : 원점 , 배율 , 이미지 인덱스 , 이미지 속도 및 색상 혼합 .

To add a parameter track in the editor you can enable "Automatically Record Changes" 트랙 패널에서 다음과 같이 현재 재생 헤드 위치에서 캔버스의 자산에 대한 변경 사항에 대한 매개 변수 키를 추가합니다.

또는 " Add Parameter Track "을 클릭해도됩니다 버튼을 누르면 사용 가능한 다양한 옵션을 보여주는 메뉴가 열립니다.

이 메뉴에서 옵션을 선택하거나 IDE에서 직접 녹음하면 기본 자산 트랙 아래에 하나 이상의 매개 변수 트랙이 생성되고 기본 자산 트랙 색상과 동일한 색상의 매개 변수 키 가 Dope Sheet에 추가됩니다.

기본 자산 트랙에 매개 변수 트랙을 추가하면 도프 시트에 자산 트랙의 길이를 프레임으로 표시하는 색상 막대가 표시되며 변경하려는 지점에 매개 변수 키 (키 프레임)가 추가됩니다. 발생할 매개 변수. 이 키는 위 이미지에서 프레임의 간단한 "점"으로 볼 수 있습니다. 트랙에 이와 같은 파라미터 키가있는 경우 트랙이 참조하는 파라미터는 기본적으로 표시된 각 포인트 사이에 보간됩니다 ( 선형 보간 사용 ). 예를 들어, 회전 매개 변수가 있고 프레임 0에서 90 °로 설정되면 프레임 30에서 180 °가되고 프레임 15에서 135 °가됩니다. 매개 변수 트랙을 선택한 다음 도프 시트에서 재생 헤드를 "스크러빙"하여 시간이 지남에 따라 값의 변화를 실제로 확인할 수 있습니다.

Note that for all parameter tracks you can disable the default linear interpretation between different keys from the right mouse button 매개 변수 트랙의 메뉴를 통해 매개 변수 값 사이의 전환이 시간이 지남에 따라 점차적으로 주어진 프레임에서 즉시 발생 함을 의미합니다.

  • 중복 트랙 : 현재 선택된 트랙의 복제본을 생성합니다. 이것이
  • 트랙 삭제 : 선택한 트랙을 삭제합니다. 자산 트랙을 삭제하면 관련된 자산 트랙도 제거되지만 매개 변수 트랙을 개별적으로 삭제할 수 있습니다.

An important feature of asset and parameter tracks is the ability to parent tracks and groups. If you drag an asset track onto another one in the track panel, the first asset becomes a child of the asset it was dropped on, which in turn becomes the parent. All parameter tracks assigned to the parent asset track will then automatically be applied to the child asset tracks, so you can have multiple sprites (for example) all using a single position asset track to move:그룹을 사용하여 동일한 작업을 수행 할 수 있습니다. 트랙 패널 하단의 버튼을 클릭하여 그룹 폴더를 생성 한 다음 그룹 폴더에서 그룹화 할 자산 트랙을 끌어서 놓을 수 있습니다. 이러한 자산은 여전히 ​​고유 한 파라미터 트랙을 유지하지만 그룹에 추가 파라미터 트랙을 제공 할 수 있으며 그룹의 모든 자산은 이러한 파라미터 트랙과 개별 파라미터 트랙의 영향을받습니다.

키 컨트롤 키 컨트롤

Once you have added a parameter track, you can add and remove parameter keys to specific frames to set the value for that parameter. This can be done using the "Record Key" 또는 " 삭제 키 " 버튼 (자세한 내용은 " 트랙 키 컨트롤 "섹션 참조) 또는 파라미터 자체의 키 컨트롤 을 사용하여 수행 할 수 있습니다. 이러한 컨트롤을 사용하려면 Dope Sheet의 재생 헤드를 편집하려는 프레임에 배치 한 다음 매개 변수 트랙 왼쪽의 원형 "키"아이콘을 클릭하십시오. 프레임에 할당 된 키가없는 경우 원은 "비어 있으며"클릭하면 새 키가 추가되고 매개 변수 값을 편집 할 수 있습니다. 프레임에 키가 이미 있으면 원이 "채워지고"클릭하면 프레임에서 키가 제거됩니다.

위 이미지에서 매개 변수 키 마커의 양쪽에있는 화살표 버튼을 사용하여 재생 헤드를 수동으로 배치하지 않고도 도프 시트의 키 프레임 사이를 빠르게 건너 뛸 수 있습니다.

 

매개 변수 값 매개 변수 값

매개 변수 트랙이 추가되면 변경하려는 키가있는 Dope Sheet의 재생 헤드를 프레임으로 설정 한 다음 다른 캔버스 위젯을 사용하여 위치, 회전, 자산의 규모 또는 원산지. 이러한 변경 사항은 해당 프레임의 키 값에 반영됩니다. 그러나 모든 매개 변수를 이러한 방식으로 편집 할 수있는 것은 아니거나 사용자의 요구에 맞게 정확하지 않을 수도 있습니다. 이러한 경우 트랙에서 매개 변수 값을 클릭하고 매개 변수 유형에 필요한 값으로 설정하여 매개 변수 값을 직접 편집 할 수 있습니다 .

Note that some parameter tracks may have a "link" icon 값 옆에 있습니다. 즉, 서로 연결할 수 있습니다. 링크를 활성화 / 비활성화하기 위해 아이콘을 클릭하면 가능하며 활성화 된 경우 한 값을 편집하면 다른 값도 설정됩니다.

 

이벤트 이벤트

일반 객체와 비슷한 방식으로 시퀀스에는 할당 된 일부 코드를 실행할 수있는 이벤트가있을 수 있습니다. 코드는 스크립트 함수 형태로 지정되며 인수를 취할 수 없으며 이벤트가 트리거 될 때 호출됩니다.

Events are added by clicking the Add Event button 다음 창이 열립니다.

여기에 나열된 이벤트는 다음과 같습니다.

  • 생성 -이 이벤트는 시퀀스 인스턴스가 처음 생성 될 때 발생하며, 룸에 들어갈 때 룸 편집기를 통해 룸에 배치 된 시퀀스 내에서 가장 먼저 발생합니다.
  • Destroy- 이 이벤트는 시퀀스의 인스턴스가 파괴 될 때 실행되며 정리 이벤트 전에 실행됩니다 (아래 참조).
  • 정리 -이 이벤트는 회의실에서 시퀀스 인스턴스를 제거하는 모든 이벤트 후에 호출됩니다. 따라서 시퀀스가 ​​파괴되거나 방이 종료되거나 게임이 종료되면 트리거되며 시퀀스 인스턴스가 게임에서 제거 될 때 한 번 수행 해야하는 작업을 수행하도록 설계되었습니다. 예를 들어 layer_sequence_destroy ()를 사용하여 시퀀스 인스턴스를 삭제 한 경우이 이벤트는 Destroy Event (위 참조) 후에 호출됩니다.
  • Begin Step / Step / End Step- 단계 이벤트는 시퀀스 인스턴스가 존재하는 동안 게임의 모든 단계 (프레임)마다 확인되는 이벤트이며 시작, 단계 및 종료의 세 부분으로 나뉩니다. 대부분의 경우 표준 단계 이벤트를 사용하는 것이 좋지만 때로는 코드 실행 및 시간을 조금 더 제어하여 Begin 및 End 단계 이벤트가 제공되므로 이러한 이벤트는 항상 게임의 모든 단계 (프레임)마다 동일한 순서로 트리거됩니다. 중대한! 시퀀스가 일시 정지되면이 이벤트는 트리거 되지 않으며 재생이 다시 시작되면 시퀀스가 ​​다시 재생을 시작한 후 다음 프레임 이 트리거 됩니다 . 또한 이벤트 순서는 재생 헤드 방향의 영향을받지 않으며 시퀀스가 ​​뒤로 재생 되더라도 이벤트는 여전히 시작, 단계 및 종료로 실행됩니다.
  • 비동기 이벤트 : 이것은 비동기 시스템 이벤트 객체와 동일합니다. 중대한! 시퀀스가 일시 중지되면이 이벤트는 트리거 되지 않습니다 .

각 이벤트에는 이벤트가 트리거 될 때 호출되는 단일 기능이 할당 될 수 있습니다. 통풍구의 입력 상자를 사용하여 기능을 할당 할 수 있으며 화살표 버튼을 클릭하면 편집 할 기능에 대한 스크립트 편집기가 열립니다. 하단에있는 새로 만들기 버튼을 클릭하여 "boilerplate"함수가 이미 정의되어 있고 채워질 준비가 된 새 스크립트 리소스를 만들 수 있습니다. 함수 이름을 필요에 따라 변경할 필요가 없습니다. 사전 정의 된 이름으로, 필요하지 않은 함수 정의를 제거 할 수도 있습니다. 시퀀스 이벤트에 사용되는 함수는 인수를 사용할 수 없습니다.

코드를 사용하여 시퀀스 이벤트를 추가하고 편집 할 수도 있습니다. 자세한 내용은 여기 를 참조 하십시오 .

  • - "선택 키 분리"버튼입니다. 도프 시트에서 하나 이상의 매개 변수 키를 선택한 경우이 단추를 클릭하면 타임 라인 재생 헤드 위치에서 두 개의 개별 키로 분할됩니다.
  • - "선택된 키 삭제"버튼입니다. 도프 시트에서 하나 이상의 매개 변수 키를 선택한 경우이 단추를 클릭하면 해당 키가 삭제됩니다.
  • - "새 키 기록"버튼입니다. 도프 시트에서 자산 키를 선택하고이 버튼을 누르면, 새로운 파라미터 키가 자산 트랙에 파라미터 트랙으로 추가되고, 파라미터 키가 재생 헤드 위치에서 도프 시트 타임 라인의 포인트로 추가됩니다.
  • - "자동으로 변경 사항 기록"버튼입니다. 이 기능을 활성화하면 캔버스 내 자산에 대한 변경 사항이 자동으로 기록되고 해당 매개 변수 트랙 및 매개 변수 키가 도프 시트의 재생 헤드 위치에 추가됩니다. 예를 들어 재생 헤드를 프레임 0에서 프레임 10으로 이동 한 다음 캔버스에서 자산을 100 픽셀 오른쪽으로 이동하면 위치에 매개 변수 트랙이 추가되고 매개 변수 키가 프레임 0에 추가됩니다 (초기) 위치) 및 프레임 10 (재생 헤드 위치)에서 시퀀스에서 "재생"을 누르면 자산이 10 프레임 이상 오른쪽으로 100 픽셀 이동합니다.

표시 / 잠금 토글 표시 / 잠금 토글

Both asset tracks and parameter tracks have a button to toggle track visibility 트랙을 잠그기 위해 . 트랙의 가시성을 비활성화하면 시퀀스가 ​​룸에서 실행될 때 표시되지 않으며 테스트 중에 시퀀스 캔버스에 표시되지 않으며 자산 트랙의 가시성을 전환하면 가시성이 토글됩니다. 파라미터 추적. 트랙을 잠그면이 트랙을 더 이상 편집 할 수 없게됩니다. 보이지 않는 트랙은 도프 시트에 희미하게 그려지고 잠긴 트랙은 오버레이로 그려 지므로 각 트랙의 상태를 즉시 확인할 수 있습니다.

자산 트랙을 잠그는 동안 관련된 트랙을 잠그면 필요에 따라 개별 매개 변수의 잠금을 해제 할 수 있으며 기본 자산 트랙의 상태에 영향을주지 않고 매개 변수 자체를 잠 그거나 잠금 해제 할 수 있습니다.

 

트랙 색상 트랙 색상

생성 한 각 자산 트랙에는 IDE에서 색상이 무작위로 지정되어 Dope 시트에서 쉽게 식별 할 수 있습니다. 이 색상은 트랙의 색상 막대를 클릭 한 다음 열리는 색상 선택기 창에서 사용하려는 색상을 선택하여 언제든지 변경할 수 있습니다. 자산 트랙에 대해 선택된 모든 색상은 연관된 모든 매개 변수 트랙에 자동으로 적용되지만 필요한 경우 각 매개 변수 트랙의 색상을 개별적으로 편집 할 수 있습니다. 그런 다음 자산 트랙 색상을 다시 변경하면 매개 변수 트랙에 대한 변경 사항이 무시됩니다.

 

파라미터 트랙 추가 파라미터 트랙 추가

The Add Parameter 버튼을 사용하여 선택한 자산 트랙에 단일 파라미터 트랙을 추가 할 수 있습니다. 이 페이지의 앞부분에서 언급했듯이 매개 변수 트랙은 관련된 자산에 따라 다양한 유형이 될 수 있습니다. 아래는 사용 가능한 모든 매개 변수 트랙과 이들이 참조하는 항목 및 적용 할 수있는 자산의 목록입니다.

  • 위치 :이 파라미터 트랙은 4 개의 시퀀스 자산 유형 ( 스프라이트 , 오브젝트 , 사운드 또는 시퀀스 ) 중 하나에 추가 할 수 있으며 자산의 시퀀스 내 공간 위치를 설정하는 데 사용됩니다. X와 Y 값이 필요합니다.
  • 회전 : 위치 매개 변수와 마찬가지로 회전 매개 변수는 모든 시퀀스 자산에 사용할 수 있으며 자산의 회전 각도를 설정하는 데 사용할 수 있습니다. GameMaker Studio 2는 오른쪽에서 0 °부터 시작하여 시계 반대 방향으로 회전을 계산하므로 90 °는 위쪽, 180 °는 왼쪽, 270 °는 아래쪽에 있습니다.
  • Origin : origin 매개 변수는 스프라이트 , 객체 및 시퀀스 자산에 사용할 수 있으며 자산 원점의 X 및 Y 위치를 설정하는 데 사용할 수 있습니다. 원점을 변경하면 자산의 위치, 회전 및 크기 조정 방법에 영향을줍니다.
  • Scale : scale 매개 변수는 스프라이트 , 객체 및 시퀀스 자산에 사용할 수 있으며 자산의 가로 및 세로 스케일을 설정하는 데 사용됩니다. 이는 자산이 IDE 및 런타임에 렌더링되는 방식에 영향을 미치며 스칼라 값으로 계산됩니다. 여기서 100 %는 기본 자산 크기이고 50 %는 절반 크기이며 200 %는 크기의 두 배입니다. "링크"를 클릭하여 가로 및 세로 스케일 값을 연결할 수 있습니다 단추. 즉, 한 값을 변경하면 다른 값도 자동으로 변경됩니다.
  • 이미지 인덱스 및 이미지 속도 :이 매개 변수는 스프라이트 및 오브젝트 자산에만 사용할 수 있으며 현재 애니메이션 프레임 (자세한 내용은 image_index 페이지 참조)과 FPS를 사용하여 설정 한 현재 애니메이션 속도를 변경하는 데 사용할 수 있습니다 스프라이트 에디터에서 사용되는 스프라이트에 설정 한 값과 같은 방식으로 작동합니다.
  • Color Multiply :이 매개 변수는 스프라이트 및 오브젝트 자산에만 사용할 수 있으며 그려지는 스프라이트의 현재 블렌드 색상을 변경하는 데 사용할 수 있습니다. 이것은 image_blend 값을 설정하는 것과 동일하며 여기에서 설정 한 색상은 스프라이트의 색상과 "곱셈"(즉, 혼합)되며 흰색 값 (#ffffffff)은 혼합이 발생하지 않아야 함을 나타냅니다. 이 유형의 트랙으로 작업 할 때는 16 진수 값을 수동으로 입력하여 색상을 설정하거나 색상 견본을 클릭하여 GameMaker Studio 2 색상 피커를 열 수 있습니다. 색상은 ARGB 형식으로 제공되므로 이러한 종류의 매개 변수 트랙을 사용하여 알파 값을 설정할 수 있습니다 (색상 선택 도구를 사용하면 알파 값을 설정할 수도 있습니다). 이것은 스프라이트에 image_alpha 를 설정하는 것과 같습니다.
  • Volume : volume 매개 변수는 사운드 자산 트랙에만 적용되며 사운드 게인을 설정하는 데 사용할 수 있습니다 (실제 사운드 자산 게인은 자산 트랙에 순서대로 사용되는 이미 터이므로 변경되지 않습니다). 볼륨 값은 백분율로 계산되며 0 %는 볼륨이없고 100 %는 전체 볼륨입니다.
  • 피치 : 피치 파라미터는 사운드 자산 트랙에만 적용되며 사운드 의 피치를 설정하는 데 사용할 수 있습니다 (볼륨과 마찬가지로 실제 사운드 자산 게인은 자산 트랙에 사용되는 이미 터이므로 변경되지 않습니다) 순서대로). 피치 값은 백분율로 계산되며 100 %는 사운드의 "자연"피치이며 100 %보다 작 으면 100 %보다 작 으면 음이 낮아집니다.
  • 폴 오프는 : 폴 오프 매개 변수는 자산 트랙 사운드에 적용하고 현재 구현되지 않았다.

 

액션 추적 액션 추적

트랙 편집기 하단의 버튼은 다른 트랙 동작이며 다음과 같은 용도로 사용할 수 있습니다.

  • -이 버튼을 누르면 자산 탐색기가 열리고 새 트랙으로 사용할 자산을 선택할 수 있습니다. 선택하면 자산 트랙이 생성되고 프레임이 현재 재생 헤드 위치에서 Dope Sheet에 추가됩니다.
  • -트랙 편집기에서 하나 이상의 자산 트랙을 선택한 경우이 버튼을 클릭하면 포함 할 수있는 모든 파라미터 트랙과 함께 해당 트랙이 복제됩니다.
  • -클릭하면 마스크 로 사용할 스프라이트와 마스크로 사용할 스프라이트를 제공 할 수있는 특수 클리핑 마스크 그룹 이 생성됩니다. 자세한 내용은 여기 를 참조하십시오.
  • -클릭하면 새로운 그룹 폴더가 트랙 편집기에 추가되어 트랙을보다 잘 구성 할 수 있습니다. 트랙을 그룹 폴더로 끌어서 트랙 그룹을 추가 할 수 있으며 폴더 밖으로 끌어서 다시 제거 할 수 있습니다. 그룹 폴더에 추가 된 자산에는 폴더 색상이 자동으로 할당되지만 필요한 경우 개별 트랙의 트랙 색상을 편집하여 개별 트랙의 트랙 색상을 편집 할 수 있습니다.
  • -예상대로 클릭하면 선택한 트랙이 삭제됩니다. 자산 트랙 인 경우 자산 트랙이 포함 된 모든 매개 변수 트랙도 삭제됩니다. 선택한 매개 변수 트랙에서 해당 트랙을 사용하여 연결된 자산 트랙을 제거하지 않고 자체적으로 삭제할 수 있습니다. 빈 그룹 폴더에서이 폴더를 사용하면 폴더가 삭제되지만 폴더에 트랙이 있으면 폴더를 삭제하면 폴더 안에있는 모든 트랙도 제거되므로 경고가 표시됩니다.
