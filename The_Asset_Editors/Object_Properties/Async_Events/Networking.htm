네트워킹

네트워킹 이벤트는 들어오는 네트워크 트래픽에 의해 트리거되며 네트워크 기능에 연결됩니다. 이 이벤트는이 이벤트 전용의 특수 DS 맵 을 생성하고 특수 변수 async_load에 저장됩니다. 이 DS 맵에는이를 생성 한 네트워크 이벤트 유형에 따라 다른 정보가 포함됩니다.

 

일반적인 네트워킹 이벤트

다음 키는 수신 된 모든 네트워크 기능에 공통이며 항상 async_load 맵에 있습니다.

  • " type ": 아래 표에 나열된 상수 중 하나를 반환 값으로 사용하여 네트워크 이벤트 유형을 지정합니다.
  • " id ": 이벤트를 수신하는 소켓 ID (network_create_server 또는 network_create_socket 함수에 의해 리턴 된 실수). 대부분의 경우, 리턴 된 소켓 ID는 이벤트를 트리거 한 TCP 또는 UDP 소켓의 ID이지만, 이벤트가 서버에서 트리거되고 데이터 이벤트 인 경우 (아래 참조) 소켓 ID는 서버를 보낸 클라이언트의 ID입니다. 데이터.
  • " ip ": 소켓의 IP 주소 (문자열).
  • " port ": IP 주소와 관련된 포트입니다 (UDP 작업시 유용함).

"type"키의 가능한 리턴 값은 아래 나열된 네 가지 상수 중 하나 일 수 있습니다.

일정한 기술
network_type_connect 이벤트가 연결에 의해 트리거되었습니다.
network_type_disconnect 연결이 끊어 이벤트가 트리거되었습니다.
network_type_data 들어오는 데이터에 의해 이벤트가 트리거되었습니다.
network_type_non_blocking_connect 비 차단으로 구성된 연결에 의해 이벤트가 트리거되었습니다 ( network_set_config () 함수를 사용할 수 있음).

 

연결 / 분리

network_type_connect 또는 network_type_disconnect 유형의 이벤트가있는 경우 async_load 맵에는 다음과 같은 추가 키가 있습니다.

  • " socket ":이 키는 연결 / 연결 해제 소켓 ID를 보유합니다.

이벤트가 network_type_non_blocking_connect 유형 인 경우 추가로 다음 키가 있습니다.

  • " 성공 ":이 키는 0 또는 1입니다. 여기서 0은 연결 시간이 초과되었음을 의미하고 1은 연결이 성공했으며 소켓을 사용할 수 있음을 의미합니다.

서버 연결이 끊어 질 때 클라이언트에서 네트워킹 이벤트가 트리거되지 않으며 연결이 TCP 기반 인 경우에도 연결이 끊어진 서버에서 연결이 끊어 질 때 network_type_ * 이벤트가 클라이언트에서 트리거되지 않습니다.

 

데이터 수신

네트워크가 데이터를 수신했음을 나타내는 network_type_data 유형 이벤트가있는 경우 작성된 ​​맵에는 다음 키가 있습니다.

  • " buffer ": 이벤트에 의해 생성 된 고유 한 "버퍼 ID"입니다. 바이트를 1로 정렬 한 "grow"유형 버퍼는 ID를 변수에 저장하기 위해 작성되며이 이벤트에서 버퍼에 대한 모든 추가 함수 호출에 사용됩니다. async_load 맵과 마찬가지로 생성 된 버퍼는이 이벤트가 끝날 때 메모리에서 자동으로 제거됩니다. 버퍼에 대한 자세한 내용은 참조 버퍼 를 참조하십시오.
  • " size ": 수신중인 버퍼 데이터의 크기 (바이트)입니다.

참고 : 생성 된 버퍼는 이러한 이벤트에서만 유효하며 이벤트가 끝나면 해제됩니다.
