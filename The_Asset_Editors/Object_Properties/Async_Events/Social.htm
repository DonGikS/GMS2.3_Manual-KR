사회적인

이 이벤트는 다양한 특정 소셜 함수 및 UWP Xbox Live 함수 의 콜백 에 의해 트리거 될 수 있으며 이벤트를 트리거 한 함수 의 콜백에 따라 다른 키 / 값 쌍을 포함하는 변수 async_load에 저장된 DS 맵 을 반환합니다.

DS 맵에는 많은 키가 포함되며 가장 중요한 키는 " id "키입니다. 이것은 이벤트에서 수신 한 많은 콜백 중 어떤 것을 콜백하기 위해 코드에서 확인할 수있는 상수를 리턴합니다. 이 키를 구문 분석하고 반환 된 값을 사용 가능한 상수와 비교 한 후에는 맵에서 나머지 정보를 계속 추출 할 수 있습니다.

Reference 섹션의 Achievements and Leaderboards 의 함수 페이지에는 함수 및 콜백과이 이벤트를 트리거하는 데 사용되는 상수가 자세히 설명되어 있지만 실행중인 게임에서 함수 호출없이 트리거되는 특정 상황도 있습니다.

  • OS 대시 보드에서 게임을 시작할 때
  • 도전을 마치면
  • 다른 플레이어가 도전을 완료하면
  • 챌린지를 선택하여 OS 대시 보드에서 게임을 시작할 때

위의 콜백 중 하나가 소셜 이벤트를 트리거하고 다음 세부 사항으로 async_load 맵이 생성됩니다 (이벤트가 호출 된 각기 다른 이유와 다른 모든 컨텐츠를 정의하기 위해 다른 " id "키 값이 있음에 유의하십시오. 지도는 동일합니다) :

  • " id "-이 키의 값은 이벤트를 트리거하는 콜백 유형에 따라 다릅니다. 다음 상수 중 하나 일 수 있습니다.
일정한 기술
achievement_challenge_received 도전이 접수되었습니다
achievement_challenge_completed 도전이 완료되었습니다.
achievement_challenge_completed_by_remote 다른 플레이어가 도전을 완료했습니다.
achievement_challenge_launched 주어진 도전 과제로 OS 대시 보드에서 게임이 시작되었습니다.

 

  • " playerid "-챌린지의 플레이어 ID입니다.
  • " issuerid "-챌린지를 발행 한 사람의 ID입니다.
  • " state "-유효하지 않은, 보류 중, 완료 또는 거부 된 경우 0-3 (문자열) 값을 갖는 챌린지 상태
  • " message "-챌린지에 대한 문자 메시지.
  • " 발행일 "-이의 제기 발행일
  • " completedate "-챌린지 완료 날짜입니다.
  • " type "-주어진 챌린지 유형. 두 상수 중 하나 일 수 있습니다.
일정한 기술
achievement_type_score_challenge 점수 값에 따른 챌린지.
achievement_type_achievement_challenge 업적에 따른 도전.

 

  • " identifier "-챌린지의 식별 문자열입니다.
  • " score "-챌린지와 관련된 점수입니다.

아래에서 사용 예제로 작은 코드 스 니펫을 찾을 수 있습니다.

var ident = ds_map_find_value(async_load, "id" );
if ident == achievement_challenge_completed;
   {
   player_id = ds_map_find_value(async_load, "playerid");
   issuer_id = ds_map_find_value(async_load, "issuerid");
 state = ds_map_find_value(async_load, "state");
   date_completed = ds_map_find_value(async_load, "completeddate");
   date_issued = ds_map_find_value(async_load, "issueddate");
   ach_type = ds_map_find_value(async_load, "type");
   ach_ident = ds_map_find_value(async_load, "identifier");
   ach_score = ds_map_find_value(async_load, "score");
   }
