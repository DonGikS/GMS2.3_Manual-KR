푸시 알림

푸시 알림 이벤트는 장치 OS의 푸시 알림 , 기능 push_local_notification ()을 사용하는 로컬 소스 또는 원격 소스 (예 : 서버)에서 콜백 에 의해 트리거되는 이벤트입니다. 이 이벤트 전용의 DS Map 을 생성하고 특수 변수 async_load에 저장됩니다 (이 이벤트의 사용법을 자세히 설명하는 코드 예제에 대해서는 비동기 이벤트를 트리거하는 개별 함수를 참조하십시오). 이 DS 맵에는 다음과 같은 키가 있습니다.

  • " type ": 장치 로컬 알림의 경우 "로컬", 원격 알림의 경우 "원격"또는 원격 알림 등록의 경우 "등록"일 수 있습니다.
  • " status ": 성공의 경우 값이 "1"이고 오류의 경우 "0"입니다.

반환 된 "유형"및 "상태"값을 기반으로 추가 키 항목이있을 수 있습니다. "상태"의 경우 오류가 반환되면 ( "0") 다음 키도 있습니다.

  • " error ": 수신 된 오류의 세부 사항을 포함합니다.

"상태"값이 1 인 경우 (예 : 오류 없음) DS 맵에는 "유형"키 값에 따라 다음과 같은 추가 값이 포함됩니다.

  • " reg_id ": 수신 된 "type"이 "register"인 경우이 키는 원격 알림에 대한 장치 등록 ID를 보유합니다.
  • " data ": 수신 된 "type"이 "local"또는 "remote"인 경우이 키는 알림 기능을 호출 할 때 정의한 문자열 페이로드를 보유합니다.

이를 더 잘 이해하기 위해 아래에 작은 예제 스 니펫을 작성했습니다. 이 예에서는 다음 코드를 사용하여 로컬 푸시 알림을 보냅니다.

var fireTime = date_inc_day(date_current_datetime(), 1);
var data = "daily_reward";
push_local_notification(fireTime, "Ahoy!", "Catch The Haggis Has A Present", data);

하루가 지나면 장치에 알림을 "푸시"하도록 타이머가 설정됩니다. 하루가 끝나면 게임이 백그라운드에 있거나 실행 중이 아닌 경우 지정된 제목과 메시지와 함께 알림이 사용자에게 표시됩니다 (iOS에서는 게임 이름이 표시되고 제목이 무시 됨). 비동기 푸시 알림 이벤트가 호출됩니다. 알림 시간이되었을 때 게임이 포 그라운드에있는 경우 표시 되지 않지만 비동기 이벤트는 여전히 트리거 됩니다. 이벤트 자체에서 다음과 같은 콜백을 처리합니다.

var type = ds_map_find_value(async_load, "type");
var status = ds_map_find_value(async_load, "status");
if status == 0
   {
   //error of some kind
   var error = ds_map_find_value(async_load, "error");
 show_debug_message("error=" + string(error));
   }
else
   {
   if type == "register"
      {
      global.reg_id = ds_map_find_value(async_load, "reg_id");
      }
   else
    {
      var data = ds_map_find_value(async_load, "data");
         if data == "daily_reward"
         {
         global.Gold += 1000;
         }
      }
   }
