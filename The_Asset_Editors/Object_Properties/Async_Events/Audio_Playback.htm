오디오 재생

이 이벤트는 오디오 큐 재생 기능에 의해서만 트리거 될 수 있으며 이벤트를 트리거 한 오디오 큐 와 관련된 다른 키 / 값 쌍을 포함하는 변수 async_load에 저장된 DS 맵 을 반환합니다. 재생할 오디오 대기열을 선택하고 해당 오디오 대기열의 버퍼 재생이 완료되면 콜백 에 의해 이벤트가 트리거됩니다.

DS 맵에서 다음 키를 사용할 수 있습니다.

  • " queue_id " -audio_create_play_queue () 함수가 리턴 한대로 재생이 완료된 큐의 큐 색인.
  • " buffer_id "-더 이상 재생되지 않는 버퍼 의 버퍼 ID
  • " queue_shutdown "-일반적인 재생 중에는 0으로 설정되고 audio_free_play_queue () 가 호출되어 이벤트가 수신되면 1로 설정됩니다. 이 값이 1로 설정되어 있으면 더 이상 데이터를 대기열에 넣고 싶지 않습니다.

여러 개의 버퍼에서 오디오 큐를 만들 수 있으므로 버퍼 된 오디오의 각 섹션 끝에 도달하여 "buffer_id"키에 도달하면이 이벤트를 한 큐에 대해 여러 번 트리거 할 수 있습니다.

참고 : async_load 변수는 비동기 이벤트에서만 유효합니다. 가리키는 DS 맵은 이벤트가 시작될 때 작성된 다음 끝에 다시 삭제 되며이 변수는 -1 값으로 재설정됩니다. 그러나 이벤트에서 작성된 모든 추가 데이터 구조는 적절한 기능을 사용하여 정리해야합니다.

아래에 주어진 사용 예제에서 10 개의 버퍼링 된 오디오 사운드가 추가 된 오디오 큐를 생성 한 다음 큐를 재생합니다.

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);
for (var i = 0; i < 10; i++;)
   {
   audio_queue_sound(audio_queue, audio_buffer[i], 0, buffer_get_size(audio_buffer[i]));
   }
audio_play_sound(audio_queue, 0, true);

녹음이 감지 된 모든 게임 단계는 다음과 같이 녹음 된 입력을 처리 할 수있는 비동기 오디오 녹음 이벤트를 트리거합니다.

var queue = async_load[? "queue_id"];
var num = 0;
if queue = audio_queue
   {
   for (var i = 0; i < 10; i++;)
      {
      if async_load[? "buffer_id"] == audio_buffer[i]
         {
 buffer_seek(audio_buffer[i], buffer_seek_start, 0);
         num = i;
         }
      }
   if num == 9
      {
      audio_stop_sound(audio_queue);
      audio_free_play_queue(audio_queue);
      }
 }

여기서 큐 ID를 확인하고 원하는 ID 인 경우 버퍼 ID를 확인하여 어떤 버퍼의 재생이 완료되었는지 확인합니다. 그런 다음 나중에 확인할 수 있도록 로컬 var를 해당 인덱스 값으로 설정하고 버퍼를 해당 사운드의 시작으로 다시 설정합니다. 로컬 변수가 9에 도달하면 큐가 완료되었음을 나타내며 사운드 재생을 중지하고 큐를 비 웁니다.
