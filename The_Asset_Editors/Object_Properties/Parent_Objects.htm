부모 객체

GameMaker Studio 2 IDE에서 객체로 작업 할 때 부모 / 자식 계층을 설정할 수 있습니다. 오브젝트 에디터에서 "Parent"버튼을 클릭 한 다음 Asset Browser에서 다른 오브젝트를 선택하여 편집중인 오브젝트의 "parent"로 선택하면됩니다.

따라서 게임 프로젝트의 모든 오브젝트는 부모 오브젝트를 가질 수 있지만 이것이 무엇을 의미합니까? 객체에 부모가 할당 된 경우 해당 부모와 코드, 작업 및 이벤트를 공유 할 수 있습니다. 이 공유를 "상속"이라고하고 부모가있는 개체를 "자식"개체라고합니다. 자식은 부모와 코드를 공유 할 수있을뿐만 아니라 부모 개체를 검사하고 코드를 실행할 수 있으며 자식 개체도 자동으로 포함하므로 많은 시간과 에너지를 절약 할 수 있습니다.

복잡하게 들리면 부모 개체를 보는 또 다른 방법은 같은 우산 아래에서 개체를 "그룹화"하여 고유 한 정체성을 잃지 않고 특정 사물을 공유하는 방법입니다. 어쩌면 그것은 여전히 ​​너무 많은 것을 명확하게하지 못하므로 몇 가지 예를 들어 보겠습니다.

"플레이어"오브젝트와 네 개의 다른 "적"오브젝트가 있다고 가정하십시오. 자, 플레이어가이 4 개의 물체 중 하나를 만지면 죽게 되길 원하며, 이것은 보통 4 개의 다른 행동 또는 코드 세트 (적의 물체마다 하나씩)로 4 개의 다른 충돌 이벤트를 수반합니다. 그러나 모든 적에 대해 부모 오브젝트를 만들면 부모 오브젝트만으로 충돌 이벤트를 하나만 만들 수 있으며 4 개의 "자식"적 오브젝트 중 하나가 플레이어에 닿아도 트리거됩니다. 편리한 물건! 실제 GameMaker Studio 2 오브젝트에서 다음과 같은 것을 볼 수 있습니다 :

왼쪽에는 4 개의 개별 충돌 이벤트가 있고 오른쪽에는 "부모"오브젝트를 만들고 모든 적 오브젝트를 할당 했으므로 단일 충돌 이벤트가 있습니다. 부모 객체에는 이벤트 나 코드가 필요하지 않습니다.

육아의 또 다른 예는 10 개의 다른 모양의 개체를 만들고 같은 방식으로 동작하도록하는 게임이있는 경우입니다. 이를 위해 하나의 상위 오브젝트를 작성하고 필요한 이벤트에 모든 동작 조치 또는 코드가있는 경우 조치 또는 코드가 아닌 스프라이트가 다른 10 개의 오브젝트를 작성하고 상위 오브젝트를 지정합니다. 이제 이러한 인스턴스를 방에 배치하면 모두 동일한 방식으로 작동하지만 부모의 이벤트를 "상속"하기 때문에 다르게 보입니다.

마지막으로, 육아를 사용하여 이벤트와 동작을 "혼합 및 일치"할 수 있습니다. 우리는 이것을 하나의 마지막 예제로 설명 할 것입니다 ... 두 개의 괴물이 필요합니다. 하나는 위아래로 움직이고 다른 하나는 왼쪽과 오른쪽으로 움직이지만 두 마리는 같은 건강을 원합니다 플레이어와 충돌하면 플레이어를 다치게합니다. 이 경우 움직임을 제어하는 ​​하나 또는 두 개를 제외하고 거의 모든 이벤트에 동일한 조치가 있어야 함을 알 수 있습니다. 다시 한 개체를 다른 개체의 부모로 만들 수 있지만이 경우 자식 개체에 대한 특정 이벤트도 정의합니다. 이러한 이벤트는 상위 이벤트를 "재정의"합니다. 즉, 하위 오브젝트의 이벤트에 조치가 포함되어 있으면 상위 이벤트에 포함 된 조치 대신 실행됩니다. 부모 이벤트도 실행하려면 event_inherited () 함수 또는 DnD ™ 액션 부모 이벤트 호출을 사용하여 소위 "상속 된"이벤트를 호출 할 수 있습니다.

왼쪽에는 5 개의 이벤트가있는 부모 개체가 있으며 오른쪽에는 "자식"개체가 있습니다. 자식 개체에도 5 개의 이벤트가 있지만 그 중 두 개는 부모에서 상속 된 이벤트 ( Step 및 Draw 이벤트)를 재정의하고 나머지 3 개는 부모에서 상속 된 이벤트이므로 회색으로 표시됩니다. 상속 된 이벤트는 이벤트 편집기에서 옆에 "parent override"아이콘이 있습니다.

클릭하면 상속 된 이벤트에서는 상속 된 부모 코드를 표시하기 위해 코드 편집기가 열리지 만 부모 개체 자체에서만 편집 할 수 있으므로이 코드를 편집 할 수 없습니다. 마우스 오른쪽 버튼을 클릭 할 수 있습니다 상위 이벤트 중 하나에서 다음 옵션 메뉴를 엽니 다.

여기에서 두 개의 상위 이벤트 열기 를 선택하여 코드를 검사하거나 이벤트 상속 또는 이벤트 대체를 선택할 수 있습니다. 상속 을 선택하면 event_inherited () 함수가 이미 추가 된 함수 (또는 DnD ™를 사용하는 경우 부모 이벤트 호출 작업 ) 와 함께 코드 편집기가 열립니다. 이 이벤트에 추가 코드를 추가하면 이제 상위 개체의 코드와 함께 실행됩니다. 이벤트를 재정의 하도록 선택하면 코드 창이 열립니다. 이제 event_inherited () 함수 만 호출되지 않으므로 여기에 추가 한 모든 항목이 상위 개체의 코드 대신 실행 됩니다 .

NOTE: From the code editor you can quickly navigate to the parent object by clicking the right mouse button 팝업 메뉴에서 객체 로 이동을 선택하거나 (이벤트가 재정의 된 경우) 상속 된 이벤트 열기 를 선택하여 상위 이벤트 코드가 포함 된 코드 편집기로 직접 이동할 수 있습니다.

코드에서 부모 개체를 대상으로 지정할 때마다 코드는 부모 개체의 "자식"에도 적용됩니다. 이것은 액션에서 특정 객체의 인스턴스에 액션을 적용해야 함을 나타내며 코드에서 with () 문을 사용할 때 발생합니다. instance_position () , instance_number () 등과 같은 코드 함수를 호출 할 때도 이와 같이 작동합니다. 여기서 부모 개체를 제공하는 경우 부모 및 자식 인스턴스의 모든 인스턴스가 검사에 포함됩니다. 마지막으로, 육체는 다른 물체에서도 변수를 참조 할 때 위의 괴물 예와 같이 적 1 속도를 10으로 설정하면 적 2 속도도 적 1의 자식 개체이므로 10으로 이동합니다.

대부분의 경우 하나의 기본 상위 오브젝트를 작성하고이 기본 오브젝트에 모든 기본 동작이 포함되지만 게임에서 해당 인스턴스를 사용하지 않는 것이 좋습니다. 오히려 모든 자식 객체를 사용하고 위에서 설명한 상황, 충돌, 변수 참조 등의 상황에서만 부모를 사용하십시오. 부모도 부모를 가질 수 있음을 알아야합니다! 분명히 당신의 사이클을 만들 수 있지만 "parent3이 parent2의 자식 상위 1의 자식" "개체 계층 구조"라는 것을 만들 수 있습니다 "부모 한 부모 2의 아이가 부모 1 자녀입니다". 게임 구성을 유지하는 데 매우 유용하며이 메커니즘을 사용하는 방법을 배우는 것이 좋습니다.
