추첨 이벤트

Draw 이벤트 범주는 게임을 실행할 때 화면에 표시되는 내용을 제어하는 ​​모든 이벤트가있는 범주입니다. 게임에 대한 다양한 그림 요구 사항을 더 잘 처리하기 위해 여러 개의 개별 이벤트로 분할되며 아래 이미지는 각 이벤트의 순서를 보여줍니다.

일반적인 Draw 이벤트는 Draw Begin , 표준 Draw 및 Draw End 이벤트의 세 가지 하위 유형으로 분류됩니다. 일반적으로 표준 Draw 이벤트 만 사용해야하며, 방의 모든 인스턴스는이 이벤트가 게임의 각 단계에서 트리거되므로 인스턴스가 표시 되는 한 항상이 이벤트가 호출됩니다. 플래그가 true로 설정되었습니다. 객체 속성에서이 하위 이벤트에 대해 아무것도 정의하지 않았더라도 (예 : 코드 나 동작 없음) 객체에 스프라이트가 할당 된 경우에도 여전히 트리거 이벤트가 발생 한다는 것을 이해하는 것이 중요 합니다 . GameMaker Studio 2에는 Draw 이벤트에서 물건을 그리는 두 가지 방법이 있기 때문입니다.

  • 기본 draw 는 오브젝트 속성에서 스프라이트를 정의하고 일반 Draw 이벤트에서 액션이나 코드를 배치하지 않는 경우이며,이 경우 GameMaker Studio 2는 할당 된 스프라이트를 자동으로 그립니다. 이미지 스케일, 인덱스, 블렌딩 등을 변경하기 위해 다른 이벤트에서 수행하는 모든 변환은 드로우 이벤트가 비어있는 한 반영됩니다.
  • 사용자 지정 draw 는 draw 이벤트에 코드 또는 작업을 배치 할 때 사용됩니다. 이것은 GameMaker Studio 2에게 " 이 객체의 인스턴스에 대해 그리는 것을 제어하고 싶다 " 고 말하고 기본 드로잉을 완전히 무시합니다. 예를 들어, 당신은 할당 된 스프라이트와 객체를 가질 수 있습니다,이 방법은, 다음 텍스트를 그릴 추첨 이벤트를 설정하면 텍스트가 화면에 그려 질 것입니다,하지만 당신은 GameMaker 스튜디오 2 말하지 않았다으로 스프라이트가 그려지지 않습니다 텍스트와 함께 그립니다.

나머지 추첨 이벤트는 아래 섹션에 설명되어 있습니다.

 

그리기 시작 및 그리기 끝 그리기 시작 및 그리기 끝

메인 Draw 이벤트 외에도 Draw Begin 및 Draw End 이벤트도 있습니다. 이것들은 표준 Draw 이벤트와 정확히 동일하게 작동합니다 (즉, 게임의 모든 단계와 모든 뷰포트에 넣은 모든 것을 그릴 것입니다). 그러나 명시 적으로 추가되지 않은 경우 아무것도 "기본 그리기"하지 않습니다 표준 Draw 이벤트 전 / 후에 항상 실행됩니다. 따라서 게임이 실행 중이면 항상 모든 인스턴스에 대해 Draw Begin 이벤트가 발생하고 모든 인스턴스에 대해 Draw 이벤트가 수행되고 마지막으로 모든 인스턴스에 대한 Draw End 이벤트가 발생합니다.

이러한 방식으로 한 인스턴스의 Draw Begin 단계에서 사물을 그리거나 그리기 속성을 설정할 수 있으며 표준 Draw 이벤트 또는 Draw End 이벤트가있는 모든 인스턴스가 이러한 속성을 사용하거나 첫 번째 인스턴스가 그린 것을 그립니다. 기본적으로 단계 시작 및 종료 단계 이벤트와 마찬가지로 특정 시점에 특정 항목이 그려 지도록하는 확실한 방법입니다.

GameMaker Studio 2에서 드로잉 할 때주의해야 할 스프라이트, 쉐이더 또는 3D 버퍼 등 몇 가지주의 할 사항이 있습니다.

  • 드로우 이벤트는 가장 많은 시간과 리소스를 필요로하는 이벤트 중 하나라는 점에서 매우 집중적 인 이벤트입니다.이를 위해, 드로우 이벤트에서 드로잉 이외의 작업을 수행하는 것은 결코 좋은 생각이 아닙니다. 따라서 Step 이벤트 또는 알람 또는 다른 이벤트에 맞는 큰 코드 나 복잡한 작업을 저장하되 드로잉에 대한 그리기 이벤트는 명확하게 유지하십시오.
  • 객체의 visible 매개 변수가 off로 설정되어 있으면 (객체의 인스턴스가 그려지지 않음) 모든 그리기 이벤트가 건너 뜁니다 ( Resize 이벤트 제외). 즉, 인스턴스를 보이지 않게 만들면 이러한 이벤트 중 하나에 배치 된 모든 코드의 실행이 중지되므로 드로우 이벤트에 필수 코드를 표시하지 않을 것입니다.
  • 그리는 것은 객체 속성과 객체에 주어진 스프라이트 (또는 마스크)에 의해 정의 된대로 선택한 충돌 엔진 (전통 또는 물리)과 아무 관련이 없습니다.

 

GUI 그리기 GUI 그리기

Draw GUI 이벤트 유형은 Draw Event 범주에 속하며 특히 뷰 카메라 배율 또는 회전에 영향을받지 않는 GUI 요소를 그리기 위해 설계되었습니다. 즉, 실 내 인스턴스 위치 또는 현재 카메라 뷰의 위치를 ​​기준으로 요소의 모든 위치를 지정하지 않고도 인스턴스가 HUD 또는 사용자 인터페이스에 대한 모든 요소를 ​​그릴 수 있습니다.

이러한 이벤트를 그릴 때 카메라 뷰가 활성화되어 있고 (0,0)이 항상 응용 프로그램 표면 또는 디스플레이의 왼쪽 상단 모서리에있을 때에도 그리기 좌표가 변경되지 않는다는 점을 이해해야합니다. 이 섹션의 맨 아래) 및 기본 너비와 높이는 응용 프로그램 표면과 1 : 1입니다. Draw GUI Begin 이벤트가 먼저 모든 인스턴스에 대해 그려 지므로 다른 레이어의 서로 다른 인스턴스간에 깊이 순서가 유지됩니다 (따라서 상위 레이어의 인스턴스는 하위 레이어의 하나 아래에 그려 짐). 표준 Draw GUI 는 그 위에있는 모든 인스턴스에 대해 Draw 하고 Draw Gui End 이벤트가 트리거됩니다.

이러한 방식으로, 하나의 인스턴스에서 Draw GUI Begin에서 사물을 그리거나 그리기 속성을 설정할 수 있으며 표준 Draw GUI 또는 Draw GUI End 이벤트가있는 모든 인스턴스가 이러한 속성을 사용하거나 첫 번째 인스턴스가 그린 것을 그립니다. 기본적으로 시작 단계 및 종료 단계 이벤트와 마찬가지로 특정 시간에 특정 항목이 그려 지도록하는 확실한 방법입니다.

이 이벤트는 일반 Draw 이벤트와 함께 사용할 수도 있습니다 (일반적으로 카메라 뷰 위치, 스케일 및 회전의 영향을 받음). Draw 이벤트가 없지만 Draw GUI 이벤트가있는 경우, GameMaker Studio 2는 기본적으로 인스턴스의 스프라이트 (있는 경우)를 정상적으로 그립니다.

다른 드로우 이벤트와 관련하여 드로우 GUI 이벤트는 항상 일반 드로우 이벤트에서 그린 항목을 그립니다. 따라서 Draw GUI 이벤트가있는 하위 레이어에 인스턴스가있는 경우 일반 Draw 이벤트가있는 상위 레이어에있는 인스턴스를 그립니다. 두 인스턴스 모두 Draw GUI 이벤트가 있으면 레이어 순서가 존중됩니다.

참고 :이 이벤트는 기본적으로 응용 프로그램 표면 크기 (일반적으로 방 또는 뷰 포트 크기)로 1 : 1을 그립니다. 즉, 게임 옵션에서 가로 세로 비율 수정을 설정하면 게임을 "레터 박스"하는 검은 막대 위에 GUI가 표시되지 않습니다. 이 동작은 display_set_gui_maximise () 함수를 사용하여 끌 수 있으며, GUI 이벤트를 특정 크기로 잠근 다음 display_set_gui_size () 함수를 사용하여 자동으로 디스플레이 또는 응용 프로그램 표면 크기에 맞게 크기를 조정할 수 있습니다.

 

사전 및 사후 그리기 사전 및 사후 그리기

프리 드로우 및 포스트 드로우 이벤트는 드로우 이벤트 범주의 일부입니다. 그러나, 다른 그리기 이벤트와는 달리 이들은 직접 그릴 디스플레이 버퍼 에만 모두에 하나 개의 뷰포트 또는 없음을 사용하는 경우 현재 보이는 모든 뷰포트에 대한 결합 된 화면 공간, 또는 창 크기의 크기가됩니다. 아래 이미지는 이것을 보여줍니다.

따라서 Pre 또는 Post 그리기 이벤트를 사용하는 경우 모든 뷰포트가 맞는 창과 크기가 같은 전체 화면 렌더 대상 (디스플레이 버퍼)에 그리게됩니다. 뷰포트가 활성화되어 있지 않으면 대신 창 자체의 크기로 설정됩니다.

프리 드로우 이벤트는 다른 드로우 이벤트보다 먼저 트리거되며, 뷰포트 또는 GUI 레이어의 크기에 대해 걱정할 필요없이 값을 설정하고, 그리기 속성을 설정하고, 그릴 수도 있습니다 (GUI 레이어 크기는 화면 버퍼, 그러나 GUI 해상도를 코드로 설정할 수 있기 때문에 아닐 수도 있습니다).

이 이벤트는 일반 그리기를 위해 디스플레이 버퍼가 지워 지기 전에 발생합니다. 즉, 룸 편집기에서 뷰 지우기를 끄지 않으면 첫 번째 뷰포트 그리기로 Pre Draw 이벤트에서 그린 것이 표시되지 않습니다. 그것을 취소합니다. 뷰포트 자체를 통해 보거나 뷰포트를 전혀 사용하지 않으려면 룸 편집기 에서 배경색의 알파가 0이되도록 설정해야합니다.

참고 :이 옵션을 끄면 게임을 테스트 할 때 화면에 원치 않는 인공물이 그려지는 것을 볼 수 있습니다 (예 : "트레일"). 디스플레이 버퍼의 이전 프레임을 지우지 않고 직접 그리려고하기 때문입니다. 그러나 draw_clear_alpha () 를 사용하여 직접 할 수 있습니다.

포스트 드로우 이벤트는 모든 표준 드로우 이벤트 이후 , 드로우 GUI 이벤트 전에 트리거됩니다. 프리 드로우 이벤트와 마찬가지로 디스플레이 버퍼의 크기를 기반으로하며 드로우 GUI 이벤트 앞에 배치되어 HUD 를 방해하지 않고 사후 처리 효과 및 기타 작업을 전체 화면으로 간단하고 쉽게 수행 할 수 있습니다. 게임에있을 수있는 / GUI 요소.

 

창 크기 조정 창 크기 조정

이 이벤트 유형은 Draw Event 범주에 속하며 실제로 아무 것도 그리지는 않지만 디스플레이 버퍼의 변화에 ​​반응합니다. 특히 게임 윈도우에서 UWP 디스플레이 버퍼 크기의 변화에 ​​반응하도록 설계되었습니다 "스냅"입니다.

UWP 대상 플랫폼에서는 게임 창의 "스냅 핑"(예 : 사용자가 창을 화면쪽으로 끌 때)에서 디스플레이 버퍼의 크기를 조정하는 것이 중요합니다. 이 이벤트는 그 작업을 수행하며 창이 변경 될 때마다 트리거되므로 여기에서보기의 크기를 조정하거나 필요에 따라 HUD 요소를 재배치하는 사용자 지정 코드를 추가 할 수 있습니다. 이 기능은 이러한 것들을 항상 확인하는 단계 이벤트 코드를 피하는 편리한 방법으로 제공됩니다.

참고 :이 이벤트에서는 그릴 수 없습니다! 창 크기를 조정하여 트리거되며이 변경 사항을 포착하기위한 것입니다. 수행해야하는 모든 그림은 다른 그리기 이벤트 중 하나에 있어야합니다.
