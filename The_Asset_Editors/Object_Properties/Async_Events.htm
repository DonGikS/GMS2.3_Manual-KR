비동기 이벤트

비동기 이벤트는 GameMaker Studio 2가 웹 또는 게임을 실행하는 장치에서 발생할 수있는 외부 소스로부터 콜백 을 수신하면 발생하는 이벤트입니다. 기본적으로 GameMaker Studio 2에 이미지로드와 같은 작업을 수행하도록 지시하면이 작업이 시작되지만 작업하는 동안 다른 작업을 계속 수행합니다. 그런 다음 요청이 완료되면 콜백이 GameMaker Studio 2로 전송되고 해당 유형의 콜백에 대해 정의 된 모든 비동기 이벤트가 시작됩니다.

키보드 이벤트와 마찬가지로 인스턴스가있는 모든 인스턴스에 대해 비동기 이벤트가 시작되므로 예를 들어 한 인스턴스에서 http_get () 호출을 수행하고 다른 인스턴스에서는 비동기 HTTP 이벤트 를 처리하여 콜백.

참고 : 브라우저의 XSS 보호로 인해 도메인 간 리소스 요청 및로드 시도가 차단되어 다음 이벤트 중 하나를 사용할 때 빈 결과가 반환 될 수 있습니다.

비동기 이벤트 범주와 관련된 다양한 유형의 이벤트가 있으며 아래 섹션에 모두 설명되어 있습니다.

참고 : 다음 섹션에서 언급 된 async_load 변수는 해당 이벤트 가 시작될 때 생성 된 DS 맵이 생성 된 후 끝에서 다시 삭제 되므로이 섹션에서 언급 된 변수 async_load 는이 이벤트에서만 유효 합니다. 다른 모든 시간에 -1의 값.

  • 오디오 재생
  • 오디오 녹음
  • 구름
  • 대화
  • HTTP
  • 인앱 구매
  • 이미지로드
  • 네트워킹
  • 푸시 알림
  • 저장 /로드
  • 사회적인
  • 증기
  • 체계
