다른 이벤트

이 페이지는 오브젝트 편집기의 기타 이벤트 카테고리에서 수집되는 각기 다른 이벤트에 대해 설명합니다.

 

외부 방 외부 방

이 이벤트는 인스턴스가 방 밖으로 나갈 때 트리거되며 인스턴스의 지정된 스프라이트 (및 해당 속성)에 대해 수행 된 검사를 기반으로하므로 이미지 x 또는 y 스케일을 다른 값으로 설정 한 경우에도 하나,이 이벤트는 전체 스프라이트가 화면을 벗어날 때만 트리거됩니다. 인스턴스에 스프라이트가 없으면 인스턴스의 위치가 사용되고 x 또는 y 위치가 방 밖에있는 순간에도 이벤트가 트리거됩니다. 이 이벤트는 일반적으로 방을 나가면 파괴되는 총알과 같은 것들에 사용됩니다. 수백만 개의 총알이 무한히 날아가서 게임 속도가 느려지지 않습니다. 이 이벤트는 인스턴스가 처음 방을 떠날 때 한 번만 트리거됩니다.

 

경계 교차

이 이벤트는 인스턴스가 룸의 내부 가장자리를 "만질"때 트리거되며 외부 룸 이벤트와 같이 인스턴스의 할당 된 스프라이트에 대해 수행 된 검사를 기반으로하므로 이미지 x 또는 y가 아닌 다른 값으로 조정되는 경우에도 스프라이트의 일부가 룸 경계와 교차하는 순간이 이벤트가 트리거됩니다. 인스턴스에 스프라이트가 없는 경우 인스턴스의 위치가 사용되고 x 또는 y 위치가 방 경계와 교차하는 순간 이벤트도 트리거됩니다. 이 이벤트를 사용하여 인스턴스가 외부로 나가지 않고 다시 방으로 다시 튀어 오도록하는 등의 작업을 수행 할 수 있습니다.

 

조회수 조회수

카메라 뷰는 일반적으로 룸 편집기에서 정의되며 한 번에 큰 룸의 작은 영역 만 표시하는 데 사용됩니다. 이 이벤트 범주에는 2 개의 개별 하위 범주 ( Outside View 및 Intersect Boundary )가 있으며 각각 8 개의 사용 가능한보기 포트에 해당하는 8 개의 서로 다른 이벤트가 있습니다. 이 두 범주는 각 룸 이벤트와 정확히 동일하게 작동하며, 카메라 뷰의 경계를 인스턴스가 아닌 룸이 아닌 확인해야하는 것으로 간주합니다.

 

게임 시작 게임 시작

이 특별 이벤트는 전체 게임에서 한 번만 트리거되며 이벤트에 조치 또는 코드가있는 인스턴스에 대해서만 트리거됩니다. 이러한 인스턴스는 게임의 첫 번째 방에 존재해야하며 코드 나 동작을 사용하여 동적으로 생성 된 것이 아니라 룸 편집기에 있어야합니다. 이 이벤트는 모든 인스턴스의 create 이벤트 이후 에 발생하므로 해당 이벤트에서 이전에 정의 된 변수가있는 코드 또는 조치를 포함 할 수 있습니다. 이 이벤트는 일반적으로 하나의 "컨트롤러"객체에만 정의되며 일반적으로 전역 변수 초기화, 음악 시작, 파일 읽기 및 일반적으로 게임 시작시 한 번만 수행되는 기타 작업에 사용됩니다.

 

게임 끝 게임 끝

위에서 언급 한 Game Start 이벤트와 유사하게이 이벤트는 전체 게임에서 한 번만 트리거되며 실제로 게임 창이 닫히기 직전입니다. 다시 말하지만,이 이벤트가있는 인스턴스는 실제로 게임이 닫힐 때 룸에 있어야하는 모든 것을 수행합니다. 모바일 장치가 GameMaker Studio 2에이 이벤트를 호출 할 기회를주지 않으면 서 게임을 닫을 수 있으므로이 이벤트는 Windows , Ubuntu (Linux) 및 macOS 게임에서만 실제로 실행됩니다. HTML5 모듈로 만든 브라우저 게임의 경우이 이벤트는 게임 탭이나 브라우저가 닫혀있을 때 작동하지만 모든 브라우저가이 기능을 지원하는 것은 아닙니다 (작성시 파이어 폭스, 크롬 및 사파리는 지원함). . 일반적으로이 이벤트를 사용하여 데이터를 저장합니다.

 

방 시작 방 시작

이 이벤트는 회의실이 시작될 때 회의실의 모든 인스턴스에서 처음 발생합니다. 작성 이벤트 이후 에 발생하므로 해당 이벤트에서 이전에 정의 된 변수 및 사물에 따라 달라질 수 있습니다.

 

방 끝 방 끝

이 이벤트는 회의실이 종료 될 때 회의실에있는 모든 인스턴스에 발생합니다. 이 이벤트는 레벨이 종료 된 후 "정리"에 매우 유용합니다. 예를 들어 여기에서로드 된 리소스를 삭제하거나 파티클 시스템을 제거하여 메모리 누수 등을 방지 할 수 있습니다.

 

애니메이션 끝 애니메이션 끝

GameMaker Studio 2에서 스프라이트는 하위 이미지를 가질 수 있고 다른 속도로 애니메이션 될 수 있기 때문에 정적 인 것은 아닙니다. 애니메이션의 각 프레임 (하위 이미지)은 0부터 시작하여 코드 또는 작업을 통해 확인할 수있는 고유 번호를 갖지만 때로는 애니메이션이 종료 된 시점 만 알면됩니다. 하위 이미지 인덱스에 마지막 프레임에 도달했음을 나타내는 애니메이션이 끝날 때이 이벤트가 트리거되는 시점입니다. 이 이벤트는 애니메이션의 마지막 프레임에 도달했을 때 인스턴스 자체를 파괴하도록 인스턴스를 설정할 수있는 폭발 오브젝트와 같은 많은 것들에 실제로 유용합니다.

 

애니메이션 업데이트 애니메이션 업데이트

이 이벤트는 Skeletal Animation Functions 와 함께 사용하도록 설계되었습니다. 스켈 레탈 애니메이션 스프라이트를 사용하는 인스턴스의 모든 단계에서 트리거되는 특수 이벤트로, 뼈의 방향이 현재 애니메이션 상태에 대해 계산 된 후이 데이터가 커밋되기 전에 뼈 데이터를 "차단"하도록 설계되었습니다. 그리기에 사용합니다. 이를 통해 적절한 기능을 사용하여 본 데이터를 수정할 수 있습니다.

 

애니메이션 이벤트 애니메이션 이벤트

이 이벤트는 Skeletal Animation Functions 와 함께 사용하도록 설계되었습니다. 이 이벤트는 스켈 레탈 애니메이션 스프라이트가 할당 된 오브젝트에서, 오브젝트가 지정된 스프라이트에 대한 기본 드로우를 사용하거나, draw_self ()를 통해 스프라이트가 그려지는 오브젝트에서 트리거됩니다. 현재 다른 방식으로 그려진 스켈레톤 애니메이션 스프라이트에서는 트리거되지 않습니다. 이 이벤트는 스프라이트 애니메이션의 특정 이벤트에 할당 된 데이터를 캡처하고 편집기에 정의 된 데이터를 전송하도록 설계되었습니다.

애니메이션 이벤트가 트리거되면 event_data 맵이라는 특수 DS 맵 이 생성됩니다 (비동기 이벤트에 대한 async_load 맵과 거의 동일). 이 맵에는 이벤트에 대한 정보를 얻기 위해 구문 분석 할 수있는 다음과 같은 키 / 값 쌍이 모두 있습니다.

  • " name ": 이벤트 이름 (애니메이션 프로그램에 정의 된대로)
  • " track ": 애니메이션이 재생되는 트랙의 색인 (기본값은 0)
  • " integer ": 이벤트와 연관된 정수 값 (애니메이션 프로그램에 정의 된대로 기본값은 0 임)
  • " float ": 이벤트와 관련된 부동 소수점 값 (애니메이션 프로그램에 정의 된대로 기본값은 0.0 임)
  • " string ": 이벤트와 관련된 문자열 값 (애니메이션 프로그램에 정의 된대로 기본값은 빈 문자열 "")
