객체 편집기

객체는 게임의 측면을 제어하고 특정 작업을 수행하는 데 사용하는 리소스입니다. 대부분의 경우 게임 룸에서 볼 수 있도록 스프라이트가 있지만 사용자와 관련된 작업이나 타이밍 등을 수행하기 위해 "장면 뒤"컨트롤러로 사용되는 경우가 있습니다. 행동에 영향을 줄 수 있고 특정 이벤트와 서로에 반응 할 수 있으며 게임에서 볼 수있는 대부분의 것은 사물과 그 상호 작용을 기반으로합니다. 이제 게임 기반에 직접 개체를 배치하지 않고 기본적으로 개체 리소스의 복제본 (또는 원하는 경우 복제) 인 이러한 개체의 인스턴스 를 배치하기 때문에 "기반"이라고 말합니다. 인스턴스와 객체 가 같지 않고 각각에 영향을 줄 수있는 고유 한 기능 세트가 있으므로 기억해야 할 것은 매우 중요합니다.

이를 명확히하기 위해 GameMaker Studio 2로 만든 게임을 생각해보십시오. 모든 캐릭터, 몬스터, 공, 벽 등은 자산 브라우저 에서 생성 한 모든 객체입니다. 그런 다음 룸 편집기 에서 객체 를 룸으로 끌어 기본 객체 템플릿에서 복사 한 인스턴스 를 만듭니다. 그런 다음 회의실의 인스턴스를 편집기 자체 또는 코드를 통해 변경하여 크기를 조정하거나 색상을 지정하거나 "기본"개체에서 다른 세부 정보를 수정할 수 있습니다. 따라서 인스턴스에 영향을 주거나 변경하는 것에 대해 이야기 할 때 , 방에있는 객체의 특정 사본 하나에 영향을 미칩니다. 나머지 하나는 영향을받지 않지만 해당 단일 인스턴스 만 영향을받습니다. 그러나 오브젝트 에 영향을 주거나 변경하는 것에 대해 이야기 할 때, 우리는 자산 브라우저에서 오브젝트를 수정하고 있고 그에 대한 모든 작업이 그 시점에서 생성 된 모든 인스턴스에도 반영됨을 의미합니다. 따라서 개체는 인스턴스의 템플릿이고 인스턴스는 게임을 만들기 위해 방에 배치하는 것입니다.

게임에서 오브젝트를 만들려면 마우스 오른쪽 버튼을 클릭하면됩니다 자산 브라우저에서 오브젝트 생성 옵션을 클릭하십시오. 그러면이 페이지 상단에 그림으로 표시된 창이 다음 섹션과 함께 열립니다.

세부 사항 세부 사항

여기에서 객체 이름을 지정할 수 있습니다. 이 이름은 공백이나 무단 문자를 포함해서는 안되며 (영숫자 문자 및 밑줄 "_"만 허용) 짧지 만 기억하기 쉬워 나중에 나중에 리소스를 많이 찾을 수있을 때 쉽게 리소스를 식별 할 수 있습니다. 예를 들어 많은 사람들이 접두사 또는 접미사를 사용하여 " obj_Ship "또는 " Ship_obj "와 같은 자원 간의 차이점을 식별합니다.

다음으로 오브젝트에 원하는 스프라이트를 선택할 수 있습니다. 마우스 왼쪽 버튼으로 클릭 스프라이트 상자에 애셋 탐색기 가 애셋 브라우저에서 사용 가능한 모든 스프라이트 목록과 함께 나타납니다.

그런 다음 객체에 사용할 것을 선택할 수 있습니다. 스프라이트가 오브젝트에 할당 된 상태에서 스프라이트 편집 버튼을 클릭 할 수 있습니다 스프라이트 속성 을 변경하거나, 오브젝트에 스프라이트가 없지만 하나를 생성하려는 경우 스프라이트 추가 버튼을 클릭하면됩니다. 새로운 스프라이트 자산을 생성하고 할당합니다. 이미지 편집 버튼을 클릭하여 할당 된 스프라이트 자산 이미지를 편집 할 수도 있습니다 이미지 편집기 를 엽니 다.

 

충돌 마스크 충돌 마스크

스프라이트 할당 영역 아래에는 특정 콜리 전 마스크 를 선택할 수있는 버튼이 있습니다. 기본적으로 객체는 지정된 스프라이트를 기반으로 충돌 속성을 생성하지만 충돌이 다른 스프라이트와 다른 모양을 기반으로하는 경우가 있습니다. 예를 들어 메인 스프라이트에 직사각형 충돌 마스크가있는 하향식 게임이있을 수 있지만 충돌을보다 부드럽게하기 위해 원형 마스크가 더 좋습니다. 이러한 경우 Collision Mask (충돌 마스크) 버튼을 클릭하고 다른 스프라이트 리소스를 선택할 수 있으므로 이제 오브젝트의 인스턴스가 할당 된 스프라이트를 그리지 만 마스크 스프라이트에서 기본 충돌이 발생 합니다. Sprite Editor 의 페이지에서 충돌 마스크에 대한 자세한 정보를 찾을 수 있습니다.

 

속성 속성

여기에 객체에 대한 다음 옵션이 제공됩니다.

  • Visible : Visible은 룸이 시작될 때이 객체의 인스턴스가 표시되는지 여부를 나타냅니다. 일반적으로 대부분의 인스턴스가 표시되지만 때로는 보이지 않는 인스턴스를 만드는 것이 유용합니다. 예를 들어, 웨이 포인트로 이동 몬스터를 제어하거나 값을 추적하고 특정 시간 동작을 수행하는 데 사용할 수 있습니다. 보이지 않는 객체는 여전히 이벤트에 반응하며, 스프라이트 또는 마스크가 할당 된 경우 다른 인스턴스와 충돌하는 경우에도 반응합니다 ... 그냥 볼 수없고 Draw 이벤트를 수행하지 않습니다 . 기본적으로 visible 플래그가 선택되어 있습니다.
  • 솔리드 : 인스턴스를 솔리드로 표시하면 충돌 이벤트 를 트리거하기 전에 GameMaker Studio 2에 충돌 해결을 시도해야합니다. 이것은 매우 기본적인 기능이며 실제로는 충돌 이벤트 자체에있는 코드 또는 DnD ™를 수행하기 전에 충돌이 감지되기 ​​전의 순간으로 인스턴스를 다시 이동시키는 것입니다. 사용이 제한되어 있습니다.
  • 지속성 : 지속성 개체는 방이 변경 될 때 사라지지 않고 오히려 "지속"되어 새 방으로 옮겨집니다. 파괴 행동이나 코드로 명시 적으로 파괴 할 때만 사라집니다. 당신이 한 방에 영속 객체를 배치하면, 아직도 이후의 모든 객실에서 사용할 수과를 제외하고는 개체 속성의 이벤트에 할당 된 작업을 수행 할 수 지속될 것으로이 방법 만 때를 한 번 해고 이벤트를 만듭니다 새 방으로 변경할 때가 아니라 처음에 생성됩니다. 예를 들어, 방에서 방으로 이동하는 메인 캐릭터가 있고 인스턴스 내에서 변수를 동일하게 유지하려고 할 때 편리하지만 지속 가능한 객체를 올바르게 작성하고 파괴하려면 쉽게주의를 기울여야합니다. 실수로 프로젝트에 오류가 발생합니다.

지속성 및 룸 레이어 지속성 및 룸 레이어

룸에서 퍼시스턴스가 활성화 된 인스턴스를 생성하면 객체의 인스턴스를 생성하는 데 사용한 기능에 따라 레이어 또는 깊이가 할당됩니다. 방이 변경 될 때 다음 방에 지정된 것과 같은 이름 또는 깊이를 가진 도면층이없는 경우 방 전체에서 지속되는 인스턴스에 대해 새 도면층이 작성됩니다 . 인스턴스를 만들 때 명명 된 레이어를 지정한 경우 새 레이어의 이름도 원래 레이어와 동일하게 지정되고 인스턴스에 깊이를 할당하면 새 레이어의 이름이 " _layer_XXX "로 지정됩니다. 여기서 " XXX " 레이어에 고유 한 이름을 부여하는 데 사용되는 16 진수 값입니다. 이 층들은 방 끝 의 방에서 제거됩니다. 레이어와 퍼시 스턴트 인스턴스에 대해 알아 두어야 할 또 다른 사항은 생성시 인스턴스에 명명 된 레이어를 할당하고 다음 방에 같은 이름의 다른 레이어가있는 경우 지속 된 인스턴스가 동일한 레이어에 할당된다는 것입니다 레이어의 깊이에 관계없이 이름. 마지막으로, 지속 된 인스턴스가 인스턴스가 생성 된 것과 같은 깊이의 레이어가있는 방으로 이동하면이 레이어에 할당 되지 않고 대신 같은 이름의 깊이로 새 레이어가 생성됩니다 (네이밍 규칙에 따름) 위에서 설명한).

지 속성 및 이벤트 지 속성 및 이벤트

앞에서 언급했듯이 영구 객체는 Create Event를 한 번만 실행하지만 적절한 시간에 Game Start , Game End , Room Start 및 Room End 이벤트가 트리거됩니다. 그러나 game_restart () 함수를 사용하여 게임을 다시 시작하면 모든 영구 객체가 제거되고 game에 의해 다시 생성 될 때만 존재합니다 . 또한 영구 객체를 비활성화 하면 Room End 이벤트가 트리거되기 전에 다시 활성화되지 않는 한 더 이상 한 방에서 다른 방으로 전달되지 않습니다. 다시 활성화 하려면 전체 게임 단계가 필요 하므로 게임에 객체를 다시 활성화하기위한 전체 단계를 제공하지 않으므로 실제 Room End 이벤트를 사용하면 작동하지 않으므로 사용할 때이 점을 명심해야합니다. 영구 객체와 함께 비활성화하고 이에 따라 코드를 계획하십시오.

  • 물리학은 사용 : 당신은 물리학이 객체가 물리적 세계의 일부가되어야한다는 GameMaker 스튜디오 2를 말하고있다 상자 용도를 확인할 때합니다 (의 세트로 회의실 편집자 . 이를 확인하면이 객체의 모든 인스턴스에 대한 물리적 속성을 정의 할 수있는 체인 창이 추가로 열립니다. 이에 대한 자세한 정보는 Physics Objects 섹션에서 찾을 수 있습니다.
