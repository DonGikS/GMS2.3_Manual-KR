스프라이트 에디터

Sprite Editor 는 스프라이트 의 일반 속성을 설정하는 곳입니다. 새 스프라이트를 생성하여이 창을 열 수 있습니다 (오른쪽 클릭 애셋 브라우저 에서 "Create Sprite"를 선택하십시오) 기존 스프라이트에서 또는 마우스 오른쪽 버튼을 클릭하여 기존 스프라이트를 선택하고 " 속성 "을 선택하십시오.

Sprite Editor를 구성하는 부분은 다음과 같습니다.

 

스프라이트 이름 스프라이트 이름

여기에서 스프라이트의 이름을 지정할 수 있습니다. 스프라이트 이름은 게임 프로젝트 전체에서 스프라이트를 리소스로 식별하는 데 사용되며 영숫자 일 수 있으며 밑줄 "_"기호 만 사용할 수 있습니다. 스프라이트 이름에 숫자를 사용할 수 있지만 이름은 숫자로 시작할 수 없으므로 GML 또는 DnD ™를 사용할 때 유효하지 않은 리소스 이름이됩니다.

 

이미지 속성 이미지 속성

In this section you can set the image size by clicking the Sprite Size button . 다음 창이 열립니다 :

여기에서 선택한 변환을 적용하기 전후에 스프라이트를 미리 볼 수 있으며 아래 섹션은 새 스프라이트 속성을 설정하기위한 것으로 두 부분으로 나뉩니다.

  • 이미지 크기 조정 :이 옵션을 사용하면 스프라이트 그리기 캔버스의 크기를 조정할 수 있으며 스프라이트에 포함 된 모든 이미지가 새 크기에 맞게 확장되고 크기가 조정됩니다. 기본 이미지의 가로 세로 비율을 조정하고 유지하도록 선택할 수 있으며 픽셀 또는 백분율로 크기를 조정할 수도 있습니다. 작업중인 이미지 유형에 따라 선형 보간을 사용하거나 사용하지 않을 수도 있습니다 (일반적으로 선형 보간은 영역에서 픽셀을 "부드럽게"만들므로 픽셀 아트에서는 가장 잘 사용하지 않는 것이 좋습니다).
  • 캔버스 크기 조정 :이 옵션은 내용의 크기를 조정하지 않고 그리기 캔버스의 크기를 새로운 크기로 조정합니다. 방향 화살표를 사용하여 크기 조정의 "앵커"지점을 설정 한 다음 크기 값을 설정할 수 있습니다. 캔버스 크기를 조정할 때 종횡비를 유지하는 옵션이 있으며 맨 아래에서 크기 조정 값을 픽셀 또는 백분율로 설정하도록 선택할 수 있습니다.

새 크기에 만족하면 적용 을 클릭하여 변경 사항을 적용하고 크기 조정 특성 창을 닫을 수 있습니다. 스프라이트에 크기 또는 크기 변경을 적용 할 때 적용 버튼 옆에있는 원점 에 적용 확인란을 선택하여 스프라이트의 원점에도 변경 사항을 적용하도록 선택할 수 있습니다. 이 옵션을 체크하면-원점이 기본 설정 중 하나가 아니고 스프라이트에 잠겨 있지 않은 경우 스프라이트의 크기 변경에 맞게 위치가 조정됩니다. 이 옵션을 선택하지 않으면 원점에 배율이 적용되지 않습니다. 원점이 사용자 정의 값이 아니거나 잠겨 있으면 회색으로 표시되어 사용할 수 없게됩니다. 스프라이트 편집기 환경 설정 에서 모든 스프라이트에 대해이 옵션을 설정할 수 있습니다.

NOTE: This option is only available for 비트 맵 sprites and will be greyed out for SWF or skeleton animation sprites.

이 섹션에서 이미지 편집기 를 열어 스프라이트를 그리거나 " 가져 오기 "버튼을 클릭하여 이전에 생성 된 스프라이트를 가져올 수 있습니다. 스프라이트는 가장 일반적인 비트 맵 형식 ( PNG , JPG , GIF 등)에서 가져올 수 있지만 GameMaker Studio 2는 벡터 스프라이트 ( SWF ) 및 골격 애니메이션 스프라이트 ( Spine )도 지원합니다. 이러한 형식에 대한 자세한 내용은 비트 맵이 아닌 스프라이트 섹션을 참조하십시오. 비트 맵 스프라이트를 가져올 때 파일 탐색기에서 여러 이미지 파일을 선택할 수 있으며 각 파일은 별도의 하위 이미지 (프레임)와 동일한 스프라이트에 추가됩니다.

 

텍스처 설정 텍스처 설정

스프라이트 속성 창의이 섹션에서는 GameMaker Studio 2가 장치 및 브라우저에서 사용하기 위해 텍스처 페이지 에 스프라이트를 구성하는 이미지를 저장하는 방법을 다룹니다. 데스크탑 플랫폼의 경우 이것이 중요하지는 않지만 모바일 또는 웹용으로 개발을 시작할 때 제대로 관리되지 않는 텍스처가 게임에 해로운 영향을 미쳐 성능 문제를 일으킬 수 있으므로 이미지 자산 (텍스처)의 적절한 관리가 매우 중요합니다.

타일 수평 타일 수직 상자는 기본적으로, 일반적으로는 타일 스프라이트에 원하지 않는 대부분의 시간으로 확인하지 않습니다 확인. 그러나 특정 상황에서는 타일링을 원할 수 있습니다. 즉, 텍스처 페이지가 생성되지 않은 경우 스케일링으로 인해 게임 그래픽에 인공물이 생길 수 있으므로 뷰 또는 룸을 스케일링하려는 경우 이러한 옵션을 확인해야합니다. 정확히.

스프라이트를 3D 게임에서 텍스쳐 맵으로 사용하려는 경우 텍스처 페이지 분리 상자를 선택하면 스프라이트에 텍스처 페이지가 모두 표시됩니다 (각 서브 이미지도 자체 텍스처 페이지를 포함하므로 스프라이트에 10 개의 이미지가 있으면 10 개의 텍스처 페이지가 생성됩니다. 이렇게하면 게임의 텍스처 메모리 요구가 증가하므로이 옵션을 사용할 때는 각별히주의해야합니다. 또한 스프라이트가 3D에이 옵션을 사용하는 경우 텍스처는 2의 거듭 제곱이어야합니다 (예 : 128x128, 256x256, 512x512 등).

다음 옵션은 스프라이트 이미지에 사전 곱셈 알파 를 활성화하는 것입니다. 활성화되면 스프라이트의 모든 하위 이미지에 알파가 미리 곱해집니다. 이것은 일반적으로 표면이나 특정 특수 효과에 필요한 스프라이트 그리기입니다. 일반 스프라이트를 사용하려면 일반 스프라이트와 미리 곱한 스프라이트 사이에 눈에 띄는 차이가 없어야합니다.

Edge Filtering 옵션도 있습니다. 이 옵션은 게임에 선형 보간이 활성화되어 있고 스프라이트 주변에 "후광"이 나타나는 경우에 사용됩니다. 텍스처 페이지의 여러 소스 텍셀 에서 한 번에 보간되는 하드웨어로 인해 발생하며, 게임을 해상도로 확장 할 때 특히 분명 할 수 있습니다. 이를 활성화하면 가장 가까운 풀 알파 스프라이트 픽셀의 색상을 취하도록 가장자리 픽셀을 필터링하므로 보간 된 픽셀을 이전에 존재했던 후광없이 블렌딩합니다.

마지막으로 스프라이트 리소스가 속하는 텍스처 그룹 을 선택할 수 있습니다. 기본적으로 텍스처 그룹 (이전 텍스처 그룹 관리자 에서 정의 됨)은 특정 룸이나 게임 레벨에 필요한 모든 이미지 리소스를 함께 저장할 수 있도록 설정할 수있는 것입니다. 예를 들어, 하나의 텍스처 그룹에 모든 레벨 1 이미지를, 다른 텍스처 그룹에 모든 레벨 2 이미지를 가질 수 있으며 GameMaker Studio 2는 그룹화 된 모든 리소스를 동일한 텍스처 페이지에 배치하여 텍스처 페이지를 줄입니다. 게임이 선택된 대상 플랫폼에서 실행되는 동안 교환.

NOTE: This may not always be necessary and the performance increase from this method will depend on whether the target device is CPU bound or GPU bound.

텍스처 페이지에 대한 자세한 내용은 여기를 참조하십시오 .

텍스처 그룹은 비트 맵 이미지에만 관련이 있으며 비트 맵이 아닌 스프라이트는 텍스처 그룹에 할당되지 않으며 위의 옵션 중 일부는 사용할 수 없습니다. 비트 맵이 아닌 스프라이트 일부에는 이미 고유 한 텍스처 페이지 (골격 애니메이션 스프라이트)가 있지만 다른 일부는 벡터 (SWF 스프라이트)로 구성되어 있으므로 텍스처가 필요하지 않습니다. 자세한 내용은 다음 섹션을 참조하십시오.

  • 비트 맵이 아닌 스프라이트

 

충돌 마스크 충돌 마스크

모든 스프라이트에는 스프라이트가 인스턴스에 할당 될 때 충돌을 감지하는 데 사용되는 스프라이트 영역에 해당하는 충돌 마스크 가 있습니다. 먼저 충돌 마스크에 대한 모드 를 설정할 수 있으며 다음 중 하나 일 수 있습니다.

  • 자동 : GameMaker Studio 2는 스프라이트 알파 값을 기반으로 충돌 마스크를 자동으로 계산합니다 (이는 아래 설명 된 공차 설정에 따라 다름).
  • Full Image : 마스크의 범위는 스프라이트의 왼쪽, 오른쪽, 위쪽 및 아래쪽 경계와 동일하게 계산됩니다.
  • Manual : 이것을 선택하면 왼쪽, 오른쪽, 위쪽 및 아래쪽의 입력 값을 사용하여 충돌 마스크의 경계를 설정합니다.

충돌 마스크 모드 외에도 드롭 다운 메뉴에서 선택할 수있는 다양한 유형 의 충돌 마스크가 있습니다.

  • Rectangle : 간단한 사각형 충돌 마스크. 프로젝트에서 사용될 때 가장 빨리 해결됩니다.
  • Rotated Rectangle : 스프라이트가 할당 된 인스턴스의 image_angle 값과 함께 회전하는 간단한 직사각형 충돌 마스크입니다. 이것은 일반적인 사각형 충돌 마스크보다 해결이 느리지 만 정확한 충돌 마스크보다 빠릅니다.
  • Ellipse : 타원형 충돌 마스크를 만듭니다. 사각형 마스크보다 해결 속도가 느립니다.
  • 다이아몬드 : "다이아몬드"충돌 마스크를 만듭니다. 이는 사각형 마스크와 엘리트 마스크보다 두 개가 해결 속도가 느리고 CPU 오버 헤드를 더 많이 생성합니다.
  • Precise : 스프라이트의 윤곽에 따라 정확한 충돌 마스크를 만듭니다. 스프라이트에 여러 개의 하위 이미지가있는 경우 이는 서로 배치 된 모든 하위 이미지의 가장자리를 합성 한 것 입니다. 모드가 수동으로 설정되어 있고 기본값을 변경하면 정확한 마스크가 경계 상자에 잘립니다. 이것은 매우 느리고 많은 CPU 오버 헤드를 생성합니다.
  • Precise (프레임 당) : 프레임 단위로 정확한 충돌 마스크를 만듭니다 (예 : 마스크가 각기 다른 프레임의 윤곽에 맞게 변경됨). 모드가 수동으로 설정되어 있고 기본값을 변경하면 정확한 마스크가 경계 상자에 잘립니다. 이는 가장 느리게 해결되는 충돌 마스크 유형으로, 매우 구체적이고 드문 경우를 제외하고는 피해야합니다.

다음 이미지는 다양한 마스크 옵션을 보여줍니다.

충돌 확인 옵션은 게임에서 매우 중요한 옵션으로, 개체의 상호 작용 방식과 게임 실행 방식에 직접적인 영향을 미치며, 잘못된 설정은 모든 성능에 부정적인 영향을 미칩니다. 왜 그런 겁니까? 글쎄, 두 인스턴스가 만나고 두 인스턴스가 모두 유효한 마스크를 가질 때마다 경계 상자 와 마스크의 겹침을 검사하여 충돌 이벤트가 생성됩니다.이 상자는 정확하거나 정확하지 않고 이미지 색인에 맞게 조정될 수 있습니다. 아래는이를 설명하기위한 이미지입니다.

충돌 마스크 설정 아래에는 알파 공차 슬라이더가 있습니다.이 공차는 스프라이트 픽셀의 투명도와 관련하여 마스크를 만드는 공차를 나타냅니다. 부분적으로 투명한 공차 픽셀은 마스크 외부에 남겨두고 낮은 공차는 유지합니다 마스크는 해당 픽셀을 고려합니다. 이는 경계 상자 (수동으로 설정되지 않은 경우)와 정확한 충돌 검사를위한 마스크에 영향을줍니다.

Finally you can set the Bounding Box limits for the mask. The bounding box is the name given to the rectangular area that the sprite mask fits in. Only pixels inside this bounding box are used for the mask and will be involved in collision checking, and by default this is automatically computed, taking the alpha tolerance into account. If you have "precise (per frame)" checked then this will be separate for each sub-image too. When setting it yourself, you can input the values in the boxes for left, right top and bottom, or you can click 스프라이트 미리보기 창에서 스프라이트 위로 드래그하여 "그립니다". 경계 상자를 "그리기"할 때 마스크는 최종 값에 맞게 자동으로 생성되며 경계 상자는 마스크를 "클립"할 수 있습니다.

미리보기 창으로 가면 각 모서리에있는 "손잡이"를 사용하여 충돌 마스크를 조정할 수 있습니다. 이를 클릭하고 드래그하면 경계 상자에 입력하는 값과 동일하게 충돌 마스크가 조정됩니다.

 

프레임 속도 프레임 속도

이 섹션에서는 왼쪽의 입력 상자에서 애니메이션 프레임을 재생하는 속도 를 설정할 수 있습니다. 프레임 속도 설정은 " 초당 프레임 수 "또는 " 게임 프레임 당 프레임 수 "를 기준으로 할 수 있습니다. 여기서 30 "초당 프레임 수"설정은 1 초에 30 개의 애니메이션 프레임을 표시하고 30 "은 "게임 프레임 당 프레임 수"는 단일 게임 단계에서 30 개의 프레임을 모두 표시하려고 시도하므로 "게임 프레임 당 프레임 수"값은 일반적으로 1 이하로 설정됩니다 (예 : 게임 프레임 속도가 0.5 인 프레임은 새 프레임마다 표시됨) 두 게임 단계).

프레임 속도 설정은 미리보기 창에서 애니메이션 재생에 영향을 줄뿐만 아니라 게임에서 스프라이트 애니메이션 방식에 영향을줍니다 . 여기에서 설정 한 속도는 이미지 속도 승수 1로 게임에서 실행될 때 스프라이트 미리보기를 표시하지만, 이는 프로그래밍 된 내용에 따라 스프라이트가 더 빠르거나 느리게 실행될 수있는 image_speed 변수를 사용하여 코드로 설정할 수 있습니다. 예를 들어 스프라이트가 10 "frames per second"를 사용하도록 설정 한 다음 image_speed 를 0.5로 설정하면 게임은 스프라이트를 절반 속도-초당 5 프레임으로 그립니다. 반대로, 같은 스프라이트의 image_speed 를 2로 설정하면 게임에서 초당 20 프레임의 두 배로 애니메이션이 그려집니다. 프레임 속도는 이미지 편집기 에서도 설정할 수 있습니다.

 

스프라이트 원산지 스프라이트 원산지

스프라이트 편집기의 상단 막대에는 스프라이트 원점 속성이 포함되어 있습니다. 룸 내 위치에 해당하는 스프라이트의 점입니다. 즉, 특정 x / y 위치에서 인스턴스를 만들 때 스프라이트의 원점이 거기에 배치됩니다.

By default the origin is the top left corner of the sprite but it is often more convenient to use the middle centre or you may wish to have some other position used (like for a platform game character you might want bottom centre), so you can click 원점 드롭 다운 메뉴에서 기본값 중 하나를 선택합니다. 클릭하여 원점을 수동으로 설정할 수도 있습니다 스프라이트 미리보기 이미지에서 원점 마커를 클릭 한 지점으로 이동 시키거나 해당 상자에 x와 y에 다른 값을 입력합니다. 음수 (왼쪽 및 위쪽) 또는 스프라이트 너비 및 높이 (오른쪽 및 아래쪽)보다 큰 양수를 사용하여 스프라이트 영역 외부에서 원점을 설정할 수도 있습니다.이 기능은 다음과 같은 객체를 처리 할 때 매우 유용합니다. 합성 스프라이트를 그릴 필요가 있습니다.

 

줌 컨트롤 줌 컨트롤

프레임 컨트롤 프레임 컨트롤

The frame controls are for playing back in the preview window the animation of the frames, with the play button 애니메이션 시작 / 중지 및 루프 버튼 스프라이트 루프 여부를 설정하는 데 사용 (마지막 프레임에 도달하면 프레임 1로 돌아갑니다) 또는 탁구 (마지막 프레임에 도달하면 프레임을 통해 뒤로 이동합니다).

루프 / 핑퐁 옵션 은 게임에서 스프라이트가 표시되는 방식에 영향을 미치므로 스프라이트에서 이러한 옵션을 설정하면 게임을 컴파일하고 실행할 때 표시되는 방식이 변경됩니다. 그러나 이것은 프레임 뷰 에서 설정할 수있는 루프 포인트에는 해당되지 않습니다 (자세한 내용은 아래 참조). 스프라이트 편집기에 순수하게 표시되며 애니메이션을 테스트하는 데 도움이되지만 효과는 없습니다. 스프라이트 애니메이션이 게임에 어떻게 표시되는지

프레임 컨트롤의 추가 부분은 현재 프레임 표시 및 입력입니다. 이 상자에는 스프라이트 애니메이션의 총 프레임 수와 현재 표시된 프레임이 표시됩니다. 입력 상자를 클릭하고 보려는 프레임 번호를 입력하여 언제든지 현재 프레임을 변경할 수 있습니다.

IMPORTANT! Frames are shown numbered from 1 to maxframe number, however in your game code, frames will be numbered from 0 to maxframe number - 1.

프레임 프레임

This part of the sprite editor shows each of the different frames that make up a single sprite set along a sequence timeline. You can click 아무 프레임에서나 왼쪽 또는 오른쪽으로 드래그하여 애니메이션 순서를 변경하고 두 번 클릭해도됩니다 아무 프레임에서나 이미지 편집기 를 시작하고 편집 할 수 있습니다. 표준 키보드 단축키를 사용하거나 마우스 오른쪽 버튼을 사용하여 프레임을 잘라 내고 복사하고 붙여 넣을 수 있습니다 메뉴 옵션을 사용하여 여러 프레임을 선택하여 사용할 수 있습니다. / + .

마우스 오른쪽 버튼 메뉴에는 다음과 같은 옵션이 있습니다.

  • 프레임 추가 -끝에 애니메이션에 새 프레임을 추가합니다.
  • 프레임 삽입 -현재 선택된 프레임 다음에 애니메이션에 새 프레임을 삽입합니다.
  • 잘라 내기 / 복사 / 붙여 넣기 -하나 이상의 프레임을 잘라 내고 복사하여 붙여 넣습니다.
  • 선택된 프레임 삭제-현재 선택된 프레임을 삭제합니다.
  • Copy To Start- 선택한 프레임을 애니메이션의 시작 부분에 복사합니다.
  • Copy To Finish- 선택한 프레임을 애니메이션의 끝에 복사합니다.
  • 자산 키 늘이기-선택한 프레임을 늘려서 빈 프레임을 채 웁니다. 이를 사용하려면 늘이려는 프레임 뒤에 빈 프레임을 추가 한 다음이 빈 프레임에서 "자산 키 삭제"옵션을 사용하여 프레임 내용을 제거하지만 실제로 프레임을 제거하지는 마십시오. 마지막으로이 옵션을 선택하면 빈 프레임을 채우기 위해 프레임이 늘어납니다.
  • 자산 키 삭제 -실제로 프레임 자체를 제거하지 않고 프레임에서 이미지를 제거하는 데 사용할 수 있습니다.

Apart from the option given above to stretch a frame, you can also position the mouse on the edge of a frame and then when the cursor changes to a double arrow, simply click 더 많은 시간이 걸리도록 드래그하여 프레임 크기를 조정하십시오.

동일한 프레임을 여러 번 복제 한 것처럼 확장 된 프레임을 표시하는 데 시간이 더 걸립니다.

Finally, you can click and drag the start and end points for the animation loop in the frame view, so that you can see how selected parts of the animation behave when looped. This is done by clicking 빨간색 "루프 마커"에서 원하는 위치로 드래그합니다.

루프 포인트는 애니메이션 작업시 시각적으로 도움 이되며 게임에서 애니메이션이 표시되는 방식에는 영향을 미치지 않습니다.
